3     * This class defines a section filter operation optimized to capture entire tables with minimum 
4     * intervention required from the application. When filtering is started, first one section 
5     * matching the specified pattern will be filtered. Once that section has been found, the 
6     * last_section_number field will be used to determine the number of Section objects required 
7     * to hold the entire table. This number of objects will be created and filtering re-started to 
8     * capture all the sections of the table. The SectionAvailableEvent will be generated each time 
9     * a Section is captured. The EndOfFilteringEvent will be generated when the complete table 
10    * has been captured.
11    *
12    * The version_number of all sections of the table will be the same. If a section is captured with a 
13    * version_number that differs from the version_number of the section first captured, a 
14    * VersionChangeDetectedEvent will be generated. The newly captured section will be ignored 
15    * and filtering will continue on the table with the version number of the first captured section. 
16    * Only one VersionChangeDetectedEvent will be sent per filtering action. 
17    *
18    * Care should be taking in setting the filter parameters, a too restrictive filter will never stop 
19    * automatically and a too wide filter can produce inconsistent results (e.g. filtering short sections 
20    * using a TableSectionFilter)
21    *
22    * When the API detects a filtering situation where the filter parameters have been incompletely defined, 
23    * resulting in a blocking filter or a non MPEG-2 compliant result, an InCompleteFilteringEvent is sent 
24    * and filtering is stopped.
