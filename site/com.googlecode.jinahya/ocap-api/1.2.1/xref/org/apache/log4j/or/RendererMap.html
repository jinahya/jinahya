64    /**
65       Search the parents of <code>clazz</code> for a renderer. The
66       renderer closest in the hierarchy will be returned. If no
67       renderers could be found, then the default renderer is returned.
68  
69       <p>The search first looks for a renderer configured for
70       <code>clazz</code>. If a renderer could not be found, then the
71       search continues by looking at all the interfaces implemented by
72       <code>clazz</code> including the super-interfaces of each
73       interface.  If a renderer cannot be found, then the search looks
74       for a renderer defined for the parent (superclass) of
75       <code>clazz</code>. If that fails, then all the interfaces
76       implemented by the parent of <code>clazz</code> are searched and
77       so on.
78  
79       <p>For example, if A0, A1, A2 are classes and X0, X1, X2, Y0, Y1
80       are interfaces where A2 extends A1 which in turn extends A0 and
81       similarly X2 extends X1 which extends X0 and Y1 extends Y0. Let
82       us also assume that A1 implements the Y0 interface and that A2
83       implements the X2 interface.
84

86       <code>get(A2.class)</code> method depending on the renderers
87       added to the map.

90       <tr><th>Added renderers</th><th>Value returned by <code>get(A2.class)</code></th>

93           <td align="center"><code>A0Renderer</code>
