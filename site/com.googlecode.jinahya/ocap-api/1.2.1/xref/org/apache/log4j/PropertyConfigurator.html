31  /**
32     Allows the configuration of log4j from an external file.  See
33     <b>{@link #doConfigure(String, LoggerRepository)}</b> for the
34     expected format.
35  
36     <p>It is sometimes useful to see how log4j is reading configuration
37     files. You can enable log4j internal logging by defining the
38     <b>log4j.debug</b> variable.
39  
40     <P>As of log4j version 0.8.5, at class initialization time class,
41     the file <b>log4j.properties</b> will be searched from the search
42     path used to load classes. If the file can be found, then it will
43     be fed to the {@link PropertyConfigurator#configure(java.net.URL)}
44     method.
45  
46     <p>The <code>PropertyConfigurator</code> does not handle the
47     advanced configuration features supported by the {@link
48     org.apache.log4j.xml.DOMConfigurator DOMConfigurator} such as
49     support custom {@link org.apache.log4j.spi.ErrorHandler ErrorHandlers},
50     nested appenders such as the {@link org.apache.log4j.AsyncAppender
51     AsyncAppender}, etc.
52  
53     This version of PropertyConfigurator has been modified to support the definition of groups as described in {@link Logger} by adding a 'group' entry using this format:
54     <p/>
55     log4j.group.NAME=LEVEL
56     <p/>
57     All loggers retrieved via {@link Logger#getLogger(String, String)} or {@link Logger#getLogger(Class, String)} with the same group name will have their effective level set to the provided level.  
58     <p/>
59     This group-defined Level verbosity threshold can be overridden by setting an explicit level on the Logger itself.  
60     <p>
61     All option <em>values</em> admit variable substitution. The
62     syntax of variable substitution is similar to that of Unix
63     shells. The string between an opening <b>&quot;${&quot;</b> and
64     closing <b>&quot;}&quot;</b> is interpreted as a key. The value of
65     the substituted variable can be defined as a system property or in
66     the configuration file itself. The value of the key is first
67     searched in the system properties, and if not found there, it is
68     then searched in the configuration file being parsed.  The
69     corresponding value replaces the ${variableName} sequence. For
70     example, if <code>java.home</code> system property is set to
71     <code>/home/xyz</code>, then every occurrence of the sequence
72     <code>${java.home}</code> will be interpreted as
73     <code>/home/xyz</code>.
74  
75  
76     @author Ceki G&uuml;lc&uuml;
77     @author Anders Kristensen
78     @since 0.8.1 */

81    /**
82      Read configuration from a file. <b>The existing configuration is
83      not cleared nor reset.</b> If you require a different behavior,
84      then call {@link  LogManager#resetConfiguration
85      resetConfiguration} method before calling
86      <code>doConfigure</code>.
87  
88      <p>The configuration file consists of statements in the format
89      <code>key=value</code>. The syntax of different configuration
90      elements are discussed below.
91  
92      <h3>Repository-wide threshold</h3>
93  
94      <p>The repository-wide threshold filters logging requests by level
95      regardless of logger. The syntax is:
96  
97      <pre>
98      log4j.threshold=[level]
99      </pre>
100 
101     <p>The level value can consist of the string values OFF, FATAL,
102     ERROR, WARN, INFO, DEBUG, ALL or a <em>custom level</em> value. A
103     custom level value can be specified in the form
104     level#classname. By default the repository-wide threshold is set
105     to the lowest possible value, namely the level <code>ALL</code>.
106     </p>
107 
108 
109     <h3>Appender configuration</h3>
110 
111     <p>Appender configuration syntax is:
112     <pre>
113     # For appender named <i>appenderName</i>, set its class.
114     # Note: The appender name can contain dots.
115     log4j.appender.appenderName=fully.qualified.name.of.appender.class
116 
117     # Set appender specific options.
118     log4j.appender.appenderName.option1=value1
119     ...
120     log4j.appender.appenderName.optionN=valueN
121     </pre>
122 
123     For each named appender you can configure its {@link Layout}. The
124     syntax for configuring an appender's layout is:
125     <pre>
126     log4j.appender.appenderName.layout=fully.qualified.name.of.layout.class
127     log4j.appender.appenderName.layout.option1=value1
128     ....
129     log4j.appender.appenderName.layout.optionN=valueN
130     </pre>
131 
132     The syntax for adding {@link Filter}s to an appender is:
133     <pre>
134     log4j.appender.appenderName.filter.ID=fully.qualified.name.of.filter.class
135     log4j.appender.appenderName.filter.ID.option1=value1
136     ...
137     log4j.appender.appenderName.filter.ID.optionN=valueN
138     </pre>
139     The first line defines the class name of the filter identified by ID;
140     subsequent lines with the same ID specify filter option - value
141     paris. Multiple filters are added to the appender in the lexicographic
142     order of IDs.
143 
144     The syntax for adding an {@link ErrorHandler} to an appender is:
145     <pre>
146     log4j.appender.appenderName.errorhandler=fully.qualified.name.of.filter.class
147     log4j.appender.appenderName.errorhandler.root-ref={true|false}
148     log4j.appender.appenderName.errorhandler.logger-ref=loggerName
149     log4j.appender.appenderName.errorhandler.appender-ref=appenderName
150     log4j.appender.appenderName.errorhandler.option1=value1
151     ...
152     log4j.appender.appenderName.errorhandler.optionN=valueN
153     </pre>
154 
155     <h3>Configuring loggers</h3>
156 
157     <p>The syntax for configuring the root logger is:
158     <pre>
159       log4j.rootLogger=[level], appenderName, appenderName, ...
160     </pre>
161 
162     <p>This syntax means that an optional <em>level</em> can be
163     supplied followed by appender names separated by commas.
164 
165     <p>The level value can consist of the string values OFF, FATAL,
166     ERROR, WARN, INFO, DEBUG, ALL or a <em>custom level</em> value. A
167     custom level value can be specified in the form
168     <code>level#classname</code>.
169 
170     <p>If a level value is specified, then the root level is set
171     to the corresponding level.  If no level value is specified,
172     then the root level remains untouched.
173 
174     <p>The root logger can be assigned multiple appenders.
175 
176     <p>Each <i>appenderName</i> (separated by commas) will be added to
177     the root logger. The named appender is defined using the
178     appender syntax defined above.
179 
180     <p>For non-root categories the syntax is almost the same:
181     <pre>
182     log4j.logger.logger_name=[level|INHERITED|NULL], appenderName, appenderName, ...
183     </pre>
184 
185     <p>The meaning of the optional level value is discussed above
186     in relation to the root logger. In addition however, the value
187     INHERITED can be specified meaning that the named logger should
188     inherit its level from the logger hierarchy.
189 
190     <p>If no level value is supplied, then the level of the
191     named logger remains untouched.
192 
193     <p>By default categories inherit their level from the
194     hierarchy. However, if you set the level of a logger and later
195     decide that that logger should inherit its level, then you should
196     specify INHERITED as the value for the level value. NULL is a
197     synonym for INHERITED.
198 
199     <p>Similar to the root logger syntax, each <i>appenderName</i>
200     (separated by commas) will be attached to the named logger.
201 
202     <p>See the <a href="../../../../manual.html#additivity">appender
203     additivity rule</a> in the user manual for the meaning of the
204     <code>additivity</code> flag.
205 
206     <h3>ObjectRenderers</h3>
207 
208     You can customize the way message objects of a given type are
209     converted to String before being logged. This is done by
210     specifying an {@link org.apache.log4j.or.ObjectRenderer ObjectRenderer}
211     for the object type would like to customize.
212 
213     <p>The syntax is:
214 
215     <pre>
216     log4j.renderer.fully.qualified.name.of.rendered.class=fully.qualified.name.of.rendering.class
217     </pre>
218 
219     As in,
220     <pre>
221     log4j.renderer.my.Fruit=my.FruitRenderer
222     </pre>
223 
224    <h3>ThrowableRenderer</h3>
225 
226    You can customize the way an instance of Throwable is
227    converted to String before being logged. This is done by
228    specifying an {@link org.apache.log4j.spi.ThrowableRenderer ThrowableRenderer}.
229 
230    <p>The syntax is:
231 
232    <pre>
233    log4j.throwableRenderer=fully.qualified.name.of.rendering.class
234    log4j.throwableRenderer.paramName=paramValue
235    </pre>
236 
237    As in,
238    <pre>
239    log4j.throwableRenderer=org.apache.log4j.EnhancedThrowableRenderer
240    </pre>
241 
242     <h3>Logger Factories</h3>
243 
244     The usage of custom logger factories is discouraged and no longer
245     documented.
246 
247     <h3>Resetting Hierarchy</h3>
248 
249     The hierarchy will be reset before configuration when
250     log4j.reset=true is present in the properties file.

252     <h3>Example</h3>
253 
254     <p>An example configuration is given below. Other configuration
255     file examples are given in the <code>examples</code> folder.
256 
257     <pre>
258 
259     # Set options for appender named "A1".
260     # Appender "A1" will be a SyslogAppender
261     log4j.appender.A1=org.apache.log4j.net.SyslogAppender
262 
263     # The syslog daemon resides on www.abc.net
264     log4j.appender.A1.SyslogHost=www.abc.net
265 
266     # A1's layout is a PatternLayout, using the conversion pattern
267     # <b>%r %-5p %c{2} %M.%L %x - %m\n</b>. Thus, the log output will
268     # include # the relative time since the start of the application in
269     # milliseconds, followed by the level of the log request,
270     # followed by the two rightmost components of the logger name,
271     # followed by the callers method name, followed by the line number,
272     # the nested disgnostic context and finally the message itself.
273     # Refer to the documentation of {@link PatternLayout} for further information
274     # on the syntax of the ConversionPattern key.
275     log4j.appender.A1.layout=org.apache.log4j.PatternLayout
276     log4j.appender.A1.layout.ConversionPattern=%-4r %-5p %c{2} %M.%L %x - %m\n
277 
278     # Set options for appender named "A2"
279     # A2 should be a RollingFileAppender, with maximum file size of 10 MB
280     # using at most one backup file. A2's layout is TTCC, using the
281     # ISO8061 date format with context printing enabled.
282     log4j.appender.A2=org.apache.log4j.RollingFileAppender
283     log4j.appender.A2.MaxFileSize=10MB
284     log4j.appender.A2.MaxBackupIndex=1
285     log4j.appender.A2.layout=org.apache.log4j.TTCCLayout
286     log4j.appender.A2.layout.ContextPrinting=enabled
287     log4j.appender.A2.layout.DateFormat=ISO8601
288 
289     # Root logger set to DEBUG using the A2 appender defined above.
290     log4j.rootLogger=DEBUG, A2
291 
292     # Logger definitions:
293     # The SECURITY logger inherits is level from root. However, it's output
294     # will go to A1 appender defined above. It's additivity is non-cumulative.
295     log4j.logger.SECURITY=INHERIT, A1
296     log4j.additivity.SECURITY=false
297 
298     # Only warnings or above will be logged for the logger "SECURITY.access".
299     # Output will go to A1.
300     log4j.logger.SECURITY.access=WARN
301 
302 
303     # The logger "class.of.the.day" inherits its level from the
304     # logger hierarchy.  Output will go to the appender's of the root
305     # logger, A2 in this case.
306     log4j.logger.class.of.the.day=INHERIT
307     </pre>
308 
309     <p>Refer to the <b>setOption</b> method in each Appender and
310     Layout for class specific options.
311 
312     <p>Use the <code>#</code> or <code>!</code> characters at the
313     beginning of a line for comments.
