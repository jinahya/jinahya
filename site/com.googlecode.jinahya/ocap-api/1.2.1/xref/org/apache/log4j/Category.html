42  /**
43    * <font color="#AA2222"><b>This class has been deprecated and
44    * replaced by the {@link Logger} <em>subclass</em></b></font>. It
45    * will be kept around to preserve backward compatibility until mid
46    * 2003.
47    * 
48    * <p><code>Logger</code> is a subclass of Category, i.e. it extends
49    * Category. In other words, a logger <em>is</em> a category. Thus,
50    * all operations that can be performed on a category can be
51    * performed on a logger. Internally, whenever log4j is asked to
52    * produce a Category object, it will instead produce a Logger
53    * object. Log4j 1.2 will <em>never</em> produce Category objects but
54    * only <code>Logger</code> instances. In order to preserve backward
55    * compatibility, methods that previously accepted category objects
56    * still continue to accept category objects.
57    * 
58    * <p>For example, the following are all legal and will work as
59    * expected.
60    * 
61     <pre>
62      &nbsp;&nbsp;&nbsp;// Deprecated form:
63      &nbsp;&nbsp;&nbsp;Category cat = Category.getInstance("foo.bar")
64     
65      &nbsp;&nbsp;&nbsp;// Preferred form for retrieving loggers:
66      &nbsp;&nbsp;&nbsp;Logger logger = Logger.getLogger("foo.bar")
67     </pre>
68     
69    *  <p>The first form is deprecated and should be avoided.
70    * 
71    *  <p><b>There is absolutely no need for new client code to use or
72    *  refer to the <code>Category</code> class.</b> Whenever possible,
73    *  please avoid referring to it or using it.
74    * 
75    * <p>See the <a href="../../../../manual.html">short manual</a> for an
76    * introduction on this class.
77    * <p>
78    * See the document entitled <a href="http://www.qos.ch/logging/preparingFor13.html">preparing
79    *  for log4j 1.3</a> for a more detailed discussion.
80    *
81    * @author Ceki G&uuml;lc&uuml;
82    * @author Anders Kristensen 
83    */

503   /**
504     *  Check whether this category is enabled for the <code>DEBUG</code>
505     *  Level.
506     *
507     *  <p> This function is intended to lessen the computational cost of
508     *  disabled log debug statements.
509     *
510     *  <p> For some <code>cat</code> Category object, when you write,
511     *  <pre>
512     *      cat.debug("This is entry number: " + i );
513     *  </pre>
514     *
515     *  <p>You incur the cost constructing the message, concatenatiion in
516     *  this case, regardless of whether the message is logged or not.
517     *
518     *  <p>If you are worried about speed, then you should write
519     *  <pre>
520     * 	 if(cat.isDebugEnabled()) {
521     * 	   cat.debug("This is entry number: " + i );
522     * 	 }
523     *  </pre>
524     *
525     *  <p>This way you will not incur the cost of parameter
526     *  construction if debugging is disabled for <code>cat</code>. On
527     *  the other hand, if the <code>cat</code> is debug enabled, you
528     *  will incur the cost of evaluating whether the category is debug
529     *  enabled twice. Once in <code>isDebugEnabled</code> and once in
530     *  the <code>debug</code>.  This is an insignificant overhead
531     *  since evaluating a category takes about 1%% of the time it
532     *  takes to actually log.
533     *
534     *  @return boolean - <code>true</code> if this category is debug
535     *  enabled, <code>false</code> otherwise.
536     *   */
537   public
538   boolean isDebugEnabled() {
539     return false;
540   }
