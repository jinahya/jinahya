40  /**
41   * The <code>Document</code> interface represents the entire HTML or XML 
42   * document. Conceptually, it is the root of the document tree, and provides 
43   * the primary access to the document's data.
44   * <p>Since elements, text nodes, comments, processing instructions, etc. 
45   * cannot exist outside the context of a <code>Document</code>, the 
46   * <code>Document</code> interface also contains the factory methods needed 
47   * to create these objects. The <code>Node</code> objects created have a 
48   * <code>ownerDocument</code> attribute which associates them with the 
49   * <code>Document</code> within whose context they were created.
50   * <p>See also the <a href='http://www.w3.org/TR/2000/REC-DOM-Level-2-Core-20001113'>Document Object Model (DOM) Level 2 Core Specification</a>
51   * and the <a href='http://www.w3.org/TR/2004/REC-DOM-Level-3-Core-20040407'>Document Object Model (DOM) Level 3 Core Specification</a>.
52   */
53  public interface Document extends Node {
54      /**
55       * The Document Type Declaration (see <code>DocumentType</code>) 
56       * associated with this document. For XML documents without a document
57       * type declaration this returns <code>null</code>. For HTML documents,
58       * a <code>DocumentType</code> object may be returned, independently of
59       * the presence or absence of document type declaration in the HTML
60       * document.
61       * <br>This provides direct access to the <code>DocumentType</code> node, 
62       * child node of this <code>Document</code>. This node can be set at 
63       * document creation time and later changed through the use of child 
64       * nodes manipulation methods, such as <code>Node.insertBefore</code>, 
65       * or <code>Node.replaceChild</code>. Note, however, that while some 
66       * implementations may instantiate different types of 
67       * <code>Document</code> objects supporting additional features than the 
68       * "Core", such as "HTML" [<a href='http://www.w3.org/TR/2003/REC-DOM-Level-2-HTML-20030109'>DOM Level 2 HTML</a>]
69       * , based on the <code>DocumentType</code> specified at creation time, 
70       * changing it afterwards is very unlikely to result in a change of the 
71       * features supported.
72       * @since DOM Level 3
73       * @return the Document Type Declaration associated with this document, or <code>null</code>
74       */
75      public DocumentType getDoctype();
76  
77      /**
78       * The <code>DOMImplementation</code> object that handles this document. A 
79       * DOM application may use objects from multiple implementations.
80       * @return the <code>DOMImplementation</code> object that handles this document
81       */
82      public DOMImplementation getImplementation();
83  
84      /**
85       * This is a convenience attribute that allows direct access to the child 
86       * node that is the root element of the document. 
87       * @return the child node that is the root element of the document
88       */
89      public Element getDocumentElement();

91      /**
92       * Creates an element of the type specified. Note that the instance 
93       * returned implements the <code>Element</code> interface, so attributes 
94       * can be specified directly on the returned object.
95       * <br>In addition, if there are known attributes with default values, 
96       * <code>Attr</code> nodes representing them are automatically created 
97       * and attached to the element.
98       * <br>To create an element with a qualified name and namespace URI, use 
99       * the <code>createElementNS</code> method.
100      * @param tagName The name of the element type to instantiate. For XML, 
101      *   this is case-sensitive. otherwise it depends on the 
102      *   case-sensitivity of the markup language in use. In that case, the 
103      *   name is mapped to the canonical form of that markup by the DOM 
104      *   implementation.
105      * @return A new <code>Element</code> object with the 
106      *   <code>nodeName</code> attribute set to <code>tagName</code>, and 
107      *   <code>localName</code>, <code>prefix</code>, and 
108      *   <code>namespaceURI</code> set to <code>null</code>.
109      * @exception DOMException
110      *   INVALID_CHARACTER_ERR: Raised if the specified name is not an XML 
111      *   name according to the XML version in use (1.0 for JSR 280).
112      */
113     public Element createElement(String tagName)
114                                  throws DOMException;

117      * Creates an empty <code>DocumentFragment</code> object. 
118      * @return A new <code>DocumentFragment</code>.
119      */
120     public DocumentFragment createDocumentFragment();

172      * @return A new <code>Attr</code> object with the <code>nodeName</code> 
173      *   attribute set to <code>name</code>, and <code>localName</code>,

175      *   <code>null</code>. The value of the attribute is the empty string.

183     /**
184      * Creates an <code>EntityReference</code> object. In addition, if the

193      * case.
194      * @param name The name of the entity to reference. Unlike 
195      *   <code>Document.createElementNS</code> or 
196      *   <code>Document.createAttributeNS</code>, no namespace well-formed 
197      *   checking is done on the entity name.
198      * @return The new <code>EntityReference</code> object.
199      * @exception DOMException
200      *   INVALID_CHARACTER_ERR: Raised if the specified name is not an XML 
201      *   name according to the XML version in use (1.0 for JSR 280).
202      *   <br>NOT_SUPPORTED_ERR: Raised if this document is an HTML document.
203      */
204     public EntityReference createEntityReference(String name)
205                                                  throws DOMException;

207     /**

214      *   markup language in use.

216      *   <code>Elements</code>.
217      */

220     /**
221      * Imports a node from another document to this document, without altering 
222      * or removing the source node from the original document; this method 
223      * creates a new copy of the source node. The returned node has no 
224      * parent; (<code>parentNode</code> is <code>null</code>).
225      * <br>For all nodes, importing a node creates a node object owned by the 
226      * importing document, with attribute values identical to the source 
227      * node's <code>nodeName</code> and <code>nodeType</code>, plus the 
228      * attributes related to namespaces (<code>prefix</code>, 
229      * <code>localName</code>, and <code>namespaceURI</code>). As in the 
230      * <code>cloneNode</code> operation, the source node is not altered.
231      * <br>Additional information is copied as appropriate to the 
232      * <code>nodeType</code>, attempting to mirror the behavior expected if 
233      * a fragment of XML or HTML source was copied from one document to 
234      * another, recognizing that the two documents may have different DTDs 
235      * in the XML case. The following list describes the specifics for each 
236      * type of node. 
237      * <dl>
238      * <dt>ATTRIBUTE_NODE</dt>
239      * <dd>The <code>ownerElement</code> attribute 
240      * is set to <code>null</code> and the <code>specified</code> flag is 
241      * set to <code>true</code> on the generated <code>Attr</code>. The 
242      * descendants of the source <code>Attr</code> are recursively imported 
243      * and the resulting nodes reassembled to form the corresponding subtree.
244      * Note that the <code>deep</code> parameter has no effect on 
245      * <code>Attr</code> nodes; they always carry their children with them 
246      * when imported.</dd>
247      * <dt>DOCUMENT_FRAGMENT_NODE</dt>
248      * <dd>If the <code>deep</code> option 
249      * was set to <code>true</code>, the descendants of the source  
250      * <code>DocumentFragment</code> are recursively imported and the 
251      * resulting nodes reassembled under the imported 
252      * <code>DocumentFragment</code> to form the corresponding subtree. 
253      * Otherwise, this simply generates an empty 
254      * <code>DocumentFragment</code>.</dd>
255      * <dt>DOCUMENT_NODE</dt>
256      * <dd><code>Document</code> 
257      * nodes cannot be imported.</dd>
258      * <dt>DOCUMENT_TYPE_NODE</dt>
259      * <dd><code>DocumentType</code> 
260      * nodes cannot be imported.</dd>
261      * <dt>ELEMENT_NODE</dt>
262      * <dd><em>Specified</em> attribute nodes of the source element are imported, and the generated 
263      * <code>Attr</code> nodes are attached to the generated 
264      * <code>Element</code>. Default attributes are <em>not</em> copied, though if the document being imported into defines default 
265      * attributes for this element name, those are assigned. If the 
266      * <code>importNode</code> <code>deep</code> parameter was set to 
267      * <code>true</code>, the descendants of the source element are 
268      * recursively imported and the resulting nodes reassembled to form the 
269      * corresponding subtree.</dd>
270      * <dt>ENTITY_NODE</dt>
271      * <dd><code>Entity</code> nodes can be 
272      * imported, however in the current release of the DOM the 
273      * <code>DocumentType</code> is readonly. Ability to add these imported 
274      * nodes to a <code>DocumentType</code> will be considered for addition 
275      * to a future release of the DOM.On import, the <code>publicId</code>, 
276      * <code>systemId</code>, and <code>notationName</code> attributes are 
277      * copied. If a <code>deep</code> import is requested, the descendants 
278      * of the the source <code>Entity</code> are recursively imported and 
279      * the resulting nodes reassembled to form the corresponding subtree.</dd>
280      * <dt>
281      * ENTITY_REFERENCE_NODE</dt>
282      * <dd>Only the <code>EntityReference</code> itself is 
283      * copied, even if a <code>deep</code> import is requested, since the 
284      * source and destination documents might have defined the entity 
285      * differently. If the document being imported into provides a 
286      * definition for this entity name, its value is assigned.</dd>
287      * <dt>NOTATION_NODE</dt>
288      * <dd>
289      * <code>Notation</code> nodes can be imported, however in the current 
290      * release of the DOM the <code>DocumentType</code> is readonly. Ability 
291      * to add these imported nodes to a <code>DocumentType</code> will be 
292      * considered for addition to a future release of the DOM.On import, the 
293      * <code>publicId</code> and <code>systemId</code> attributes are copied.
294      * Note that the <code>deep</code> parameter has no effect on this type
295      * of nodes since they cannot have any children.</dd>
296      * <dt>
297      * PROCESSING_INSTRUCTION_NODE</dt>
298      * <dd>The imported node copies its 
299      * <code>target</code> and <code>data</code> values from those of the 
300      * source node. Note that the <code>deep</code> parameter has no effect 
301      * on this type of nodes since they cannot have any children.</dd>
302      * <dt>TEXT_NODE, CDATA_SECTION_NODE, COMMENT_NODE</dt>
303      * <dd>These three 
304      * types of nodes inheriting from <code>CharacterData</code> copy their 
305      * <code>data</code> and <code>length</code> attributes from those of 
306      * the source node.Note that the <code>deep</code> parameter has no effect 
307      * on this type of nodes since they cannot have any children.</dd>
308      * </dl> 
309      * @param importedNode The node to import.
310      * @param deep If <code>true</code>, recursively import the subtree under 
311      *   the specified node; if <code>false</code>, import only the node 
312      *   itself, as explained above. This has no effect on nodes that cannot 
313      *   have any children, and on <code>Attr</code>, and 
314      *   <code>EntityReference</code> nodes.
315      * @return The imported node that belongs to this <code>Document</code>.
316      * @exception DOMException
317      *   NOT_SUPPORTED_ERR: Raised if the type of node being imported is not 
318      *   supported.
319      *   <br>INVALID_CHARACTER_ERR: Raised if one of the imported names is not 
320      *   an XML name according to the XML version in use (1.0 for JSR 280).
321      *   This may happen when importing an XML 1.1 
322      *   [<a href='http://www.w3.org/TR/2004/REC-xml11-20040204/'>XML 1.1</a>]
323      *   element into an XML 1.0 document, for instance.
324      * @since DOM Level 2
325      */
326     public Node importNode(Node importedNode, boolean deep)
327                                                  throws DOMException;

473     /**
474      * Returns the <code>Element</code>that has an ID attribute with the 
475      *given value. If no such element exists, this returns <code>null</code>.
476      * If more than one element has an ID attribute with that value, what 
477      * is returned is undefined.
478      * <br>The DOM implementation is expected to use the attribute 
479      * <code>Attr.isId</code> to determine if an attribute is of type ID. 
480      * <p ><b>Note:</b> Attributes with the name "ID" or "id" are not of type 
481      * ID unless so defined. 
482      * Implementations that do not know whether attributes are of type ID or
