41  /**
42   * The <code>Node</code> interface is the primary datatype for the entire 
43   * Document Object Model. It represents a single node in the document tree. 
44   * While all objects implementing the <code>Node</code> interface expose 
45   * methods for dealing with children, not all objects implementing the 
46   * <code>Node</code> interface may have children. For example, 
47   * <code>Text</code> nodes may not have children, and adding children to 
48   * such nodes results in a <code>DOMException</code> being raised.
49   * <p>The attributes <code>nodeName</code>, <code>nodeValue</code> and 
50   * <code>attributes</code> are included as a mechanism to get at node 
51   * information without casting down to the specific derived interface. In 
52   * cases where there is no obvious mapping of these attributes for a 
53   * specific <code>nodeType</code> (e.g., <code>nodeValue</code> for an 
54   * <code>Element</code> or <code>attributes</code> for a <code>Comment</code>
55   * ), this returns <code>null</code>. Note that the specialized interfaces 
56   * may contain additional and more convenient mechanisms to get and set the 
57   * relevant information.
58   * <p>The values of <code>nodeName</code>, 
59   * <code>nodeValue</code>, and <code>attributes</code> vary according to the 
60   * node type as follows: 
61   * <table border='1' cellpadding='3'>
62   * <tr>
63   * <th>Interface</th>

66   * <th>attributes</th>
67   * </tr>

72   * <td valign='top' rowspan='1' colspan='1'>same as 
73   * <code>Attr.value</code></td>
74   * <td valign='top' rowspan='1' colspan='1'><code>null</code></td>

314     /**
315      * Inserts the node <code>newChild</code> before the existing child node 
316      * <code>refChild</code>. If <code>refChild</code> is <code>null</code>, 
317      * insert <code>newChild</code> at the end of the list of children.
318      * <br>If <code>newChild</code> is a <code>DocumentFragment</code> object, 
319      * all of its children are inserted, in the same order, before 
320      * <code>refChild</code>. If the <code>newChild</code> is already in the 
321      * tree, it is first removed.
322      * <p ><b>Note:</b>  Inserting a node before itself is implementation 
323      * dependent. 
324      * @param newChild The node to insert.
325      * @param refChild The reference node, i.e., the node before which the new 
326      *   node must be inserted.
327      * @return The node being inserted.
328      * @exception DOMException
329      *   HIERARCHY_REQUEST_ERR: Raised if this node is of a type that does not 
330      *   allow children of the type of the <code>newChild</code> node, or if 
331      *   the node to insert is one of this node's ancestors  or this node 
332      *   itself, or if this node is of type <code>Document</code> and the 
333      *   DOM application attempts to insert a second 
334      *   <code>DocumentType</code> or <code>Element</code> node.
335      *   <br>WRONG_DOCUMENT_ERR: Raised if <code>newChild</code> was created 
336      *   from a different document than the one that created this node.
337      *   <br>NO_MODIFICATION_ALLOWED_ERR: Raised if this node is readonly or 
338      *   if the parent of the node being inserted is readonly.
339      *   <br>NOT_FOUND_ERR: Raised if <code>refChild</code> is not a child of 
340      *   this node.
341      *   <br>NOT_SUPPORTED_ERR: if this node is of type <code>Document</code>, 
342      *   this exception might be raised if the DOM implementation doesn't 
343      *   support the insertion of a <code>DocumentType</code> or 
344      *   <code>Element</code> node.
345      * @since DOM Level 3
346      */
347     public Node insertBefore(Node newChild, 
348                              Node refChild)
349                              throws DOMException;
350 
351     /**
352      * Replaces the child node <code>oldChild</code> with <code>newChild</code>
353      *  in the list of children, and returns the <code>oldChild</code> node.
354      * <br>If <code>newChild</code> is a <code>DocumentFragment</code> object, 
355      * <code>oldChild</code> is replaced by all of the 
356      * <code>DocumentFragment</code> children, which are inserted in the 
357      * same order. If the <code>newChild</code> is already in the tree, it 
358      * is first removed.
359      * @param newChild The new node to put in the child list.
360      * @param oldChild The node being replaced in the list.
361      * @return The node replaced.
362      * @exception DOMException
363      *   HIERARCHY_REQUEST_ERR: Raised if this node is of a type that does not 
364      *   allow children of the type of the <code>newChild</code> node, or if 
365      *   the node to put in is one of this node's ancestors or this node 
366      *   itself, or if this node is of type <code>Document</code> and the 
367      *   result of the replacement operation would add a second 
368      *   <code>DocumentType</code> or <code>Element</code> on the 
369      *   <code>Document</code> node.
370      *   <br>WRONG_DOCUMENT_ERR: Raised if <code>newChild</code> was created 
371      *   from a different document than the one that created this node.
372      *   <br>NO_MODIFICATION_ALLOWED_ERR: Raised if this node or the parent of 
373      *   the new node is readonly.
374      *   <br>NOT_FOUND_ERR: Raised if <code>oldChild</code> is not a child of 
375      *   this node.
376      *   <br>NOT_SUPPORTED_ERR: if this node is of type <code>Document</code>, 
377      *   this exception might be raised if the DOM implementation doesn't 
378      *   support the replacement of the <code>DocumentType</code> child or 
379      *   <code>Element</code> child.
380      * @since DOM Level 3
381      */
382     public Node replaceChild(Node newChild, 
383                              Node oldChild)
384                              throws DOMException;
385 
386     /**
387      * Removes the child node indicated by <code>oldChild</code> from the list 
388      * of children, and returns it.
389      * @param oldChild The node being removed.
390      * @return The node removed.
391      * @exception DOMException
392      *   NO_MODIFICATION_ALLOWED_ERR: Raised if this node is readonly.
393      *   <br>NOT_FOUND_ERR: Raised if <code>oldChild</code> is not a child of 
394      *   this node.
395      *   <br>NOT_SUPPORTED_ERR: if this node is of type <code>Document</code>, 
396      *   this exception might be raised if the DOM implementation doesn't 
397      *   support the removal of the <code>DocumentType</code> child or the 
398      *   <code>Element</code> child.
399      * @since DOM Level 3
400      */
401     public Node removeChild(Node oldChild)
402                             throws DOMException;
403 
404     /**
405      * Adds the node <code>newChild</code> to the end of the list of children 
406      * of this node. If the <code>newChild</code> is already in the tree, it 
407      * is first removed.
408      * @param newChild The node to add.If it is a <code>DocumentFragment</code>
409      *   object, the entire contents of the document fragment are moved 
410      *   into the child list of this node
411      * @return The node added.
412      * @exception DOMException
413      *   HIERARCHY_REQUEST_ERR: Raised if this node is of a type that does not 
414      *   allow children of the type of the <code>newChild</code> node, or if 
415      *   the node to append is one of this node's ancestors or this node 
416      *   itself, or if this node is of type <code>Document</code> and the 
417      *   DOM application attempts to append a second 
418      *   <code>DocumentType</code> or <code>Element</code> node.
419      *   <br>WRONG_DOCUMENT_ERR: Raised if <code>newChild</code> was created 
420      *   from a different document than the one that created this node.
421      *   <br>NO_MODIFICATION_ALLOWED_ERR: Raised if this node is readonly or 
422      *   if the previous parent of the node being inserted is readonly.
423      *   <br>NOT_SUPPORTED_ERR: if the <code>newChild</code> node is a child 
424      *   of the <code>Document</code> node, this exception might be raised 
425      *   if the DOM implementation doesn't support the removal of the 
426      *   <code>DocumentType</code> child or <code>Element</code> child.
427      * @since DOM Level 3
428      */
429     public Node appendChild(Node newChild)
430                             throws DOMException;
431 
432     /**
433      * Returns whether this node has any children.
434      * @return  <code>true</code> if this node has any children, 
435      *   <code>false</code> otherwise.
436      */
437     public boolean hasChildNodes();

439     /**
440      * Returns a duplicate of this node, i.e., serves as a generic copy 
441      * constructor for nodes. The duplicate node has no parent (
442      * <code>parentNode</code> is <code>null</code>) and no user data. User 
443      * data associated to the imported node is not carried over. However, if 
444      * any <code>UserDataHandlers</code> has been specified along with the 
445      * associated data these handlers will be called with the appropriate 
446      * parameters before this method returns.
447      * <br>Cloning an <code>Element</code> copies all attributes and their 
448      * values, including those generated by the XML processor to represent 
449      * defaulted attributes, but this method does not copy any children it 
450      * contains unless it is a deep clone. This includes text contained in 
451      * an the <code>Element</code> since the text is contained in a child 
452      * <code>Text</code> node. Cloning an <code>Attr</code> directly, as 
453      * opposed to be cloned as part of an <code>Element</code> cloning 
454      * operation, returns a specified attribute (<code>specified</code> is 
455      * <code>true</code>). Cloning an <code>Attr</code> always clones its 
456      * children, since they represent its value, no matter whether this is a 
457      * deep clone or not. Cloning an <code>EntityReference</code> 
458      * automatically constructs its subtree if a corresponding 
459      * <code>Entity</code> is available, no matter whether this is a deep 
460      * clone or not. Cloning any other type of node simply returns a copy of 
461      * this node.
462      * <br>Note that cloning an immutable subtree results in a mutable copy, 
463      * but the children of an <code>EntityReference</code> clone are readonly
464      * . In addition, clones of unspecified <code>Attr</code> nodes are 
465      * specified. And, cloning <code>Document</code>, 
466      * <code>DocumentType</code>, <code>Entity</code>, and 
467      * <code>Notation</code> nodes is implementation dependent.
468      * @param deep If <code>true</code>, recursively clone the subtree under 
469      *   the specified node; if <code>false</code>, clone only the node 
470      *   itself (and its attributes, if it is an <code>Element</code>).
471      * @return The duplicate node.
472      */
473     public Node cloneNode(boolean deep);
474 
475     /**
476      * Puts all <code>Text</code> nodes in the full depth of the sub-tree 
477      * underneath this <code>Node</code>, including attribute nodes, into a 
478      * "normal" form where only structure (e.g., elements, comments, 
479      * processing instructions, CDATA sections, and entity references) 
480      * separates <code>Text</code> nodes, i.e., there are neither adjacent 
481      * <code>Text</code> nodes nor empty <code>Text</code> nodes. This can 
482      * be used to ensure that the DOM view of a document is the same as if 
483      * it were saved and re-loaded, and is useful when operations (such as 
484      * XPointer [<a href='http://www.w3.org/TR/2003/REC-xptr-framework-20030325/'>XPointer</a>]
485      * lookups) that depend on a particular document tree 
486      * structure are to be used. 
487      * <p><b>Note:</b> In cases where the document contains 
488      * <code>CDATASections</code>, the normalize operation alone may not be 
489      * sufficient, since XPointers do not differentiate between 
490      * <code>Text</code> nodes and <code>CDATASection</code> nodes.
491      * @since DOM Level 2
492      */
493     public void normalize();

496      * Tests whether the DOM implementation implements a specific feature and 
497      * that feature is supported by this node.
498      * @param feature The name of the feature to test. This is the same name 
499      *   which can be passed to the method <code>hasFeature</code> on 
500      *   <code>DOMImplementation</code>.
501      * @param version This is the version number of the feature to test. In 
502      *   Level 2, version 1, this is the string "2.0". If the version is not 
503      *   specified, supporting any version of the feature will cause the 
504      *   method to return <code>true</code>.
505      * @return Returns <code>true</code> if the specified feature is 
506      *   supported on this node, <code>false</code> otherwise.
507      * @since DOM Level 2
508      */
509     public boolean isSupported(String feature, 
510                                String version);

512     /**
513      * The namespace URI of this node, or <code>null</code> if it is 
514      * unspecified.
515      * <br>This is not a computed value that is the result of a namespace
