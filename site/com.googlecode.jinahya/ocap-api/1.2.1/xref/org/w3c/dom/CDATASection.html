40  /**
41   * CDATA sections are used to escape blocks of text containing characters that 
42   * would otherwise be regarded as markup. The only delimiter that is 
43   * recognized in a CDATA section is the "]]&gt;" string that ends the CDATA 
44   * section. CDATA sections cannot be nested. Their primary purpose is for 
45   * including material such as XML fragments, without needing to escape all 
46   * the delimiters.
47   * <p>The <code>DOMString</code> attribute of the <code>Text</code> node holds 
48   * the text that is contained by the CDATA section. Note that this may 
49   * contain characters that need to be escaped outside of CDATA sections and 
50   * that, depending on the character encoding ("charset") chosen for 
51   * serialization, it may be impossible to write out some characters as part 
52   * of a CDATA section. 
53   * <p> The <code>CDATASection</code> interface inherits from the 
54   * <code>CharacterData</code> interface through the <code>Text</code> 
55   * interface. Adjacent <code>CDATASection</code> nodes are not merged by use 
56   * of the <code>normalize</code> method of the <code>Node</code> interface.
57   * Because no markup is recognized within a <code>CDATASection</code>, 
58   * character numeric references cannot be used as an escape mechanism when 
59   * serializing. Therefore, action needs to be taken when serializing a 
60   * <code>CDATASection</code> with a character encoding where some of the 
61   * contained characters cannot be represented. Failure to do so would not 
62   * produce well-formed XML.One potential solution in the serialization 
63   * process is to end the CDATA section before the character, output the 
64   * character using a character reference or entity reference, and open a new 
65   * CDATA section for any further characters in the text node. Note, however, 
66   * that some code conversion libraries at the time of writing do not return 
67   * an error or exception when a character is missing from the encoding, 
68   * making the task of ensuring that data is not corrupted on serialization 
69   * more difficult.
70   * <p>See also the <a href='http://www.w3.org/TR/2000/REC-DOM-Level-2-Core-20001113'>Document Object Model (DOM) Level 2 Core Specification</a>.
71   */
