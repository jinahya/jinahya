38  package org.w3c.dom;
39  
40  /**
41   *  The <code>Attr</code> interface represents an attribute in an 
42   * <code>Element</code> object. Typically the allowable values for the 
43   * attribute are defined in a document type definition.
44   * <p><code>Attr</code> objects inherit the <code>Node</code> interface, but 
45   * since they are not actually child nodes of the element they describe, the 
46   * DOM does not consider them part of the document tree. Thus, the 
47   * <code>Node</code> attributes <code>parentNode</code>, 
48   * <code>previousSibling</code>, and <code>nextSibling</code> have a 
49   * <code>null</code> value for <code>Attr</code> objects. The DOM takes the 
50   * view that attributes are properties of elements rather than having a 
51   * separate identity from the elements they are associated with; this should 
52   * make it more efficient to implement such features as default attributes 
53   * associated with all elements of a given type. Furthermore, 
54   * <code>Attr</code> nodes may not be immediate children of a 
55   * <code>DocumentFragment</code>. However, they can be associated with 
56   * <code>Element</code> nodes contained within a 
57   * <code>DocumentFragment</code>. In short, users and implementors of the 
58   * DOM need to be aware that <code>Attr</code> nodes have some things in 
59   * common with other objects inheriting the <code>Node</code> interface, but 
60   * they also are quite distinct.
61   * <p> The attribute's effective value is determined as follows: if this 
62   * attribute has been explicitly assigned any value, that value is the 
63   * attribute's effective value; otherwise, if there is a declaration for 
64   * this attribute, and that declaration includes a default value, then that 
65   * default value is the attribute's effective value; otherwise, the 
66   * attribute does not exist on this element in the structure model until it 
67   * has been explicitly added. Note that the <code>Node.nodeValue</code> 
68   * attribute on the <code>Attr</code> instance can also be used to retrieve
69   * the string version of the attribute's value(s). 
70   * <p>In XML, where the value of an attribute can contain entity references, 
71   * the child nodes of the <code>Attr</code> node may be either 
72   * <code>Text</code> or <code>EntityReference</code> nodes (when these are 
73   * in use; see the description of <code>EntityReference</code> for 
74   * discussion).
75   * <p>The DOM Core represents all attribute values as simple strings, even if 
76   * the DTD or schema associated with the document declares them of some 
77   * specific type such as tokenized.  
78   * <p>See also the <a href='http://www.w3.org/TR/2000/REC-DOM-Level-2-Core-20001113'>Document Object Model (DOM) Level 2 Core Specification</a> and the <a href='http://www.w3.org/TR/2004/REC-DOM-Level-3-Core-20040407'>Document Object Model (DOM) Level 3 Core Specification</a>.
79   */
80  public interface Attr extends Node {
81      /**
82       * Returns the name of this attribute. If <code>Node.localName</code> is 
83       * different from <code>null</code>, this attribute is a qualified name.
84       * @return the attribute name
85       */
86      public String getName();
87  
88      /**
89       * If this attribute was explicitly given a value in the original 
90       * document, this is <code>true</code>; otherwise, it is 
91       * <code>false</code>. Note that the implementation is in charge of this 
92       * attribute, not the user. If the user changes the value of the 
93       * attribute (even if it ends up having the same value as the default 
94       * value) then the <code>specified</code> flag is automatically flipped 
95       * to <code>true</code>. To re-specify the attribute as the default 
96       * value from the DTD, the user must delete the attribute. The 
97       * implementation will then make a new attribute available with 
98       * <code>specified</code> set to <code>false</code> and the default 
99       * value (if one exists).
100      * <br>In summary:  If the attribute has an assigned value in the document 
101      * then <code>specified</code> is <code>true</code>, and the value is 
102      * the assigned value.  If the attribute has no assigned value in the 
103      * document and has a default value in the DTD, then 
104      * <code>specified</code> is <code>false</code>, and the value is the 
105      * default value in the DTD. If the attribute has no assigned value in 
106      * the document and has a value of #IMPLIED in the DTD, then the 
107      * attribute does not appear in the structure model of the document. If 
108      * the <code>ownerElement</code> attribute is <code>null</code> (i.e. 
109      * because it was just created or was set to <code>null</code> by the 
110      * various removal and cloning operations) <code>specified</code> is 
111      * <code>true</code>. 
112      * @return <code>true</code> if this attribute was explicitly specified, otherwise <code>false</code>
113      */
114     public boolean getSpecified();
115 
116     /**
117      * Returns the value of this attribute. 
118      * On retrieval, the value of the attribute is returned as a string. 
119      * Character and general entity references are replaced with their 
120      * values. See also the method <code>getAttribute</code> on the 
121      * <code>Element</code> interface.
122      * <br>On setting, this creates a <code>Text</code> node with the unparsed 
123      * contents of the string. I.e. any characters that an XML processor 
124      * would recognize as markup are instead treated as literal text. See 
125      * also the method <code>Element.setAttribute</code>.
126      * <br> Some specialized implementations, such as some [<a href='http://www.w3.org/TR/2003/REC-SVG11-20030114/'>SVG 1.1</a>] 
127      * implementations, may do normalization automatically, even after 
128      * mutation; in such case, the value on retrieval may differ from the 
129      * value on setting.
130      * @return a String containing the value of this attribute
131      */
132     public String getValue();
133 
134     /**
135      * Sets the value of this attribute. 
136      * On setting, this creates a <code>Text</code> node with the unparsed 
137      * contents of the string. I.e. any characters that an XML processor 
138      * would recognize as markup are instead treated as literal text. See 
139      * also the method <code>Element.setAttribute</code>.
140      * <br> Some specialized implementations, such as some [<a href='http://www.w3.org/TR/2003/REC-SVG11-20030114/'>SVG 1.1</a>] 
141      * implementations, may do normalization automatically, even after 
142      * mutation; in such case, the value on retrieval may differ from the 
143      * value on setting.
144      * @param value a String containing the value of this attribute
145      * @exception DOMException
146      *   NO_MODIFICATION_ALLOWED_ERR: Raised when the node is readonly.
147      */
148     public void setValue(String value)
149                             throws DOMException;
150 
151     /**
152      * The <code>Element</code> node this attribute is attached to or 
153      * <code>null</code> if this attribute is not in use.
154      * @return the <code>Element</code> node this attribute is attached to, or <code>null</code>
155      * @since DOM Level 2
156      */
157     public Element getOwnerElement();
158 
159     /**
160      * Returns whether this attribute is known to be of type ID or not. 
161      * In other words, whether this attribute 
162      * contains an identifier for its owner element or not. When it is and 
163      * its value is unique, the <code>ownerElement</code> of this attribute 
164      * can be retrieved using the method <code>Document.getElementById</code>.
165      * <p>Note: The JSR 280 DOM subset does not support XML schema or
166      * <code>Document.normalizeDocument()</code>, and thus supports only 
167      * a subset of the DOM 3 mechanisms for identifying ID attributes:
168      * <ul>
169      * <li> the use of the methods <code>Element.setIdAttribute()</code>, 
170      * <code>Element.setIdAttributeNS()</code>, or 
171      * <code>Element.setIdAttributeNode()</code>, i.e. it is an 
172      * user-determined ID attribute.</li>
173      * </ul>
174      * @return <code>true</code> if the attribute is of type ID, otherwise <code>false</code>
175      * @since DOM Level 3
176      */
177     public boolean isId();
178 }
