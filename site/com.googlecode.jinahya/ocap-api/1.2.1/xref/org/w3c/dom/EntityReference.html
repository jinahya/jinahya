40  /**
41   * <code>EntityReference</code> objects may be inserted into the structure 
42   * model when an entity reference is in the source document, or when the 
43   * user wishes to insert an entity reference. Note that character references 
44   * and references to predefined entities are considered to be expanded by 
45   * the HTML or XML processor so that characters are represented by their 
46   * Unicode equivalent rather than by an entity reference. Moreover, the XML 
47   * processor may completely expand references to entities while building the 
48   * structure model, instead of providing <code>EntityReference</code> 
49   * objects. If it does provide such objects, then for a given 
50   * <code>EntityReference</code> node, it may be that there is no 
51   * <code>Entity</code> node representing the referenced entity. If such an 
52   * <code>Entity</code> exists, then the subtree of the 
53   * <code>EntityReference</code> node is in general a copy of the 
54   * <code>Entity</code> node subtree. However, this may not be true when an 
55   * entity contains an unbound namespace prefix. In such a case, because the 
56   * namespace prefix resolution depends on where the entity reference is, the 
57   * descendants of the <code>EntityReference</code> node may be bound to 
58   * different namespace URIs.
59   * <p>As for <code>Entity</code> nodes, <code>EntityReference</code> nodes and 
60   * all their descendants are readonly.
61   * <p>See also the <a href='http://www.w3.org/TR/2000/REC-DOM-Level-2-Core-20001113'>Document Object Model (DOM) Level 2 Core Specification</a>.
