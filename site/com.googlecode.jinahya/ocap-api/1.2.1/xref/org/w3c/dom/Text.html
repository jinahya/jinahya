40  /**
41   * The <code>Text</code> interface inherits from <code>CharacterData</code> 
42   * and represents the textual content (termed character data in XML) of an 
43   * <code>Element</code> or <code>Attr</code>. If there is no markup inside 
44   * an element's content, the text is contained in a single object 
45   * implementing the <code>Text</code> interface that is the only child of 
46   * the element. If there is markup, it is parsed into the information items 
47   * (elements, comments, etc.) and <code>Text</code> nodes that form the list 
48   * of children of the element.
49   * <p>When a document is first made available via the DOM, there is only one 
50   * <code>Text</code> node for each block of text. Users may create adjacent 
51   * <code>Text</code> nodes that represent the contents of a given element 
52   * without any intervening markup, but should be aware that there is no way 
53   * to represent the separations between these nodes in XML or HTML, so they 
54   * will not (in general) persist between DOM editing sessions. The 
55   * <code>normalize()</code> method on <code>Node</code> merges any such 
56   * adjacent <code>Text</code> objects into a single node for each block of 
57   * text.
58   * <p>See also the <a href='http://www.w3.org/TR/2000/REC-DOM-Level-2-Core-20001113'>Document Object Model (DOM) Level 2 Core Specification</a>.
59   */
60  public interface Text extends CharacterData {
61      /**
62       * Breaks this node into two nodes at the specified <code>offset</code>, 
63       * keeping both in the tree as siblings. After being split, this node 
64       * will contain all the content up to the <code>offset</code> point. A 
65       * new node of the same type, which contains all the content at and 
66       * after the <code>offset</code> point, is returned. If the original 
67       * node had a parent node, the new node is inserted as the next sibling 
68       * of the original node. When the <code>offset</code> is equal to the 
69       * length of this node, the new node has no data.
70       * @param offset The 16-bit unit offset at which to split, starting from 
71       *   <code>0</code>.
72       * @return The new node, of the same type as this node.
73       * @exception DOMException
74       *   INDEX_SIZE_ERR: Raised if the specified offset is negative or greater 
75       *   than the number of 16-bit units in <code>data</code>.
76       *   <br>NO_MODIFICATION_ALLOWED_ERR: Raised if this node is readonly.
77       */
78      public Text splitText(int offset)
79                            throws DOMException;
80  
81  }
