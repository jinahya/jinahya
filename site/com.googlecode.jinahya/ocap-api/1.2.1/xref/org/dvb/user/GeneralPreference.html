3   /**
4    * This class defines a set of general preferences. These preferences 
5    * are read from the receiver and each application (downloaded or not) 
6    * can access them through the <code>UserPreferenceManager.read</code> method. The 
7    * standardized preferences are "User Language", "Parental Rating", 
8    * "User Name", "User Address", "User @", "Country Code", "Default Font Size".<p>
9    * When constructed, objects of this class are empty and have no values
10   * defined. Values may be added using the add methods inherited from the
11   * Preference class or by calling <code>UserPreferenceManager.read</code>.<p>
12   * The encodings of these standardized preferences are as follows.<ul>
13   * <li>User Language: 3 letter ISO 639 language codes; 
14   * <li>Parental Rating: string using the same encoding as returned by <code>javax.tv.service.guide.ContentRatingAdvisory.getDisplayText</code>; 
15   * <li>User Name: Name of the user. This shall be in an order that is appropriate for 
16   * presentation directly to the user, e.g. in Western Europe, listing the first name first 
17   * and the family name last is recommended as being culturally appropriate in many locales.
18   * <li>User Address: postal address of the user, may contain multiple lines separated by 
19   * carriage return characters (as defined in table D-4). 
20   * <li>User @: e-mail address of the user in the SMTP form as defined in RFC821; 
21   * <li>Country Code: two letter ISO 3166-1 country code; 
22   * <li>Default Font Size: preferred font size for normal body text expressed in points, 
23   * decimal integer value encoded as a string (26 is the default; differing size indicates a 
24   * preference of different font size than usual)
25   * </ul>
26   * The preference names are treated as case-insensitive. The preference names shall be 
27   * considered equal at least when the method java.lang.String.equalsIgnoreCase() returns 
28   * true for the strings when the locale "EN.UK" is used. Depending on the locale used in 
29   * the implementation, implementations are allowed to consider equal also other upper 
30   * and lower case character pairs in addition to those defined by the "EN.UK" locale.
31   * <p>
32   * The standardized preference names in the present document shall only use such letters 
33   * where the upper and lower case characters are recognized by the "EN.UK" locale.
