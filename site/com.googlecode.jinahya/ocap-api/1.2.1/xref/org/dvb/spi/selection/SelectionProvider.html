14  /**
15   * A provider of service selection.  For example, a SelectionProvider
16   * can be installed in an always-running service-unbound xlet to
17   * provide the ability to reach services that do not have standarized
18   * SI signalling in an IPTV network.
19   * <p>
20   * The process for determining which Provider (if any) shall be used
21   * to present a transport independent service is defined by the following 
22   * steps in the order given.
23   * <p>
24   * 1) Consult the service list. If the service appears in the service
25   * list then make an selection between the available sources of that
26   * Service as defined by ServiceContext.select(Service).
27   * <p>
28   * 2) If there one Provider registered to support the locator scheme
29   * of the locator for the Service, ask that Provider to present that
30   * service. The service selection operation shall fail if the provider
31   * cannot present it.
32   * <p>
33   * 3) If more than one Provider has registed as supporting the locator
34   * scheme of the locator for the Service, offer the Service to each 
35   * Provider in turn until either one of them does not fail or no more
36   * Providers are available in which case the service selection operation
37   * shall fail.
38   * <p>
39   * NOTE: If the performance penalty of polling multiple Providers
40   * for the same locator scheme is undesirable, service providers / operators
41   * using multiple Providers should ensure they use different Locator schemes.
42   * <p>
43   * Providers should select the transport dependent locators
44   * returned such that there is no collision the transport dependent
45   * locators returned by other Providers that may offer the same 
46   * transport independent service. Where an MHP terminal is managed
47   * by a service provider / operator, the service provider / operator
48   * should enforce this if they permit more than one Provider to be
49   * installed.
50   * <p>
51   * Where there is a collision and a transport dependent locator
52   * can be accessed by more than one Provider, the selection of 
53   * which to use is implementation dependent.
54   * 
55   * <p>A SelectionProvider may inform the implementation of the
56   * actual location of a service by two mechanisms.
57   * <ul>
58   * <li>For services which are already present in the network,
59   * the actual location can be passed in ServiceReference
60   * objects. These can be provided by the getServiceList
61   * method when a provider is first installed and later by
62   * the SelectionProviderContext. For these services, the
63   * implementation shall use this actual location to present
64   * the service without calling the select method. A session
65   * shall be created using the ServiceReference used to present
66   * the service in order that the provider can be notified
67   * when the service is no longer used. The session instance
68   * should be created once the platform has started finding
69   * the service (e.g. during tuning or after sending an IGMP
70   * join request) in order to avoid delaying the presentation
71   * of the service. 
72   * <li>For services whose actual location is not passed
73   * in a ServiceReference (e.g. services not already present
74   * in the network), the implementation shall use the select
75   * method to obtain the actual location.
76   * </ul>
77   * The mechanism by which the actual location of a service
78   * is determined may change dynamically, e.g. as the set of
79   * services available in the network changes.
80   *
81   * @see org.dvb.spi.Provider#getServiceProviderInterfaces()
82   * @since MHP 1.1.3
83   **/

98      /**
99       * Give a list of the services provided by this provider.  
100      * The services returned from this method shall be merged into the platform
101      * service list as returned by javax.tv.service.SIManager.filterServices (null).
102      * Where the transport independent identification of a service is equal to one
103      * already in the service list then that transport independent service shall
104      * acquire an additional transport dependent service.
105      * Where the transport independent identification of a service is not equal to
106      * one already in the service list, a new service shall be added to the service
107      * list and SIChangeEvents generated to appropriate listeners.
108      * <p>The list of services returned shall include both those
109      * where the actual location is returned in the ServiceReference
110      * and those where the actual location is to be returned from
111      * a later call to the select method.
112      * @return a list of ServiceReference instances.
113      *
114      * @see  org.dvb.spi.selection.SelectionProviderContext#serviceListChanged(ServiceReference[])
