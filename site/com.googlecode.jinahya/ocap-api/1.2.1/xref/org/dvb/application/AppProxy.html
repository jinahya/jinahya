5   /**
6    * An <code>AppProxy</code> Object is a proxy to an application.
7    * A call to the start, stop or pause will cause the resident Application 
8    * Manager to respectively start, stop or pause the application bound to 
9    * this AppProxy object. Each of these three method calls can throw a Security
10   * Exception if the calling application is not entitled to do so.
11   * <p>Each of these method calls are asynchronous and will result in exactly one
12   * AppStateChangedEvent to be generated whether the method call was successful
13   * or not. If the method call was not successful, any call to the <code>hasFailed</code>
14   * method of the corresponding <code>AppStateChangeEvent</code> will return true.<p>
15   * Some of the methods here allow the AppProxy to transition through several states
16   * before the final state is reached. If this compound state transition is unsuccessful 
17   * at any point, the resulting AppStateChangedEvent shall have a fromstate which is 
18   * the last state in this transition which the AppProxy successfully entered and a 
19   * toState which would have been the next state in the compound state transition. 
20   * <p>For instance, if an application were to call start on an AppProxy  for a DVB-J 
21   * application in the NOT_LOADED state and that DVB-J application was to throw a 
22   * XletStateChangeException from its startXlet method, the getFromState will return 
23   * PAUSED and getToState will return STARTED. If an application were to call start on 
24   * an AppProxy  for a DVB-J application in the NOT_LOADED state and that DVB-J 
25   * application was to throw a XletStateChangeException from its initXlet method, the 
26   * getFromState will return NOT_LOADED and getToState will return PAUSED.
27   * Calling the start method for an application which is already running shall fail 
28   * and generate an AppStateChangeEvent with hasFailed returning true and both 
29   * fromstate and tostate being STARTED.Â”<p>
30   * See the definition of <code>AppStateChangeEvent</code> for more information.
31   * @see AppStateChangeEvent
32   */
