5   /**
6    * The <code>AppsDatabase</code> is an abstract view of the currently 
7    * available applications. The entries will be provided by the 
8    * application manager, and gleaned from the AIT signaling.
9    * When the service context in which an application is running undergoes
10   * service selection, instances of <code>AppsDatabase</code> used by that application 
11   * shall be updated from the new service before an <code>AppsDatabaseEvent</code> 
12   * is sent to the <code>newDatabase</code> method of any registered 
13   * <code>AppsDatabaseEventListeners</code>.
14   * For applications fully signalled in the current service (i.e. excluding 
15   * externally authorised ones), the attributes entries shall be the ones from 
16   * the signalling of the current service even if the application was originally 
17   * launched from another service and then survived service selection. For running 
18   * externally authorised applications, the entries will be those from the last service 
19   * in which they ran fully signalled.<p>
20   * Externally authorized applications shall not appear unless
21   * an instance of that application is actually running.
22   * <p> A generic launcher may be written which uses the database to display 
23   * information in AppAttributes and uses an AppProxy to launch it
24   * <p> Methods on classes in this package do not block, they return the 
25   * information the system currently has. Therefore applications should be 
26   * aware that data may be stale, to within one refresh period of the AIT.<p>
27   * e.g.:
28   * <pre>	
29   * <code>
30   *   AppsDatabase theDatabase = AppsDatabase.getDatabase();
31   *   if (theDatabase != null ) {
32   *       Enumeration attributes = theDatabase.getAppAttributes();
33   *       if(attributes != null) {
34   *          while(attributes.hasMoreElements()) {
35   *              AppAttributes info ;
36   *              AppProxy proxy ;
37   *    
38   *              info = (AppAttributes)attributes.nextElement();
39   *              proxy = (AppProxy)theDatabase.getAppProxy(info.getIdentifier());
40   *              URL icon = info.getIcon();
41   *              // blah blah..
42   *              // lets start it.
43   *              proxy.start(false, null);
44   *          }
45   *       }
46   *   }
47   * </code>
48   * </pre>
49   * Where methods on this class as specified as working on "available" applications or
50   * "currently available" applications the following definition shall apply. 
51   * An application is "currently available" if and only if one of the following
52   * applies in the service context within which the application calling the method
53   * is executing and the visibility of the application is not '00'..<ul>
54   * <li>it is signalled as being present or autostart in the currently selected
55   * service of that service context and could be started.
56  
57   * <li>it is currently running in that service context.
58   * </ul>
59   * In addition to the methods listed below, all calls made using an 
60   * AppsDatabaseFilter shall only use that filter to test "currently available"
61   * applications as defined here.<p>
62   * Applications whose information (e.g. signaling) is invalid
63   * (e.g. one or more mandatory descriptors are missing or incorrect) may not be listed in the AppsDatabase.
64   * Where applications are signalled in a broadcast AIT and the MHP terminal
65   * tunes away from the service on which the AIT is carried, but
66   * without selecting a new service, the <code>AppsDatabase</code> shall retain
67   * the entries as signalled in that AIT until a new service is selected.
68   */
