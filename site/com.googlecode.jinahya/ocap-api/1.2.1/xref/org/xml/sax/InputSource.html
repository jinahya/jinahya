36  /**
37   * A single input source for an XML entity.
38   *
39   * <p>This class allows a SAX application to encapsulate information
40   * about an input source in a single object, which may include
41   * a public identifier, a system identifier, a byte stream (possibly
42   * with a specified encoding), and/or a character stream.</p>
43   *
44   * <p>There are two places that the application can deliver an
45   * input source to the parser: as the argument to the XMLReader.parse
46   * method, or as the return value of the EntityResolver.resolveEntity
47   * method.</p>
48   *
49   * <p>The SAX parser will use the InputSource object to determine how
50   * to read XML input.  If there is a character stream available, the
51   * parser will read that stream directly, disregarding any text
52   * encoding declaration found in that stream.
53   * If there is no character stream, but there is
54   * a byte stream, the parser will use that byte stream, using the
55   * encoding specified in the InputSource or else (if no encoding is
56   * specified) auto-detecting the character encoding using an algorithm
57   * such as the one in the XML specification.  If neither a character
58   * stream nor a
59   * byte stream is available, the parser will attempt to open a URI
60   * connection to the resource identified by the system
61   * identifier.</p>
62   *
63   * <p>An InputSource object belongs to the application: the SAX parser
64   * shall never modify it in any way (it may modify a copy if 
65   * necessary).  However, standard processing of both byte and
66   * character streams is to close them on as part of end-of-parse cleanup,
67   * so applications should not attempt to re-use such streams after they
68   * have been handed to a parser.  </p>
69   *
70   * @since SAX 1.0
71   * @author David Megginson
72   * @see org.xml.sax.XMLReader#parse(org.xml.sax.InputSource)
73   * @see org.xml.sax.EntityResolver#resolveEntity
74   * @see java.io.InputStream
75   * @see java.io.Reader
76   */
