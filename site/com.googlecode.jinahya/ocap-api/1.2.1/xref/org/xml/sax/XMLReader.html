35  /**
36   * Interface for reading an XML document using callbacks.
37   *
38   * <p><strong>Note:</strong> despite its name, this interface does 
39   * <em>not</em> extend the standard Java {@link java.io.Reader Reader} 
40   * interface, because reading XML is a fundamentally different activity 
41   * than reading character data.</p>
42   *
43   * <p>XMLReader is the interface that an XML parser's SAX2 driver must
44   * implement.  This interface allows an application to set and
45   * query features and properties in the parser, to register
46   * event handlers for document processing, and to initiate
47   * a document parse.</p>
48   *
49   * <p>All SAX interfaces are assumed to be synchronous: the
50   * {@link #parse parse} methods must not return until parsing
51   * is complete, and readers must wait for an event-handler callback
52   * to return before reporting the next event.</p>
53   *
54   * <p>This interface replaces the (now deprecated) SAX 1.0 
55   * interface.  The XMLReader interface
56   * contains two important enhancements over the old Parser
57   * interface (as well as some minor ones):</p>
58   *
59   * <ol>
60   * <li>it adds a standard way to query and set features and 
61   *  properties; and</li>
62   * <li>it adds Namespace support, which is required for many
63   *  higher-level XML standards.</li>
64   * </ol>
65   *
66   * @since SAX 2.0
67   * @author David Megginson
68   * @see org.xml.sax.XMLFilter
69   */
70  public interface XMLReader

78      /**
79       * Look up the value of a feature flag.
80       *
81       * <p>The feature name is any fully-qualified URI.  It is
82       * possible for an XMLReader to recognize a feature name but
83       * temporarily be unable to return its value.
84       * Some feature values may be available only in specific
85       * contexts, such as before, during, or after a parse.
86       * Also, some feature values may not be programmatically accessible.</p>
87       *
88       * <p>All XMLReaders are required to recognize the
89       * http://xml.org/sax/features/namespaces and the
90       * http://xml.org/sax/features/namespace-prefixes feature names.</p>
91       *
92       * <p>Typical usage is something like this:</p>
93       *
94       * <pre>
95       * XMLReader r = new MySAXDriver();
96       *
97       *                         // try to activate validation
98       * try {
99       *   r.setFeature("http://xml.org/sax/features/validation", true);
100      * } catch (SAXException e) {
101      *   System.err.println("Cannot activate validation."); 
102      * }
103      *
104      *                         // register event handlers
105      * r.setContentHandler(new MyContentHandler());
106      * r.setErrorHandler(new MyErrorHandler());
107      *
108      *                         // parse the first document
109      * try {
110      *   r.parse("http://www.foo.com/mydoc.xml");
111      * } catch (IOException e) {
112      *   System.err.println("I/O exception reading XML document");
113      * } catch (SAXException e) {
114      *   System.err.println("XML exception reading document.");
115      * }
116      * </pre>
117      *
118      * <p>Implementors are free (and encouraged) to invent their own features,
119      * using names built on their own URIs.</p>
120      *
121      * @param name The feature name, which is a fully-qualified URI and
122      *            which may not be null.
123      * @return The current value of the feature (<code>true</code> or <code>false</code>).
124      * @exception org.xml.sax.SAXNotRecognizedException If the feature
125      *            value can't be assigned or retrieved.
126      * @exception org.xml.sax.SAXNotSupportedException When the
127      *            XMLReader recognizes the feature name but 
128      *            cannot determine its value at this time.
129      * @throws java.lang.NullPointerException
130      * @see #setFeature
131      */
132     public boolean getFeature (String name)
133         throws SAXNotRecognizedException, SAXNotSupportedException;
134

136     /**
137      * Set the value of a feature flag.
138      *
139      * <p>The feature name is any fully-qualified URI.  It is
140      * possible for an XMLReader to expose a feature value but
141      * to be unable to change the current value.
142      * Some feature values may be immutable or mutable only 
143      * in specific contexts, such as before, during, or after 
144      * a parse.</p>
145      *
146      * <p>All XMLReaders are required to support setting
147      * http://xml.org/sax/features/namespaces to <code>true</code> and
148      * http://xml.org/sax/features/namespace-prefixes to <code>false</code>.</p>
149      *
150      * @param name The feature name, which is a fully-qualified URI and 
151      *            which may not be null.
152      * @param value The requested value of the feature (<code>true</code> or <code>false</code>).
153      * @exception org.xml.sax.SAXNotRecognizedException If the feature
154      *            value can't be assigned or retrieved.
155      * @exception org.xml.sax.SAXNotSupportedException When the
156      *            XMLReader recognizes the feature name but 
157      *            cannot set the requested value.
158      * @throws java.lang.NullPointerException
159      * @see #getFeature
160      */
161     public void setFeature (String name, boolean value)
162         throws SAXNotRecognizedException, SAXNotSupportedException;

340     /**
341      * Parse an XML document.
342      *
343      * <p>The application can use this method to instruct the XML
344      * reader to begin parsing an XML document from any valid input
345      * source (a character stream, a byte stream, or a URI).
346      * </p>
347      *
348      * <p>Applications may not invoke this method while a parse is in
349      * progress (they should create a new XMLReader instead for each
350      * nested XML document).  Once a parse is complete, an
351      * application may reuse the same XMLReader object, possibly with a
352      * different input source.
353      * Configuration of the XMLReader object (such as handler bindings and
354      * values established for feature flags and properties) is unchanged
355      * by completion of a parse, unless the definition of that aspect of
356      * the configuration explicitly specifies other behavior.
357      * (For example, feature flags or properties exposing
358      * characteristics of the document being parsed.)
359      * </p>
360      *
361      * <p>During the parse, the XMLReader will provide information
362      * about the XML document through the registered event
363      * handlers.</p>
364      *
365      * <p>This method is synchronous: it will not return until parsing
366      * has ended.  If a client application wants to terminate 
367      * parsing early, it should throw an exception.</p>
368      *
369      * @param input The input source for the top-level of the
370      *        XML document, may not be null.
371      * @exception org.xml.sax.SAXException Any SAX exception, possibly
372      *            wrapping another exception.
373      * @exception java.io.IOException An IO exception from the parser,
374      *            possibly from a byte stream or character stream
375      *            supplied by the application.
376      * @see org.xml.sax.InputSource
377      * @see #parse(java.lang.String)
378      * @see #setEntityResolver
379      * @see #setDTDHandler
380      * @see #setContentHandler
381      * @see #setErrorHandler 
382      */
383     public void parse (InputSource input)
384         throws IOException, SAXException;
385

387     /**
388      * Parse an XML document from a system identifier (URI).
389      *
390      * <p>This method is a shortcut for the common case of reading a
391      * document from a system identifier.  It is the exact
392      * equivalent of the following:</p>
393      *
394      * <pre>
395      * parse(new InputSource(systemId));
396      * </pre>
397      *
398      * <p>If the system identifier is a URL, it must be fully resolved
399      * by the application before it is passed to the parser.</p>
400      *
401      * @param systemId The system identifier (URI); may not be null.
402      * @exception org.xml.sax.SAXException Any SAX exception, possibly
403      *            wrapping another exception.
404      * @exception java.io.IOException An IO exception from the parser,
405      *            possibly from a byte stream or character stream
406      *            supplied by the application.
407      * @see #parse(org.xml.sax.InputSource)
408      */
409     public void parse (String systemId)
410         throws IOException, SAXException;
