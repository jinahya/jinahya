31  package org.xml.sax;

33  /**
34   * Receive notification of the logical content of a document.
35   *
36   * <p>This is the main interface that most SAX applications
37   * implement: if the application needs to be informed of basic parsing 
38   * events, it implements this interface and registers an instance with 
39   * the SAX parser using the {@link org.xml.sax.XMLReader#setContentHandler 
40   * setContentHandler} method.  The parser uses the instance to report 
41   * basic document-related events like the start and end of elements 
42   * and character data.</p>
43   *
44   * <p>The order of events in this interface is very important, and
45   * mirrors the order of information in the document itself.  For
46   * example, all of an element's content (character data, processing
47   * instructions, and/or sub-elements) will appear, in order, between
48   * the startElement event and the corresponding endElement event.</p>
49   *
50   * <p>This interface is similar to the now-deprecated SAX 1.0
51   * DocumentHandler interface, but it adds support for Namespaces
52   * and for reporting skipped entities (in non-validating XML
53   * processors).</p>
54   *
55   * <p>Implementors should note that on CDC there is also a 
56   * <code>ContentHandler</code> class in the <code>java.net</code>
57   * package; that means that it's probably a bad idea to do</p>
58   *
59   * <pre>import java.net.*;
60   * import org.xml.sax.*;
61   * </pre>
62   *
63   * <p>In fact, "import ...*" is usually a sign of sloppy programming
64   * anyway, so the user should consider this a feature rather than a
65   * bug.</p>
66   *
67   * @since SAX 2.0
68   * @author David Megginson
69   * @see org.xml.sax.XMLReader
70   * @see org.xml.sax.DTDHandler
71   * @see org.xml.sax.ErrorHandler
72   */
73  public interface ContentHandler
74  {

76      /**
77       * Receive an object for locating the origin of SAX document events.
78       *
79       * <p>SAX parsers are strongly encouraged (though not absolutely
80       * required) to supply a locator: if it does so, it must supply
81       * the locator to the application by invoking this method before
82       * invoking any of the other methods in the ContentHandler
83       * interface.</p>
84       *
85       * <p>The locator allows the application to determine the end
86       * position of any document-related event, even if the parser is
87       * not reporting an error.  Typically, the application will
88       * use this information for reporting its own errors (such as
89       * character content that does not match an application's
90       * business rules).  The information returned by the locator
91       * is probably not sufficient for use with a search engine.</p>
92       *
93       * <p>Note that the locator will return correct information only
94       * during the invocation SAX event callbacks after
95       * {@link #startDocument startDocument} returns and before
96       * {@link #endDocument endDocument} is called.  The
97       * application should not attempt to use it at any other time.</p>
98       *
99       * @param locator An object that can return the location of
100      *                any SAX document event
101      * @see org.xml.sax.Locator
102      */
103     public void setDocumentLocator (Locator locator);

112      *

121     /**
122      * Receive notification of the end of a document.
123      *
124      * <p><strong>There is an apparent contradiction between the
125      * documentation for this method and the documentation for {@link
126      * org.xml.sax.ErrorHandler#fatalError}.  Until this ambiguity is
127      * resolved in a future major release, clients should make no

133      * be the last method invoked during the parse.  The parser shall
134      * not invoke this method until it has either abandoned parsing
135      * (because of an unrecoverable error) or reached the end of
136      * input.</p>
137      *
138      * @throws org.xml.sax.SAXException any SAX exception, possibly
139      *            wrapping another exception
140      * @see #startDocument
141      */
142     public void endDocument()
143         throws SAXException;

146     /**
147      * Begin the scope of a prefix-URI Namespace mapping.
148      *
149      * <p>The information from this event is not necessary for
150      * normal Namespace processing: the SAX XML reader will 
151      * automatically replace prefixes for element and attribute
152      * names when the <code>http://xml.org/sax/features/namespaces</code>
153      * feature is <code>true</code> (the default).</p>
154      *
155      * <p>There are cases, however, when applications need to
156      * use prefixes in character data or in attribute values,
157      * where they cannot safely be expanded automatically; the
158      * start/endPrefixMapping event supplies the information
159      * to the application to expand prefixes in those contexts
160      * itself, if necessary.</p>
161      *
162      * <p>Note that start/endPrefixMapping events are not
163      * guaranteed to be properly nested relative to each other:
164      * all startPrefixMapping events will occur immediately before the
165      * corresponding {@link #startElement startElement} event, 
166      * and all {@link #endPrefixMapping endPrefixMapping}
167      * events will occur immediately after the corresponding
168      * {@link #endElement endElement} event,
169      * but their order is not otherwise 
170      * guaranteed.</p>
171      *
172      * <p>There should never be start/endPrefixMapping events for the
173      * "xml" prefix, since it is pre-declared and immutable.</p>
174      *
175      * @param prefix The Namespace prefix being declared.
176      *            An empty string is used for the default element namespace,
177      *            which has no prefix.
178      * @param uri the Namespace URI the prefix is mapped to
179      * @throws org.xml.sax.SAXException the client may throw
180      *            an exception during processing
181      * @see #endPrefixMapping
182      * @see #startElement
183      */
184     public void startPrefixMapping (String prefix, String uri)
185         throws SAXException;
186 
187 
188     /**
189      * End the scope of a prefix-URI mapping.
190      *
191      * <p>See {@link #startPrefixMapping startPrefixMapping} for 
192      * details.  These events will always occur immediately after the
193      * corresponding {@link #endElement endElement} event, but the order of 
194      * {@link #endPrefixMapping endPrefixMapping} events is not otherwise
195      * guaranteed.</p>
196      *
197      * @param prefix the prefix that was being mapped.
198      *	This is the empty string when a default mapping scope ends.
199      * @throws org.xml.sax.SAXException the client may throw
200      *            an exception during processing
201      * @see #startPrefixMapping
202      * @see #endElement
203      */
204     public void endPrefixMapping (String prefix)
205         throws SAXException;

208     /**
209      * Receive notification of the beginning of an element.
210      *
211      * <p>The parser will invoke this method at the beginning of every
212      * element in the XML document; there will be a corresponding
213      * {@link #endElement endElement} event for every startElement event
214      * (even when the element is empty). All of the element's content will be
215      * reported, in order, before the corresponding endElement
216      * event.</p>
217      *
218      * <p>This event allows up to three name components for each
219      * element:</p>
220      *
221      * <ol>
222      * <li>the Namespace URI;</li>
223      * <li>the local name; and</li>
224      * <li>the qualified (prefixed) name.</li>
225      * </ol>
226      *
227      * <p>Any or all of these may be provided, depending on the
228      * values of the <var>http://xml.org/sax/features/namespaces</var>
229      * and the <var>http://xml.org/sax/features/namespace-prefixes</var>
230      * properties:</p>
231      *
232      * <ul>
233      * <li>the Namespace URI and local name are required when 
234      * the namespaces property is <code>true</code> (the default), and are
235      * optional when the namespaces property is <code>false</code> (if one is
236      * specified, both must be);</li>
237      * <li>the qualified name is required when the namespace-prefixes property
238      * is <code>true</code>, and is optional when the namespace-prefixes property
239      * is <code>false</code> (the default).</li>
240      * </ul>
241      *
242      * <p>Note that the attribute list provided will contain only
243      * attributes with explicit values (specified or defaulted):
244      * #IMPLIED attributes will be omitted.  The attribute list
245      * will contain attributes used for Namespace declarations
246      * (xmlns* attributes) only if the
247      * <code>http://xml.org/sax/features/namespace-prefixes</code>
248      * property is <code>true</code> (it is <code>false<code> by default, and support for a 
249      * <code>true</code> value is optional).</p>
250      *
251      * <p>Like {@link #characters characters()}, attribute values may have
252      * characters that need more than one <code>char</code> value.  </p>
253      *
254      * @param uri The Namespace URI, or the empty string if the
255      *        element has no Namespace URI or if Namespace
256      *        processing is not being performed
257      * @param localName The local name (without prefix), or the
258      *        empty string if Namespace processing is not being
259      *        performed
260      * @param qName The qualified name (with prefix), or the
261      *        empty string if qualified names are not available
262      * @param atts The attributes attached to the element.  If
263      *        there are no attributes, it shall be an empty
264      *        Attributes object.  The value of this object after
265      *        startElement returns is undefined
266      * @throws org.xml.sax.SAXException any SAX exception, possibly
267      *            wrapping another exception
268      * @see #endElement
269      * @see org.xml.sax.Attributes
270      */
271     public void startElement (String uri, String localName, String qName, 
272         Attributes atts)
273         throws SAXException;
274

288      *        processing is not being performed
289      * @param localName The local name (without prefix), or the

301     /**
302      * Receive notification of character data.
303      *
304      * <p>The parser will call this method to report each chunk of
305      * character data.  SAX parsers may return all contiguous character
306      * data in a single chunk, or they may split it into several
307      * chunks; however, all of the characters in any single event
308      * must come from the same external entity so that the Locator
309      * provides useful information.</p>
310      *
311      * <p>The application must not attempt to read from the array
312      * outside of the specified range.</p>
313      *
314      * <p>Individual characters may consist of more than one Java
315      * <code>char</code> value.  There are two important cases where this
316      * happens, because characters can't be represented in just sixteen bits.
317      * In one case, characters are represented in a <em>Surrogate Pair</em>,
318      * using two special Unicode values. Such characters are in the so-called
319      * "Astral Planes", with a code point above U+FFFF.  A second case involves
320      * composite characters, such as a base character combining with one or
321      * more accent characters. </p>
322      *
323      * <p> Your code should not assume that algorithms using
324      * <code>char</code>-at-a-time idioms will be working in character
325      * units; in some cases they will split characters.  This is relevant
326      * wherever XML permits arbitrary characters, such as attribute values,
327      * processing instruction data, and comments as well as in data reported
328      * from this method.  It's also generally relevant whenever Java code
329      * manipulates internationalized text; the issue isn't unique to XML.</p>
330      *
331      * <p>Note that some parsers will report whitespace in element
332      * content using the {@link #ignorableWhitespace ignorableWhitespace}
333      * method rather than this one (validating parsers <em>must</em> 
334      * do so).</p>
335      *
336      * @param ch the characters from the XML document
337      * @param start the start position in the array
338      * @param length the number of characters to read from the array
339      * @throws org.xml.sax.SAXException any SAX exception, possibly
340      *            wrapping another exception
341      * @see #ignorableWhitespace 
342      * @see org.xml.sax.Locator
343      */
344     public void characters (char ch[], int start, int length)
345         throws SAXException;
346

348     /**
349      * Receive notification of ignorable whitespace in element content.
350      *
351      * <p>Validating parsers must use this method to report each chunk
352      * of whitespace in element content (see the W3C XML 1.0
353      * recommendation, section 2.10): non-validating parsers may also
354      * use this method if they are capable of parsing and using
355      * content models.</p>
356      *
357      * <p>SAX parsers may return all contiguous whitespace in a single
358      * chunk, or they may split it into several chunks; however, all of
359      * the characters in any single event must come from the same
360      * external entity, so that the Locator provides useful
361      * information.</p>
362      *
363      * <p>The application must not attempt to read from the array
364      * outside of the specified range.</p>
365      *
366      * @param ch the characters from the XML document
367      * @param start the start position in the array
368      * @param length the number of characters to read from the array
369      * @throws org.xml.sax.SAXException any SAX exception, possibly
370      *            wrapping another exception
371      * @see #characters
372      */
373     public void ignorableWhitespace (char ch[], int start, int length)
374         throws SAXException;

378      * Receive notification of a processing instruction.
379      *
380      * <p>The parser will invoke this method once for each processing
381      * instruction found: note that processing instructions may occur

390      * value. </p>
391      *
392      * @param target the processing instruction target
393      * @param data The processing instruction data, or null if
394      *        none was supplied.  The data does not include any
395      *        whitespace separating it from the target
396      * @throws org.xml.sax.SAXException any SAX exception, possibly
397      *            wrapping another exception
398      */
399     public void processingInstruction (String target, String data)
400         throws SAXException;

403     /**
404      * Receive notification of a skipped entity.
405      *
406      * <p>This is not called for entity references within markup constructs
407      * such as element start tags or markup declarations.  (The XML
408      * recommendation requires reporting skipped external entities.
409      * SAX also reports internal entity expansion/non-expansion, except
410      * within markup constructs.)
411      *
412      * <p>The parser will invoke this method each time the entity is
413      * skipped.  Non-validating processors may skip entities if they
414      * have not seen the declarations (because, for example, the
415      * entity was declared in an external DTD subset).  All processors
416      * may skip external entities, depending on the values of the
417      * <code>http://xml.org/sax/features/external-general-entities</code>
418      * and the
419      * <code>http://xml.org/sax/features/external-parameter-entities</code>
420      * properties.</p>
421      *
422      * @param name The name of the skipped entity.  If it is a 
423      *        parameter entity, the name will begin with '%', and if
424      *        it is the external DTD subset, it will be the string
425      *        "[dtd]"
426      * @throws org.xml.sax.SAXException any SAX exception, possibly
427      *            wrapping another exception
428      */
429     public void skippedEntity (String name)
430         throws SAXException;
