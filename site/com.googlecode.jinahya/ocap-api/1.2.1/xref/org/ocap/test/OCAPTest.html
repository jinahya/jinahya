11  /**
12   * The purpose of this class is to provide a very simple communication 
13   * channel between the OCAP implementation under test (IUT) and the test server.
14   * The functionality of this class is intentionally limited to 
15   * sending/receiving raw messages via TCP/IP or UDP/IP protocol. 
16   * This approach decouples the lower 
17   * communications channel from the higher messaging protocol used to exchange messages 
18   * between the test client and the test server.  As a result, the implementation will 
19   * be isolated from any future messaging protocol changes which may be required 
20   * for future testing needs.
21   * <br>
22   * This class does not provide a means for concurrently executing test applications
23   * to negotiate reservation of the interaction channel used to exchange message with 
24   * the test environment host.  Test application authors should ensure that 
25   * concurrently executing test applications correctly interoperate in their use of
26   * this channel.
27   * <br>
28   *
29   * The implementation of this class MUST use TCP/IP or UDP/IP protocol to establish a 
30   * socket connection to the test server via the RF connector labeled 'Cable In'
31   * on the OCAP Host Device. See the {@link OCAPTest#getProtocol} method. 
32   *
33   * @author Piotr Czapla
34   * @author Shigeaki Watanabe
35   */
