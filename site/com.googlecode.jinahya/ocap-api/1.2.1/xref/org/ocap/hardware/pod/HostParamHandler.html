5   /**
6    * <p>
7    * A class that implements this interface can reject the update of the 
8    * Feature parameter in the Host device. Feature parameter is defined for 
9    * the Generic Feature Control Support in the OpenCable CableCARD Interface 
10   * specification. 
11   * An OCAP-J application can set only one instance of such classes to the 
12   * OCAP implementation via the 
13   * {@link org.ocap.hardware.pod.POD#setHostParamHandler} method. 
14   * </p><p>
15   * Before Feature parameter in the Host is modified, the
16   * {@link HostParamHandler#notifyUpdate} method shall be called with the 
17   * Feature ID to be modified and its Feature parameter value. And only if 
18   * the HostParamHandler.notifyUpdate() method returns true, the Feature 
19   * parameter value in the Host device will be modified. 
20   * Note that the Host device may reject the update of Feature parameter 
21   * even if the HostParamHandler.notifyUpdate() method returns true. 
22   * </p><p>
23   * The Feature ID and the Feature parameter value format are defined in 
24   * the CableCARD Interface 2.0 Specification [4]. 
25   * For example, the Feature ID of "RF Output Channel" Feature is 0x1, and 
26   * its parameter value format is <br>
27   * <code>
28   * Rf_output_channel() {<br>
29   * &nbsp;&nbsp;&nbsp;&nbsp;Output_channel<br>
30   * &nbsp;&nbsp;&nbsp;&nbsp;Output_channel_ui<br>
31   * }<br>
32   * </code>
33   * </p><p>
34   * The Feature parameters in the Host device will be modified by the 
35   * following cases. 
36   * <ul>
37   * <li> The CableCARD sends feature_parameters APDU to the Host.  
38   *      (See the [CCIF 2.0].)
39   * <li> The Host modifies its own Feature parameters. 
40   * <li> An OCAP-J application calls the 
41   *      {@link org.ocap.hardware.pod.POD#updateHostParam} method.
42   * </ul>
43   * In every cases, the HostParamHandler.notifyUpdate() method shall be called. 
44   * </p>
45   *
46   */
47  public interface HostParamHandler
48  {
49      /**
50       * <p>
51       * This is a call back method to notify an update of the Feature 
52       * parameter in the Host device. This method shall be called every time 
53       * before the Feature parameter is modified. Only if this method returns 
54       * true, the Host device can modify its Feature parameter by the specified 
55       * value. 
56       * </p><p>
57       * Note that the Host device may reject the update of Feature parameter 
58       * even if the HostParamHandler.notifyUpdate() method returns true. 
59       * </p><p>
60       * This method should return immediately without blocking. 
61       * </p>
62       *
63       * @param featureID  a Feature ID for the Generic Feature Control Support 
64       *                in the CableCARD Interface 2.0 Specification [4]. 
65       *                The Feature ID reserved for proprietary use (0x70 - 0xFF) 
66       *                can be specified. 
67       *
68       * @param value  a Feature parameter value for the specified featureID. 
69       *                An actual format of each Feature parameter is defined 
70       *                in the CableCARD Interface 2.0 Specification [4]. 
71       *                For example, if the featureID is 0x1, the value is <br>
72       *                <code>
73       *                Rf_output_channel() {<br>
74       *                &nbsp;&nbsp;&nbsp;&nbsp;Output_channel<br>
75       *                &nbsp;&nbsp;&nbsp;&nbsp;Output_channel_ui<br>
76       *                }
77       *                </code>
78       *
79       * @return <code>true</code> to accept the modification of the specified 
80       *                value. <code>false</code> to reject it.
81       *
82       * @see org.ocap.hardware.pod.POD#setHostParamHandler
83       **/
84      public boolean notifyUpdate(int featureID, byte[] value);
85  }
