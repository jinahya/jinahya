6   /**
7    * A class implementing this interface decides which application shall be 
8    * allowed to reserve a resource. 
9    * <P>
10   * An application which has a MonitorAppPermission("handler.resource") 
11   * may have a class implementing this interface, and may set an instance of 
12   * it in the ResourceContentionManager. 
13   * The {@link ResourceContentionHandler#resolveResourceContention} method decides 
14   * the how to resolve resource conflicts between the new request and existing resource
15   * allocations. 
16   * See the {@link ResourceContentionManager} for the details. 
17   */
18  public interface ResourceContentionHandler
19  {
20  
21      /**
22       * This method notifies the ResourceContentionHandler that one to many
23       * resource contentions have occurred between one or more applications
24       * and system modules, except the Emergency Alert System (EAS) module.
25       * EAS system module resource requests SHALL be given the highest
26       * priority by the implementation and resource requests by this module
27       * SHALL not be reported to the ResourceContentionHandler.
28       * In the case of one application, the same application is conflicting
29       * with itself and a registered ResourceContentionHandler SHALL be
30       * notified in this case.
31       * <p>
32       * This method notifies the ResourceContentionHandler that one to many
33       * resource contentions have occurred between two or more applications.
34       * Each entry in the <i>currentReservations</i> indicates a set of resources 
35       * reserved by an application for a single activity such as a resource usage 
36       * by a single service context.  There may be multiple 
37       * entries in this list from a single application.  An entry may correspond 
38       * to a current resource usage or resource reservations for a future activity.
39       * <p>
40       * A prioritized array of {@link ResourceUsage} instances is returned.  
41       * The array is in priority order from highest to lowest indicating the 
42       * priority order to be followed by the implementation while resolving the 
43       * conflicts.  When this method returns the implementation will iterate 
44       * through each entry in the array in the order of priority, awarding resources 
45       * as required by the activity represented by the resourceUsage. The 
46       * ResourceContentionHandler may use information such as Application Priority
47       * to prioritize the array of ResourceUsages returned. When the value
48       * returned is not null the ResourceContentionHandler MAY return an array
49       * containing all of the <code>ResourceUsage</code> objects passed to it,
50       * or it MAY return a subset of those objects.
51       * </p>
52       *
53       * @param   newRequest The resource usage object containing the attributes of
54       *     the resource request(s).
55       *
56       * @param   currentReservations The set of resource usage objects that describe
57       *     current resource reservations which are in conflict with the newRequest.  
58       *     A <code>ResourceUsage</code> associated
59       *     with a current reservation MAY belong to an application that has been
60       *     destroyed.  Use of the <code>AppID</code> contained within such a 
61       *     <code>ResourceUsage</code>
62       *     with any of the methods in <code>org.dvb.application.AppsDatabase</code>
63       *     MAY cause a failure status to be returned.
64       *
65       * @return  A prioritized array of resource usage objects. The first entry has the highest 
66       *     priority. This function returns null if the contention handler wants the 
67       *     implementation to resolve the conflict.
68       */
69      public ResourceUsage [] resolveResourceContention(
70              ResourceUsage newRequest,
71              ResourceUsage [] currentReservations);
72  
73      /**
74       * Warns the resource contention handler of an impending contention with
75       * a presenting ServiceContext (e.g., scheduled recording as defined by
76       * the OCAP DVR specification).  If a ResourceContentionHandler is
77       * registered the implementation SHALL call this method as defined by
78       * the {@link ResourceContentionManager#setWarningPeriod} method.
79       *
80       * @param	newRequest The resource usage object containing the attributes of
81       *     the resource[s] request.
82       *
83       * @param	currentReservations The resource usage objects currently owned by applications
84       *     which are in conflict with the newRequest.  A ResourceUsage associated
85       *     with a current reservation may belong to an application that has been
86       *     destroyed.  Use of the AppID contained within such a ResourceUsage
87       *     with any of the methods in <code>org.dvb.application.AppsDatabase</code>
88       *     MAY cause a failure status to be returned.
89       */
90      public void resourceContentionWarning(ResourceUsage newRequest,
91                                            ResourceUsage [] currentReservations);
92  }
