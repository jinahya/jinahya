47      /**
48       * Registers a MIB object by adding the OID and listener to the MIB tables.  
49       * OID must be unique and not pre-existing.
50       * OID must include a trailing ".0" for leaf items or but not include the instance ID
51       * for table column items.  (The same listener may be installed to will be invoked
52       * for all row instances for table column items. any number of OIDs, if desired.)
53       * Table OIDs may be the OID of the start of the table or a column 
54       * within the table, depending upon whether the implementation desires
55       * handling all table values with a single listener, or have different
56       * listeners for each column.  
57       * 
58       * @param oid The Object Identifier of the MIB being registered.  The
59       *      format of the string is based on the format defined by RFC 2578 for
60       *      OBJECT IDENTIFIER definition.  Terms in the string are period
61       *      delimited, e.g. "1.3.6.1.4.1".
62       * @param access Indicates allowed access to the MIB being registered.  See
63       *      access constants in the class for valid values.
64       * @param leaf When true indicates if the object is a leaf, otherwise the
65       *      object is a non-leaf table column.
66       * @param dataType The data type of the MIB being registered.  See constants
67       *      in {@link MIBDefinition} for valid values.
68       * @param listener Listener to the MIB being registered.  Must not be null.
69       * 
70       * @throws IllegalArgumentException if oid is an invalid oid string, 
71       *      if oid is already installed, or any other parameter has an 
72       *      invalid value.
73       */

93      /**
94       * Makes a query for all MIB objects matching the oid parameter, as
95       * well as any descendants in the MIB tree.  If the object to be searched
96       * for is a leaf the trailing ".0" must be included for an exact match.
97       * A query for a leaf object SHALL return just that object if found.  A
98       * query for a non-leaf OID SHALL return all MIB objects below that OID.
99       * Existing leaf and table items SHALL be included in the results;
100      * branch-nodes without data SHALL NOT.  For example; If a query is for
101      * OID 1.2.3.4 then all table items and leafs below that OID are returned.
102      * If OIDs 1.2.3.4.1 and 1.2.3.4.2 are the only items below the query object
103      * they would be returned.  The query SHALL NOT return items outside the
104      * OID.  For example; if 1.2.3.4 is the query OID then 1.2.3.5 is not
105      * returned.</p><p>
106      * 
107      * If the CableCARD supports MIB APDUs then this method SHALL be able to
108      * query the CableCARD MIB objects.</p>
109      *
110      * @param oid The object identifier to search for.  The format of the
111      *      string is based on the format defined by RFC 2578 for
112      *      OBJECT IDENTIFIER definition.  Terms in the string
113      *      are period delimited, e.g. "1.3.6.1.4.1".
114      * 
115      * @return An array of MIB definitions.  The array is lexographically ordered
116      *      by increasing value of OID with the lowest value in the first element
117      *      of the array.
118      */ 
119     public abstract MIBDefinition [] queryMibs(String oid);

135      *
136      * If ECM_SUBDEVICE is the subDevice argument value the implementation SHALL

150      *      by increasing value of OID with the lowest value in the first element
151      *      of the array.
152      *
153      * @throws IllegalArgumentException if the subDevice is not one of the
154      *      ECM_SUBDEVICE or ESTB_SUBDEVICE.
155      */
