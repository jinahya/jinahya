3   /**
4    * Represents an environment that provides the context in which applications run.
5    * <h4>Environment state machine</h4>
6    * Environments SHALL be in one of four states; inactive, selected, presenting and 
7    * background. These are defined as follows;
8    * <ul>
9    * <li>Environments in the inactive state SHALL have no running applications at all.
10   * <li>Environments in the selected state have all applications running to the maximum
11   * extent possible and able to interact with the end-user.
12   * <li>Environments in the presenting state may have running applications which are
13   * visible to the end-user
14   * but these SHALL NOT be able to receive input from the remote control. 
15   * <li>Environments in the background state may have running applications but these 
16   * applications SHALL NOT be in the normal mode.
17   * </ul>
18   * <h4>Transitions from selected or presenting to background</h4>
19   * When an environment changes state from either selected or presenting to 
20   * background, the following SHALL apply:<ul>
21   * <li>applications able to run in cross-environment mode SHALL be
22   * put in cross-environment mode by the implementation
23   * <li>applications able to run in background mode SHALL be
24   * put in background mode by the implementation
25   * <li>applications signaled as pauseable MAY be put in the paused state
26   * by the implementation
27   * <li>all other applications SHALL be terminated
28   * <li>The implementation SHALL hide the user interfaces of applications which 
29   * are put in background or paused mode or which are terminated.
30   * HScene instances shall have their visibility set to false.
31   * <p>NOTE: Applicable events announcing such changes are generated as expected.
32   * For example, org.dvb.application.AppStateChangeEvents announcing that applications
33   * have been paused or terminated; and java.awt.event.ComponentEvents announcing that
34   * an HScene has been hidden.
35   * </ul>
36   * <h4>Transitions from background to selected or presenting</h4>
37   * When an environment changes state from background to either selected or 
38   * presenting, the following SHALL apply:<ul>
39   * <li>all auto-start unbound applications which were terminated due to their
40   * environment going into the background state SHALL be started 
41   * if their service is still selected
42   * <li>all auto-start bound applications which were terminated due to their
43   * environment going into the background state SHALL be started if still signaled in a
44   * selected service
45   * <li>all applications from the newly selected environment that are running in 
46   * cross-environment or background mode SHALL be returned to normal mode and 
47   * restrictions on them as a consequence of them running in those modes SHALL be lifted
48   * <li>visible user interfaces of cross-environment applications whose
49   * environment becomes selected SHALL continue to remain visible
50   * <li>the policy of the newly selected environment is 
51   * responsible for determining which of the applications in that environment
52   * should be the first to have focus. 
53   * <li>Any pauseable applications which were paused when this environment
54   * went into the background state and which are still paused shall be returned 
55   * to the active state
56   * </ul>
57   */
