6   /**
7    * The Program Association Table (PAT) manager is used to discover and 
8    * listen for PATs. 
9    * To retrieve the PAT, an application add the TableChangeListener to 
10   * ProgramAssociationTableManager via the addInBandChangeListener() or 
11   * the addOutOfBandChangeListener(), and call the retrieveInBand() or the 
12   * retrieveOutOfBand(). If PAT has changed, ProgramAssociationTableManager 
13   * calls the TableChangeListener.notifyChange() to notify it. 
14   * The application must get the ProgramAssociationTable object via the 
15   * SIChangeEvent.getSIElement() method to keep the PAT table fresh when the 
16   * PAT change is notified, i.e., ProgramAssociationTable is not updated 
17   * automatically.
18   */
19  public abstract class ProgramAssociationTableManager

45       * the inband PAT for the channel (transport stream) identified
46       * by the Locator parameter changes.

49       * <tt>javax.tv.service.SIRequestFailureType.DATA_UNAVAILABLE</tt>

97      /** 
98       * <P>
99       * Retrieve a PAT from the in-band channel (transport stream)
100      * identified by the Locator parameter.
101      * <P></P>
102      * The OCAP implementation does not automatically tune to the
103      * transport stream specified by the Locator.
104      * Hence, the calling application must tune to the corresponding
105      * transport stream before calling this method.
106      * <P></P>
107      * The attempt to retrieve a PAT stops silently and permanently
108      * when the network interface starts tuning to another transport
109      * stream.  In this case, the registered
110      * <tt>SIRequestor.notifyFailure()</tt>
111      * method is invoked with a failure type of
112      * <tt>javax.tv.service.SIRequestFailureType.DATA_UNAVAILABLE</tt>.
113      * </P><P>
114      * It is not guaranteed that the transport stream specified 
115      * by the Locator is still tuned when the method of the SIRequestor 
116      * is called back.
117      * <P></P>
118      * Note:  If an application has added a listener via the
119      * <tt>addInBandChangeListener()</tt> method,
120      * the <tt>TableChangeListener.notifyChange()</tt> method
121      * is called when the specified PAT is updated.  In this case,
122      * the PAT returned to the <tt>SIRequestor</tt> registered with
123      * this method may have expired.
124      * <tt>javax.tv.service.SIRequestFailureType.DATA_UNAVAILABLE</tt>
125      * is returned via <tt>SIRequestor.notifyFailure()</tt>
126      * if the locator refers to an analog service.
127      * 
128      *
129      * @param requestor  The SIRequestor object to be called back with 
130      *                   the retrieval result.
131      *
132      * @param locator    A locator to specify the channels (transport 
133      *                   streams) carrying the PATs.   Should correspond
134      *                  to one of the following OCAP URL forms:
135      *                  ocap://source_id, ocap://f=frequency.program_number
136      *
137      * @return           The SIRequest object that identifies this 
138      *                   asynchronous retrieval request and allows the 
139      *                   request to be cancelled.
140      */
141     public abstract javax.tv.service.SIRequest retrieveInBand(
142                             javax.tv.service.SIRequestor requestor, 
143                             javax.tv.locator.Locator locator);
