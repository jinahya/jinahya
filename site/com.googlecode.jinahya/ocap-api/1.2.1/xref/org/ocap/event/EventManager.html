7   /**
8    * <p>
9    * The event manager allows an application to receive events coming from 
10   * the user. These events can be sent exclusively to an application or can 
11   * be shared between applications. The EventManager allows an application 
12   * to ask for exclusive access to some events, these events being received 
13   * either from the standard java.awt event mechanism or by the mechanism 
14   * defined in this package. The EventManager is a singleton, and the 
15   * instance is gotten from the getInstance() method. (Note that a type 
16   * cast is necessary to gain reference to object of type 
17   * org.ocap.event.EventManager.) 
18   * </p><p>
19   * The right to receive events is considered as the same resource regardless 
20   * of whether it is being handled exclusively or shared. An application 
21   * successfully obtaining exclusive access to an event results in all other 
22   * applications losing access to that event, whether the access of those 
23   * applications was shared or exclusive. 
24   * </p><p>
25   * If an UserEventFilter instance is set via 
26   * EventManager.setUserEventFilter(), EventManager shall call the 
27   * UserEventFilter.filterUserEvent() method before delivering events to the 
28   * listening applications that are specified by the UserEventFilter. Note 
29   * that EventManager shall call the filterUserEvent() method for only the 
30   * events specified by an UserEventRepository instance which is set via 
31   * EventManager.setFilteredRepository(). EventFilter may modify the key 
32   * value of the userEvent and/or may direct the platform to forward the 
33   * userEvent to a specific set of applications. Then EventManager gets the 
34   * (possibly modified) event via UserEventAction.getEvent() and the list of 
35   * AppIDs of  the  applications to receive the forwarded event via 
36   * UserEventAction.getAppIDs(). EventManager shall call the 
37   * UserEventListener.userEventReceived() of the applications which have the 
38   * AppIDs specified by UserEventAction.getAppIDs(). For this purpose, 
39   * EventManager shall track and keep the AppID of the applications which 
40   * call the addExclusiveAccessToAWTEvent() and addUserEventListener() methods 
41   * in a proprietary manner (manufacture dependent).
42   * </p>
43   */
