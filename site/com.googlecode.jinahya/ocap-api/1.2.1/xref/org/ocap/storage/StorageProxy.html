7   /**
8    * This interface represents a persistent storage device.  The current set of storage
9    * proxies is queried from the {@link org.ocap.storage.StorageManager}.
10   * <p>
11   * A StorageProxy may contain one or more logical volumes. See {@link
12   * org.ocap.storage.LogicalStorageVolume}.  A LogicalStorageVolume is a construct for organizing files on a
13   * disk and corresponds to a directory subtree that is treated as a whole for some
14   * purposes.  A StorageProxy only represents the application visible storage on the device,
15   * i.e. it does not include portions of the device reserved for internal system use.
16   * <p>
17   * If this proxy represents a detachable or hot-pluggable device, the proxy is not listed
18   * until the device is connected.  When a storage device is attached, the proxy is added
19   * to the list returned by the StorageManager and an appropriate event is sent to any
20   * {@link org.ocap.storage.StorageManagerListener} registered with the {@link org.ocap.storage.StorageManager}.  When a
21   * storage device is no longer available for use (or for reactivation after being made
22   * detachable), the corresponding proxy is removed from the StorageManager's list and a
23   * {@link org.ocap.storage.StorageManagerEvent} is sent to any StorageManagerListeners registered with the
24   * StorageManager.
25   * <p>
26   * StorageProxy extends ResourceProxy as an implementation convenience for resource
27   * contention handling. This ResourceProxy is not meant for access by applications and
28   * the StorageProxy.getClient method SHALL always return null.
29   *( </p>
30   **/
