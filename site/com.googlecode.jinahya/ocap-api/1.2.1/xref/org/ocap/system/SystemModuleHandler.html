3   /**
4    * The SystemModuleHandler is used by an OCAP-J application  
5    * for the following purposes: 
6    * 1) receive an APDU from the CableCARD device, 
7    * 2) detect an unsent APDU to the POD in case of an error, 
8    * and 3) notification of becoming registered and unregistered. 
9    * 
10   * @see SystemModuleRegistrar
11   *
12   * @author Patrick Ladd
13   * @author Shigeaki Watanabe (modified by ECN 03.0531-4)
14   */
15  public interface SystemModuleHandler {
16      /**
17       * <p>
18       * This is a call back method to notify an APDU received from the 
19       * CableCARD device.
20       * The APDU structure is defined in Table 16 in Section 8.3 of EIA-679-B 
21       * referred by [CCIF 2.0] and SCTE 28 2003. 
22       * The APDU structure consists of apdu_tag, length_field and data_byte.
23        * </p><p>
24       * For the Private Host Application on the SAS Resource, the 
25       * SystemModuleHandler is bound to a specific session number (and 
26       * a specific Private Host Application ID) when it is registered 
27       * via the {@link SystemModuleRegistrar#registerSASHandler} method. 
28       * Only the receiveAPDU() method that is bound to the session of the 
29       * received APDU shall be called only once by the OCAP implementation. 
30       * </p><p>
31       * For the MMI Resource and the Application Information Resource, 
32       * the OCAP-J application can receive APDUs for both Resources by 
33       * a single SystemModuleHandler. 
34       * The OCAP implementation shall call the receiveAPDU() method of 
35       * the SystemModuleHandler registered via the 
36       * {@link SystemModuleRegistrar#registerMMIHandler} method 
37       * only once for both the MMI and Application Information APDU. 
38       * </p><p>
39       * The OCAP implementation extract the APDU from an SPDU from 
40       * the CableCARD device according to the OpenCable CableCARD Interface 
41       * Specification, and then call this method. Note that the OCAP 
42       * implementation simply retrieves the field values from the APDU 
43       * and call this method. No validity check is done by the OCAP 
44       * implementation. Though SPDU and TPDU mechanism may detect a 
45       * destruction of the APDU structure while transmitting, the OCAP 
46       * shall call this method every time when it receives an APDU. In 
47       * such case, the parameters may be invalid so that the OCAP-J 
48       * application can detect an error. 
49       * </p><p>
50       * Note that if the CableCARD device returns an APDU indicating an error 
51       * condition, this method is called instead of the sendAPDUFailed() 
52       * method. 
53       * </p><p>
54       * This method shall return immediately. 
55       * <p>
56       *
57       * @param apduTag  an apdu_tag value in the APDU coming from the 
58       *             CableCARD device. I.e., first 3 bytes. If the corresponding 
59       *             bytes are missed, they are filled by zero. Note that 
60       *             the OCAP implementation calls this method according 
61       *             to the session number, so the apdu_tag value may be 
62       *             out of the valid range. 
63       *
64       * @param lengthField  a length_field value in the APDU coming from 
65       *             the CableCARD device.  This is a decimal int value 
66       *             converted from a length field encoded in ASN.1 BER.
67       *             If the corresponding bytes are missing, the value of
68       *             this parameter is set to 0.
69       *
70       * @param dataByte  an data_byte bytes in the APDU coming from the 
71       *             CableCARD device. If the corresponding bytes are missed since 
72       *             signaling trouble, only existing bytes are specified. 
73       *             If they are more than expected length, all existing 
74       *             bytes are specified. 
75       *             The APDU consists of the specified apdu_tag, dataByte 
76       *             and length_field. The APDU format is defined in [CCIF 2.0]. 
77       *
78       */
79      public void receiveAPDU(int apduTag, int lengthField, byte[] dataByte);
80  
81  
82      /**
83       * This is a call back method to notify an error has occurred 
84       * while sending an APDU via the {@link SystemModule#sendAPDU} 
85       * method. This method shall return immediately. 
86       *
87       * @param apduTag  an apdu_tag of the APDU that was failed to 
88       *             be sent. 
89       *             This is the apduTag value specified in the 
90       *             SystemModule.sendAPDU() method. 
91       *
92       * @param dataByte  an data_byte of the APDU that was failed 
93       *             to be sent. 
94       *             This is dataByte value specified in the 
95       *             SystemModule.sendAPDU() method. 
96       *
97       */
98      public void sendAPDUFailed(int apduTag, byte[] dataByte);

101     /**
102      * This is a call back method to notify that the 
103      * SystemModuleHandler is being unregistered and give a chance 
104      * to do a termination procedure. This method returns after the 
105      * termination procedure has finished. 
106      */
107     public void notifyUnregister();

110     /**
111      * This is a call back method to notify that this 
112      * SystemModuleHandler is ready to receive an APDU, and returns a 
113      * SystemModule to send an APDU to the CableCARD device. 
114      *
115      * @param systemModule  a SystemModule instance corresponding to 
116      *             this SystemModuleHandler. The returned SystemModule 
117      *             sends an APDU using the same session that this 
118      *             SystemModuleHandler receives an APDU. 
119      *             Null is specified, if the OCAP implementation fails 
120      *             to establish a SAS connection or fails to create an 
121      *             SystemModule instance due to lack of resource. 
122      *
123      */
124     public void ready(SystemModule systemModule);
