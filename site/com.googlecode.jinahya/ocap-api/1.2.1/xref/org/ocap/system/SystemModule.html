3   /**
4    * <p>
5    * The SystemModule is used by an OCAP-J application 
6    * to send an APDU to the CableCARD device. 
7    * A SystemModule instance is provided by the 
8    * {@link SystemModuleHandler#ready} method after calling the 
9    * {@link SystemModuleRegistrar#registerSASHandler} method or the 
10   * {@link SystemModuleRegistrar#registerMMIHandler} method. 
11   * </p>
12   *
13   * @see SystemModuleRegistrar
14   *
15   * @author Patrick Ladd
16   * @author Shigeaki Watanabe (modified by ECN 03.0531-4)
17   */
18  public interface SystemModule {
19      /**
20       * <p>
21       * This method sends an APDU to the CableCARD device. 
22       * The APDU structure is defined in Table 16 in Section 8.3 of EIA-679-B 
23       * referred by [CCIF 2.0] and SCTE 28 2003. 
24       * The APDU structure consists of apdu_tag, length_field and data_byte.
25       * </p><p>
26       * For the Private Host Application of the SAS Resource, the 
27       * session number for sending the APDU is decided by the OCAP 
28       * implementation automatically when registered via the 
29       * {@link SystemModuleRegistrar#registerSASHandler} method. 
30       * Sending APDU is delegated to the SAS Resource. 
31       * </p><p>
32       * For the MMI Resource and Application Information Resource, 
33       * sending APDU is delegated to the resident MMI and Application 
34       * Information Resources.
35       * The OCAP-J application can send APDUs of either MMI Resource 
36       * or Application Information Resource via a single SystemModule. 
37       * The OCAP implementation SHALL investigate the apdu_tag field 
38       * in the APDU and send the APDU to the CableCARD device using the 
39       * session of the Resource specified by the apdu_tag. 
40       * The session established by the resident MMI Resource and 
41       * Application Information Resource is used to send the APDU. 
42       * </p><p>
43       * For both above, the delegated Resource encodes the specified 
44       * APDU into an SPDU complementing a length_field and sends it 
45       * to the CableCARD device according to the OpenCable CableCARD Interface 
46       * Specification. 
47       * </p><p>
48       * The OCAP implementation doesn't have to confirm the validity 
49       * of the specified dataByte parameter, but SHALL confirm the 
50       * validity of the specified apduTag value. 
51       * </p><p>
52       * This method returns immediately and doesn't confirm success of 
53       * sending the APDU. Errors detected while sending the APDU are 
54       * notified via the {@link SystemModuleHandler#sendAPDUFailed} 
55       * method. 
56       * </p>
57       *
58       * @param apduTag  an apdu_tag value for the APDU to be sent to the 
59       *             CableCARD device. 
60       *
61       * @param dataByte  a data_byte binary for the APDU to be sent to 
62       *             the CableCARD device. 
63       *             This value shall contain only the data_byte part of 
64       *             an APDU structure defined in the OpenCable CableCARD 
65       *             Interface Specification. The APDU consists of the 
66       *             specified apduTag and dataByte and a length_field 
67       *             complemented by the OCAP implementation. 
68       *
69       * @throws IllegalArgumentException if the specified apdu_tag value 
70       *             is invalid (i.e., the value is not among possible tag 
71       *             values for MMI Resource or Application Information 
72       *             Resource). 
73       *             Possible apdu_tag values and possible direction for 
74       *             each Resource are defined in the OpenCable CableCARD 
75       *             Interface Specification. 
76       *
77       */
78      public void sendAPDU(int apduTag, byte[] dataByte);
79  }
