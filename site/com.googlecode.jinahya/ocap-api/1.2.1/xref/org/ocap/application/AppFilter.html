7   /**
8    * <em>AppFilter</em> provides a means of filtering AppIDs.  As a
9    * subclass of {@link AppsDatabaseFilter}, the method {@link #accept}
10   * makes a <code>true</code>/<code>false</code> decision based on an AppID.
11   *
12   * <p>An AppFilter contains a list of zero or more {@link AppPattern}s.  Each
13   * AppPattern has the attributes: <em>pattern</em>, <em>action</em>, and
14   * <em>priority</em>.  <em>pattern</em> specifies a group of AppIDs with
15   * a pair of ranges for organization ID and application ID.
16   * <em>action</em> specifies an action assigned to the AppID group;
17   * either {@link AppPattern#ALLOW}, {@link AppPattern#DENY}, or {@link
18   * AppPattern#ASK}.  <em>priority</em> specifies this AppPattern's
19   * position in the search order: the biggest number comes first.
20   * Applications can insert an AppPattern anywhere in the search
21   * order by using the priority attribute effectively
22   * (<code>AppFilter.add</code>).  When two or more AppPatterns in an
23   * AppFilter have the same priority, the search order among them is
24   * undefined.  It is not recommendable to use AppPatterns that have the
25   * same priority but different actions.
26   *
27   * <p>When <code>accept</code> is called, the given AppID is compared to
28   * the AppID group of each AppPattern in the search order until a match
29   * is found.  Then, it returns <code>true</code> or <code>false</code>
30   * if the action of matching AppPattern is <code>ALLOW</code> or
31   * <code>DENY</code> respectively.  If no match is found,
32   * <code>accept</code> returns <code>true</code>.
33   *
34   * <p>If the action of matching AppPattern is <code>ASK</code>, then
35   * AppFilter calls <code>AppFilterHandler.accept</code> for the final
36   * decision; the matching AppPattern is handed over to this method.
37   * Applications can specify the <code>AppFilterHandler</code> with
38   * <code>AppFilter.setAskHandler</code>. If no AppFilterHandler is set,
39   * AppFilter returns <code>true</code>.
40   *
41   *
42   * <p>AppPatterns can have an expiration time and MSO-private
43   * information (<em>expirationTime</em> and <em>info</em>).
44   * <code>accept</code> and <code>getAppPatterns</code> methods ignore
45   * AppPatterns that have expired.  The implementation may delete expired
46   * AppPatterns from AppFilter.
47   *
48   * <p><b>Example:</b>
49   *
50   * <blockquote><pre>
51   * import org.ocap.application.*;
52   * import org.dvb.application.AppID;
53   *
54   * AppManagerProxy am = ...;
55   * AppPattern[] patterns = {
56   *     &#47;* note that search order is dictated by "priority" *&#47;
57   *     new AppPattern("10-5f:1-ff", AppPattern.ALLOW, 40),     // #3
58   *     new AppPattern("30:2c-34", AppPattern.ALLOW, 100),      // #1
59   *     new AppPattern("20-40", AppPattern.DENY, 80),           // #2
60   * };
61   * AppFilter af = new AppFilter(patterns);
62   * 
63   * &#47;* false - matches "20-40" *&#47;
64   * boolean badOne = af.accept(new AppID(0x30, 0x10));
65   *
66   * &#47;* true - matches "30:2c-34" *&#47;
67   * boolean goodOne = af.accept(new AppID(0x30, 0x30));
68   *
69   * &#47;* will be the second entry: priority between 100 and 80 *&#47;
70   * af.add(new AppPattern("40-4f:1000-1fff", DENY, 90));
71   *
72   * &#47;* register af with the system *&#47;
73   * am.setAppFilter(af);
74   * </pre></blockquote>
75   *
76   * @see AppPattern
77   * @see AppFilterHandler
78   * @see AppManagerProxy
79   * @see org.dvb.application.AppID
80   * @see org.dvb.application.AppsDatabaseFilter
81   */
