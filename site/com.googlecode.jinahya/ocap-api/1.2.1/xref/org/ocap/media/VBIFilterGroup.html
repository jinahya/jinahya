12  /**
13   * <p>
14   * This class represents a group of VBI data filters. The VBI filters in 
15   * a group are attached to a ServiceContext. Current video played back 
16   * on the ServiceContext is the target of filtering. If the current video is 
17   * analog, the filters retrieve specific data in a specified VBI line of 
18   * the video. The data format is defined in each VBI data specification. 
19   * If the current video is MPEG video, the filters retrieve specific data 
20   * in the user_data found in the MPEG picture headers of the video. Data 
21   * format in user_data is defined in ANSI/SCTE 20 and ANSI/SCTE 21. 
22   * </p><p>
23   * When a new VBIFilterGroup is constructed, an application specifies the 
24   * total number of VBIFilters held by it. The application can create a 
25   * specified number of new VBIFilter instances via a 
26   * {@link VBIFilterGroup#newVBIFilter} method. VBI lines and data format for 
27   * filtering are specified in this method. After configuring all 
28   * filters, the VBIFilterGroup is attached to a specific ServiceContext. 
29   * Current video on the ServiceContext is the target of filtering. 
30   * If the startFiltering() method of a VBI filter is called, the filter 
31   * starts filtering immediately. 
32   * The application can retrieve VBI data via a {@link VBIFilter#getVBIData} 
33   * method. The host may be able to filter VBI data by using separated fields
34   * (specify either field 1 or field 2 and retrieve data in only the specified 
35   * field) or by using mixed fields (retrieve data in both field 1 and 2 in 
36   * arrival order). 
37   * </p><p>
38   * The data unit is defined as follows. 
39   * </p><p>
40   * <b>XDS (VBI_DATA_FORMAT_XDS)</b><br>
41   * XDS packets in line 21 field 2 is defined in EIA-608-B. 
42   * It has a start/continue/end code and is interleaved with CC3, CC4, T3 and 
43   * T4 service. One XDS packet is a single data unit of this data format. 
44   * </p><p>
45   * <b>Text service (VBI_DATA_FORMAT_T1/T2/T3/T4)</b><br>
46   * Text service data in line 21 field 1 or 2 is defined in EIA-608-B. 
47   * It has start/end codes and is interleaved with the other closed captioning 
48   * service and Text service according to Section 7 of EIA-608-B. 
49   * A single data unit of Text service is a character sequence between a text 
50   * mode in code (RTD or TR) and a text mode out code (EOC, RCL, RDC, RU2, 
51   * RU3 or RU4). 
52   * Note that Text service data unit returned from the VBIFilter doesn't contain 
53   * these text mode in/out codes, but contains another control code like EDM etc. 
54   * </p><p>
55   * <b>Generic EIA-608-B service (VBI_DATA_FORMAT_EIA608B)</b><br>
56   * Generic EIA-608-B service represents a character stream that consists of 
57   * Character One and Character Two in a field of line 21. 
58   * Character One and Character Two are defined in Figure 1 of EIA-608-B. 
59   * Note that the VBIFilter only supports separated field filtering since 
60   * Character One and Character Two make a separate data stream in each field. 
61   * A single data unit of this format is a set of Character One and Character 
62   * Two in one VBI line and field (i.e., only two characters). 
63   * </p><p>
64   * <b>NABTS (VBI_DATA_FORMAT_NABTS)</b><br>
65   * NABTS is defined in EIA-516. It is a 33 byte fixed length packet and 
66   * consists of 5 byte prefix and 28 byte data block. 
67   * Note that the filter retrieves only the "Data packet" part. I.e., one 
68   * NABTS packet is a single data unit. 
69   * The VBIFilter retrieves multiple NABTS packets in arrival order for all 
70   * specified lines and fields. 
71   * </p><p>
72   * <b>AMOL I/II (VBI_DATA_FORMAT_AMOL_I/II)</b><br>
73   * AMOL I and II is defined in ACN 403-1218-024. 
74   * It specifies header bits and the number of bits in a VBI line. 
75   * The bits in one VBI line/field is a single data unit for this data format, 
76   * i.e., the header bits are included in the data unit. 
77   * </p><p>
78   * <b>UNKNOWN (VBI_DATA_FORMAT_UNKNOWN) </b><br>
79   * UNKNOWN represents a VBI waveform that is unknown. To retrieve UNKNOWN 
80   * data unit, the OCAP implementation needs to synchronize to the VBI's bit 
81   * rate automatically. Filtering success is not guaranteed even if 
82   * the OCAP implementation supports the UNKNOWN format, since an analog waveform 
83   * is not standardized. 
84   * The bits in one VBI line and field is a data unit for UNKNOWN format. 
85   * </p><p>
86   * Support of the other VBI lines is optional. And support of the ANSI/SCTE 
87   * 20 and ANSI/SCTE 21 is also optional. An application can check the supported 
88   * filtering lines, data format and filtering techniques (field separated or 
89   * field mixed) via capability methods. 
90   * </p>
91   *
92   * @author  Shigeaki Watanabe (Panasonic)
93   */

181     /**
182      * Create a new VBIFilter instance within a VBIFilterGroup instance. 
183      * The VBIFilter filters the specified VBI line/field and retrieves 
184      * data units of the specified format. 
185      * The filter has an internal buffer of the specified size to store the 
186      * filtered data units. If the filtered data size exceeds the buffer size, 
187      * the oversized part of data will be lost. 
188      *
189      * @return  a new VBIFilter instance. Total number of filters 
190      *            created by a VBIFilterGroup shall not exceed the 
191      *            number of filters specified by a constructor. 
192      *
193      * @param lineNumber  an array of VBI line numbers that will be filtered 
194      *            by a returned SimpleVBIFilter. 
195      *            If line 21 is specified, the filter also filters user_data 
196      *            defined in ANSI/SCTE 20 and ANSI/SCTE 21, when a current 
197      *            video is MPEG. Note that filtering of user_data is optional. 
198      *
199      * @param field  a field number that will be filtered by a returned 
200      *            VBIFilter. One of the constants that has a FIELD_ prefix. 
201      *            <br>
202      *            If FIELD_MIXED is specified, the VBIFilter filters 
203      *            both field 1 and 2 of the specified lineNumber lines. 
204      *            Filtered data is concatenated in arrival order. <br>
205      *            If FIELD_1 is specified, the VBIFilter filters only 
206      *            field 1 of the specified lineNumber lines. <br>
207      *            If FIELD_2 is specified, the VBIFilter filters only 
208      *            field 2 of the specified lineNumber lines. <br>
209      *
210      * @param dataFormat  one of constants that has a VBI_DATA_FORMAT_ prefix 
211      *            to specify a data format to be filtered. 
212      *
213      * @param unitLength  specify the number of bits (not bytes) of a single 
214      *            data unit for the VBI_DATA_FORMAT_UNKNOWN format. 
215      *            This value shall be ignored for the other well known 
216      *            formats. 
217      *
218      * @param bufferSize  number of bytes to specify size of an internal 
219      *            buffer used to hold a VBI data unit. 
220      *
221      * @throws IllegalStateException  if the total number of filters created 
222      *            by this VBIFilterGroup exceeds the number specified by the 
223      *            constructor. 
224      *
225      * @throws IllegalArgumentException  if this VBIFilterGroup can't create 
226      *            a VBIFilter for specified parameters due to hardware 
227      *            or software restrictions, or the lineNumber is not capable 
228      *            line, or if the field is not defined in constants, 
229      *            or the specified dataFormat is not defined in constants, 
230      *            or if the bufferSize is 0. 
231      *            For example, if this method is called with lineNumber=21, 
232      *            field=FIELD_1 and dataFormat=VBI_DATA_FORMAT_XDS, this 
233      *            exception is thrown and null returns. 
234      */
