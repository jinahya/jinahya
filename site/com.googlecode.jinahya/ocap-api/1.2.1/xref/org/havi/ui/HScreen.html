8   /**
9      This class describes the final output composition of a device.  It
10     ties together all the (MPEG) video decoders, all the graphics
11     sub-systems and backgrounds which are all combined together before
12     finally being displayed. A platform with two independent displays
13     would support two instances of this class. Where a device outputs
14     audio closely bound with video, that audio output can also be
15     represented through this class.
16  
17     <p>
18  
19     Since an HScreen represents a single video output signal from a
20     device, all the devices which contribute to that signal must have
21     certain properties in common. It is not possible to select
22     conflicting configurations for different devices on the same
23     HScreen - for example having a video device whose logical output
24     has a 4:3 picture aspect ratio and a graphics device whose logical
25     output has a 16:9 picture aspect ratio. This specification
26     intentionally does not define configurations, or which
27     configurations would be conflicting, since these are essentially
28     region or market dependent.
29

32    parameters are not used, then the constructor should use the default
