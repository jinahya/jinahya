12  /**
13     The {@link org.havi.ui.HLook HLook} interface defines the
14     &quot;look&quot; of a component and may be regarded as a mechanism
15     to allow a &quot;pluggable&quot; paint method to be attached to the
16     component. Instead of having to subclass the entire
17     component to change its look, it is possible to simply implement an
18     {@link org.havi.ui.HLook HLook} that will render the component
19     &quot;look&quot; and then associate this {@link org.havi.ui.HLook
20     HLook} implementation with the component.  
21  
22     <h3>Borders</h3>
23  
24     An implementation of {@link org.havi.ui.HLook HLook} will also
25     include code to draw implementation-specific borders. The
26     application or component can query the reserved space for an {@link
27     org.havi.ui.HLook HLook} with the {@link
28     org.havi.ui.HLook#getInsets getInsets} method.
29  
30     <p>
31     Since the border area is included in the overall size of the
32     component, the border effectively constrains the area available for
33     rendering content to the rectangle which has an upper-left corner
34     location of <code>(insets.left, insets.top)</code>, and has a size
35     of <code>width - (insets.left + insets.right)</code> by
36     <code>height - (insets.top + insets.bottom)</code>.
37  
38     <h3>Invocation Mechanism</h3>
39  
40     The {@link org.havi.ui.HLook#showLook showLook} method of the
41     {@link org.havi.ui.HLook HLook} interface will be called by the
42     havi.ui framework in response to the paint method of the {@link
43     org.havi.ui.HVisible HVisible} being called by the AWT lightweight
44     component framework. Applications should simply invoke the
45     component repaint method as in normal AWT, rather than calling the
46     {@link org.havi.ui.HLook#showLook showLook} method directly.
47  
48     <h3>Content Rendering</h3>
49  
50     Some looks make use of content stored on an associated
51     {@link org.havi.ui.HVisible HVisible}. These looks are:
52  
53     <p><ul>
54     <li>{@link org.havi.ui.HAnimateLook HAnimateLook} - for animated
55     graphical content
56     <li>{@link org.havi.ui.HGraphicLook HGraphicLook} - for static
57     graphical content
58     <li>{@link org.havi.ui.HTextLook HTextLook} - for static textual
59     content
60     <li>{@link org.havi.ui.HSinglelineEntryLook HSinglelineEntryLook} -
61     for single line text entry content
62     <li>{@link org.havi.ui.HMultilineEntryLook HMultilineEntryLook} -
63     for multiline text entry content 
64     <li>{@link org.havi.ui.HListGroupLook HListGroupLook} -
65     for list element content
66     </ul>
67  
68     <p>  
69     Some of these looks may support the scaling and alignment of their
70     content as an implementation option. The scaling and alignment
71     modes supported are specified in the class description of {@link
72     org.havi.ui.HVisible HVisible}. The table below details which
73     features the content-based platform looks should support:
74  
75     <p><table border>
76     <tr><th>HLook</th><th>Scaling</th><th>Alignment</th></tr>
77     <tr><td>HAnimateLook</td><td>Optional</td><td>Mandatory</td></tr>
78     <tr><td>HGraphicLook</td><td>Optional</td><td>Mandatory</td></tr>
79     <tr><td>HListGroupLook</td><td>Optional</td><td>Mandatory</td></tr>
80     <tr><td>HTextLook</td><td>Not used</td><td>Mandatory</td></tr>
81     <tr><td>HSinglelineEntryLook</td><td>Not used</td><td>Not used</td></tr>
82     <tr><td>HMultilineEntryLook</td><td>Not used</td><td>Not used</td></tr>
83     </table>
84  
85     <p>
86     Where scaling support is optional all implementations must
87     as a minimum support the {@link org.havi.ui.HVisible#RESIZE_NONE
88     RESIZE_NONE} scaling mode. Platforms are
89     <em>not</em> required to support scaling of textual content by default.
90        
91     <p>
92     Looks should use the {@link
93     org.havi.ui.HVisible#getHorizontalAlignment getHorizontalAlignment}
94     and {@link org.havi.ui.HVisible#getVerticalAlignment
95     getVerticalAlignment} methods to retrieve the current alignment
96     modes, and the {@link org.havi.ui.HVisible#getResizeMode
97     getResizeMode} method to determine the active scaling mode, where
98     supported. However, note that HAVi platform looks which render text
99     content using the {@link org.havi.ui.HDefaultTextLayoutManager
100    HDefaultTextLayoutManager} class shall delegate the alignment of
101    text content to the layout manager.
102 
103    <h3>Rendering Hints</h3>
104    
105    HLook provides a method, {@link org.havi.ui.HLook#widgetChanged
106    widgetChanged} which can be called by an {@link
107    org.havi.ui.HVisible HVisible} with one or more hints to inform the
108    look that something has changed. This method provides information
109    to the look about what has changed, which allows smarter repainting
110    than having the HVisible simply call its <code>repaint</code>
111    method. The hint constants are defined on {@link
112    org.havi.ui.HVisible HVisible}. See the class definition for {@link
113    org.havi.ui.HVisible HVisible} for more information.
114    
115    <h3>Private Data</h3>
116 
117    Implementations of {@link org.havi.ui.HLook HLook} may store
118    private data on each instance of an {@link org.havi.ui.HVisible
119    HVisible} to optimize the drawing of that component. However, this
120    is an implementation option. Furthermore such data may be
121    invalidated by another part of the system, for example if 
122    {@link org.havi.ui.HVisible#setLook setLook}
123    is called on {@link org.havi.ui.HVisible HVisible}. 
124    Therefore if this mechanism is used by implementations
125    of {@link org.havi.ui.HLook HLook} those implementations
126    <em>must</em> be capable of regenerating such data on the fly,
127    according to the current state of the {@link org.havi.ui.HVisible
128    HVisible}.
129 
130    <h3>Platform Looks</h3>
131 
132    The HAVi UI provides a number of classes implementing the {@link
133    org.havi.ui.HLook HLook} interface. Applications wishing to provide
134    their own {@link org.havi.ui.HLook HLooks} may directly implement
135    this interface or may subclass those provided by the platform.
136 
137    <h3>Default Behavior </h3>
138 
139    Unless already specified in a particular {@link org.havi.ui.HLook
140    HLook}, implementations of {@link org.havi.ui.HLook HLook} should
141    use:
142 
143    <ul>
144    <li> the foreground color of each associated {@link org.havi.ui.HVisible 
145    HVisible} (using the java.awt.Component method getForeground) to
146    determine the Color to render both the content and border (as necessary) 
147    <li> the background color of each associated {@link
148    org.havi.ui.HVisible HVisible} (using the java.awt.Component method
149    getBackground) to determine the Color used to render a rectangular
150    area to fill the component and erase any previous content, as
151    specified by the current background drawing mode for that component
152    - see {@link org.havi.ui.HVisible#setBackgroundMode
153    setBackgroundMode} on {@link org.havi.ui.HVisible HVisible}.
154 
155    <li> the current alignment mode of each associated {@link
156    org.havi.ui.HVisible HVisible} (using {@link
157    org.havi.ui.HVisible#getHorizontalAlignment getHorizontalAlignment}
158    and {@link org.havi.ui.HVisible#getVerticalAlignment
159    getVerticalAlignment}) to determine how to align content.
160 
161    <li> the current scaling mode of each associated {@link
162    org.havi.ui.HVisible HVisible} (using to {@link
163    org.havi.ui.HVisible#getResizeMode getResizeMode}) to determine how
164    to scale content.
165    </ul>
166  
167    @see org.havi.ui.HVisible#setLook
168    @see org.havi.ui.HVisible#setLookData
169    @see org.havi.ui.HVisible#paint
170    @see org.havi.ui.HVisible#setBackgroundMode
171    @see org.havi.ui.HVisible#setHorizontalAlignment
172    @see org.havi.ui.HVisible#setVerticalAlignment
173    @see org.havi.ui.HVisible#setResizeMode
174    @see org.havi.ui.HTextLook
175    @see org.havi.ui.HGraphicLook
176    @see org.havi.ui.HAnimateLook
177    @see org.havi.ui.HRangeLook
178    @see org.havi.ui.HSinglelineEntryLook
179    @see org.havi.ui.HMultilineEntryLook
180    
181 */

185     /**
186      * The {@link org.havi.ui.HLook#showLook showLook} method is
187      * responsible for repainting the entire {@link
188      * org.havi.ui.HVisible HVisible} component, (including any
189      * content set on the component, and the component background), subject to
190      * the clipping rectangle of the Graphics object passed to it.
191      * <p>
192      * The {@link org.havi.ui.HLook#showLook showLook} method should
193      * not modify the clipRect (clipping rectangle) of the
194      * <code>Graphics</code> object that is passed to it in a way
195      * which includes any area not part of that original clipRect. If any
196      * modifications are made, the original clipRect shall be restored.
197      * <p> 
198      * For looks which draw content (e.g.  {@link
199      * org.havi.ui.HTextLook HTextLook}, {@link
200      * org.havi.ui.HGraphicLook HGraphicLook} and {@link
201      * org.havi.ui.HAnimateLook HAnimateLook}), if no content is
202      * associated with the component, the {@link
203      * org.havi.ui.HLook#showLook showLook} method paints the component
204      * with its current background <code>Color</code> according to the {@link
205      * org.havi.ui.HVisible#setBackgroundMode setBackgroundMode}
206      * method of {@link org.havi.ui.HVisible HVisible} and draws any
207      * (implementation-specific) borders. Note that by default the
208      * background mode is set so as to <b>not</b> paint a
209      * background. Furthermore on platforms which support transparent
210      * colors the background <code>Color</code> may be partially or completely
211      * transparent.
212      * <p> 
213      * Any resources <b>explicitly</b> associated with an {@link
214      * org.havi.ui.HLook HLook} should be loaded by the {@link
215      * org.havi.ui.HLook HLook} during its creation, etc. 
216      * Note that the &quot;standard&quot; looks don't load content by default. 
217      * <p>
218      * This method is called from the {@link
219      * org.havi.ui.HVisible#paint paint} method of {@link
220      * org.havi.ui.HVisible HVisible} and must never be called from
221      * elsewhere. Components wishing to redraw themselves should call
222      * their repaint method in the usual way.
223      *
224      * @param g            the graphics context.
225      * @param visible      the visible.
226      * @param state the state parameter indicates the state of the
227      * visible, allowing the look to render the appropriate content
228      * for that state. Note that some components (e.g. HStaticRange,
229      * HRange, HRangeValue) do not use state-based content.
230      */
231     public void showLook(java.awt.Graphics g, HVisible visible, int state);

235       * its content, state, or any other data changes. See the class
236       * description of {@link org.havi.ui.HVisible HVisible} for more
237       * information about the <code>changes</code> parameter.
238       * <p>
239       * The implementation of this method should work out which
240       * graphical areas of the {@link org.havi.ui.HVisible HVisible}
241       * have changed and make any relevant calls to trigger the
242       * repainting of those areas.
243       * <p>
244       * A minimum implementation of this method could simply call
245       * <pre>
246       * visible.repaint()
247       * </pre>
248       * 
249       * @param visible the {@link org.havi.ui.HVisible HVisible} which
250       * has changed

252       * objects. If this argument is <code>null</code> a full repaint
253       * will be triggered.  
254       */
255    public void widgetChanged (HVisible visible, HChangeData[] changes);

260      * Gets the minimum size of the {@link org.havi.ui.HVisible
261      * HVisible} component when drawn with this {@link
262      * org.havi.ui.HLook HLook}. 
263      * <p>
264      * This size may be determined in several ways depending on the
265      * information available to the look. These steps are performed in
266      * order and the first available result is returned. For the
267      * purposes of this algorithm {@link org.havi.ui.HLook HLook}
268      * classes that do not use content (e.g. {@link
269      * org.havi.ui.HRangeLook HRangeLook}) are treated as if no
270      * content was present.
271      * <p>
272      * The extra space required for border decoration can be determined from 
273      * the {@link org.havi.ui.HLook#getInsets getInsets} method.
274      * <p><ol>
275      * 
276      * <li> If this look is an {@link org.havi.ui.HTextLook HTextLook} and 
277      * if {@link org.havi.ui.HVisible#getTextLayoutManager
278      * HVisible.getTextLayoutManager()} returns an {@link
279      * org.havi.ui.HDefaultTextLayoutManager HDefaultTextLayoutManager}, then
280      * this method should delegate the call to its {@link
281      * org.havi.ui.HDefaultTextLayoutManager#getMinimumSize getMinimumSize()}
282      * method plus any additional dimensions that the HLook requires
283      * for border decoration etc. If the HDefaultTextLayoutManager
284      * returns a zero size, then proceed with the following steps.
285      * <li> If the HLook supports the scaling of its content (e.g.
286      * an HGraphicLook) and scaling is requested and content is set,
287      * then the return value is a size containing the width of the
288      * narrowest content and the height of the shortest content plus
289      * any additional dimensions that the HLook requires for border
290      * decoration etc.
291      * <li> If the {@link org.havi.ui.HLook HLook} does not support
292      * scaling of content or no scaling is requested, <em>and</em>
293      * content is set then the return value is a size sufficiently
294      * large to hold each
295      * piece of content plus any additional dimensions that the HLook
296      * requires for border decoration etc.
297      * <li> If no content is available but a default preferred size
298      * has been set using {@link org.havi.ui.HVisible#setDefaultSize
299      * setDefaultSize} has been called to set then the return value is
300      * this value (as obtained with {@link
301      * org.havi.ui.HVisible#getDefaultSize getDefaultSize}) plus any
302      * additional dimensions that the HLook requires for border
303      * decoration etc.
304      * <li> If there is no content or default size set then the
305      * return value is an implementation-specific minimum size plus
306      * any additional dimensions that the HLook requires for border
307      * decoration etc.
308      * </ol>
309      * 
310      * @param hvisible {@link org.havi.ui.HVisible HVisible} to which
311      * this {@link org.havi.ui.HLook HLook} is attached.
312      * @return A dimension object indicating this {@link
313      * org.havi.ui.HLook HLook's} minimum size.
314      * @see org.havi.ui.HVisible#getMinimumSize 
315      */
316     public Dimension getMinimumSize(HVisible hvisible);
