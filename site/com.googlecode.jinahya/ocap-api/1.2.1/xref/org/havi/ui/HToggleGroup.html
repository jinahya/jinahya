8   /**
9      {@link org.havi.ui.HToggleButton HToggleButtons} within the same
10     {@link org.havi.ui.HToggleGroup HToggleGroup} will behave so that a
11     maximum of one {@link org.havi.ui.HToggleButton HToggleButton} has
12     switchable state true, as returned by {@link
13     org.havi.ui.HToggleButton#getSwitchableState getSwitchableState},
14     so as to achieve a &quot;radio button&quot; effect. When an {@link
15     org.havi.ui.HToggleButton HToggleButton} is acted upon to change
16     its switchable state to true, then if any other {@link
17     org.havi.ui.HToggleButton HToggleButton} within the {@link
18     org.havi.ui.HToggleGroup HToggleGroup} currently has switchable
19     state true, it will have its switchable state set to
20     false. Similarly, if an {@link org.havi.ui.HToggleButton
21     HToggleButton} is added which has switchable state true, then any
22     current {@link org.havi.ui.HToggleButton HToggleButton} within the
23     {@link org.havi.ui.HToggleGroup HToggleGroup} with switchable state
24     true, shall have its switchable state modified to false.
25     <p> 
26     If the forced selection mode for the {@link
27     org.havi.ui.HToggleGroup HToggleGroup} is set via a call to
28     <code>setForcedSelection(true)</code> then there will always be one
29     {@link org.havi.ui.HToggleButton HToggleButton} selected (i.e. with
30     switchable state <code>true</code>), and if necessary the {@link
31     org.havi.ui.HToggleGroup HToggleGroup} will automatically force a
32     selection to ensure this. If forced selection mode is not set it
33     is valid for there to be no selection, i.e. all {@link
34     org.havi.ui.HToggleButton HToggleButtons} may have switchable state
35     <code>false</code>. By default forced selection mode is not set.
36     <p>  
37     Note that when an {@link org.havi.ui.HToggleButton HToggleButton}
38     has switchable state <code>true</code> this implies that the
39     interaction state as returned by {@link
40     org.havi.ui.HVisible#getInteractionState getInteractionState} will
41     be either the {@link org.havi.ui.HState#ACTIONED_STATE ACTIONED_STATE} or
42     {@link org.havi.ui.HState#ACTIONED_FOCUSED_STATE ACTIONED_FOCUSED_STATE} state.
43     <p>
44     Similarly, a switchable state of <code>false</code> implies that
45     the interaction state is any other state for which the {@link
46     org.havi.ui.HState#ACTIONED_STATE_BIT ACTIONED_STATE_BIT} is not
47     set. See the {@link org.havi.ui.HSwitchable HSwitchable} class
48     description for more information about the valid interaction
49     states.
