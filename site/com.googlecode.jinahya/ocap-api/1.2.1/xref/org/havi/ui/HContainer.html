12  /**  
13       The {@link org.havi.ui.HContainer HContainer} class extends the
14       <code>java.awt.Container</code> class by implementing the {@link
15       org.havi.ui.HMatteLayer HMatteLayer} interface and providing
16       additional Z-ordering capabilities, which are required since
17       components in the HAVi user-interface are explicitly allowed to
18       overlap each other.
19       <p>
20       Note that these Z-ordering capabilities (<code>addBefore,
21       addAfter, pop, popInFrontOf, popToFront, push, pushBehind and
22       pushToBack</code>) must be implemented by (implicitly) reordering
23       the child Components within the {@link org.havi.ui.HContainer
24       HContainer}, so that the standard AWT convention that the Z-order
25       is defined as the order in which Components are added to a given
26       Container is maintained. For example, one implementation of
27       <code>popToFront</code> might be to make the specified Component
28       become the first Component added to the parent Container by
29       removing all Components from that Container, adding the specified
30       Container first, and then adding the remaining Components in
31       their current relative order to that Container.
32

35    parameters are not used, then the constructor should use the default
36    values.
