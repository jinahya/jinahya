10  /**
11     The {@link org.havi.ui.HSceneFactory HSceneFactory} class provides
12     a generic mechanism for an application to request {@link
13     org.havi.ui.HScene HScene} resources from a (conceptual) window
14     management system. The {@link org.havi.ui.HSceneFactory
15     HSceneFactory} is the single entry to potentially multiple
16     HGraphicsDevice centric window management policies.
17     
18     <p>
19     The {@link org.havi.ui.HSceneFactory HSceneFactory} class provides
20     an opaque interface between any application (or window) management
21     scheme and the Java application, itself.
22  
23     <p> 
24     Note that only one {@link org.havi.ui.HScene HScene} per {@link
25     org.havi.ui.HGraphicsDevice HGraphicsDevice} can be acquired at any
26     one time for each application.
27  
28     <p> 
29  
30     {@link org.havi.ui.HScene HScenes} follow the design pattern of the
31     <code>java.awt.Window</code> class. They are not a scarce resource
32     on the platform. On platforms which only support one {@link
33     org.havi.ui.HScene HScene} being visible at one time the current
34     {@link org.havi.ui.HScene HScene} both loses the input focus and is
35     hidden (e.g. iconified) when another application successfully
36     requests the input focus. Two
37     <code>java.awt.event.WindowEvent</code> events, with ids
38     <code>WINDOW_DEACTIVATED</code> and <code>WINDOW_ICONIFIED</code>,
39     shall be generated and sent to the {@link org.havi.ui.HScene
40     HScene} which has lost the focus and the {@link
41     org.havi.ui.HScene#isShowing isShowing} method for that HScene
42     shall return false.
43  
44     <p>
45     The constraints on the sizing and positioning of the {@link
46     org.havi.ui.HScene HScene} returned by the methods are dependent on
47     the platform-specific {@link org.havi.ui.HScene HScene} support.
48     The three scenarios are defined below:
49     
50     <h4>Platforms Supporting a Full Multi-Window System</h4>
51     
52     Platforms where windows may obscure each other shall allow
53     applications to create {@link org.havi.ui.HScene HScene} objects
54     which are fully within the area of the supporting {@link
55     org.havi.ui.HGraphicsDevice HGraphicsDevice} without any restriction
56     on size or location. Whether successful creation of HScenes which
57     are wholly or partially outside the area of the HGraphicsDevice is
58     supported is implementation dependent.
59     
60     <h4>Platforms Supporting a Single Window System</h4>
61     
62     Platforms supporting a simple &quot;full-screen&quot; view on a single
63     application at any one time are required to allow applications to
64     create HScenes which cover the full area of the supporting
65     HGraphicsDevice. HScenes matching this description shall be returned
66     as the default {@link org.havi.ui.HScene HScene} for the supporting
67     {@link org.havi.ui.HGraphicsDevice HGraphicsDevice}. It is
68     implementation dependent whether requests to create HScene
69     objects which cover less than the full area of the supporting
70     HGraphicsDevice succeed and if they succeed, what the consequences
71     are of their being displayed.
72     
73     <h4>Platforms Supporting a Restricted Multi-Window System</h4>
74     
75     Platforms supporting a &quot;paned&quot; system where each application
76     occupies an area on-screen that is always visible make fewer
77     guarantees to applications.  When an {@link org.havi.ui.HScene
78     HScene} is created on such a system, the platform shall return an
79     HScene which, if visible at that time, would cover as much as
80     possible of the requested area of the supporting HGraphicsDevice
81     considering all other visible HScenes at that time. HScenes which
82     are not visible at this time shall not be considered when fixing the
83     location & size of the new HScene.  HScenes which are visible at
84     this time shall not be effected by the creation of the new HScene.
85     
86     <p>
87     When {@link org.havi.ui.HScene#setVisible setVisible(true)} is
88     called on such an {@link org.havi.ui.HScene HScene}, the platform
89     shall attempt to make the HScene visible using its currently set
90     position and size. If this would conflict with HScenes which are
91     already visible (e.g.  because of changes between when the HScene
92     was created and when <code>setVisible</code> was called) then the
93     call to <code>setVisible</code> shall fail silently. Applications
94     are responsible for testing for failure using the {@link
95     org.havi.ui.HScene#isVisible isVisible} method. The HScenes of
96     already visible applications shall not be impacted by this method
97     call.
98     
99     <p>
100    The above text specifies the relationship between an {@link
101    org.havi.ui.HScene HScene} and its supporting {@link
102    org.havi.ui.HGraphicsDevice HGraphicsDevice} . This specification
103    intentionally does not define minimum requirements for
104    HGraphicsDevices or for their relationship with other HScreenDevices
105    of any type.
106    
107    <p>
108    Calling {@link org.havi.ui.HSceneFactory#resizeScene resizeScene}
109    for an {@link org.havi.ui.HScene HScene} shall apply the same
110    policies as described above for newly created HScenes when deciding
111    whether the method call is possible.
112
