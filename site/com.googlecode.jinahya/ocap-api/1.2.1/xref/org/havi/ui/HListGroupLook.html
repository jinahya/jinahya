14  /**
15     The {@link org.havi.ui.HListGroupLook HListGroupLook} class is used by the {@link
16     org.havi.ui.HListGroup HListGroup} component to display both the
17     {@link org.havi.ui.HListGroup HListGroup} itself (potentially
18     including a scrollbar component) and graphical or textual list
19     content held on the {@link org.havi.ui.HListGroup HListGroup}. This
20     look will be provided by the platform and the exact way in which it
21     is rendered will be platform dependent.
22  
23     <p> 
24     The {@link org.havi.ui.HListGroupLook HListGroupLook} class draws the HListGroup
25     and any look-specific borders around the component, and then
26     renders the content set on the {@link org.havi.ui.HListGroup
27     HListGroup}. It uses the {@link
28     org.havi.ui.HListGroup#getListContent getListContent} method to
29     determine the content to render. The content of the HListGroup does
30     not depend on the interaction state.
31  
32     <p> 
33     {@link org.havi.ui.HListGroupLook HListGroupLook} should use the 
34     following properties of {@link org.havi.ui.HListGroup HListGroup}
35     to lay out and render the {@link org.havi.ui.HListElement
36     HListElement} content:
37     
38     <p><table border>   
39     <tr><th>Item</th><th>Method</th><th>Purpose</th></tr>
40     <tr><td>Orientation</td><td>{@link
41     org.havi.ui.HListGroup#getOrientation
42     getOrientation}</td><td>direction to lay out elements</td></tr>
43     <tr><td>Content</td><td>{@link
44     org.havi.ui.HListGroup#getListContent
45     getListContent}</td><td>elements to display</td></tr>
46     <tr><td>Scroll position</td><td>{@link
47     org.havi.ui.HListGroup#getScrollPosition
48     getScrollPosition}</td><td>first element to draw</td></tr>
49     <tr><td>Selection</td><td>{@link
50     org.havi.ui.HListGroup#isItemSelected isItemSelected}</td><td>mark
51     an element as selected</td></tr>
52     <tr><td>Current item</td><td>{@link
53     org.havi.ui.HListGroup#getCurrentItem
54     getCurrentItem}</td><td>highlight an element</td></tr>
55     </table><p>
56  
57     {@link org.havi.ui.HListGroupLook HListGroupLook} 
58     should draw a scrollbar as necessary when there are more {@link
59     org.havi.ui.HListElement HListElements} than can be displayed. It
60     is an implementation option to leave border space between each
61     element. The insets used for the element borders can be retrieved
62     using {@link org.havi.ui.HListGroupLook#getElementInsets
63     getElementInsets}
64  
65     <p>
66     Implementations of {@link org.havi.ui.HListGroupLook HListGroupLook} 
67     should use the appropriate methods on {@link org.havi.ui.HListGroup
68     HListGroup} to determine which scaling and alignment modes to use
69     when rendering content.  See the class description for {@link
70     org.havi.ui.HLook HLook} for more details. 
71     <p>
72     {@link org.havi.ui.HListGroupLook HListGroupLook} may support scalable graphical
73     content. As a minimum, all implementations must support the {@link
74     org.havi.ui.HVisible#RESIZE_NONE RESIZE_NONE} scaling mode for
75     graphical content, and all alignment modes for text content.
76     However, Note that {@link org.havi.ui.HListGroupLook HListGroupLook} behaves
77     slightly differently from other HAVi {@link org.havi.ui.HLook
78     HLook} classes, as follows.
79     <p><ul>
80     <li> Where supported, scaling applies to the icon (graphical
81     content) of each {@link org.havi.ui.HListElement HListElement},
82     based on the area allocated to that {@link org.havi.ui.HListElement
83     HListElement} rather than the entire area of the {@link
84     org.havi.ui.HListGroup HListGroup}.
85     <li> Alignment mode applies to the content of the {@link
86     org.havi.ui.HListElement HListElement} within the area allocated to
87     that {@link org.havi.ui.HListElement HListElement} rather than the
88     entire area of the {@link org.havi.ui.HListGroup HListGroup}.
89     </ul><p> 
90     Note that the results of applying the {@link
91     org.havi.ui.HVisible#VALIGN_JUSTIFY VALIGN_JUSTIFY} and {@link
92     org.havi.ui.HVisible#HALIGN_JUSTIFY HALIGN_JUSTIFY} alignment modes
93     to graphical content are defined to identical to {@link
94     org.havi.ui.HVisible#VALIGN_CENTER VALIGN_CENTER} and {@link
95     org.havi.ui.HVisible#HALIGN_CENTER HALIGN_CENTER} modes
96     respectively, as justification is meaningless in this context.
97  
98     <p>   
99     This is the default look that is used by {@link
100    org.havi.ui.HListGroup HListGroup}.

253     /**
254      * The {@link org.havi.ui.HExtendedLook#renderVisible} method
255      * paints any content or other data associated with the look's HVisible. This
256      * method shall not be used to render a background nor any other decoration.
257      * Its purpose is purely to render content or other value data stored on
258      * the HVisible.
259      * This may be set via HVisible methods such as <code>setTextContent</code>
260      * and <code>setGraphicContent</code>. Rendering shall take place within the
261      * bounds returned by the <code>getInsets</code> method.
262      * <p> 
263      * This method is only called from <code>showLook</code> within this
264      * <code>HExtendedLook</code> implementation. It's not the intention to call
265      * this method directly from outside of the <code>HExtendedLook</code>.
266      * <p> 
267      * For looks which draw content (e.g.  {@link org.havi.ui.HTextLook}, {@link
268      * org.havi.ui.HGraphicLook} and {@link org.havi.ui.HAnimateLook}),
269      * if no content is associated with the component, this method does nothing.
270      * <p> 
271      * The {@link org.havi.ui.HExtendedLook#renderVisible}
272      * method should not modify the clipRect (clipping rectangle) of the
273      * <code>Graphics</code> object that is passed to it in a way
274      * which includes any area not part of that original clipRect. If any
275      * modifications are made, the original clipRect shall be restored.
276      * <p> 
277      * The labels of the associated <code>HListElement</code>s shall
278      * be rendered by using the current text layout manager of the
279      * <code>HListGroup</code>. For each visible label is the
280      * <code>render()</code> method of <code>HTextLayoutManager</code> 
281      * called. The position and size per label are specified as insets
282      * relatively to the bounds of <code>HListGroup</code>. Note that
283      * the bounds are independent of any borders of the
284      * <code>HListGroup</code>, but the insets have to include the
285      * borders per element, if any. The look shall use the method
286      * <code>getLabelSize()</code> of <code>HListGroup</code> to
287      * determine the size for each label. If the returned dimension
288      * object has <code>DEFAULT_LABEL_WIDTH</code> for the width and/or
289      * <code>DEFAULT_LABEL_HEIGHT</code> for the height as values, then
290      * this method shall use implementation specific value(s) as default(s)
291      * for the missing dimension(s) instead. If
292      * <code>getTextLayoutManager()</code> returns <code>null</code>,
293      * then labels shall not be rendered.
294      * <p>
295      * If supported, scaling of icons shall reflect the resize mode of the
296      * visible within the area of the respective list element. The look
297      * shall use the method <code>getIconSize()</code> of
298      * <code>HListGroup</code> to determine the size for each icon. If
299      * the returned dimension object has <code>DEFAULT_ICON_WIDTH</code>
300      * for the width and/or <code>DEFAULT_ICON_HEIGHT</code> for the
301      * height as values, then this method shall use implementation specific
302      * value(s) as default(s) for the missing dimension(s) instead. 
303      * <p>
304      * Except for the alignment of labels and sizes of labels and icons, it
305      * is explicitly not defined, how this look arranges icons and labels
306      * within the elements' areas. Additionally, it is an implementation
307      * option to render labels and icons in other sizes than specified, if
308      * the available size per element is smaller or larger than label and
309      * icon size. It is also not defined, how the look presents the current
310      * item and selected items, or the current selection mode. The
311      * elements shall be layed out as specified by
312      * <code>getOrientation()</code> of the associated <code>HListGroup</code>.
313      * <p>
314      * When the associated <code>HListGroup</code> contains more elements than
315      * presentable, the look shall make the user aware of that condition,
316      * e.g. by displaying an additional scrollbar reflecting the current
317      * scroll position. Again, the visible means by which this information
318      * is conveyed is not defined and implementation dependent. It is an
319      * implementation option for <code>HListGroupLook</code> to draw
320      * elements before the scroll position, in order to fill the
321      * available space.
322      * <p>
323      * The behavior of this method, when a subclass overrides the methods
324      * <code>getInsets()</code> or <code>getElementInsets()</code>, is
325      * not defined. Application developers shall not assume that the
326      * corresponding borders will appear as specified.
327      * <p>
328      * The {@link org.havi.ui.HExtendedLook#renderVisible} method is
329      * responsible for painting any implementation specific borders for
330      * each HListElement as well as drawing of an additional scrollbar
331      * if required.
332      *
