62  /** 
63   * Abstract class for X.509 v1 certificates. This provides a standard
64   * way to access all the version 1 attributes of an X.509 certificate.
65   * Attributes that are specific to X.509 v2 or v3 are not available
66   * through this interface. Future API evolution will provide full access to
67   * complete X.509 v3 attributes.
68   * <p>
69   * The basic X.509 format was defined by
70   * ISO/IEC and ANSI X9 and is described below in ASN.1:
71   * <pre>
72   * Certificate  ::=  SEQUENCE  {
73   *     tbsCertificate       TBSCertificate,
74   *     signatureAlgorithm   AlgorithmIdentifier,
75   *     signature            BIT STRING  }
76   * </pre>
77   * <p>
78   * These certificates are widely used to support authentication and
79   * other functionality in Internet security systems. Common applications
80   * include Privacy Enhanced Mail (PEM), Transport Layer Security (SSL),
81   * code signing for trusted software distribution, and Secure Electronic
82   * Transactions (SET).
83   * <p>
84   * These certificates are managed and vouched for by <em>Certificate
85   * Authorities</em> (CAs). CAs are services which create certificates by
86   * placing data in the X.509 standard format and then digitally signing
87   * that data. CAs act as trusted third parties, making introductions
88   * between principals who have no direct knowledge of each other.
89   * CA certificates are either signed by themselves, or by some other
90   * CA such as a "root" CA.
91   * <p>
92   * The ASN.1 definition of <code>tbsCertificate</code> is:
93   * <pre>
94   * TBSCertificate  ::=  SEQUENCE  {
95   *     version         [0]  EXPLICIT Version DEFAULT v1,
96   *     serialNumber         CertificateSerialNumber,
97   *     signature            AlgorithmIdentifier,
98   *     issuer               Name,
99   *     validity             Validity,
100  *     subject              Name,
101  *     subjectPublicKeyInfo SubjectPublicKeyInfo,
102  *     }
103  * </pre>
104  * <p>
105  * Here is sample code to instantiate an X.509 certificate:
106  * <pre> 
107  * InputStream inStream = new FileInputStream("fileName-of-cert");
108  * X509Certificate cert = X509Certificate.getInstance(inStream);
109  * inStream.close();
110  * </pre>
111  * OR
112  * <pre>
113  * byte[] certData = &lt;certificate read from a file, say&gt;
114  * X509Certificate cert = X509Certificate.getInstance(certData);
115  * </pre>
116  * <p>
117  * In either case, the code that instantiates an X.509 certificate
118  * consults the Java security properties file to locate the actual
119  * implementation or instantiates a default implementation.
120  * <p>
121  * The Java security properties file is located in the file named
122  * &lt;JAVA_HOME&gt;/lib/security/java.security, where &lt;JAVA_HOME&gt;
123  * refers to the directory where the JDK was installed.
124  * In the Security properties file, a default implementation
125  * for X.509 v1 may be given such as:
126  * <pre>
127  * cert.provider.x509v1=com.sun.security.cert.internal.x509.X509V1CertImpl
128  * </pre>
129  * <p>
130  * The value of this <code>cert.provider.x509v1</code> property has to be 
131  * changed to instatiate another implementation. If this security
132  * property is not set, a default implementation will be used.
133  * Currently, due to possible security restrictions on access to
134  * Security properties, this value is looked up and cached at class
135  * initialization time and will fallback on a default implementation if
136  * the Security property is not accessible.
137  *
138  * <p><em>Note: The classes in the package <code>javax.security.cert</code>
139  * exist for compatibility with earlier versions of the
140  * Java Secure Sockets Extension (JSSE). New applications should instead
141  * use the standard J2SE certificate classes located in
142  * <code>java.security.cert</code>.</em></p>
143  *
144  * @author Hemma Prafullchandra
145  * @version 1.28
146  * @since 1.4
147  * @see Certificate
148  * @see java.security.cert.X509Extension
149  */

279      * Gets the <code>issuer</code> (issuer distinguished name) value from 
280      * the certificate. The issuer name identifies the entity that signed (and
281      * issued) the certificate. 
282      * 
283      * <p>The issuer name field contains an
284      * X.500 distinguished name (DN).
285      * The ASN.1 definition for this is:
286      * <pre>
287      * issuer    Name<p>
288      *
289      * Name ::= CHOICE { RDNSequence }
290      * RDNSequence ::= SEQUENCE OF RelativeDistinguishedName
291      * RelativeDistinguishedName ::=
292      *     SET OF AttributeValueAssertion
293      *
294      * AttributeValueAssertion ::= SEQUENCE {
295      *                               AttributeType,
296      *                               AttributeValue }
297      * AttributeType ::= OBJECT IDENTIFIER
298      * AttributeValue ::= ANY
299      * </pre>
300      * The <code>Name</code> describes a hierarchical name composed of
301      * attributes, such as country name, and corresponding values, such as US.
302      * The type of the <code>AttributeValue</code> component is determined by
303      * the <code>AttributeType</code>; in general it will be a 
304      * <code>directoryString</code>. A <code>directoryString</code> is usually 
305      * one of <code>PrintableString</code>,
306      * <code>TeletexString</code> or <code>UniversalString</code>.
