42  /** 
43   * A class that allows a very simple, interoperable form of 
44   * compositing.  This is achieved by setting an alpha value for alpha
45   * blending on a color.  Higher alpha values indicate greater opacity
46   * of the color; lower values indicate greater transparency.  The
47   * alpha value will be respected by all instances of java.awt.Graphics 
48   * given to applications.  <p>
49   *
50   * In the final composition between the graphics and video, the
51   * underlying video stream will be alpha-blended with the AWT graphics
52   * plane using that pixel's alpha value by default, i.e.  <em>source
53   * over</em> compositing will be used between the video plane and the
54   * AWT graphics plane by default.  This behavior can be changed using
55   * other APIs, possibly APIs defined outside of Java TV.  <p>
56   *
57   * This API supports up to 256 levels of alpha blending.  However, an
58   * individual graphics system may support fewer levels.  Such systems
59   * will round the alpha value specified in an <code>AlphaColor</code>
60   * constructor to some nearest value when the <code>AlphaColor</code>
61   * instance is used, e.g. rounding to the nearest implemented alpha
62   * value. <p>
63   *
64   * Systems on which alpha blending is not supported will interpret 
65   * alpha values other than 255 as if they were 255  (opaque)
66   * instead.<p>
67   *
68   * The actual color used in rendering will depend on finding the 
69   * best match given the color space available for a given output 
70   * device.<p>
71   *
72   * Within the AWT graphics plane, the actual compositing done will be 
73   * platform-dependent.
74   *
75   *
76   * @see #getAlpha
77   *
78   * @version 1.13, 10/24/05
79   */
