44  /** 
45   * A <code>ServiceContext</code> represents an environment in which
46   * services are presented in a broadcast receiver.  Applications may
47   * use <code>ServiceContext</code> objects to select new services to
48   * be presented.  Content associated with a selected service is
49   * presented by one or more <code>ServiceContentHandler</code> objects
50   * managed by the <code>ServiceContext</code>.<p>
51   *
52   * A <code>ServiceContext</code> can exist in four states -
53   * <em>presenting</em>, <em>not presenting</em>, <em>presentation
54   * pending</em> and <em>destroyed</em>. The initial state is
55   * <em>not presenting</em>. <p>
56   *
57   * The <code>select()</code> method can be called from any state
58   * except <em>destroyed</em>. Assuming no exception is thrown, the
59   * service context then enters the <em>presentation pending</em>
60   * state. No event is generated on this state transition. If a call to
61   * <code>select()</code> completes successfully, either a
62   * <code>NormalContentEvent</code> or an
63   * <code>AlternativeContentEvent</code> is generated and the
64   * <code>ServiceContext</code> moves into the <em>presenting</em>
65   * state.<p>
66   *
67   * If the selection operation fails, a
68   * <code>SelectionFailedEvent</code> is generated.  If the
69   * <code>select()</code> method is called during the <em>presentation
70   * pending</em> state, a <code>SelectionFailedEvent</code> with reason
71   * code <code>INTERRUPTED</code> is generated, and the
72   * <code>ServiceContext</code> will proceed in the <em>presentation
73   * pending</em> state for the most recent <code>select()</code> call.
74   * Otherwise, if the state before the failed select operation was
75   * <em>not presenting</em>, the <code>ServiceContext</code> will
76   * return to that state and a <code>PresentationTerminatedEvent</code>
77   * be generated.
78   * Likewise, if the state before the failed select operation was
79   * <em>presentation pending</em>, the <code>ServiceContext</code> will
80   * move to the <em>not presenting</em> state and generate a
81   * <code>PresentationTerminatedEvent</code>.
82   * If the state before the failed select operation was
83   * <em>presenting</em>, it will attempt to return to that previous
84   * state, which can result in a <code>NormalContentEvent</code> or
85   * <code>AlternativeContentEvent</code> if this is possible or a
86   * <code>PresentationTerminatedEvent</code> if it is not possible.<p>
87   *
88   * The <em>not presenting</em> state is entered due to service
89   * presentation being stopped which is reported by a
90   * {@link PresentationTerminatedEvent}. Service
91   * presentation stops when an application calls the
92   * <code>stop()</code> method or
93   * when continued presentation becomes impossible -- due, for example,
94   * to a a change in the environment or a failed service selection attempt.
95   * <p>
96   *
97   * The <em>destroyed</em> state is entered by calling the
98   * <code>destroy()</code> method, and is signaled by a
99   * <code>ServiceContextDestroyedEvent</code>. Once this state is
100  * entered, the <code>ServiceContext</code> can no longer be used for
101  * any purpose.  A destroyed <code>ServiceContext</code> will be
102  * eligible for garbage collection once all references to it by any
103  * applications have been removed.<p>
104  *
105  * Note that the ability to select a service for presentation does not
106  * imply exclusive rights to the resources required for that
107  * presentation.  Subsequent attempts to select the same service may
108  * fail.<p>
109  *
110  * Applications may also use this interface to register for events
111  * associated with <code>ServiceContext</code> state changes.<p>
112  *
113  * @see javax.tv.service.Service
114  * @see ServiceContentHandler
115  *
116  * @see NormalContentEvent
117  * @see AlternativeContentEvent
118  * @see SelectionFailedEvent
119  * @see PresentationTerminatedEvent
120  * @see ServiceContextDestroyedEvent
121  * @see ServiceContextListener 
122  */
123 public interface ServiceContext

126     /** 
127      * Selects a service to be presented in this
128      * <code>ServiceContext</code>.  If the <code>ServiceContext</code>
129      * is already presenting content, the new selection replaces the
130      * content being presented. If the <code>ServiceContext</code> is
131      * not presenting, successful conclusion of this operation results in
132      * the <code>ServiceContext</code> entering the <em>presenting</em>
133      * state. <p>
134      *
135      * This method is asynchronous and successful completion of the
136      * selection is notified by either a <code>NormalContentEvent</code>
137      * or a <code>AlternativeContentEvent</code>.  If an exception is
138      * thrown when this method is called, the state of the service
139      * context does not change. In such a case, any service being
140      * presented before this method was called will continue to be
141      * presented.<p>
142      *
143      * If the selection process fails after this method returns, a
144      * <code>SelectionFailedEvent</code> will be generated. In this
145      * case, the system will attempt to return to presenting the
146      * original service (if any). If this is not possible (due, for
147      * example, to issues such as tuning or CA) the
148      * <code>ServiceContext</code> will enter the <em>not
149      * presenting</em> state and a
150      * <code>PresentationTerminatedEvent</code> will be generated.<p>
151      *
152      * If the <code>ServiceContext</code> is currently presenting a
153      * service and components of the current service are also to be
154      * presented in the newly selected service, these components will
155      * continue to be presented and will not be restarted.  If the
156      * calling application is not a part of the newly selected service
157      * and the application lifecycle policy on the receiver dictates
158      * that the calling application be terminated, termination of the
159      * application will be affected through the application lifecycle
160      * API.
161      * <p>
162      * If the provided <code>Service</code> is transport-independent,
163      * this method will resolve the <code>Service</code> to a
164      * transport-dependent <code>Service</code> before performing the
165      * selection. The actual <code>Service</code> selected will be
166      * reported through the <code>getService()</code> method.<p>
167      *
168      * Successful completion of a select operation using this method
169      * provides <code>ServiceContentHandler</code> instances for all
170      * components that are signaled as "auto-start" in the selected
171      * service.  Upon entering the <em>presenting</em> state, these
172      * <code>ServiceContentHandler</code> instances will have begun
173      * presenting their respective content;
174      * <code>ServiceMediaHandler</code> instances will be in the
175      * <em>started</em> state.
176      * <p>
177      * <span style="color: red;">
178      * If the <code>Service</code> that is currently selected in this
179      * <code>ServiceContext</code> is re-selected, its auto-start
180      * components are ignored and no change is made to the content
181      * being presented.
182      * A {@link NormalContentEvent} is generated immediately.
183      * </span>
184      * <p>
185      * If a service with no auto-start components is selected, the
186      * <code>ServiceContext</code> generates a
187      * <code>NormalContentEvent</code> and enters the <em>presenting</em>
188      * state, but no <code>ServiceContentHandler</code>s are created.
189      * 
190      * @param selection The <code>Service</code> the service to be
191      * selected.
192      *
193      * @throws SecurityException If the caller does not have
194      * <code>SelectPermission(selection.getLocator(), "own")</code>.
195      *
196      * @throws IllegalStateException If the <code>ServiceContext</code>
197      * has been destroyed.
198      *
199      * @see NormalContentEvent
200      * @see AlternativeContentEvent
201      * @see SelectionFailedEvent
202      * @see PresentationTerminatedEvent
203      * @see javax.tv.service.Service
204      * @see ServiceContext#getService
205      * @see ServiceContentHandler
206      * @see ServiceContext#destroy 
207      * @see SelectPermission
208      */
209     public void select(Service selection) throws SecurityException;

211     /** 
212      * Selects content by specifying the parts of a service to be
213      * presented.  If the <code>ServiceContext</code> is
214      * already presenting content, the new selection replaces the
215      * content being presented. If the
216      * <code>ServiceContext</code> is not presenting,
217      * successful conclusion of this operation results in the
218      * <code>ServiceContext</code> entering the
219      * <em>presenting</em> state. <p>
220      *
221      * If failure of the selection operation
222      * is determined in the execution of this method, an
223      * exception is generated and the state of the
224      * <code>ServiceContext</code> does not change.
225      * (Note that such sychronous failure may result from a failure
226      * to select the content corresponding to <em>any</em> locator
227      * in the array.)
228      * In this case, any
229      * service being presented before this method was called will
230      * continue to be presented.
231      * <p>
232      * If the method returns successfully, then the selection operation
233      * proceeds asynchronously.
234      * Successful completion of the operation will be signaled by either
235      * a single
236      * <code>NormalContentEvent</code> or a single
237      * <code>AlternativeContentEvent</code>.  
238      * If the content corresponding to any of the specified locators can
239      * be successfully presented then the selection operation is
240      * considered successful, i.e., even if attempts to present content
241      * corresponding to some locators fail.
242      * <p>
243      * If failure of the selection is determined asynchronously,
244      * a <code>SelectionFailedEvent</code> will be
245      * generated indicating the reason for the failure.
246      * (Note that such asychronous failure must result from a failure
247      * to select the content corresponding to <em>all</em> locators
248      * in the array.)
249      * If the selection fails due to multiple reasons, then the
250      * reason code will be <code>SelectionFailedEvent.OTHER</code>.
251      * After a <code>SelectionFailedEvent</code> is generated,
252      * the <code>ServiceContext</code> will
253      * try to return to presenting the original service, if any. If
254      * this is not possible (due, for example, to issues such as tuning
255      * or conditional access), then the <code>ServiceContext</code>
256      * will enter the <em>not presenting</em> state and a
257      * <code>PresentationTerminatedEvent</code> will be generated.<p>
258      *
259      * If the <code>ServiceContext</code> is currently presenting a service and
260      * components of the current service are also to be presented in the
261      * newly selected content, these components will continue to be
262      * presented and will not be restarted.  If the calling application
263      * is not a part of the newly selected content and the application
264      * lifecycle policy on the receiver dictates that the calling
265      * application be terminated, termination of the application
266      * will be affected through the application lifecycle API.<p>
267      *
268      * Successful completion of a select operation using this method
269      * provides <code>ServiceContentHandler</code> instances for all
270      * components indicated in the <code>components</code>
271      * parameter that were successfully presented.
272      * Upon entering the <em>presenting</em> state, these
273      * <code>ServiceContentHandler</code> instances will have begun
274      * presenting their respective content;
275      * <code>ServiceMediaHandler</code> instances will be in the
276      * <em>started</em> state.  This method will not provide
277      * <code>ServiceContentHandler</code> instances for service
278      * components for which a locator is not specified.
279      * <p>
280      * For locators referencing a service component that is not
281      * selectable and for which there is no <code>ServiceContentHandler</code>
282      * defined, <code>InvalidLocatorException</code> is thrown if the
283      * condition can be detected without causing the method to block.
284      * For example, implementations of this method do not block while
285      * waiting for network access or tuning if such is required to determine
286      * whether a specific component can be succesfully presented.
287      * When such an error condition is discovered after this method returns,
288      * a <code>SelectionFailedEvent</code> with reason code
289      * <code>MISSING_HANDLER</code> is generated.
290      * <p>
291      * In addition, selection of content involving service-bound xlets is
292      * subject to some <a href="doc-files/select-xlets.html">special rules</a>.
293      *
294      * @param components An array of <code>Locator</code> instances
295      * representing the parts of this service to be selected.  Each
296      * array element must be a locator to either a
297      * <code>ServiceComponent</code> or content within a service
298      * component (such as an Xlet).
299      *
300      * @throws InvalidLocatorException If a <code>Locator</code> provided
301      * does not reference a selectable service component or selectable
302      * content within a service component.
303      *
304      * @throws InvalidServiceComponentException If a
305      * specified service component must be presented in conjunction
306      * with another service component not contained in
307      * <code>components</code>, if the specified
308      * components are not all members of the same service, or if the
309      * specified set of components cannot be presented as a coherent
310      * whole.
311      *
312      * @throws SecurityException If, for any valid <code>i</code>, the
313      * caller does not have
314      * <code>SelectPermission(components[i], "own")</code>.
315      *
316      * @throws IllegalStateException If the <code>ServiceContext</code>
317      * has been destroyed.
318      *
319      * @throws IllegalArgumentException If <code>components</code> is a
320      * zero-length array.
321      *
322      * @see NormalContentEvent
323      * @see AlternativeContentEvent
324      * @see SelectionFailedEvent
325      * @see PresentationTerminatedEvent
326      * @see ServiceContentHandler
327      * @see javax.tv.service.navigation.ServiceComponent
328      * @see SelectionPermission
329      */
330     public void select(Locator[] components)
331         throws InvalidLocatorException, InvalidServiceComponentException,
332         SecurityException;

334     /** 
335      * Causes the <code>ServiceContext</code> to stop presenting content
336      * and enter the <em>not presenting</em> state.  Resources used
337      * in the presentation will be released, associated
338      * <code>ServiceContentHandlers</code> will cease presentation
339      * (<code>ServiceMediaHandlers</code> will no longer be in the
340      * <em>started</em> state), and a
341      * <code>PresentationTerminatedEvent</code> will be posted.<p>
342      *
343      * This operation completes asynchronously. No action is performed
344      * if the <code>ServiceContext</code> is already in the <em>not
345      * presenting</em> state.
346      *
347      * @throws SecurityException If the caller does not have
348      * <code>ServiceContextPermission("stop", "own")</code>.
349      *
350      * @throws IllegalStateException If the <code>ServiceContext</code>
351      * has been destroyed.
352      *
353      * @see ServiceContextPermission
354      */
355     public void stop() throws SecurityException;

357     /** 
358      * Causes the <code>ServiceContext</code> to release all resources
359      * and enter the <em>destroyed</em> state. This method indicates
360      * that the the <code>ServiceContext</code> must cease further
361      * activity, and that it will no longer be used.  After completion
362      * of this method, <code>ServiceMediaHandler</code> instances
363      * associated with this <code>ServiceContext</code> will have become
364      * <em>unrealized</em> or will have been closed.
365      *
366      * If the <code>ServiceContext</code> is currently in the
367      * <em>presenting</em> or <em>presentation pending</em> state, this
368      * method will first stop the <code>ServiceContext</code>, causing a
369      * <code>PresentationTerminatedEvent</code> to be posted.  After the
370      * <code>ServiceContext</code> has moved to the <em>destroyed</em>
371      * state, a <code>ServiceContextDestroyedEvent</code> will be
372      * posted.<p>
373      *
374      * This operation completes asynchronously.  No action is performed
375      * if the <code>ServiceContext</code> is already in the
376      * <em>destroyed</em> state.
377      *
378      * @throws SecurityException If the caller does not have permission
379      * to call {@link #stop} on this <code>ServiceContext</code>,
380      * or if the caller does
381      * not have <code>ServiceContextPermission("destroy", "own")</code>.
382      *
383      * @see #stop
384      * @see ServiceContextPermission
385      */
386     public void destroy() throws SecurityException;

389      * Reports the current collection of ServiceContentHandlers. The order

401      * </ul>
402      *
403      * @throws SecurityException If the caller does not have
404      * <code>ServiceContextPermission("getServiceContentHandlers",
405      * "own")</code>.
406      *
407      * @return The current <code>ServiceContentHandler</code> instances.
408      *
409      * @throws IllegalStateException If the <code>ServiceContext</code>
410      * has been destroyed.
411      *
412      * @see ServiceContextPermission
413      */

417     /** 
418      * Reports the <code>Service</code> being presented in this

426      *
427      * @return The service currently being presented.
