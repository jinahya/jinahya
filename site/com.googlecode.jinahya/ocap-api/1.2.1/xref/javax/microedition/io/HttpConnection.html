38  
39  import java.io.IOException;
40  import java.io.InputStream;
41  import java.io.OutputStream;
42  import java.lang.String;

45  /** 
46   * This interface defines the necessary methods and constants
47   * for an HTTP connection.
48   * <P>
49   * HTTP is a request-response protocol in which the parameters of
50   * request must be set before the request is sent.
51   * The connection exists in one of three states:
52   * <UL>
53   * <LI> Setup, in which the request parameters can be set
54   * <LI> Connected, in which request parameters have been sent and the
55   *      response is expected
56   * <LI> Closed, the final state, in which the HTTP connection as been
57   * terminated
58   * </UL>
59   * The following methods may be invoked only in the Setup state:
60   * <UL>
61   * <LI> <CODE> setRequestMethod</CODE>
62   * <LI> <CODE> setRequestProperty</CODE>
63   * </UL>
64   *
65   * The transition from Setup to Connected is caused by any method that
66   * requires data to be sent to or received from the server. <p>
67   *
68   * The following methods cause the transition to the Connected state when
69   * the connection is in Setup state.
70   * <UL>
71   * <LI> <CODE> openInputStream</CODE>
72   * <LI> <CODE> openDataInputStream</CODE>
73   * <LI> <CODE> getLength</CODE>
74   * <LI> <CODE> getType</CODE>
75   * <LI> <CODE> getEncoding</CODE>
76   * <LI> <CODE> getHeaderField</CODE>
77   * <LI> <CODE> getResponseCode</CODE>
78   * <LI> <CODE> getResponseMessage</CODE>
79   * <LI> <CODE> getHeaderFieldInt</CODE>
80   * <LI> <CODE> getHeaderFieldDate</CODE>
81   * <LI> <CODE> getExpiration</CODE>
82   * <LI> <CODE> getDate</CODE>
83   * <LI> <CODE> getLastModified</CODE>
84   * <LI> <CODE> getHeaderField</CODE>
85   * <LI> <CODE> getHeaderFieldKey</CODE>
86   * </UL>
87   * <P>
88   * The following methods may be invoked while the connection is in Setup or
89   * Connected state.
90   * <UL>
91   * <LI> <CODE> close</CODE>
92   * <LI> <CODE> getRequestMethod</CODE>
93   * <LI> <CODE> getRequestProperty</CODE>
94   * <LI> <CODE> getURL</CODE>
95   * <LI> <CODE> getProtocol</CODE>
96   * <LI> <CODE> getHost</CODE>
97   * <LI> <CODE> getFile</CODE>
98   * <LI> <CODE> getRef</CODE>
99   * <LI> <CODE> getPort</CODE>
100  * <LI> <CODE> getQuery</CODE>
101  * </UL>
102  *
103  * <P>
104  * After an output stream has been opened by the
105  * <code>openOutputStream</code> or <code>openDataOutputStream</code>
106  * methods, attempts to change the request parameters via
107  * <code>setRequestMethod</code> or the <code>setRequestProperty</code>
108  * are ignored. Once the request parameters have been sent,
109  * these methods will throw an <code>IOException</code>.
110  *
111  * When an output stream is closed via the
112  * <code>OutputStream.close</code> or <code>DataOutputStream.close</code>
113  * methods, the connection enters the Connected state.
114  * When the output stream is flushed via the  
115  * <code>OutputStream.flush</code> or <code>DataOutputStream.flush</code>
116  * methods, the request parameters MUST be sent along with any data
117  * written to the stream.
118  * </P>
119  *
120  * The transition to Closed state from any other state is caused by the
121  * <code>close</code> method and the closing all of the streams that were
122  * opened from the connection. 
123  * <P>
124  * <STRONG>Example using StreamConnection</STRONG>
125  * <p>
126  * Simple read of a URL using <code>StreamConnection</code>.
127  * No HTTP specific behavior is needed or used.
128  * (<strong>Note:</strong> this example ignores all HTTP response
129  * headers and the HTTP response code. Since a proxy or server
130  * may have sent an error response page, an application
131  * can not distinquish which data is retreived in
132  * the <code>InputStream</code>.)
133  *
134  * <p>
135  * <code>Connector.open</code> is used to open URL and a
136  * <code>StreamConnection</code> is returned.
137  * From the <code>StreamConnection</code> the
138  * <code>InputStream</code> is opened.
139  * It is used to read every character until end of file (-1).
140  * If an exception is thrown the connection and stream are closed.
141  * <p>
142  * <pre>
143  *     void getViaStreamConnection(String url) throws IOException {
144  *         StreamConnection c = null;
145  *         InputStream s = null;
146  *         try {
147  *             c = (StreamConnection)Connector.open(url);
148  *             s = c.openInputStream();
149  *             int ch;
150  *             while ((ch = s.read()) != -1) {
151  *                 ...
152  *             }
153  *         } finally {
154  *             if (s != null)
155  *                 s.close();
156  *             if (c != null)
157  *                 c.close();
158  *         }
159  *     }
160  * </pre>
161  * <p>
162  * <STRONG>Example using ContentConnection</STRONG>
163  * <p>
164  * Simple read of a URL using <code>ContentConnection</code>.
165  * No HTTP specific behavior is needed or used.
166  * <p>
167  * <code>Connector.open</code> is used to open url and
168  * a <code>ContentConnection</code> is returned.
169  * The <code>ContentConnection</code> may be able to provide the length.
170  * If the length is available, it is used to read the data in bulk.
171  * From the <code>ContentConnection</code> the
172  * <code>InputStream</code> is opened.
173  * It is used to read every character until end of file (-1).
174  * If an exception is thrown the connection and stream are closed.
175  * <p>
176  * <pre>
177  *     void getViaContentConnection(String url) throws IOException {
178  *         ContentConnection c = null;
179  *         DataInputStream is = null;
180  *         try {
181  *             c = (ContentConnection)Connector.open(url);
182  *             int len = (int)c.getLength();
183  *             dis = c.openDataInputStream();
184  *             if (len &gt; 0) {
185  *                 byte[] data = new byte[len];
186  *                 dis.readFully(data);
187  *             } else {
188  *                 int ch;
189  *                 while ((ch = dis.read()) != -1) {
190  *                     ...
191  *                 }
192  *             }
193  *         } finally {
194  *             if (dis != null)
195  *                 dis.close();
196  *             if (c != null)
197  *                 c.close();
198  *         }
199  *     }
200  * </pre>
201  * <p>
202  * <STRONG>Example using HttpConnection</STRONG>
203  * <p>
204  * Read the HTTP headers and the data using <code>HttpConnection</code>.
205  * <p>
206  * <code>Connector.open</code> is used to open url and a
207  * <code>HttpConnection</code> is returned.
208  * The HTTP headers are read and processed.
209  * If the length is available, it is used to read the data in bulk.
210  * From the <code>HttpConnection</code> the
211  * <code>InputStream</code> is opened.
212  * It is used to read every character until end of file (-1).
213  * If an exception is thrown the connection and stream are closed.
214  * <p>
215  * <pre>
216  *     void getViaHttpConnection(String url) throws IOException {
217  *         HttpConnection c = null;
218  *         InputStream is = null;
219  *         int rc;
220  *
221  *         try {
222  *             c = (HttpConnection)Connector.open(url);
223  *
224  *             // Getting the response code will open the connection,
225  *             // send the request, and read the HTTP response headers.
226  *             // The headers are stored until requested.
227  *             rc = c.getResponseCode();
228  *             if (rc != HttpConnection.HTTP_OK) {
229  *                 throw new IOException("HTTP response code: " + rc);
230  *             }
231  *
232  *             is = c.openInputStream();
233  *
234  *             // Get the ContentType
235  *             String type = c.getType();
236  *
237  *             // Get the length and process the data
238  *             int len = (int)c.getLength();
239  *             if (len &gt; 0) {
240  *                 int actual = 0;
241  *                 int bytesread = 0 ;
242  *                 byte[] data = new byte[len];
243  *                 while ((bytesread != len) && (actual != -1)) {
244  *                    actual = is.read(data, bytesread, len - bytesread);
245  *                    bytesread += actual;
246  *                 }
247  *             } else {
248  *                 int ch;
249  *                 while ((ch = is.read()) != -1) {
250  *                     ...
251  *                 }
252  *             }
253  *         } catch (ClassCastException e) {
254  *             throw new IllegalArgumentException("Not an HTTP URL");
255  *         } finally {
256  *             if (is != null)
257  *                 is.close();
258  *             if (c != null)
259  *                 c.close();
260  *         }
261  *     }
262  * </pre>
263  * <p>
264  * <STRONG>Example using POST with HttpConnection</STRONG>
265  * <p>
266  *
267  * Post a request with some headers and content to the server and
268  * process the headers and content.
269  * <p>
270  * <code>Connector.open</code> is used to open url and a
271  * <code>HttpConnection</code> is returned.
272  * The request method is set to POST and request headers set.
273  * A simple command is written and flushed.
274  * The HTTP headers are read and processed.
275  * If the length is available, it is used to read the data in bulk.
276  * From the <code>HttpConnection</code> the
277  * <code>InputStream</code> is opened.
278  * It is used to read every character until end of file (-1).
279  * If an exception is thrown the connection and stream is closed.
280  * <p>
281  * <pre>
282  *    void postViaHttpConnection(String url) throws IOException {
283  *        HttpConnection c = null;
284  *        InputStream is = null;
285  *        OutputStream os = null;
286  *        int rc;
287  *
288  *        try {
289  *            c = (HttpConnection)Connector.open(url);
290  *
291  *            // Set the request method and headers
292  *            c.setRequestMethod(HttpConnection.POST);
293  *            c.setRequestProperty("If-Modified-Since",
294  *                "29 Oct 1999 19:43:31 GMT");
295  *            c.setRequestProperty("User-Agent",
296  *                "Profile/MIDP-2.0 Configuration/CLDC-1.0");
297  *            c.setRequestProperty("Content-Language", "en-US");
298  *
299  *            // Getting the output stream may flush the headers
300  *            os = c.openOutputStream();
301  *            os.write("LIST games\n".getBytes());
302  *            os.flush();           // Optional, getResponseCode will flush
303  *
304  *            // Getting the response code will open the connection,
305  *            // send the request, and read the HTTP response headers.
306  *            // The headers are stored until requested.
307  *            rc = c.getResponseCode();
308  *            if (rc != HttpConnection.HTTP_OK) {
309  *                throw new IOException("HTTP response code: " + rc);
310  *            }
311  *
312  *            is = c.openInputStream();
313  *
314  *            // Get the ContentType
315  *            String type = c.getType();
316  *            processType(type);
317  *
318  *            // Get the length and process the data
319  *            int len = (int)c.getLength();
320  *            if (len &gt; 0) {
321  *                 int actual = 0;
322  *                 int bytesread = 0 ;
323  *                 byte[] data = new byte[len];
324  *                 while ((bytesread != len) && (actual != -1)) {
325  *                    actual = is.read(data, bytesread, len - bytesread);
326  *                    bytesread += actual;
327  *                 }
328  *                process(data);
329  *            } else {
330  *                int ch;
331  *                while ((ch = is.read()) != -1) {
332  *                    process((byte)ch);
333  *                }
334  *            }
335  *        } catch (ClassCastException e) {
336  *            throw new IllegalArgumentException("Not an HTTP URL");
337  *        } finally {
338  *            if (is != null)
339  *                is.close();
340  *            if (os != null)
341  *                os.close();
342  *            if (c != null)
343  *                c.close();
344  *        }
345  *    }
346  * </pre>
347  * <hr>
348  * <p>
349  * <STRONG>Simplified Stream Methods on Connector</STRONG>
350  * <p>
351  * Please note the following: The
352  * <code>Connector</code> class defines the following
353  * convenience methods for retrieving an input or output stream directly
354  * for a specified URL:
355  *
356  * <UL>
357  * <LI> <CODE> InputStream openInputStream(String url) </CODE>
358  * <LI> <CODE> DataInputStream openDataInputStream(String url) </CODE>
359  * <LI> <CODE> OutputStream openOutputStream(String url) </CODE>
360  * <LI> <CODE> DataOutputStream openDataOutputStream(String url) </CODE>
361  * </UL>
362  *
363  * Please be aware that using these methods implies certain restrictions.
364  * You will not get a reference to the actual connection, but rather just
365  * references to the input or output stream of the connection. Not having
366  * a reference to the connection means that you will not be able to manipulate
367  * or query the connection directly. This in turn means that you will not
368  * be able to call any of the following methods:
369  *
370  * <UL>
371  * <LI> <CODE> getRequestMethod() </CODE>
372  * <LI> <CODE> setRequestMethod() </CODE>
373  * <LI> <CODE> getRequestProperty() </CODE>
374  * <LI> <CODE> setRequestProperty() </CODE>
375  * <LI> <CODE> getLength() </CODE>
376  * <LI> <CODE> getType() </CODE>
377  * <LI> <CODE> getEncoding() </CODE>
378  * <LI> <CODE> getHeaderField() </CODE>
379  * <LI> <CODE> getResponseCode() </CODE>
380  * <LI> <CODE> getResponseMessage() </CODE>
381  * <LI> <CODE> getHeaderFieldInt</CODE>
382  * <LI> <CODE> getHeaderFieldDate</CODE>
383  * <LI> <CODE> getExpiration</CODE>
384  * <LI> <CODE> getDate</CODE>
385  * <LI> <CODE> getLastModified</CODE>
386  * <LI> <CODE> getHeaderField</CODE>
387  * <LI> <CODE> getHeaderFieldKey</CODE>
388  * </UL>
389  * @since MIDP 1.0
390  */
