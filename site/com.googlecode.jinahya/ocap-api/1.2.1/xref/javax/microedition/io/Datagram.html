41  /** 
42   * This class defines an abstract interface for datagram packets.
43   * The implementations of this interface hold data to be
44   * sent or received from a <code>DatagramConnection</code> object.
45   * <p>
46   * Since this is an interface class, the internal structure
47   * of the datagram packets is not defined here.  However, it is
48   * assumed that each implementation of this interface will
49   * provide the following fields / state variables (the actual
50   * implementation and the names of these fields may vary):
51   * <ul>
52   * <li><i>buffer</i>: the internal buffer in which data is stored
53   * <li><i>offset</i>: the read/write offset for the internal buffer
54   * <li><i>length</i>: the length of the data in datagram packet
55   * <li><i>address</i>: the destination or source address
56   * <li><i>read/write pointer</i>: a pointer that is added to the
57   *   <i>offset</i> to point to the current data location during a
58   *   read or write operation
59   * </ul>
60   * <p>
61   * <strong>Reading and Writing</strong>
62   * <p>
63   * The <code>Datagram</code> interface extends interfaces
64   * <code>DataInput</code> and <code>DataOutput</code> in order
65   * to provide a simple way to read and write binary data in and out of
66   * the datagram buffer instead of using <code>getData</code> and
67   * <code>setData</code> methods. Writing automatically increments
68   * <i>length</i> and reading will continue while the <i>read/write pointer</i>
69   * is less than <i>length</i>. Before any writing is done reset must be called.
70   * If <code>setData()</code> is to be used when reading or writing, any value
71   * for the <code>offset</code> parameter other than 0 is not supported.
72   * <p>
73   * For example to write to datagram:
74   * <pre>
75   *    datagram = connection.newDatagram(max);
76   *
77   *    // Reset prepares the datagram for writing new message.
78   *    datagram.reset();
79   *
80   *    // writeUTF automatically increases the datagram length.
81   *    datagram.writeUTF("hello world");
82   *
83   *    connection.send(datagram);
84   * </pre>
85   * For example to read from a datagram (single use only):
86   * <pre>
87   *    datagram = connection.newDatagram(max);
88   *
89   *    connection.receive(datagram);
90   *
91   *    message = datagram.readUTF();
92   * </pre>
93   * <strong>Reusing Datagrams</strong>
94   * <p>
95   * It should be noted the <i>length</i> above is returned
96   * from <code>getLength</code> and can have different meanings at different
97   * times. When sending <i>length</i> is the number of bytes to send. Before
98   * receiving <i>length</i> is the maximum number of bytes to receive.
99   * After receiving <i>length</i> is the number of bytes that were received.
100  * So when reusing a datagram to receive after sending or receiving, length
101  * must be set back to the maximum using <code>setLength</code>.
102  * <pre>
103  *    datagram = connection.newDatagram(max);
104  *
105  *    while (notDone) {
106  *
107  *        // The last receive in the loop changed the length
108  *        // so put it back to the maximum length.
109  *        datagram.setLength(max);
110  *        connection.receive(datagram);
111  *
112  *        data = datagram.getData();
113  *        bytesReceived = datagram.getLength();
114  *
115  *        // process datagram ...
116  *    }
117  * </pre>
118  * When reading instead of using <code>getData</code> the <code>reset</code>
119  * method must be used.
120  * <pre>
121  *    datagram = connection.newDatagram(max);
122  *
123  *    while (notDone) {
124  *
125  *        // The last read in the loop changed the read pointer
126  *        // so reset the pointer.
127  *        datagram.reset();
128  *        datagram.setLength(max);
129  *        connection.receive(datagram);
130  *
131  *        message = datagram.readUTF(message);
132  *
133  *        // process message ...
134  *    }
135  * </pre>
136  * For example to reread a datagram:
137  * <pre>
138  *    connection.receive(datagram);
139  *
140  *    message = datagram.readUTF(message);
141  *
142  *    len = datagram.getLength();
143  *
144  *    datagram.reset();
145  *
146  *    datagram.setLength(len);
147  *
148  *    copy = datagram.readUTF(message);
149  * </pre>
150  *
151  * @author  Brian Modra, Nik Shaylor, Stephen Flores
152  * @version 12/17/01 (CLDC 1.1)
153  * @since   CLDC 1.0
154  */

264      */
