42  /** 
43   * This interface defines the necessary methods
44   * and constants to establish a secure network connection.
45   * The URI format with scheme <CODE>https</CODE> when passed to
46   * <CODE>Connector.open</CODE> will return a 
47   * <code>HttpsConnection</code>.
48   * <A HREF="http://www.ietf.org/rfc/rfc2818.txt">RFC 2818</A> 
49   * defines the scheme.
50   * <p>
51   * A secure connection MUST be implemented by one or more
52   * of the following specifications:
53   * <UL>
54   * <LI>HTTP over TLS as documented in
55   *  <A HREF="http://www.ietf.org/rfc/rfc2818.txt">RFC 2818</A>
56   * and TLS Protocol Version 1.0 as specified in
57   * <A HREF="http://www.ietf.org/rfc/rfc2246.txt">RFC 2246</A>.
58   *
59   * <LI>SSL V3 as specified in
60   *    <A HREF="http://home.netscape.com/eng/ssl3/draft302.txt">
61   *    The SSL Protocol Version 3.0</A>
62   * </LI>
63   *
64   * <LI>WTLS as specified in
65   * 	<A HREF="http://www.wapforum.org/what/technical_1_2_1.htm">
66   * 	WAP Forum Specifications June 2000 (WAP 1.2.1) conformance release</A>
67   * 	- Wireless Transport Layer Security document WAP-199.
68   * <LI>WAP(TM) TLS Profile and Tunneling Specification as specified
69   *	in <A HREF="http://www.wapforum.com/what/technical.htm">
70   *	WAP-219-TLS-20010411-a</A>
71   * </UL>
72   * <p>
73   * HTTPS is the secure version of HTTP (IETF RFC2616), 
74   * a request-response protocol in which the parameters of the request must 
75   * be set before the request is sent.
76   * <p>
77   * In addition to the normal <code>IOExceptions</code> that may occur during 
78   * invocation of the various methods that cause a transition to the Connected
79   * state, <code>CertificateException</code>
80   * (a subtype of <code>IOException</code>) may be thrown to indicate various
81   * failures related to establishing the secure link.  The secure link
82   * is necessary in the <code>Connected</code> state so the headers
83   * can be sent securely. The secure link may be established as early as the
84   * invocation of <code>Connector.open()</code> and related  
85   * methods for opening input and output streams and failure related to
86   * certificate exceptions may be reported.
87   *
88   * </p><br>
89   * <b>Example</b><br>
90   *
91   * <p>Open a HTTPS connection, set its parameters, then read the HTTP
92   * response.</p>
93   *
94   * <code>Connector.open</code> is used to open the URL 
95   * and an <code>HttpsConnection</code> is returned.
96   * The HTTP
97   * headers are read and processed. If the length is available, it is used
98   * to read the data in bulk. From the 
99   * <code>HttpsConnection</code> the <code>InputStream</code> is
100  * opened. It is used to read every character until end of file (-1). If
101  * an exception is thrown the connection and stream are closed.
102  * 
103  * <code><PRE>
104  *     void getViaHttpsConnection(String url) 
105  *            throws CertificateException, IOException {
106  *         HttpsConnection c = null;
107  *         InputStream is = null;
108  *         try {
109  *             c = (HttpsConnection)Connector.open(url);
110  * 
111  *             // Getting the InputStream ensures that the connection
112  *             // is opened (if it was not already handled by
113  *             // Connector.open()) and the SSL handshake is exchanged,
114  *             // and the HTTP response headers are read.
115  *             // These are stored until requested.
116  *             is = c.openDataInputStream();
117  * 
118  *             if c.getResponseCode() == HttpConnection.HTTP_OK) {
119  *                 // Get the length and process the data
120  *                 int len = (int)c.getLength();
121  *                 if (len &gt; 0) {
122  *                     byte[] data = new byte[len];
123  *                     int actual = is.readFully(data);
124  *                     ...
125  *                 } else {
126  *                     int ch;
127  *                     while ((ch = is.read()) != -1) {
128  *                         ...
129  *                     }
130  *                 }
131  *             } else {
132  *               ...
133  *             }
134  *         } finally {
135  *             if (is != null)
136  *                 is.close();
137  *             if (c != null)
138  *                 c.close();
139  *         }
140  *     }
141  * </PRE> </code>
142  * @see CertificateException
143  * @since MIDP 2.0
144  */
145 public interface HttpsConnection extends HttpConnection
146 {
147 
148     /** 
149      * Return the security information associated with this
150      * successfully opened connection.
151      * If the connection is still in <CODE>Setup</CODE> state then
152      * the connection is initiated to establish the secure connection
153      * to the server.  The method returns when the connection is
154      * established and the <CODE>Certificate</CODE> supplied by the
155      * server has been validated.
156      * The <CODE>SecurityInfo</CODE> is only returned if the
157      * connection has been successfully made to the server.
158      *
159      * @return the security information associated with this open connection.
160      *
161      * @exception IOException if an arbitrary connection failure occurs
162      */
163     public SecurityInfo getSecurityInfo() throws IOException;
164 
165     /** 
166      * Returns the network port number of the URL for this
167      * <code>HttpsConnection</code>.
168      *
169      * @return  the network port number of the URL for this
170      * <code>HttpsConnection</code>.
171      * The default HTTPS port number (443) is returned if there was
172      * no port number in the string passed to <code>Connector.open</code>.
173      */
174     public int getPort();
175 }
