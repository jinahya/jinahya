28  /**
29   *  The XMLStreamReader interface allows forward, read-only access to XML.
30   *  It is designed to be the lowest level and most efficient way to
31   *  read XML data.
32   *
33   * <p> The XMLStreamReader is designed to iterate over XML using
34   * next() and hasNext().  The data can be accessed using methods such as 
35   * getEventType(), getNamespaceURI(), getLocalName() and getText();
36   *
37   * <p> The <a href="#next()">next()</a> method causes the reader to read the 
38   * next parse event. The next() method returns an integer which identifies the 
39   * type of event just read.
40   * <p> The event type can be determined using <a href="#getEventType()">
41   * getEventType()</a>.
42   * <p> Parsing events are defined as the XML Declaration, a DTD,
43   * start tag, character data, white space, end tag, comment,
44   * or processing instruction.  An attribute or namespace event may be 
45   * encountered at the root level of a document as the result of a query 
46   * operation.
47   *
48   * <p>For XML 1.0 compliance an XML processor must pass the
49   * identifiers of declared unparsed entities, notation declarations and their
50   * associated identifiers to the application.  This information was
51   * provided through the property API on this interface.
52   * The following two properties allowed access to this information:
53   * "javax.xml.stream.notations" and "javax.xml.stream.entities".
54   * These properties are not supported in the JSR 280 subset of StAX;
55   * calls to <code>getProperty("javax.xml.stream.notations");</code>
56   * or <code>getProperty("javax.xml.stream.entities");</code> will
57   * always return <code>null</code>.
58   *
59   * <p>The following table describes which methods are valid in what state.
60   * If a method is called in an invalid state the method will throw a
61   * java.lang.IllegalStateException.
62   *
63   * <table border="2" rules="all" cellpadding="4" width=100%>
64   *     <tr>
65   *       <th align="center" colspan="2">
66   *         Valid methods for each state
67   *       </th>
68   *     </tr>
69   *     <tr>
70   *       <th width=13%>Event Type</th>
71   *       <th width=28%>Valid Methods</th>
72   *     </tr>
73   *     <tr>
74   *       <td> All States  </td>
75   *       <td> getProperty(), hasNext(), require(), close(),
76   *            getNamespaceURI(), isWhiteSpace(),
77   *            getEventType(),getLocation()</td>
78   *     </tr>
79   *     <tr>
80   *       <td> START_ELEMENT  </td>
81   *       <td> next(), getLocalName(), getPrefix(),
82   *            getAttributeXXX(), isAttributeSpecified(),
83   *            getNamespaceXXX(), getElementText(),
84   *            nextTag()
85   *       </td>
86   *     </tr>
87   *       <td> ATTRIBUTE  </td>
88   *       <td> next(), nextTag()
89   *            getAttributeXXX(), isAttributeSpecified()
90   *       </td>
91   *     </tr>
92   *     </tr>
93   *       <td> NAMESPACE  </td>
94   *       <td> next(), nextTag()
95   *            getNamespaceXXX()
96   *       </td>
97   *     </tr>
98   *     <tr>
99   *       <td> END_ELEMENT  </td>
100  *       <td> next(), getLocalName(), getPrefix(),
101  *            getNamespaceXXX(), nextTag()
102  *      </td>
103  *     </tr>
104  *     <tr>
105  *       <td> CHARACTERS  </td>
106  *       <td> next(), getTextXXX(), nextTag() </td>
107  *     </tr>
108  *     <tr>
109  *       <td> CDATA  </td>
110  *       <td> next(), getTextXXX(), nextTag() </td>
111  *     </tr>
112  *     <tr>
113  *       <td> COMMENT  </td>
114  *       <td> next(), getTextXXX(), nextTag() </td>
115  *     </tr>
116  *     <tr>
117  *       <td> SPACE  </td>
118  *       <td> next(), getTextXXX(), nextTag() </td>
119  *     </tr>
120  *     <tr>
121  *       <td> START_DOCUMENT  </td>
122  *       <td> next(), getEncoding(), getVersion(), isStandalone(), 
123  *            standaloneSet(), getCharacterEncodingScheme(), nextTag()</td>
124  *     </tr>
125  *     <tr>
126  *       <td> END_DOCUMENT  </td>
127  *       <td> close()</td>
128  *     </tr>
129  *     <tr>
130  *       <td> PROCESSING_INSTRUCTION  </td>
131  *       <td> next(), getPITarget(), getPIData(), nextTag() </td>
132  *     </tr>
133  *     <tr>
134  *       <td> ENTITY_REFERENCE  </td>
135  *       <td> next(), getLocalName(), getText(), nextTag() </td>
136  *     </tr>
137  *     <tr>
138  *       <td> DTD  </td>
139  *       <td> next(), getText(), nextTag() </td>

141  *  </table>

159   /**
160    * Get next parsing event - a processor may return all contiguous
161    * character data in a single chunk, or it may split it into several chunks.
162    * If the property javax.xml.stream.isCoalescing is set to <code>true</code>
163    * element content must be coalesced and only one CHARACTERS event
164    * must be returned for contiguous element content or
165    * CDATA Sections.  
166    *
167    * By default entity references must be 
168    * expanded and reported transparently to the application.
169    * An exception will be thrown if an entity reference cannot be expanded.
170    * If element content is empty (i.e. content is "") then no CHARACTERS event 
171    * will be reported.
172    *
173    * <p>Given the following XML:<br>
174    * &lt;foo>&lt;!--description-->content text&lt;![CDATA[&lt;greeting>Hello&lt;
175    * /greeting>]]>other content&lt;/foo><br>
176    * The behavior of calling next() when being on foo will be:<br>
177    * 1- the comment (COMMENT)<br>
178    * 2- then the characters section (CHARACTERS)<br>
179    * 3- then the CDATA section (another CHARACTERS)<br>
180    * 4- then the next characters section (another CHARACTERS)<br>
181    * 5- then the END_ELEMENT<br>
182    *
183    * <p><b>NOTE:</b> empty element (such as &lt;tag/>) will be reported
184    *  with  two separate events: START_ELEMENT, END_ELEMENT - This preserves
185    *   parsing equivalency of empty element to &lt;tag>&lt;/tag>.
186    *
187    * @return the integer code corresponding to the current parse event
188    * @throws NoSuchElementException if this is called when hasNext() returns 
189    *   <code>false</code>
190    * @throws XMLStreamException  if there is an error processing the underlying 
191    *   XML source
192    */
193   public int next() throws XMLStreamException;

196    * Test if the current event is of the given type and if the namespace and

198    * namespaceURI is <code>null</code> it is not checked for equality, if the 
199    * localName is <code>null</code> it is not checked for equality.
200    * @param type the event type
201    * @param namespaceURI the uri of the event, may be <code>null</code>
202    * @param localName the localName of the event, may be <code>null</code>
203    * @throws XMLStreamException if the required values are not matched.
204    */

208   /**
209    * Reads the content of a text-only element, an exception is thrown if this 
210    * is not a text-only element.
211    * Regardless of value of javax.xml.stream.isCoalescing this method always 
212    * returns coalesced content.
213    * <br /> Precondition: the current event is START_ELEMENT.
214    * <br /> Postcondition: the current event is the corresponding END_ELEMENT.
215    *
216    * <br />The method does the following (implementations are free to optimize 
217    * but must do equivalent processing):
218    * <pre>
219    * if(getEventType() != XMLStreamConstants.START_ELEMENT) {
220    *     throw new XMLStreamException(
221    *         "parser must be on START_ELEMENT to read next text", 
222    *         getLocation());
223    * }
224    * int eventType = next();
225    * StringBuffer content = new StringBuffer();
226    * while(eventType != XMLStreamConstants.END_ELEMENT ) {
227    *     if(eventType == XMLStreamConstants.CHARACTERS
228    *             || eventType == XMLStreamConstants.CDATA
229    *             || eventType == XMLStreamConstants.SPACE
230    *             || eventType == XMLStreamConstants.ENTITY_REFERENCE) {
231    *         buf.append(getText());
232    *     } else if(eventType == XMLStreamConstants.PROCESSING_INSTRUCTION
233    *             || eventType == XMLStreamConstants.COMMENT) {
234    *             // skipping
235    *     } else if(eventType == XMLStreamConstants.END_DOCUMENT) {
236    *         throw new XMLStreamException(
237    *             "unexpected end of document when reading element text content",
238    *             this);
239    *     } else if(eventType == XMLStreamConstants.START_ELEMENT) {
240    *         throw new XMLStreamException(
241    *             "element text content may not contain START_ELEMENT", 
242    *             getLocation());
243    *     } else {
244    *         throw new XMLStreamException(
245    *             "Unexpected event type "+eventType, getLocation());
246    *     }
247    *     eventType = next();
248    * }
249    * return buf.toString();
250    * </pre>
251    *
252    * @throws XMLStreamException if the current event is not a START_ELEMENT 
253    * or if a non text element is encountered
254    */
255   public String getElementText() throws XMLStreamException;

257   /**
258    * Skips any white space (isWhiteSpace() returns <code>true</code>), COMMENT,
259    * or PROCESSING_INSTRUCTION,
260    * until a START_ELEMENT or END_ELEMENT is reached.
261    * If other than white space characters, COMMENT, PROCESSING_INSTRUCTION, 
262    * START_ELEMENT, END_ELEMENT are encountered, an exception is thrown. This 
263    * method should be used when processing element-only content separated by 
264    * white space. 
265    *
266    * <br /> Precondition: none
267    * <br /> Postcondition: the current event is START_ELEMENT or END_ELEMENT
268    * and cursor may have moved over any whitespace event.
269    *
270    * <br />Essentially it does the following (implementations are free to 
271    * optimize but must do equivalent processing):
272    * <pre>
273    * int eventType = next();
274    * // skip whitespace
275    * while((eventType == XMLStreamConstants.CHARACTERS 
276    *         &amp;&amp; isWhiteSpace())
277    *         || (eventType == XMLStreamConstants.CDATA &amp;&amp; isWhiteSpace()) 
278    *         // skip whitespace
279    *         || eventType == XMLStreamConstants.SPACE
280    *         || eventType == XMLStreamConstants.PROCESSING_INSTRUCTION
281    *         || eventType == XMLStreamConstants.COMMENT) {
282    *     eventType = next();
283    * }
284    * if (eventType != XMLStreamConstants.START_ELEMENT 
285    *         &amp;&amp; eventType != XMLStreamConstants.END_ELEMENT) {
286    *     throw new String XMLStreamException(
287    *         "expected start or end tag", getLocation());
288    * }
289    * return eventType;
290    * </pre>
291    *
292    * @return the event type of the element read (START_ELEMENT or END_ELEMENT)
293    * @throws XMLStreamException if the current event is not white space, 
294    *   COMMENT, PROCESSING_INSTRUCTION, START_ELEMENT or END_ELEMENT
295    * @throws NoSuchElementException if this is called when hasNext() returns 
296    *   <code>false</code>
297    */
298   public int nextTag() throws XMLStreamException;
