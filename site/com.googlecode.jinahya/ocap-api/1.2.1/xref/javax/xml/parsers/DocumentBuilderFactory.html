52       * Obtain a new instance of a

55       * This method uses the following ordered lookup procedure to determine
56       * the <code>DocumentBuilderFactory</code> implementation class to

58       * <ul>
59       * <li>

73       * <h2>Tip for Trouble-shooting on CDC</h2>
74       * <p>Setting the <code>jaxp.debug</code> system property will cause

77       * 
78       * <p> If you have problems loading {@link DocumentBuilder}s, try:</p>
79       * <pre>
80       * java -Djaxp.debug=1 YourProgram ....
81       * </pre>
82       * 
83       * @return New instance of a <code>DocumentBuilderFactory</code>
84       *
85       * @exception FactoryConfigurationError if the implementation is not
86       * available or cannot be instantiated.
87       */

89      public static DocumentBuilderFactory newInstance()
90      {
91          try {
92              String fname = System.getProperty(DEFAULT_PROPERTY_NAME);
93              if(fname == null)
94                  fname = "com.sun.ukit.dom.DocBuilderFactory";
95              return (DocumentBuilderFactory) Class.forName(fname).newInstance();
96          } catch (java.lang.Throwable ex) {
97              throw new FactoryConfigurationError(ex.toString());
98          }
99      }

317     /**
318      * <p>Set a feature for this <code>DocumentBuilderFactory</code> and 
319      * <code>DocumentBuilder</code>s created by this factory.</p>
320      * 
321      * <p>
322      * Feature names are fully qualified URIs.
323      * Implementations may define their own features.
324      * An {@link ParserConfigurationException} is thrown if this 
325      * <code>DocumentBuilderFactory</code> or the
326      * <code>DocumentBuilder</code>s it creates cannot support the feature.
327      * It is possible for an <code>DocumentBuilderFactory</code> to expose
328      * a feature value but be unable to change its state. A null
329      * feature name parameter causes the XML reader to throw a 
330      * <code>NullPointerException</code>. </p>
331      * 
332      * <p>
333      * All implementations are required to support the 
334      * {@link javax.xml.XMLConstants#FEATURE_SECURE_PROCESSING} feature.
335      * When the feature is:</p>
336      * <ul>
337      *   <li>
338      *     <code>true</code>: the implementation will limit XML processing to 
339      *     conform to implementation limits. Examples include entity expansion 
340      *     limits and XML Schema constructs that would consume large amounts of 
341      *     resources. If XML processing is limited for security reasons, it will 
342      *     be reported via a call to the registered
343      *     {@link org.xml.sax.ErrorHandler#fatalError(SAXParseException exception)}.
344      *     See {@link DocumentBuilder#setErrorHandler(org.xml.sax.ErrorHandler errorHandler)}.
345      *   </li>
346      *   <li>
347      *     <code>false</code>: the implementation will processing XML according 
348      *     to the XML specifications without regard to possible implementation 
349      *     limits.
350      *   </li>
351      * </ul>
352      * 
353      * @param name Feature name.
354      * @param value Is feature state <code>true</code> or <code>false</code>.
355      *  
356      * @throws ParserConfigurationException if this <code>DocumentBuilderFactory</code> 
357      *   or the <code>DocumentBuilder</code>s it creates cannot support this 
358      *   feature.
359      * @throws NullPointerException If the <code>name</code> parameter is null.
360      */
361 
362     public abstract void setFeature(String name, boolean value)
363                 throws ParserConfigurationException;
