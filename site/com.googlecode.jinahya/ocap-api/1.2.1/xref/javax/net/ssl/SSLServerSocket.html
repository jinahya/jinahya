52  /** 
53   * This class extends <code>ServerSocket</code>s and
54   * provides secure server sockets using protocols such as the Secure
55   * Sockets Layer (SSL) or Transport Layer Security (TLS) protocols.
56   * <P>
57   * Instances of this class are generally created using a
58   * <code>SSLServerSocketFactory</code>.  The primary function
59   * of <code>SSLServerSocket</code>s
60   * is to create <code>SSLSocket</code>s by <code>accept</code>ing
61   * connections.
62   * <P>
63   * <code>SSLServerSocket</code>s contain several pieces of state data
64   * which are inherited by the <code>SSLSocket</code> at
65   * socket creation.  These include the enabled cipher
66   * suites and protocols, whether client
67   * authentication is necessary, and whether created sockets should
68   * begin handshaking in client or server mode.  The state
69   * inherited by the created <code>SSLSocket</code> can be
70   * overriden by calling the appropriate methods.
71   *
72   * @see java.net.ServerSocket
73   * @see SSLSocket
74   *
75   * @since 1.4
76   * @version 1.22
77   * @author David Brownell
78   */

295      * Controls whether accepted connections are in the (default) SSL

301      * connections. For example, FTP clients acquire server sockets

306      * connection. During the resulting handshake, existing SSL
307      * sessions may be reused.
308      * <P>
