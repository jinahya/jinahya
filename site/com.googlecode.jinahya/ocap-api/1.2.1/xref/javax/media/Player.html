38  package javax.media;
39  
40  import java.awt.Component;
41  import javax.media.protocol.DataSource;
42  import java.io.IOException;
43  
44  /** 
45   * <code>Player</code> is a <code>MediaHandler</code> for rendering
46   * and controlling time based media data.
47   * <code>Player</code> extends both the <code>Controller</code>
48   * and <code>Duration</code> interfaces 
49   * <code>Player</code> provides methods for
50   * obtaining AWT components, media processing controls, and a way to
51   * manage other <code>Controllers</code>.  
52   *
53   * <h2>How a Player Differs from a Controller</h2>
54   * 
55   * <code>Player</code> relaxes some restrictions that a
56   * <code>Controller</code> imposes 
57   * on what methods can be called on a <i>Started</i>, <i>Stopped</i>, or 
58   * <i>Unrealized</i>&nbsp;<code>Controller</code>.
59   * It also provides a way to manage groups of <CODE>Controllers</CODE>.
60   *
61   * <h3>Methods Restricted to <i>Stopped</i>&nbsp;Players</h3>
62   *
63   * The following methods can only be called on a <CODE>Player</CODE> in
64   * one of the <i>Stopped</i> states.
65   * If they are invoked on a <i>Started</i>&nbsp;<code>Player</code>, 
66   * a <code>ClockStartedError</code> is thrown.
67   * <ul>
68   * <li> <code>setTimeBase</code>
69   * <li> <code>syncStart</code>
70   * <li> <code>deallocate</code>
71   * <li> <code>addController</code>
72   * <li> <code>removeController</code>
73   * </ul>
74   * <p>
75   * 
76   * <h3>Methods Allowed on <i>Started</i>&nbsp;Players</h3>
77   * Unlike a <code>Controller</code>, the following methods are <i>legal</i> on 
78   * a <code>Player</code> in the <i>Started</i> state:
79   * <ul>
80   * <li> <code>setMediaTime</code>
81   * <li> <code>setRate</code>
82   * </ul>
83   * Invoking these methods on a <i>Started</i>&nbsp;<code>Player</code> might
84   * initiate significant and time-consuming processing, depending
85   * on the location and type of media being processed.
86   * 
87   * These methods might also cause the state of the <code>Player</code> to
88   * change.
89   * If this happens, the appropriate <code>TransitionEvents</code> are posted 
90   * by the <code>Player</code> when its state changes.
91   * <p>
92   * 
93   * For example, a <code>Player</code> might have to enter
94   * the <i>Prefetching</i> state to process a <code>setMediaTime</code>
95   * invocation.
96   * In this case, the <code>Player</code> posts a <code>RestartingEvent</code>,
97   * a <code>PrefetchCompleteEvent</code>, and a <code>StartEvent</code> as 
98   * it moves from the <i>Started</i> state to <i>Prefetching</i>, back to 
99   * <i>Prefetched</i>, and finally back to the <i>Started</i> state.
100  * 
101  * <h3>Methods that are Illegal on <i>Unrealized</i> Players</h3>
102  * 
103  * As with <code>Controller</code>, it is illegal to call the following methods
104  * on  an <i>Unrealized</i>&nbsp;<code>Player</code>:
105  * <ul>
106  * <li> <code>getTimeBase</code>
107  * <li> <code>setTimeBase</code>
108  * <li> <code>setMediaTime</code>
109  * <li> <code>setRate</code>
110  * <li> <code>setStopTime</code> 
111  * <li> <code>getStartLatency</code>
112  * </ul>
113  * <p>
114  * It is also illegal to call the following <code>Player</code> methods on an 
115  * <i>Unrealized</i>&nbsp;<code>Player</code>:
116  * <ul>
117  * <li> <code>getVisualComponent</code>
118  * <li> <code>getControlPanelComponent</code> 
119  * <li> <code>getGainControl</code>
120  * <li> <code>addController</code>
121  * <li> <code>removeController</code>
122  * </ul>
123  * <p>
124  * 
125  * The <code>Player</code> throws a <code>NotRealizedError</code>
126  * if any of these methods are called while the <code>Player</code>
127  * is in the <i>Unrealized</i> state.
128  *
129  * <h3>Start Method </h3>
130  *
131  * As a convenience, <code>Player</code> provides a <code>start</code>
132  * method that can be invoked before a <code>Player</code>
133  * is <i>Prefetched</i>.
134  * This method attempts to transition the <code>Player</code> to
135  * the <i>Started</i> state from whatever state it's currently in.
136  * For example, if the <code>Player</code> is <i>Unrealized</i>,
137  * <code>start</code> implicitly calls <code>realize</code>,
138  * <code>prefetch</code>, and <code>Clock.syncStart</code>.
139  * The appropriate <code>TransitionEvents</code> are posted as
140  * the <code>Player</code> moves through each state on its way
141  * to <i>Started</i>.
142  *
143  * <h3>RestartingEvent</h3>
144  *
145  * If <code>setMediaTime</code> or <code>setRate</code> cause a perceptible
146  * delay  in the presentation of the media, the <code>Player</code> posts a 
147  * <code>RestartingEvent</code> and transitions to the <i>Prefetching</i>
148  * state.
149  * The previous state and target state of a <code>RestartingEvent</code>  
150  * is always <i>Started</i>. <code>RestartingEvent</code> is a subclass
151  * of <code>StopEvent</code>.
152  *
153  * <h3>DurationUpdateEvent</h3>
154  * 
155  * Because a <code>Player</code> cannot always know the duration of the media
156  * it is playing, the <code>Duration</code> interface defines that
157  * <code>getDuration</code> returns <code>Duration.DURATION_UNKNOWN</code>
158  * until the duration can be determined.
159  * A <code>DurationUpdateEvent</code> is generated when the <code>Player</code>
160  * can determine its duration or the if its duration
161  * changes, which can happen at any time. When the end of the media
162  * is reached, the duration should be known.
163  *
164  * <h2>Managing other Controllers</h2>
165  *
166  * In some situations, an application might want to use a single
167  * <code>Player</code> to
168  * control other <code>Players</code> or <code>Controllers</code>.
169  * A single controlling <code>Player</code> can be used to
170  * invoke <code>start</code>, <code>stop</code>, <code>setMediaTime</code>,
171  * and other methods on the entire group. The controlling
172  * <code>Player</code> manages all of the state transitions and event posting.
173  * <p> 
174  * It is also possible to construct a simple <code>Controller</code>
175  * to update animations, report on media time-line progress, or
176  * provide other timing-related functions. Such <code>Controllers</code> can 
177  * operate in sync with a controlling <code>Player</code>.
178  *
179  * <h3>Adding a Controller</h3>
180  *
181  * To have a <CODE>Player</CODE> assume control over a <code>Controller</code>,
182  * use the <code>addController</code> method.
183  * A <code>Controller</code> can only be added to a <i>Stopped</i>&nbsp;
184  * <code>Player</code>. If <code>addController</code> is called on
185  * a <i>Started</i>&nbsp;<code>Player</code>,
186  * a  <code>ClockStartedError</code> is thrown.
187  * An <i>Unrealized</i>&nbsp;<code>Controller</code> cannot be added to a
188  * <code>Player</code>; a <code>NotRealizedError</code> is thrown if the
189  * <code>Controller</code> is <i>Unrealized</i>.
190  * <p>
191  *
192  * Once a <code>Controller</code> has been added, the <code>Player</code>:
193  * <ul>
194  * <li>Invokes <code>setTimeBase</code> on the <code>Controller</code> with the
195  * <code>Player's</code>&nbsp;<code>TimeBase</code>.
196  *
197  * If this fails, <code>addController</code> throws
198  * an <code>IncompatibleTimeBaseException</code>.
199  *
200  * <li>Synchronizes the <code>Controller</code> with the <code>Player</code>
201  * using <code>setMediaTime</code>, <code>setStopTime</code>,
202  * and <code>setRate</code>.
203  *
204  * <li>Takes the added <code>Controller's</code> latency into account when
205  * computing the <code>Player's</code> start latency.
206  * When <code>getStartLatency</code> is called,
207  * the <code>Player</code> returns the greater of:
208  * its latency before the <code>Controller</code> was added and
209  * the latency of the added <code>Controller</code>.
210  *
211  * <li>Takes the added <code>Controller's</code> duration into account when
212  * computing the <code>Player's</code>
213  * duration.  When <code>getDuration</code> is called,
214  * the <code>Player</code> returns the greater of:
215  * its duration before the <code>Controller</code> was added and
216  * the duration of the added <code>Controller</code>.
217  * If either of these values is DURATION_UNKNOWN,
218  * <CODE>getDuration</CODE> returns DURATION_UNKNOWN.
219  * If either of these values is DURATION_UNBOUNDED <CODE>getDuration</CODE>
220  * returns DURATION_UNBOUNDED.
221  * 
222  * <li> Adds itself as a <code>ControllerListener</code> for the
223  * added <code>Controller</code> so that it can
224  * manage the events that the <code>Controller</code> generates.
225  * (See the <a href="#events">Events</a> section below for more information.)
226  *
227  * <li>Invokes control methods on the added <code>Controller</code>
228  * in response to methods invoked on the <code>Player</code>. The 
229  * methods that affect
230  * managed <code>Controllers</code> are discussed below.
231  * 
232  * </ul>
233  *
234  * Once a <code>Controller</code> has been added to a <code>Player</code>,
235  * methods should only be called on the <code>Controller</code> through the
236  * managing <code>Player</code>.
237  * It is  not defined how the <code>Controller</code> or <code>Player</code>
238  * will behave if methods are called directly on an added
239  * <code>Controller</code>.
240  * You cannot place a controlling <CODE>Player</CODE> under the control of a
241  * <CODE>Player</CODE> that it is managing; the resulting behavior is
242  * undefined.
243  * <p>
244  *
245  * When a <code>Controller</code> is added to a
246  * <code>Player</code>, the <code>Player</code>
247  * does not transition the added <code>Controller</code> to
248  * new state, nor does the <code>Player</code> transition itself
249  * forward.
250  * The <code>Player</code> either transitions back to the
251  * <i>realized</i> state if the added <code>Controller</code>
252  * is <i>realized</i> or <i>prefetching</i> or it stays
253  * in the <i>prefetched</i> state if the both the <code>Player</code>
254  * and the added <code>Controller</code> are in the <i>prefetched</i>
255  * state. If the <code>Player</code> makes a state transition
256  * as a result of adding a <code>Controller</code> the <code>Player</code>
257  * posts a <code>TransitionEvent</code>.
258  * 
259  * <h3>Removing a Controller</h3>
260  *
261  * To stop a <code>Player</code> from managing another
262  * <code>Controller</code>, call <code>removeController</code>.
263  * The managing <code>Player</code> must be <i>Stopped</i> before
264  * <code>removeController</code> can be called.
265  * A <code>ClockStartedError</code> is thrown if <code>removeController</code>
266  * is called on a <i>Started</i>&nbsp;<code>Player</code>.
267  * <p>
268  *
269  * When a <code>Controller</code> is removed from a <code>Player's</code>
270  * control, the <code>Player</code>:
271  * <ul>
272  * <li> Resets the <code>Controller's</code>&nbsp;<code>TimeBase</code>
273  * to its default.
274  * <li> Recalculates its duration and posts a
275  * <code>DurationUpdateEvent</code> if the <code>Player's</code> duration
276  * is different without the <CODE>Controller</CODE> added.
277  * <li> Recalculates its start latency.
278  * </ul>
279  *
280  * <h3>Setting the Media Time and Rate of a Managing Player</h3>
281  *
282  * When you call <code>setMediaTime</code> on a <code>Player</code> that's
283  * managing other <code>Controllers</code>,
284  * its actions differ depending on whether or not the <code>Player</code>
285  * is <I>Started</I>.
286  * If the <code>Player</code> is not <i>Started</i>, it simply
287  * invokes <code>setMediaTime</code> on all of the
288  * <code>Controllers</code> it's managing.
289  * <p>
290  *
291  * If the <code>Player</code> is <i>Started</i>,
292  * it posts a <code>RestartingEvent</code> and
293  * performs the following tasks for each managed <code>Controller</code>:
294  * 
295  * <ul>
296  * <li>Invokes <code>stop</code> on the <code>Controller</code>.
297  * <li>Invokes <code>setMediaTime</code> on the <code>Controller</code>.
298  * <li>Invokes <code>prefetch</code> on the <code>Controller</code>.
299  * <li>Waits for a <code>PrefetchCompleteEvent</code> from
300  * the <code>Controller</code>.
301  * <li>Invokes <code>syncStart</code> on the <code>Controller</code>
302  * </ul>
303  *
304  * <p>
305  *
306  * The same is true when <code>setRate</code> is called on a
307  * managing <code>Player</code>.
308  * The <code>Player</code> attempts to set the specified rate
309  * on all managed <code>Controllers</code>, stopping and restarting
310  * the <code>Controllers</code> if necessary.
311  * If some of the <code>Controllers</code> do not support the requested rate,
312  * the <code>Player</code> returns the rate that was actually set.
313  * All <code>Controllers</code> are guaranteed to have been successfully
314  * set to the rate returned. 
315  * 
316  * <h3>Starting a Managing Player</h3>
317  * When you call <code>start</code> on a managing <code>Player</code>,
318  * all of the  <code>Controllers</code> managed by
319  * the <code>Player</code> are transitioned to
320  * the <i>Prefetched</i> state. When the <code>Controllers</code>
321  * are <i>Prefetched</i>,
322  * the managing <code>Player</code> calls <code>syncStart</code>
323  * with a time consistent with the latencies of each of the managed
324  * <code>Controllers</code>.
325  *  
326  * 
327  * <h3>Calling realize, prefetch, stop, or deallocate on a Managing Player</h3>
328  * 
329  * When you call <code>realize</code>, <code>prefetch</code>, 
330  * <code>stop</code>,  or <code>deallocate</code> on a managing
331  * <code>Player</code>, 
332  * the <code>Player</code> calls that method on all of the 
333  * <code>Controllers</code> that it is managing.
334  * The <code>Player</code> moves from one state to the
335  * next when all of its <code>Controllers</code> have reached that state. 
336  * For example, a <code>Player</code> in the <i>Prefetching</i>
337  * state does not transition into the <i>Prefetched</i>
338  * state until all of its managed <code>Controllers</code>
339  * are <i>Prefetched</i>.
340  * The  <code>Player</code> posts <code>TransitionEvents</code> normally
341  * as it changes state.
342  * <p>
343  *
344  * <h3>Calling syncStart or setStopTime on a Managing Player</h3>
345  * When you call <code>syncStart</code> or <code>setStopTime</code> on a
346  * managing <code>Player</code>, the <code>Player</code>
347  * calls that method  on all of the <code>Controllers</code> that it
348  * is managing. (The <code>Player</code> 
349  * must be in the correct state or an error is thrown.
350  * For example, the <code>Player</code> must be <I>Prefetched</I>
351  * before you can call <code>syncStart</code>.)
352  * 
353  * <h3>Setting the Time Base of a Managing Player</h3>
354  * When <code>setTimeBase</code> is called on a managing <code>Player</code>,
355  * the <code>Player</code> calls <code>setTimeBase</code> on all of
356  * the <code>Controllers</code> it's managing. 
357  * If <code>setTimeBase</code> fails on any of the <code>Controllers</code>,
358  * an <code>IncompatibleTimeBaseException</code> is thrown
359  * and the  <code>TimeBase</code> last used
360  * is restored for all of the <code>Controllers</code>.
361  * 
362  * <h3>Getting the Duration of a Managing Player</h3>
363  * Calling <code>getDuration</code> on a managing <code>Player</code>
364  * returns the maximum duration of all of the added
365  * <code>Controllers</code> and the managing <code>Player</code>.
366  * If the <CODE>Player</CODE>  or any <CODE>Controller</CODE>
367  * has not resolved its duration, <code>getDuration</code>
368  * returns <code>Duration.DURATION_UNKNOWN</code>.
369  *
370  * <h3> Closing a Managing Player</h3>
371  * When <code>close</code> is called on a managing <code>Player</code>
372  * all managed <code>Controllers</code> are closed as well.
373  * 
374  * <a name="events"><h3>Events</h3></a>
375  * Most events posted by a managed <code>Controller</code> are filtered
376  * by the managing <code>Player</code>. Certain events are sent directly
377  * from the <code>Controller</code> through the <code>Player</code> and to the
378  * listeners registered with the <code>Player</code>.
379  * <p>
380  *
381  * To handle the events that a managed <code>Controller</code> can generate,
382  * the <code>Player</code> registers a listener with the
383  * <code>Controller</code> when it is added.
384  * Other listeners that are registered with the <code>Controller</code>
385  * must be careful not to invoke methods on the <code>Controller</code>
386  * while it is being managed by the <code>Player</code>.
387  * Calling a control method on a managed <code>Controller</code> directly
388  * will produce unpredictable results.
389  * <p>
390  *
391  * When a <CODE>Controller</CODE> is removed from the <code>Player's</code>
392  * list of managed <code>Controllers</code>,
393  * the <code>Player</code> removes itself from the <code>Controller's</code>
394  * listener list.
395  * 
396  * <h4>Transition Events</h4>
397  * A managing <code>Player</code> posts <code>TransitionEvents</code> normally 
398  * as it moves between states, but
399  * the managed <code>Controllers</code> affect when the <code>Player</code>
400  * changes state.
401  * In general, 
402  * a <code>Player</code> does not post a transition event until all of its
403  * managed <code>Controllers</code> have posted the event.
404  * 
405  * <h4>Status Change Events</h4>
406  * The managing <code>Player</code> collects the
407  * <code>RateChangeEvents</code>, <code>StopTimeChangeEvents</code>,
408  * and <code>MediaTimeSetEvents</code> posted by its
409  * managed <code>Controllers</code> and posts a single event for the group.
410  * 
411  * <h4>DurationUpdateEvent</h4>
412  * A <code>Player</code> posts a <code>DurationUpdateEvent</code> when
413  * it determines its duration or its duration changes.
414  * A managing <code>Player's</code> duration might change if a managed
415  * <code>Controller</code> updates or discovers its duration.
416  * In general, if a managed <code>Controller</code> 
417  * posts a <code>DurationUpdateEvent</code> and the new duration
418  * changes the managing <code>Player's</code> duration,
419  * the <code>Player</code> posts a <code>DurationUpdateEvent</code>
420  *
421  * <h4>CachingControlEvent</h4>
422  * A managing <CODE>Player</CODE> reposts <CODE>CachingControlEvents</CODE>
423  * received from a <CODE>Players</CODE> that it manages, but otherwise
424  * ignores the events.
425  *
426  * <h4>ControllerErrorEvents</h4>
427  * A managing <CODE>Player</CODE> immediately reposts
428  * any <CODE>ControllerErrorEvent</CODE> received from a
429  * <CODE>Controller</CODE> that it is managing.
430  * After a <CODE>ControllerErrorEvent</CODE> has been
431  * received from a managed <CODE>Controller</CODE>, a
432  * managing <CODE>Player</CODE> no longer invokes any methods
433  * on the managed <CODE>Controller</CODE>; the
434  * managed <CODE>Controller</CODE> is ignored from that point on.
435  * 
436  * @see Manager
437  * @see GainControl
438  * @see Clock 
439  * @see TransitionEvent
440  * @see RestartingEvent
441  * @see DurationUpdateEvent
442  * @see java.awt.Component
443  *
444  * @version 1.76, 07/09/19
445  */
446 public interface Player extends MediaHandler, Controller, Duration
447 {
448 
449     /** 
450      * Obtain the display <code>Component</code> for this <code>Player</code>.
451      * The display <code>Component</code> is where visual media
452      * is rendered.
453      * If this <code>Player</code> has no visual component,
454      * <code>getVisualComponent</code> returns <CODE>null</CODE>.
455      * For example, <code>getVisualComponent</code> might return
456      * <CODE>null</CODE> if the <code>Player</code> only plays audio.
457      *
458      *
459      * @return The media display <code>Component</code> for this
460      * <code>Player</code>.
461      */
462     public Component getVisualComponent();
463 
464     /** 
465      * Obtain the object for controlling this <code>Player's</code>
466      * audio gain. 
467      * If this player does not have a 
468      * <code>GainControl</code>, <code>getGainControl</code> returns
469      * <CODE>null</CODE>.  
470      * For example, <code>getGainControl</code> might return
471      * <CODE>null</CODE> if the <code>Player</code> does not play audio data.
472      *
473      * @return The <code>GainControl</code> object for this
474      * <code>Player</code>.
475      */
476     public GainControl getGainControl();
477 
478     /** 
479      * Obtain the <code>Component</code> that provides the default user
480      * interface for controlling this <code>Player</code>.
481      * If this <code>Player</code> has no default control panel,
482      * <code>getControlPanelComponent</code> returns <CODE>null</CODE>.
483      *
484      * @return The default control panel GUI for this <code>Player</code>.
485      */
486     public Component getControlPanelComponent();
487 
488     /** 
489      * Start the <code>Player</code> as soon as possible.
490      * The <CODE>start</CODE> method attempts to transition the
491      * <code>Player</code> to the <i>Started</i> state.
492      * If the <CODE>Player</CODE> has not been <i>Realized</i> or
493      * <i>Prefetched</i>, <code>start</code> automatically performs
494      * those actions. The appropriate events
495      * are posted as the <code>Player</code> moves through each state.
496      */
497     public void start();
498 
499     /** 
500      * Assume control of another <code>Controller</code>.
501      * <p>
502      * Note that in some implementations of Java TV, there may be no two
503      * instances of {@link Controller} with compatible timebases.
504      * In such cases, {@link IncompatibleTimeBaseException} will always
505      * be thrown.
506      * 
507      * @param newController The <code>Controller</code> to be managed.
508      *
509      * @exception IncompatibleTimeBaseException Thrown if the added
