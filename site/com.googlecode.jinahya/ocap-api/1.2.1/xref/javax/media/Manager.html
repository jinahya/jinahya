48  /** 
49   * <code>Manager</code> is the access point for obtaining
50   * system dependent resources such as <code>Players</code>,
51   * <code>DataSources</code>, and the system <code>TimeBase</code>.
52   * <p>
53   *
54   * A <code>Player</code> is an object used to
55   * control and render multimedia data that
56   * is specific to the content type of the data.
57   * A <code>DataSource</code> is an object used to 
58   * deliver time-based multimedia data that is specific
59   * to a delivery protocol.
60   * A <code>DataSource</code> provides
61   * a <code>Player</code> with media data;
62   * a <CODE>Player</CODE> must have a <CODE>DataSource</CODE>.
63   * <code>Manager</code> provides access to a protocol and media independent
64   * mechanism for constructing <code>Players</code> and
65   * <code>DataSources</code>.
66   *
67   * <h2>Creating Players and DataSources</h2>
68   *
69   * <code>Manager</code> will create<code>Players</code> from a
70   * <code>URL</code>, a <CODE>MediaLocator</CODE> or a <code>DataSource</code>.
71   * Creating a <code>Player</code> requires the following:
72   * <ul>
73   * <li> Obtain the connected <code>DataSource</code> for the specified
74   * protocol
75   * <li> Obtain the <code>Player</code> for the content-type
76   * specified by the <code>DataSource</code>
77   * <li> Attach the <code>DataSource</code> to the <code>Player</code>
78   * using the <code>setSource</code> method.
79   * </ul>
80   *
81   * It is possible to create multiple Players for the same scarce resource
82   * (such as a hardware video decoder).
83   * However, doing so may affect the state of other Players which depend on
84   * that resource.
85   *
86   * <h3>Finding DataSources by Protocol</h3>
87   *
88   * A <code>MediaLocator</code> defines a protocol for obtaining
89   * content.
90   * <code>DataSources</code> are identified by the protocol
91   * that they support. <code>Manager</code> uses the protocol
92   * name to find <code>DataSource</code> classes.
93   * <p>
94   * 
95   * To find a <code>DataSource</code> using a <code>MediaLocator</code>,
96   * <code>Manager</code> constructs a list of class names from the protocol
97   * package-prefix list and the protocol name obtained
98   * from the <code>MediaLocator</code>.
99   * For each class name in the constructed list a new <code>DataSource</code>
100  * is instanced, the <code>MediaLocator</code> is attached,
101  * and the <code>DataSource</code> is connected.
102  * If no errors have occurred, the procces is considered finished and the
103  * connected <code>DataSource</code> is used by
104  * <code>Manager</code> in any following operations.
105  * If there was an error then the next class name in the list
106  * is tried.
107  * The exact details of the search algorithm is described in
108  * the method documentation below.
109  *
110  * <h3>Finding Players by Content Type</h3>
111  *
112  * A <code>Player</code> is a <code>MediaHandler</code>.
113  * A <code>MediaHandler</code> is a an object that reads
114  * data from a <code>DataSource</code>. There are two types
115  * of supported <code>MediaHandler</code>: <code>MediaProxy</code>,
116  * and <code>Player</code>.
117  * <p>
118  *
119  * <code>MediaHandlers</code> are identified by the content type that they
120  * support. A <code>DataSource</code> identifies the content type
121  * of the data it produces with the <code>getContentType</code> method.
122  * <code>Manager</code> uses the content type name to
123  * find instances of <code>MediaHandler</code>.
124  * <p>
125  *
126  * To find a <code>MediaHandler</code> using a content type name,
127  * <code>Manager</code> constructs a list of class names from
128  * the content package-prefix list and the content type name.
129  * For each class name in the constructed list a new <code>MediaHandler</code>
130  * is instanced, and the <code>DataSource</code> is attached to
131  * the <code>MediaHandler</code> using <coded>MediaHandler.setSource</code>.
132  * <p>
133  *
134  * If the <code>MediaHandler</code> is a <code>Player</code> and the
135  * <code>setSource</code> was successful the process is finished
136  * and the <code>Player</code> is returned.
137  * If the <code>setSource</code> failed, another name in the
138  * list is tried.
139  * <p>
140  *
141  * If the <code>MediaHandler</code> is a <code>MediaProxy</code>
142  * then a new <code>DataSource</code> is obtained from the
143  * <code>MediaProxy</code>, a new list is created for the
144  * content type the <code>DataSource</code> supports and the
145  * whole thing is tried again.
146  * <p>
147  * 
148  * If a valid <code>Player</code>, is not found then the whole
149  * procedure is repeated is repeated with "unknown" substituted
150  * for the content-type name. The "unknown" content type is supported
151  * by generic <code>Players</code> that are capable of handling
152  * a large variety of media types, often in a platform dependent
153  * way.
154  *<p>
155  *
156  * The detailed creation algorithm is specified in the methods below.
157  * <p>
158  *
159  * <h2>Player Threads</h2>
160  *
161  * <code>Players</code> render media data asynchronously from
162  * the main program flow.
163  * This implies that a <code>Player</code> must often manage one
164  * or more threads.
165  * The threads managed by the <code>Player</code> are not
166  * in the thread group of the application that calls
167  * <code>createPlayer</code>.
168  *
169  * <h2>System Time Base</h2>
170  *
171  * All <code>Players</code> need a <code>TimeBase</code>. Many  
172  * use a system-wide <code>TimeBase</code>, often based on
173  * a time-of-day clock.
174  * <code>Manager</code> provides access to the system <code>TimeBase</code>
175  * through <code>getSystemTimeBase</code>.
176  * 
177  *
178  * @see java.net.URL
179  * @see MediaLocator
180  * @see PackageManager
181  * @see javax.media.protocol.DataSource
182  * @see javax.media.protocol.URLDataSource
183  * @see MediaHandler
184  * @see Player
185  * @see MediaProxy
186  * @see TimeBase
187  *
188  * @version 1.58, 07/09/19.
189  */

219     /** 
220      * Create a <code>Player</code> for the specified media.
221      * <p>
222      * The algorithm for creating a <CODE>Player</CODE> from
223      * a <code>MediaLocator</code> is:
224      * <ol>
225      * <li>Get the protocol from the <code>MediaLocator</code>.
226      * <li>Get a list of <code>DataSource</code> classes that
227      * support the protocol, using the protocol package-prefix-list.
228      * <li> For each source class in the list:
229      * <ol>
230      * <li>Instantiate a new <code>DataSource</code>,
231      * <li>Call the <code>connect</code> method to connect the source.
232      * <li>Get the media content-type-name (using <code>getContentType</code>)
233      * from the source.
234      * <li>Get a list of <code>MediaHandler</code> classes that support the
235      * media-content-type-name, using the content package-prefix-list.
236      * <li>For each <code>MediaHandler</code> class in the list:
237      * <ol>
238      * <li>Instantiate a new <code>MediaHandler</code>.
239      * <li>Attach the source to the <code>MediaHandler</code> by calling
240      * <code>MediaHandler.setSource</code>.
241      * <li>If there are no failures, determine the type of
242      * the <code>MediaHandler</code>; otherwise try the next
243      * <coded>MediaHandler</code> in the list.
244      * <li>If the <code>MediaHandler</code> is a <code>Player</code>,
245      * return the new <code>Player</code>.
246      * <li>If the <code>MediaHandler</code> is a <code>MediaProxy</code>,
247      * obtain a new <code>DataSource</code> from the <code>MediaProxy</code>,
248      * obtain the list of <code>MediaHandlers</code> that support the new
249      * <code>DataSource</code>, and continue searching the new list.
250      * </ol>
251      * <li>If no <code>MediaHandler</code> is found for this source,
252      * try the next source in the list.
253      * </ol>
254      * <li>If no <code>Player</code> is found after trying all of the sources,
255      * reuse the source list.<br>
256      * This time, for each source class in the list:
257      * <ol>
258      * <li>Instantiate the source.
259      * <li>Call the <code>connect</code> method to connect to the source.
260      * <li>Use the content package-prefix-list to create a list of 
261      * <code>MediaHandler</code> classes that support the
262      * "unknown" content-type-name.
263      * <li>For each <code>MediaHandler</code> class in the list,
264      * search for a <code>Player</code> as in the previous search.
265      * <ol>
266      * <li>If no <code>Player</code> is found after trying all of the sources,
267      * a <CODE>NoPlayerException</CODE> is thrown.
268      * </ol>
269      * </ol>
270      * @param sourceLocator A <CODE>MediaLocator</CODE> that describes
271      * the media content.
272      * @return A <CODE>Player</CODE> for the media described by the source.
273      * @exception NoPlayerException Thrown if no <CODE>Player</CODE> can
274      * be found.
275      * @exception IOException Thrown if there was a problem connecting
276      * with the source.
277      */
