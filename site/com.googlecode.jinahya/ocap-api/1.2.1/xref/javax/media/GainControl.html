40  /** 
41   * <code>GainControl</code> is an interface for manipulating audio signal
42   * gain.
43   *
44   * <h3>Gain and Gain Measures</h3>
45   * Gain is a multiplicative value applied to
46   * an audio signal that modifies the amplitude
47   * of the signal. 
48   * This interface allows the gain to be specified in either decibels
49   * or using a floating point value that varies between 0.0 and 
50   * 1.0.
51   *
52   * <h4>Specifying Gain in Decibels</h4>
53   * The decibel scale is valid over all <code>float</code>
54   * values. A gain of 0.0 dB implies that the audio signal
55   * is neither amplified nor attenuated. Positive values
56   * amplify the audio signal, negative values attenuate
57   * the audio signal. The relationship between a linear
58   * gain multiplier and the gain specified in decibels is:
59   * <p>
60   *
61   *<CENTER><CODE> value = pow(10.0, gainDB/20.0) </CODE></CENTER>
62   *
63   *<h4>Specifying Gain in the Level Scale</h4>
64   * The level scale ranges from 0.0 to 1.0, where 0.0 represents
65   * a gain that is virtually indistinguishable from silence and
66   * 1.0 represents the value that is, in some sense,
67   * the maximum gain. In other words, 1.0 represents the highest gain value that 
68   * produces "useful" results. The mapping for
69   * producing a linear multiplicative value is
70   * implementation dependent.
71   *
72   * <h4>Decibel and Level Interactions</h4>
73   * The dB and level scales are representations
74   * of the same gain value. Calling <code>setLevel</code> 
75   * will affect subsequent <code>getDB</code> invocations.
76   * Level and dB are interrelated in the following ways:
77   *
78   * <ul>
79   * <li>Level Silence Threshold. After <code>setLevel(0.0)</code>, <code>getDB</code> 
80   * returns the value for which smaller values are not usefully 
81   * distinguishable from silence.
82   * Calling <code>setDB</code> with values equal to or less than this silence
83   * threshold causes <code>getLevel</code> to return a value of 0.0.
84   *
85   * <li>Level Maximum Threshold. After <code>setLevel(1.0)</code>, <code>getDB</code>
86   * returns the value for which  larger values are not useful. 
87   * Calling <code>setDB</code> with values equal to or greater than this
88   * threshold causes <code>getLevel</code> to return a value of 1.0.
89   *
90   * <li>The decibel interface is not limited to the thresholds
91   * described by the level interface.
92   * For example, if you call <code>setDB</code> with a value that is greater than the
93   * maximum level threshold and then immediately call 
94   * <code>getDB</code>,  <code>getDB</code>  returns the gain 
95   * that was returned by the <code>setDB</code>,   <i>not</i> the value that
96   * would be returned if you called <code>setLevel(1.0)</code> and then
97   * called <code>getDB</code>.
98   * 
99   * <li>Both measures increase gain monotonically with increasing
100  *  measure values.
101  *  </ul>
102  *
103  * <h4>Defaults</h4>
104  * Gain defaults to a value of 0.0 dB. The corresponding level
105  * is implementation dependent. Note that for some implementations,
106  * the default level might change on a per-instance basis.
107  *
108  * <h3>Mute</h3>
109  * Muting is independent of the gain. If mute is <CODE>true</CODE>,
110  * no audio signal is produced by this object; if mute is <CODE>false</CODE>
111  * an audio signal is produced and the gain is applied to the
112  * signal.
113  *
114  * <h3>Gain Change Events</h3>
115  * When the state of the <code>GainControl</code> changes, a
116  * <code>GainChangeEvent</code> is posted.
117  * This event is delivered through an object
118  * that implements <code>GainChangeListener</code> and has been registered as a listener
119  * with the <code>GainControl</code> using <code>addGainChangeListener</code>.
120  *
121  * @see GainChangeEvent
122  * @see GainChangeListener
123  * @see Control
124  * @version 1.34, 05/10/18
125  */
