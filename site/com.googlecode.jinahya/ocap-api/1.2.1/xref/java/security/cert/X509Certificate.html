47  /** 
48   * <p>
49   * Abstract class for X.509 certificates. This provides a standard
50   * way to access all the attributes of an X.509 certificate.
51   * <p>
52   * In June of 1996, the basic X.509 v3 format was completed by
53   * ISO/IEC and ANSI X9, which is described below in ASN.1:
54   * <pre>
55   * Certificate  ::=  SEQUENCE  {
56   *     tbsCertificate       TBSCertificate,
57   *     signatureAlgorithm   AlgorithmIdentifier,
58   *     signature            BIT STRING  }
59   * </pre>
60   * <p>
61   * These certificates are widely used to support authentication and
62   * other functionality in Internet security systems. Common applications
63   * include Privacy Enhanced Mail (PEM), Transport Layer Security (SSL),
64   * code signing for trusted software distribution, and Secure Electronic
65   * Transactions (SET).
66   * <p>
67   * These certificates are managed and vouched for by <em>Certificate
68   * Authorities</em> (CAs). CAs are services which create certificates by
69   * placing data in the X.509 standard format and then digitally signing
70   * that data. CAs act as trusted third parties, making introductions
71   * between principals who have no direct knowledge of each other.
72   * CA certificates are either signed by themselves, or by some other
73   * CA such as a "root" CA.
74   * <p>
75   * More information can be found in RFC 2459,
76   * "Internet X.509 Public Key Infrastructure Certificate and CRL
77   * Profile" at <A HREF="http://www.ietf.org/rfc/rfc2459.txt"> 
78   * http://www.ietf.org/rfc/rfc2459.txt </A>.
79   * <p>
80   * The ASN.1 definition of <code>tbsCertificate</code> is:
81   * <pre>
82   * TBSCertificate  ::=  SEQUENCE  {
83   *     version         [0]  EXPLICIT Version DEFAULT v1,
84   *     serialNumber         CertificateSerialNumber,
85   *     signature            AlgorithmIdentifier,
86   *     issuer               Name,
87   *     validity             Validity,
88   *     subject              Name,
89   *     subjectPublicKeyInfo SubjectPublicKeyInfo,
90   *     issuerUniqueID  [1]  IMPLICIT UniqueIdentifier OPTIONAL,
91   *                          -- If present, version must be v2 or v3
92   *     subjectUniqueID [2]  IMPLICIT UniqueIdentifier OPTIONAL,
93   *                          -- If present, version must be v2 or v3
94   *     extensions      [3]  EXPLICIT Extensions OPTIONAL
95   *                          -- If present, version must be v3
96   *     }
97   * </pre>
98   * <p>
99   * Certificates are instantiated using a certificate factory. The following is
100  * an example of how to instantiate an X.509 certificate:
101  * <pre> 
102  * InputStream inStream = new FileInputStream("fileName-of-cert");
103  * CertificateFactory cf = CertificateFactory.getInstance("X.509");
104  * X509Certificate cert = (X509Certificate)cf.generateCertificate(inStream);
105  * inStream.close();
106  * </pre>
107  *
108  * @author Hemma Prafullchandra
109  *
110  * @version 1.27
111  *
112  * @see Certificate
113  * @see CertificateFactory
114  * @see X509Extension
115  */
116 public abstract class X509Certificate extends Certificate
117     implements X509Extension

132      * The validity period consists of two date/time values: 
133      * the first and last dates (and times) on which the certificate 
134      * is valid. It is defined in
135      * ASN.1 as:
136      * <pre>
137      * validity             Validity<p>
138      * Validity ::= SEQUENCE {
139      *     notBefore      CertificateValidityDate,

494      * Gets the certificate constraints path length from the
495      * critical <code>BasicConstraints</code> extension, (OID = 2.5.29.19).
496      * <p>
497      * The basic constraints extension identifies whether the subject
498      * of the certificate is a Certificate Authority (CA) and 
499      * how deep a certification path may exist through that CA. The 
500      * <code>pathLenConstraint</code> field (see below) is meaningful
501      * only if <code>cA</code> is set to TRUE. In this case, it gives the
502      * maximum number of CA certificates that may follow this certificate in a
503      * certification path. A value of zero indicates that only an end-entity
504      * certificate may follow in the path.
505      * <p>
506      * Note that for RFC 2459 this extension is always marked
507      * critical if <code>cA</code> is TRUE, meaning this certificate belongs
508      * to a Certificate Authority.
509      * <p>
510      * The ASN.1 definition for this is:
511      * <pre>
512      * BasicConstraints ::= SEQUENCE {
513      *     cA                  BOOLEAN DEFAULT FALSE,
514      *     pathLenConstraint   INTEGER (0..MAX) OPTIONAL }
515      * </pre>
516      *
517      * @return the value of <code>pathLenConstraint</code> if the
518      * BasicConstraints extension is present in the certificate and the
519      * subject of the certificate is a CA, otherwise -1.
520      * If the subject of the certificate is a CA and
521      * <code>pathLenConstraint</code> does not appear,
522      * <code>Integer.MAX_VALUE</code> is returned to indicate that there is no
523      * limit to the allowed length of the certification path.

528      * Gets an immutable collection of subject alternative names from the
529      * <code>SubjectAltName</code> extension, (OID = 2.5.29.17).
530      * <p>
531      * The ASN.1 definition of the <code>SubjectAltName</code> extension is:
532      * <pre>
533      * SubjectAltName ::= GeneralNames
534      *
535      * GeneralNames :: = SEQUENCE SIZE (1..MAX) OF GeneralName
536      *
537      * GeneralName ::= CHOICE {
538      *      otherName                       [0]     OtherName,
539      *      rfc822Name                      [1]     IA5String,
540      *      dNSName                         [2]     IA5String,
541      *      x400Address                     [3]     ORAddress,
542      *      directoryName                   [4]     Name,
543      *      ediPartyName                    [5]     EDIPartyName,
544      *      uniformResourceIdentifier       [6]     IA5String,
545      *      iPAddress                       [7]     OCTET STRING,
546      *      registeredID                    [8]     OBJECT IDENTIFIER}
547      * </pre>
548      * <p>
549      * If this certificate does not contain a <code>SubjectAltName</code>
550      * extension, <code>null</code> is returned. Otherwise, a 
551      * <code>Collection</code> is returned with an entry representing each 
552      * <code>GeneralName</code> included in the extension. Each entry is a 
553      * <code>List</code> whose first entry is an <code>Integer</code> 
554      * (the name type, 0-8) and whose second entry is a <code>String</code> 
555      * or a byte array (the name, in string or ASN.1 DER encoded form, 
556      * respectively).
557      * <p>
558      * RFC 822, DNS, and URI names are returned as <code>String</code>s, 
559      * using the well-established string formats for those types (subject to
560      * the restrictions included in RFC 2459). IPv4 address names are 
561      * returned using dotted quad notation. IPv6 address names are returned
562      * in the form "a1:a2:...:a8", where a1-a8 are hexadecimal values 
563      * representing the eight 16-bit pieces of the address. OID names are 
564      * returned as <code>String</code>s represented as a series of nonnegative 
565      * integers separated by periods. And directory names (distinguished names) 
566      * are returned in RFC 2253 string format. No standard string format is 
567      * defined for otherNames, X.400 names, EDI party names, or any 
568      * other type of names. They are returned as byte arrays 
569      * containing the ASN.1 DER encoded form of the name.
570      * <p>
571      * Note that the <code>Collection</code> returned may contain more
572      * than one name of the same type. Also, note that the returned
573      * <code>Collection</code> is immutable and any entries containing byte 
574      * arrays are cloned to protect against subsequent modifications.
575      * <p>
576      * This method was added to version 1.4 of the Java 2 Platform Standard 
577      * Edition. In order to maintain backwards compatibility with existing 
578      * service providers, this method is not <code>abstract</code>
579      * and it provides a default implementation. Subclasses
580      * should override this method with a correct implementation.
581      *
582      * @return an immutable <code>Collection</code> of subject alternative 
583      * names (or <code>null</code>)
584      * @throws CertificateParsingException if the extension cannot be decoded
585      * @since 1.4
586      */

588         throws CertificateParsingException
