46  /** 
47   * An immutable sequence of certificates (a certification path).
48   * <p>
49   * This is an abstract class that defines the methods common to all
50   * <code>CertPath</code>s. Subclasses can handle different kinds of
51   * certificates (X.509, PGP, etc.).
52   * <p>
53   * All <code>CertPath</code> objects have a type, a list of 
54   * <code>Certificate</code>s, and one or more supported encodings. Because the 
55   * <code>CertPath</code> class is immutable, a <code>CertPath</code> cannot 
56   * change in any externally visible way after being constructed. This 
57   * stipulation applies to all public fields and methods of this class and any 
58   * added or overridden by subclasses.
59   * <p>
60   * The type is a <code>String</code> that identifies the type of
61   * <code>Certificate</code>s in the certification path. For each
62   * certificate <code>cert</code> in a certification path <code>certPath</code>,
63   * <code>cert.getType().equals(certPath.getType())</code> must be
64   * <code>true</code>.
65   * <p>
66   * The list of <code>Certificate</code>s is an ordered <code>List</code> of
67   * zero or more <code>Certificate</code>s. This <code>List</code> and all
68   * of the <code>Certificate</code>s contained in it must be immutable.
69   * <p>
70   * Each <code>CertPath</code> object must support one or more encodings
71   * so that the object can be translated into a byte array for storage or
72   * transmission to other parties. Preferably, these encodings should be
73   * well-documented standards (such as PKCS#7). One of the encodings supported
74   * by a <code>CertPath</code> is considered the default encoding. This
75   * encoding is used if no encoding is explicitly requested (for the
76   * {@link #getEncoded() getEncoded()} method, for instance).
77   * <p>
78   * All <code>CertPath</code> objects are also <code>Serializable</code>. 
79   * <code>CertPath</code> objects are resolved into an alternate 
80   * {@link CertPathRep CertPathRep} object during serialization. This allows
81   * a <code>CertPath</code> object to be serialized into an equivalent
82   * representation regardless of its underlying implementation.
83   * <p>
84   * By convention, X.509 <code>CertPath</code>s (consisting of
85   * <code>X509Certificate</code>s), are ordered starting with the target 
86   * certificate and ending with a certificate issued by the trust anchor. That 
87   * is, the issuer of one certificate is the subject of the following one. 
88   * Unvalidated X.509 <code>CertPath</code>s 
89   * may not follow these conventions.
90   * <p>
91   * <b>Concurrent Access</b>
92   * <p>
93   * All <code>CertPath</code> objects must be thread-safe. That is, multiple
94   * threads may concurrently invoke the methods defined in this class on a
95   * single <code>CertPath</code> object (or more than one) with no
96   * ill effects. This is also true for the <code>List</code> returned by
97   * <code>CertPath.getCertificates</code>.
98   * <p>
99   * Requiring <code>CertPath</code> objects to be immutable and thread-safe
100  * allows them to be passed around to various pieces of code without worrying
101  * about coordinating access.  Providing this thread-safety is
102  * generally not difficult, since the <code>CertPath</code> and
103  * <code>List</code> objects in question are immutable.
104  *
105  * @see CertificateFactory
106  *
107  * @version 1.9 03/12/05
108  * @author	Yassir Elley
109  * @since	1.4
110  */
