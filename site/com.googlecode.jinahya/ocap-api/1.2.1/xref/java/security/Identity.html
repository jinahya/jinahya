43  /** 
44   * <p>This class represents identities: real-world objects such as people,
45   * companies or organizations whose identities can be authenticated using 
46   * their public keys. Identities may also be more abstract (or concrete) 
47   * constructs, such as daemon threads or smart cards.
48   *
49   * <p>All Identity objects have a name and a public key. Names are
50   * immutable. Identities may also be scoped. That is, if an Identity is
51   * specified to have a particular scope, then the name and public
52   * key of the Identity are unique within that scope.
53   *
54   * <p>An Identity also has a set of certificates (all certifying its own
55   * public key). The Principal names specified in these certificates need 
56   * not be the same, only the key.
57   *
58   * <p>An Identity can be subclassed, to include postal and email addresses,
59   * telephone numbers, images of faces and logos, and so on.
60   *
61   * @see IdentityScope
62   * @see Signer
63   * @see Principal
64   *
