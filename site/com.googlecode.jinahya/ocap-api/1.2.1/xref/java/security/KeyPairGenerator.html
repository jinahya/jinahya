41  /** 
42   * The KeyPairGenerator class is used to generate pairs of
43   * public and private keys. Key pair generators are constructed using the
44   * <code>getInstance</code> factory methods (static methods that
45   * return instances of a given class).
46   *
47   * <p>A Key pair generator for a particular algorithm creates a public/private
48   * key pair that can be used with this algorithm. It also associates
49   * algorithm-specific parameters with each of the generated keys.
50   * 
51   * <p>There are two ways to generate a key pair: in an algorithm-independent
52   * manner, and in an algorithm-specific manner.
53   * The only difference between the two is the initialization of the object:
54   * 
55   * <ul>
56   * <li><b>Algorithm-Independent Initialization</b>
57   * <p>All key pair generators share the concepts of a keysize and a
58   * source of randomness. The keysize is interpreted differently for different
59   * algorithms (e.g., in the case of the <i>DSA</i> algorithm, the keysize 
60   * corresponds to the length of the modulus).
61   * There is an 
62   * {@link #initialize(int, java.security.SecureRandom) initialize}
63   * method in this KeyPairGenerator class that takes these two universally
64   * shared types of arguments. There is also one that takes just a
65   * <code>keysize</code> argument, and uses the <code>SecureRandom</code>
66   * implementation of the highest-priority installed provider as the source
67   * of randomness. (If none of the installed providers supply an implementation
68   * of <code>SecureRandom</code>, a system-provided source of randomness is
69   * used.)
70   * 
71   * <p>Since no other parameters are specified when you call the above
72   * algorithm-independent <code>initialize</code> methods, it is up to the
73   * provider what to do about the algorithm-specific parameters (if any) to be
74   * associated with each of the keys.
75   * 
76   * <p>If the algorithm is the <i>DSA</i> algorithm, and the keysize (modulus
77   * size) is 512, 768, or 1024, then the <i>Sun</i> provider uses a set of
78   * precomputed values for the <code>p</code>, <code>q</code>, and
79   * <code>g</code> parameters. If the modulus size is not one of the above
80   * values, the <i>Sun</i> provider creates a new set of parameters. Other
81   * providers might have precomputed parameter sets for more than just the
82   * three modulus sizes mentioned above. Still others might not have a list of
83   * precomputed parameters at all and instead always create new parameter sets.
84   * <p>
85   *
86   * <li><b>Algorithm-Specific Initialization</b>
87   * <p>For situations where a set of algorithm-specific parameters already
88   * exists (e.g., so-called <i>community parameters</i> in DSA), there are two
89   * {@link #initialize(java.security.spec.AlgorithmParameterSpec)
90   * initialize} methods that have an <code>AlgorithmParameterSpec</code>
91   * argument. One also has a <code>SecureRandom</code> argument, while the
92   * the other uses the <code>SecureRandom</code>
93   * implementation of the highest-priority installed provider as the source
94   * of randomness. (If none of the installed providers supply an implementation
95   * of <code>SecureRandom</code>, a system-provided source of randomness is
96   * used.)
97   * </ul>
98   *
99   * <p>In case the client does not explicitly initialize the KeyPairGenerator
100  * (via a call to an <code>initialize</code> method), each provider must
101  * supply (and document) a default initialization.
102  * For example, the <i>Sun</i> provider uses a default modulus size (keysize)
103  * of 1024 bits.
104  *
105  * <p>Note that this class is abstract and extends from
106  * <code>KeyPairGeneratorSpi</code> for historical reasons.
107  * Application developers should only take notice of the methods defined in
108  * this <code>KeyPairGenerator</code> class; all the methods in
109  * the superclass are intended for cryptographic service providers who wish to
110  * supply their own implementations of key pair generators.
111  *
112  * @author Benjamin Renaud
113  *
114  * @version 1.49, 02/02/00
115  *
116  * @see java.security.spec.AlgorithmParameterSpec
117  */
