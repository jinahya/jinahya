39  /** 
40   * The Key interface is the top-level interface for all keys. It
41   * defines the functionality shared by all key objects. All keys
42   * have three characteristics:
43   *
44   * <UL>
45   *
46   * <LI>An Algorithm
47   *
48   * <P>This is the key algorithm for that key. The key algorithm is usually
49   * an encryption or asymmetric operation algorithm (such as DSA or
50   * RSA), which will work with those algorithms and with related
51   * algorithms (such as MD5 with RSA, SHA-1 with RSA, Raw DSA, etc.)
52   * The name of the algorithm of a key is obtained using the
53   * {@link #getAlgorithm() getAlgorithm} method.<P>
54   *
55   * <LI>An Encoded Form
56   *
57   * <P>This is an external encoded form for the key used when a standard
58   * representation of the key is needed outside the Java Virtual Machine,
59   * as when transmitting the key to some other party. The key
60   * is encoded according to a standard format (such as 
61   * X.509 <code>SubjectPublicKeyInfo</code> or PKCS#8), and
62   * is returned using the {@link #getEncoded() getEncoded} method.
63   * Note: The syntax of the ASN.1 type <code>SubjectPublicKeyInfo</code> 
64   * is defined as follows:
65   *
66   * <pre>
67   * SubjectPublicKeyInfo ::= SEQUENCE {
68   *   algorithm AlgorithmIdentifier,
69   *   subjectPublicKey BIT STRING }
70   *
71   * AlgorithmIdentifier ::= SEQUENCE {
72   *   algorithm OBJECT IDENTIFIER,
73   *   parameters ANY DEFINED BY algorithm OPTIONAL }
74   * </pre>
75   * 
76   * For more information, see 
77   * <a href="http://www.ietf.org/rfc/rfc2459.txt">RFC 2459: 
78   * Internet X.509 Public Key Infrastructure Certificate and CRL Profile</a>.
79   * <P>
80   *
81   * <LI>A Format
82   *
83   * <P>This is the name of the format of the encoded key. It is returned
84   * by the {@link #getFormat() getFormat} method.<P>
85   *
86   * </UL>
87   *
88   * Keys are generally obtained through key generators, certificates,
89   * or various Identity classes used to manage keys.
90   * Keys may also be obtained from key specifications (transparent
91   * representations of the underlying key material) through the use of a key
92   * factory (see {@link KeyFactory}).
93   * NOTE: <B>java.security.Identity</B> is found in J2ME CDC profiles such as 
94   * J2ME Foundation Profile.
95   *
96   * @see PublicKey
97   * @see PrivateKey
98   * @see KeyPair
99   * @see KeyPairGenerator
100  * @see KeyFactory
101  * @see java.security.spec.KeySpec
102  * @see Identity
103  * @see Signer
104  *
105  * @version 1.47 00/02/02
106  * @author Benjamin Renaud
107  */

117     /** 
118      * Returns the standard algorithm name for this key. For
119      * example, "DSA" would indicate that this key is a DSA key.
120      * See Appendix A in the <a href=
121      * "../../../guide/security/CryptoSpec.html#AppA">
122      * Java Cryptography Architecture API Specification &amp; Reference </a>
123      * for information about standard algorithm names.
124      *
125      * @return the name of the algorithm associated with this key.
126      */
127     public String getAlgorithm();
128 
129     /** 
130      * Returns the name of the primary encoding format of this key,
131      * or null if this key does not support encoding.
132      * The primary encoding format is
133      * named in terms of the appropriate ASN.1 data format, if an
134      * ASN.1 specification for this key exists.
135      * For example, the name of the ASN.1 data format for public
136      * keys is <I>SubjectPublicKeyInfo</I>, as
137      * defined by the X.509 standard; in this case, the returned format is
138      * <code>"X.509"</code>. Similarly,
139      * the name of the ASN.1 data format for private keys is
140      * <I>PrivateKeyInfo</I>,
141      * as defined by the PKCS #8 standard; in this case, the returned format is
142      * <code>"PKCS#8"</code>.
143      *
144      * @return the primary encoding format of the key.
145      */
146     public String getFormat();

151      *
152      * @return the encoded key, or null if the key does not support
153      * encoding.
154      */
155     public byte[] getEncoded();
