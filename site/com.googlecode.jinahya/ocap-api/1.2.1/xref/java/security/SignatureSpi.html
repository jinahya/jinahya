44  /** 
45   * This class defines the <i>Service Provider Interface</i> (<b>SPI</b>)
46   * for the <code>Signature</code> class, which is used to provide the
47   * functionality of a digital signature algorithm. Digital signatures are used
48   * for authentication and integrity assurance of digital data.
49   *.
50   * <p> All the abstract methods in this class must be implemented by each
51   * cryptographic service provider who wishes to supply the implementation
52   * of a particular signature algorithm.

58   * @see Signature

151     /** 
152      * Finishes this signature operation and stores the resulting signature

156      * signature scheme.
157      *

181      *
182      * @return the number of bytes placed into <code>outbuf</code>
