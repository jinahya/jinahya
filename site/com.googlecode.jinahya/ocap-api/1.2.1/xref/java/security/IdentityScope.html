43  /** 
44   * <p>This class represents a scope for identities. It is an Identity 
45   * itself, and therefore has a name and can have a scope. It can also 
46   * optionally have a public key and associated certificates.
47   *
48   * <p>An IdentityScope can contain Identity objects of all kinds, including
49   * Signers. All types of Identity objects can be retrieved, added, and 
50   * removed using the same methods. Note that it is possible, and in fact
51   * expected, that different types of identity scopes will
52   * apply different policies for their various operations on the
53   * various types of Identities.
54   *
55   * <p>There is a one-to-one mapping between keys and identities, and 
56   * there can only be one copy of one key per scope. For example, suppose
57   * <b>Acme Software, Inc</b> is a software publisher known to a user.
58   * Suppose it is an Identity, that is, it has a public key, and a set of
59   * associated certificates. It is named in the scope using the name 
60   * "Acme Software". No other named Identity in the scope has the same 
61   * public  key. Of course, none has the same name as well.
62   *
63   * @see Identity
64   * @see Signer
65   * @see Principal
66   * @see Key
67   *
68   * @version 1.46 00/02/02
69   * @author Benjamin Renaud
70   *
71   * @deprecated This class is no longer used. Its functionality has been
72   * replaced by <code>java.security.KeyStore</code>, the
73   * <code>java.security.cert</code> package, and
74   * <code>java.security.Principal</code>.
75   */
