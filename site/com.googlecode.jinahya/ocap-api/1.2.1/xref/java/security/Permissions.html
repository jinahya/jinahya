54  /** 
55   * This class represents a heterogeneous collection of Permissions. That is,
56   * it contains different types of Permission objects, organized into
57   * PermissionCollections. For example, if any
58   * <code>java.io.FilePermission</code> objects are added to an instance of
59   * this class, they are all stored in a single
60   * PermissionCollection. It is the PermissionCollection returned by a call to
61   * the <code>newPermissionCollection</code> method in the FilePermission class.
62   * Similarly, any <code>java.lang.RuntimePermission</code> objects are
63   * stored in the PermissionCollection returned by a call to the 
64   * <code>newPermissionCollection</code> method in the
65   * RuntimePermission class. Thus, this class represents a collection of
66   * PermissionCollections.
67   * 
68   * <p>When the <code>add</code> method is called to add a Permission, the 
69   * Permission is stored in the appropriate PermissionCollection. If no such 
70   * collection exists yet, the Permission object's class is determined and the
71   * <code>newPermissionCollection</code> method is called on that class to create
72   * the PermissionCollection and add it to the Permissions object. If
73   * <code>newPermissionCollection</code> returns null, then a default 
74   * PermissionCollection that uses a hashtable will be created and used. Each 
75   * hashtable entry stores a Permission object as both the key and the value.
76   *
77   * <p> Enumerations returned via the <code>elements</code> method are
78   * not <em>fail-fast</em>.  Modifications to a collection should not be
79   * performed while enumerating over that collection.
80   * 
81   * @see Permission
82   * @see PermissionCollection
83   * @see AllPermission
