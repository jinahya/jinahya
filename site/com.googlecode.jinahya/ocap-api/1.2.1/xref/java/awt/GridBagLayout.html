41  /** 
42   * The <code>GridBagLayout</code> class is a flexible layout
43   * manager that aligns components vertically and horizontally,
44   * without requiring that the components be of the same size.
45   * Each <code>GridBagLayout</code> object maintains a dynamic,
46   * rectangular grid of cells, with each component occupying
47   * one or more cells, called its <em>display area</em>.
48   * <p>
49   * Each component managed by a <code>GridBagLayout</code> is associated with
50   * an instance of {@link GridBagConstraints}.  The constraints object
51   * specifies where a component's display area should be located on the grid
52   * and how the component should be positioned within its display area.  In
53   * addition to its constraints object, the <code>GridBagLayout</code> also
54   * considers each component's minimum and preferred sizes in order to
55   * determine a component's size.
56   * <p>
57   * <!-- PBP/PP -->
58   * <!-- 6187232 -->
59   *  Grid coordinate (0,0) is in the upper left corner of the
60   * container with x increasing to the right and y increasing downward. 
61   * <p>
62   * To use a grid bag layout effectively, you must customize one or more
63   * of the <code>GridBagConstraints</code> objects that are associated
64   * with its components. You customize a <code>GridBagConstraints</code>
65   * object by setting one or more of its instance variables:
66   * <p>
67   * <!-- PBP/PP -->
68   * <!-- 6187232 -->
69   * <dl>
70   * <dt>{@link GridBagConstraints#gridx},
71   * {@link GridBagConstraints#gridy}
72   * <dd>Specifies the cell  <em>at the upper left</em> of the component's 
73   * display area, where the cell at the origin of the grid has address
74   * <code>gridx&nbsp;=&nbsp;0</code>,
75   * <code>gridy&nbsp;=&nbsp;0</code>.  
76   * Use <code>GridBagConstraints.RELATIVE</code> (the default value)
77   * to specify that the component be placed immediately following
78   * (along the x axis for <code>gridx</code> or the y axis for 
79   * <code>gridy</code>) the component that was added to the container
80   * just before this component was added.
81   * <dt>{@link GridBagConstraints#gridwidth},
82   * {@link GridBagConstraints#gridheight}
83   * <dd>Specifies the number of cells in a row (for <code>gridwidth</code>)
84   * or column (for <code>gridheight</code>)
85   * in the component's display area.
86   * The default value is 1.
87   * Use <code>GridBagConstraints.REMAINDER</code> to specify
88   * that the component be the last one in its row (for <code>gridwidth</code>)
89   * or column (for <code>gridheight</code>).
90   * Use <code>GridBagConstraints.RELATIVE</code> to specify
91   * that the component be the next to last one
92   * in its row (for <code>gridwidth</code>)
93   * or column (for <code>gridheight</code>).
94   * <dt>{@link GridBagConstraints#fill}
95   * <dd>Used when the component's display area
96   * is larger than the component's requested size
97   * to determine whether (and how) to resize the component.
98   * Possible values are
99   * <code>GridBagConstraints.NONE</code> (the default),
100  * <code>GridBagConstraints.HORIZONTAL</code>
101  * (make the component wide enough to fill its display area
102  * horizontally, but don't change its height),
103  * <code>GridBagConstraints.VERTICAL</code>
104  * (make the component tall enough to fill its display area
105  * vertically, but don't change its width), and
106  * <code>GridBagConstraints.BOTH</code>
107  * (make the component fill its display area entirely).
108  * <dt>{@link GridBagConstraints#ipadx},
109  * {@link GridBagConstraints#ipady}
110  * <dd>Specifies the component's internal padding within the layout,
111  * how much to add to the minimum size of the component.
112  * The width of the component will be at least its minimum width
113  * plus <code>(ipadx&nbsp;*&nbsp;2)</code> pixels (since the padding
114  * applies to both sides of the component). Similarly, the height of
115  * the component will be at least the minimum height plus
116  * <code>(ipady&nbsp;*&nbsp;2)</code> pixels.
117  * <dt>{@link GridBagConstraints#insets}
118  * <dd>Specifies the component's external padding, the minimum
119  * amount of space between the component and the edges of its display area.
120  *
121  * <!-- PBP/PP-->
122  * <dt>{@link GridBagConstraints#anchor}
123  * <dd>Used when the component is smaller than its display area
124  * to determine where (within the display area) to place the component.
125  *  
126  * <p>
127  * Valid values are:</dd>
128  * <p>
129  * <center><table BORDER=0 COLS=2 WIDTH=800 SUMMARY="absolute and relative values as described above">
130  * <!- PBP/PP 6187232 ->
131  * 
132  * <tr>
133  * <td>
134  * <li><code>GridBagConstraints.NORTH</code></li>
135  * <li><code>GridBagConstraints.SOUTH</code></li>
136  * <li><code>GridBagConstraints.WEST</code></li>
137  * <li><code>GridBagConstraints.EAST</code></li>
138  * <li><code>GridBagConstraints.NORTHWEST</code></li>
139  * <li><code>GridBagConstraints.NORTHEAST</code></li>
140  * <li><code>GridBagConstraints.SOUTHWEST</code></li>
141  * <li><code>GridBagConstraints.SOUTHEAST</code></li>
142  * <li><code>GridBagConstraints.CENTER</code> (the default)</li>
143  * </td>
144  * 
145  * 
146  * 
147  * 
148  *

153  * 
154  * 
155  * </tr>
156  * </table></center><p>
157  * <dt>{@link GridBagConstraints#weightx},
158  * {@link GridBagConstraints#weighty}
159  * <dd>Used to determine how to distribute space, which is
160  * important for specifying resizing behavior.
161  * Unless you specify a weight for at least one component
162  * in a row (<code>weightx</code>) and column (<code>weighty</code>),
163  * all the components clump together in the center of their container.
164  * This is because when the weight is zero (the default),
165  * the <code>GridBagLayout</code> object puts any extra space
166  * between its grid of cells and the edges of the container.
167  * </dl>
