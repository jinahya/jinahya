42  /** 
43   * An event which indicates that the mouse wheel was rotated in a component.
44   * <P>
45   * A wheel mouse is a mouse which has a wheel in place of the middle button.
46   * This wheel can be rotated towards or away from the user.  Mouse wheels are
47   * most often used for scrolling, though other uses are possible.
48   * <P>
49   * A MouseWheelEvent object is passed to every <code>MouseWheelListener</code>
50   * object which registered to receive the "interesting" mouse events using the
51   * component's <code>addMouseWheelListener</code> method.  Each such listener
52   * object gets a <code>MouseEvent</code> containing the mouse event.
53   * <P>
54   * Due to the mouse wheel's special relationship to scrolling Components,
55   * MouseWheelEvents are delivered somewhat differently than other MouseEvents.
56   * This is because while other MouseEvents usually affect a change on 
57   * the Component directly under the mouse
58   * cursor (for instance, when clicking a button), MouseWheelEvents often have
59   * an effect away from the mouse cursor.
60   * <P>
61   * MouseWheelEvents start delivery from the Component underneath the
62   * mouse cursor.  If MouseWheelEvents are not enabled on the
63   * Component, the event is delivered to the first ancestor 
64   * Container with MouseWheelEvents enabled.  The source
65   * Component and x,y coordinates will be relative to the event's
66   * final destination.
67   * <P>
68   * Some AWT Components are implemented using native widgets which
69   * display their own scrollbars and handle their own scrolling.  
70   * The particular Components for which this is true will vary from
71   * platform to platform.  When the mouse wheel is
72   * moved over one of these Components, the event is delivered straight to
73   * the native widget, and not propagated to ancestors.
74   * <P>
75   * Platforms offer customization of the amount of scrolling that
76   * should take place when the mouse wheel is moved.  The two most
77   * common settings are to scroll a certain number of "units"
78   * (commonly lines of text in a text-based component) or an entire "block"
79   * (similar to page-up/page-down).  The MouseWheelEvent offers
80   * methods for conforming to the underlying platform settings.  These
81   * platform settings can be changed at any time by the user.  MouseWheelEvents
82   * reflect the most recent settings.
83   *
84   * <p>
85   * <em>
86   * Note: The generation of mouse wheel events is not guaranteed.
87   * Applications should not be written to rely on the receipt of such events
88   * for proper operation. Nonetheless, applications may still
89   * create MouseWheelEvent objects and post them to the system event queue.
90   * </em>
91   * @author Brent Christian
92   * @version 1.7 01/23/03
93   * @see MouseWheelListener
94   * 
95   * 
96   * @since 1.4
97   */
