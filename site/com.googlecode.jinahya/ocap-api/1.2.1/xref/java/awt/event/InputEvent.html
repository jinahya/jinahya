45   *
46   * Input events are delivered to listeners before they are

48   * This allows listeners and component subclasses to "consume"
49   * the event so that the source will not process them in their
50   * default manner.  For example, consuming mousePressed events

52   * activated.

243     /** 
244      * Returns the extended modifier mask for this event.
245      * Extended modifiers represent the state of all modal keys, 
246      * such as ALT, CTRL, META, and the mouse buttons just after 
247      * the event occurred
248      * <P> 
249      * For example, if the user presses <b>button 1</b> followed by
250      * <b>button 2</b>, and then releases them in the same order,
251      * the following sequence of events is generated:
252      * <PRE>
253      *    <code>MOUSE_PRESSED</code>:  <code>BUTTON1_DOWN_MASK</code>
254      *    <code>MOUSE_PRESSED</code>:  <code>BUTTON1_DOWN_MASK | BUTTON2_DOWN_MASK</code>
255      *    <code>MOUSE_RELEASED</code>: <code>BUTTON2_DOWN_MASK</code>
256      *    <code>MOUSE_CLICKED</code>:  <code>BUTTON2_DOWN_MASK</code>
257      *    <code>MOUSE_RELEASED</code>: 
258      *    <code>MOUSE_CLICKED</code>:  
259      * </PRE>
260      * <P>
261      * It is not recommended to compare the return value of this method
262      * using <code>==</code> because new modifiers can be added in the future.
263      * For example, the appropriate way to check that SHIFT and BUTTON1 are
264      * down, but CTRL is up is demonstrated by the following code:
265      * <PRE>
266      *    int onmask = SHIFT_DOWN_MASK | BUTTON1_DOWN_MASK;
267      *    int offmask = CTRL_DOWN_MASK;
268      *    if (event.getModifiersEx() & (onmask | offmask) == onmask) {
269      *        ...
270      *    }
271      * </PRE>
272      * The above code will work even if new modifiers are added.
273      * 
274      * @since 1.4
275      */
