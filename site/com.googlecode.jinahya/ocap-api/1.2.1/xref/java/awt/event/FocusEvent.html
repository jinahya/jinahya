45  /** 
46   * A low-level event which indicates that a Component has gained or lost the
47   * input focus. This low-level event is generated by a Component (such as a
48   * TextField). The event is passed to every <code>FocusListener</code> or
49   * <code>FocusAdapter</code> object which registered to receive such events
50   * using the Component's <code>addFocusListener</code> method. (<code>
51   * FocusAdapter</code> objects implement the <code>FocusListener</code>
52   * interface.) Each such listener object gets this <code>FocusEvent</code> when
53   * the event occurs.
54   * <p>
55   * There are two levels of focus events: permanent and temporary. Permanent
56   * focus change events occur when focus is directly moved from one Component to
57   * another, such as through a call to requestFocus() or as the user uses the
58   * TAB key to traverse Components. Temporary focus change events occur when
59   * focus is temporarily lost for a Component as the indirect result of another
60   * operation, such as Window deactivation. In this case,
61   * the original focus state will automatically be restored once that operation
62   * is finished, or, for the case of Window deactivation, when the Window is
63   * reactivated. Both permanent and temporary focus events are delivered using
64   * the FOCUS_GAINED and FOCUS_LOST event ids; the level may be distinguished in
65   * the event using the isTemporary() method.
66   *  
67   * @see FocusAdapter
68   * @see FocusListener
69   * @see <a href="http://java.sun.com/docs/books/tutorial/post1.0/ui/focuslistener.html">Tutorial: Writing a Focus Listener</a>
70   * @see <a href="http://www.awl.com/cp/javaseries/jcl1_2.html">Reference: The Java Class Libraries (update file)</a>
71   *
72   * @author Carl Quinn
73   * @author Amy Fowler
74   * @version 1.28 01/23/03
75   * @since 1.1
76   */
