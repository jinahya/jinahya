46  /** 
47   * An event which indicates that a mouse action occurred in a component.
48   * A mouse action is considered to occur in a particular component if and only
49   * if the mouse cursor is over the unobscured part of the component's bounds
50   * when the action happens.
51   * Component bounds can be obscurred by the visible component's children or by a
52   * menu or by a top-level window.
53   * This event is used both for mouse events (click, enter, exit) and mouse 
54   * motion events (moves and drags). 
55   * <P>
56   * This low-level event is generated by a component object for:
57   * <ul>
58   * <li>Mouse Events
59   *     <ul>
60   *     <li>a mouse button is pressed
61   *     <li>a mouse button is released
62   *     <li>a mouse button is clicked (pressed and released)
63   *     <li>the mouse cursor enters the unobscured part of component's geometry
64   *     <li>the mouse cursor exits the unobscured part of component's geometry
65   *     </ul>
66   * <li> Mouse Motion Events
67   *     <ul>
68   *     <li>the mouse is moved
69   *     <li>the mouse is dragged
70   *     </ul>
71   * </ul>
72   * <P>
73   * A <code>MouseEvent</code> object is passed to every
74   * <code>MouseListener</code>
75   * or <code>MouseAdapter</code> object which is registered to receive 
76   * the "interesting" mouse events using the component's 
77   * <code>addMouseListener</code> method.
78   * (<code>MouseAdapter</code> objects implement the 
79   * <code>MouseListener</code> interface.) Each such listener object 
80   * gets a <code>MouseEvent</code> containing the mouse event.
81   * <P>
82   * A <code>MouseEvent</code> object is also passed to every
83   * <code>MouseMotionListener</code> or
84   * <code>MouseMotionAdapter</code> object which is registered to receive 
85   * mouse motion events using the component's
86   * <code>addMouseMotionListener</code>
87   * method. (<code>MouseMotionAdapter</code> objects implement the 
88   * <code>MouseMotionListener</code> interface.) Each such listener object 
89   * gets a <code>MouseEvent</code> containing the mouse motion event.
90   * <P>
91   * When a mouse button is clicked, events are generated and sent to the
92   * registered <code>MouseListener</code>s.
93   * The state of modal keys can be retrieved using {@link InputEvent#getModifiers}
94   * and {@link InputEvent#getModifiersEx}.
95   * The button mask returned by {@link InputEvent#getModifiers} reflects
96   * only the button that changed state, not the current state of all buttons.
97   * (Note: Due to overlap in the values of ALT_MASK/BUTTON2_MASK and
98   * META_MASK/BUTTON3_MASK, this is not always true for mouse events involving
99   * modifier keys).
100  * To get the state of all buttons and modifier keys, use
101  * {@link InputEvent#getModifiersEx}.
102  * The button which has changed state is returned by {@link MouseEvent#getButton}
103  * <P> 
104  * For example, if the first mouse button is pressed, events are sent in the
105  * following order:
106  * <PRE>
107  *    <b   >id           </b   >   <b   >modifiers   </b   > <b   >button </b   >          
108  *    <code>MOUSE_PRESSED</code>:  <code>BUTTON1_MASK</code> <code>BUTTON1</code>
109  *    <code>MOUSE_RELEASED</code>: <code>BUTTON1_MASK</code> <code>BUTTON1</code>
110  *    <code>MOUSE_CLICKED</code>:  <code>BUTTON1_MASK</code> <code>BUTTON1</code>
111  * </PRE>
112  * When multiple mouse buttons are pressed, each press, release, and click
113  * results in a separate event. 
114  * <P> 
115  * For example, if the user presses <b>button 1</b> followed by
116  * <b>button 2</b>, and then releases them in the same order,
117  * the following sequence of events is generated:
118  * <PRE>
119  *    <b   >id           </b   >   <b   >modifiers   </b   > <b   >button </b   >          
120  *    <code>MOUSE_PRESSED</code>:  <code>BUTTON1_MASK</code> <code>BUTTON1</code>
121  *    <code>MOUSE_PRESSED</code>:  <code>BUTTON2_MASK</code> <code>BUTTON2</code>
122  *    <code>MOUSE_RELEASED</code>: <code>BUTTON1_MASK</code> <code>BUTTON1</code>
123  *    <code>MOUSE_CLICKED</code>:  <code>BUTTON1_MASK</code> <code>BUTTON1</code>
124  *    <code>MOUSE_RELEASED</code>: <code>BUTTON2_MASK</code> <code>BUTTON2</code>
125  *    <code>MOUSE_CLICKED</code>:  <code>BUTTON2_MASK</code> <code>BUTTON2</code>
126  * </PRE>
127  * If <b>button 2</b> is released first, the
128  * <code>MOUSE_RELEASED</code>/<code>MOUSE_CLICKED</code> pair
129  * for <code>BUTTON2_MASK</code> arrives first,
130  * followed by the pair for <code>BUTTON1_MASK</code>.
131  * <p>
132  *
133  * <code>MOUSE_DRAGGED</code> events are delivered to the <code>Component</code> 
134  * in which the mouse button was pressed until the mouse button is released 
135  * (regardless of whether the mouse position is within the bounds of the 
136  * <code>Component</code>).  Due to platform-dependent Drag&Drop implementations, 
137  * <code>MOUSE_DRAGGED</code> events may not be delivered during a native 
138  * Drag&Drop operation.  
139  * 
140  * In a multi-screen environment mouse drag events are delivered to the
141  * <code>Component</code> even if the mouse position is outside the bounds of the
142  * <code>GraphicsConfiguration</code> associated with that 
143  * <code>Component</code>. However, the reported position for mouse drag events
144  * in this case may differ from the actual mouse position: 
145  * <ul>
146  * <li>In a multi-screen environment without a virtual device:
147  * <br>
148  * The reported coordinates for mouse drag events are clipped to fit within the
149  * bounds of the <code>GraphicsConfiguration</code> associated with 
150  * the <code>Component</code>.
151  * <li>In a multi-screen environment with a virtual device:
152  * <br>
153  * The reported coordinates for mouse drag events are clipped to fit within the
154  * bounds of the virtual device associated with the <code>Component</code>.   
155  * </ul>
156  *
157  * <p>
158  * <a name="restrictions">
159  * <h4>Restrictions</h4>
160  * <em>
161  * Implementations of this Profile exhibit certain restrictions with respect
162  * to mouse events, specifically:
163  * <ul>
164  * <li> An implementation need not support the generation of all mouse
165  * events.  In such a case, it may partially support the generation of
166  * mouse events or not support them at all.
167  * <p>
168  * Regardless of this restriction, applications may still
169  * create MouseEvent objects and post them to the system event queue.
170  * </ul>
171  * For more information, see <a href="../../../doc-files/properties.html">Profile-specific properties</a>.
172  * <p>
173  * <b>Note:</b> These restrictions and their corresponding properties
174  * apply only to the <code>MouseEvent</code> class, and not to its subclass,
175  * {@link MouseWheelEvent}.
176  * </em>
177  * @author Carl Quinn
178  * 1.45, 01/23/03
179  *   
180  * @see MouseAdapter
181  * @see MouseListener
182  * @see MouseMotionAdapter
183  * @see MouseMotionListener
184  * @see <a href="http://java.sun.com/docs/books/tutorial/post1.0/ui/mouselistener.html">Tutorial: Writing a Mouse Listener</a>
185  * @see <a href="http://java.sun.com/docs/books/tutorial/post1.0/ui/mousemotionlistener.html">Tutorial: Writing a Mouse Motion Listener</a>
186  * @see <a href="http://www.awl.com/cp/javaseries/jcl1_2.html">Reference: The Java Class Libraries (update file)</a>
187  *
188  * @since 1.1
