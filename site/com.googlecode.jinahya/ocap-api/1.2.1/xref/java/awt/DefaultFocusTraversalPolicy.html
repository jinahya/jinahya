42  /** 
43   * A FocusTraversalPolicy that determines traversal order based on the order
44   * of child Components in a Container. From a particular focus cycle root, the
45   * policy makes a pre-order traversal of the Component hierarchy, and traverses
46   * a Container's children according to the ordering of the array returned by
47   * <code>Container.getComponents()</code>. Portions of the hierarchy that are
48   * not visible and displayable will not be searched.
49   * <p>
50   * If client code has explicitly set the focusability of a Component by either
51   * overriding <code>Component.isFocusTraversable()</code> or
52   * <code>Component.isFocusable()</code>, or by calling
53   * <code>Component.setFocusable()</code>, then a DefaultFocusTraversalPolicy
54   * behaves exactly like a ContainerOrderFocusTraversalPolicy. If, however, the
55   * Component is relying on default focusability, then a
56   * DefaultFocusTraversalPolicy will reject all Components with non-focusable
57   * peers. This is the default FocusTraversalPolicy for all AWT Containers.
58   * <p>
59   * The focusability of a peer is implementation-dependent. Sun recommends that
60   * all implementations for a particular native platform construct peers with
61   * the same focusability. The recommendations for Windows and Unix are that
62   * Canvases, Labels, Panels, Scrollbars, ScrollPanes, Windows, and lightweight
63   * Components have non-focusable peers, and all other Components have focusable
64   * peers. These recommendations are used in the Sun AWT implementations. Note
65   * that the focusability of a Component's peer is different from, and does not
66   * impact, the focusability of the Component itself.
67   *
68   * @author David Mendenhall
69   * @version 1.3, 01/23/03
70   *
71   * @see Container#getComponents
72   * @see Component#isFocusable
73   * @see Component#setFocusable
74   * @since 1.4
75   */
76  public class DefaultFocusTraversalPolicy
77      extends ContainerOrderFocusTraversalPolicy
78  {
79  
80      public DefaultFocusTraversalPolicy() { }
81  
82      /** 
83       * Determines whether a Component is an acceptable choice as the new
84       * focus owner. The Component must be visible, displayable, and enabled
85       * to be accepted. If client code has explicitly set the focusability
86       * of the Component by either overriding
87       * <code>Component.isFocusTraversable()</code> or
88       * <code>Component.isFocusable()</code>, or by calling
89       * <code>Component.setFocusable()</code>, then the Component will be
90       * accepted if and only if it is focusable. If, however, the Component is
91       * relying on default focusability, then all Canvases, Labels, Panels,
92       *  Windows, and lightweight Components will be
93       * rejected.
94       *
95       * @param aComponent the Component whose fitness as a focus owner is to
96       *        be tested
97       * @return <code>true</code> if aComponent meets the above requirements;
98       *         <code>false</code> otherwise
99       */
100     protected boolean accept(Component aComponent) { return false; }
