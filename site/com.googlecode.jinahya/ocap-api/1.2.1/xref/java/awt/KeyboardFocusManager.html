60  /** 
61   * The KeyboardFocusManager is responsible for managing the active and focused
62   * Windows, and the current focus owner. The focus owner is defined as the
63   * Component in an application that will typically receive all KeyEvents
64   * generated by the user. The focused Window is the Window that is, or
65   * contains, the focus owner. Only a Frame can be the active
66   * Window. The native windowing system may denote the active Window or its
67   * children with special decorations, such as a highlighted title bar. The
68   * active Window is always either the focused Window, or the first Frame that is an owner of the focused Window.
69   * <p>
70   * The KeyboardFocusManager is both a centralized location for client code to
71   * query for the focus owner and initiate focus changes, and an event
72   * dispatcher for all FocusEvents, WindowEvents related to focus, and
73   * KeyEvents.
74   * <p>
75   * Some browsers partition applets in different code bases into separate
76   * contexts, and establish walls between these contexts. In such a scenario,
77   * there will be one KeyboardFocusManager per context. Other browsers place all
78   * applets into the same context, implying that there will be only a single,
79   * global KeyboardFocusManager for all applets. This behavior is
80   * implementation-dependent. Consult your browser's documentation for more
81   * information. No matter how many contexts there may be, however, there can
82   * never be more than one focus owner, focused Window, or active Window, per
83   * ClassLoader.
84   *
85   * @author David Mendenhall
86   * @version 1.40, 01/23/03 
87   *
88   * @see Window
89   * @see Frame
90   * 
91   * @see java.awt.event.FocusEvent
92   * @see java.awt.event.WindowEvent
93   * @see java.awt.event.KeyEvent
94   * @since 1.4
95   */

477      * Sets the default focus traversal keys for a given traversal operation.
478      * This traversal key <code>Set</code> will be in effect on all
479      * <code>Window</code>s that have no such <code>Set</code> of
480      * their own explicitly defined. This <code>Set</code> will also be
481      * inherited, recursively, by any child <code>Component</code> of
482      * those <code>Windows</code> that has
483      * no such <code>Set</code> of its own explicitly defined.
484      * <p>
485      * The default values for the default focus traversal keys are
486      * implementation-dependent. Sun recommends that all implementations for a
487      * particular native platform use the same default values. The
488      * recommendations for Windows and Unix are listed below. These
489      * recommendations are used in the Sun AWT implementations.
490      *
491      * <table border=1 summary="Recommended default values for focus traversal keys">
492      * <tr>
493      *    <th>Identifier</th>
494      *    <th>Meaning</th>
495      *    <th>Default</th>
496      * </tr>
497      * <tr>
498      *    <td><code>KeyboardFocusManager.FORWARD_TRAVERSAL_KEYS</code></td>
499      *    <td>Normal forward keyboard traversal</td>
500      *    <td><code>TAB</code> on <code>KEY_PRESSED</code>,
501      *        <code>CTRL-TAB</code> on <code>KEY_PRESSED</code></td>
502      * </tr>
503      * <tr>
504      *    <td><code>KeyboardFocusManager.BACKWARD_TRAVERSAL_KEYS</code></td>
505      *    <td>Normal reverse keyboard traversal</td>
506      *    <td><code>SHIFT-TAB</code> on <code>KEY_PRESSED</code>,
507      *        <code>CTRL-SHIFT-TAB</code> on <code>KEY_PRESSED</code></td>
508      * </tr>
509      * <tr>
510      *    <td><code>KeyboardFocusManager.UP_CYCLE_TRAVERSAL_KEYS</code></td>
511      *    <td>Go up one focus traversal cycle</td>
512      *    <td>none</td>
513      * </tr>
514      * <tr>
515      *    <td><code>KeyboardFocusManager.DOWN_CYCLE_TRAVERSAL_KEYS</code></td>
516      *    <td>Go down one focus traversal cycle</td>
517      *    <td>none</td>
518      * </tr>
519      * </table>
520      *
521      * To disable a traversal key, use an empty <code>Set</code>;
522      * <code>Collections.EMPTY_SET</code> is recommended.

525      * specify on which of two
526      * specific <code>KeyEvent</code>s, <code>KEY_PRESSED</code> or
527      * <code>KEY_RELEASED</code>, the focus traversal operation will
528      * occur. Regardless of which <code>KeyEvent</code> is specified,

532      * to any <code>Component</code>. It is a runtime error to

546      * @see Component#getFocusTraversalKeys

548      *         <code>KeyboardFocusManager.FORWARD_TRAVERSAL_KEYS</code>,
549      *         <code>KeyboardFocusManager.BACKWARD_TRAVERSAL_KEYS</code>,
550      *         <code>KeyboardFocusManager.UP_CYCLE_TRAVERSAL_KEYS</code>, or
551      *         <code>KeyboardFocusManager.DOWN_CYCLE_TRAVERSAL_KEYS</code>,

595     /** 
596      * Returns the current focus cycle root, if the current focus cycle root is
597      * in the same context as the calling thread. If the focus owner is itself
598      * a focus cycle root, then it may be ambiguous as to which Components
599      * represent the next and previous Components to focus during normal focus
