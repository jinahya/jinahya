43  /** 
44   * This class is for AWT permissions.
45   * An <code>AWTPermission</code> contains a target name but
46   * no actions list; you either have the named permission
47   * or you don't.
48   *
49   * <P>
50   * The target name is the name of the AWT permission (see below). The naming
51   * convention follows the hierarchical property naming convention.
52   * Also, an asterisk could be used to represent all AWT permissions.
53   *
54   * <P>
55   * The following table lists all the possible <code>AWTPermission</code>
56   * target names, and for each provides a description of what the
57   * permission allows and a discussion of the risks of granting code
58   * the permission.
59   * <P>
60   *
61   * <table border=1 cellpadding=5 summary="AWTPermission target names, descriptions, and associated risks.">
62   * <tr>
63   * <th>Permission Target Name</th>
64   * <th>What the Permission Allows</th>
65   * <th>Risks of Allowing this Permission</th>
66   * </tr>
67   *
68   * <tr>
69   *   <td>accessClipboard</td>
70   *   <td>Posting and retrieval of information to and from the AWT clipboard</td>
71   *   <td>This would allow malfeasant code to share
72   * potentially sensitive or confidential information.</td>
73   * </tr>
74   *
75   * <tr>
76   *   <td>accessEventQueue</td>
77   *   <td>Access to the AWT event queue</td>
78   *   <td>After retrieving the AWT event queue,
79   * malicious code may peek at and even remove existing events
80   * from its event queue, as well as post bogus events which may purposefully
81   * cause the application or applet to misbehave in an insecure manner.</td>
82   * </tr>
83   *
84   *
85   *
86   * <tr>
87   *   <td>fullScreenExclusive</td>
88   *   <td>Enter full-screen exclusive mode</td>
89   *   <td>Entering full-screen exclusive mode allows direct access to
90   * low-level graphics card memory.  This could be used to spoof the
91   * system, since the program is in direct control of rendering.</td>
92   * </tr>
93   *
94   * <tr>
95   *   <td>listenToAllAWTEvents</td>
96   *   <td>Listen to all AWT events, system-wide</td>
97   *   <td>After adding an AWT event listener,
98   * malicious code may scan all AWT events dispatched in the system,
99   * allowing it to read all user input (such as passwords).  Each
100  * AWT event listener is called from within the context of that
101  * event queue's EventDispatchThread, so if the accessEventQueue
102  * permission is also enabled, malicious code could modify the
103  * contents of AWT event queues system-wide, causing the application
104  * or applet to misbehave in an insecure manner.</td>
105  * </tr>
106  *
107 <!-- PBP/PP -->
108 <!-- [6187202] -->
109  * 
110  * <tr>
111  *   <td>readDisplayPixels</td>
112  *   <td>Readback of pixels from the display screen</td>
113  *   <td>Interfaces  <em>which</em> allow arbitrary code to examine pixels on the 
114  * display enable malicious code to snoop on the activities of the user.</td>
115  * </tr>
116  *
117  * <tr>
118  *   <td>replaceKeyboardFocusManager</td>
119  *   <td>Sets the <code>KeyboardFocusManager</code> for
120  *       a particular thread.
121  *   <td>When <code>SecurityManager</code> is installed, the invoking
122  *       thread must be granted this permission in order to replace
123  *       the current <code>KeyboardFocusManager</code>.  If permission
124  *       is not granted, a <code>SecurityException</code> will be thrown.
125  * </tr>
126  *
127  * <tr>
128  *   <td>showWindowWithoutWarningBanner</td>
129  *   <td>Display of a window without also displaying a banner warning
130  * that the window was created by an applet</td>
131  *   <td>Without this warning,
132  * an applet may pop up windows without the user knowing that they
133  * belong to an applet.  Since users may make security-sensitive
134  * decisions based on whether or not the window belongs to an applet
135  * (entering a username and password into a dialog box, for example),
136  * disabling this warning banner may allow applets to trick the user
137  * into entering such information.</td>
138  * </tr>
139  * </table>
140  *
141  * @see java.security.BasicPermission
142  * @see java.security.Permission
143  * @see java.security.Permissions
144  * @see java.security.PermissionCollection
145  * @see java.lang.SecurityManager
146  *
147  * @version 	1.24, 01/28/03
148  *
149  * @author Marianne Mueller
150  * @author Roland Schemers
151  */
