64  /** 
65   * The <code>Font</code> class represents fonts, which are used to
66   * render text in a visible way.
67   * A font provides the information needed to map sequences of
68   * <em>characters</em> to sequences of <em>glyphs</em>
69   * and to render sequences of glyphs on <code>Graphics</code> and
70   * <code>Component</code> objects.
71   * 
72   * <h4>Characters and Glyphs</h4>
73   * 
74   * A <em>character</em> is a symbol that represents an item such as a letter,
75   * a digit, or punctuation in an abstract way. For example, <code>'g'</code>,
76   * <font size=-1>LATIN SMALL LETTER G</font>, is a character.
77   * <p>
78   * A <em>glyph</em> is a shape used to render a character or a sequence of
79   * characters. In simple writing systems, such as Latin, typically one glyph
80   * represents one character. In general, however, characters and glyphs do not
81   * have one-to-one correspondence. For example, the character '&aacute;'
82   * <font size=-1>LATIN SMALL LETTER A WITH ACUTE</font>, can be represented by
83   * two glyphs: one for 'a' and one for '&acute;'. On the other hand, the
84   * two-character string "fi" can be represented by a single glyph, an
85   * "fi" ligature. In complex writing systems, such as Arabic or the South
86   * and South-East Asian writing systems, the relationship between characters
87   * and glyphs can be more complicated and involve context-dependent selection
88   * of glyphs as well as glyph reordering.
89   *
90   * A font encapsulates the collection of glyphs needed to render a selected set
91   * of characters as well as the tables needed to map sequences of characters to
92   * corresponding sequences of glyphs.
93   *
94   * <h4>Physical and Logical Fonts</h4>
95   *
96   * The Java 2 platform distinguishes between two kinds of fonts:
97   * <em>physical</em> fonts and <em>logical</em> fonts.
98   * <p>
99   * <em>Physical</em> fonts are the actual font libraries containing glyph data
100  * and tables to map from character sequences to glyph sequences, using a font
101  * technology such as TrueType or PostScript Type 1.
102  * 
103  * Physical fonts may use names such as Helvetica, Palatino, HonMincho, or
104  * any number of other font names.
105  * Typically, each physical font supports only a limited set of writing
106  * systems, for example, only Latin characters or only Japanese and Basic
107  * Latin.
108  * The set of available physical fonts varies between configurations.
109  * 
110  * <p>
111  * <em>Logical</em> fonts are the five font families defined by the Java
112  * platform which must be supported by any Java runtime environment:
113  * Serif, SansSerif, Monospaced, Dialog, and DialogInput.
114  * These logical fonts are not actual font libraries. Instead, the logical
115  * font names are mapped to physical fonts by the Java runtime environment.
116  * The mapping is implementation and usually locale dependent, so the look
117  * and the metrics provided by them vary.
118  * Typically, each logical font name maps to several physical fonts in order to
119  * cover a large range of characters.
120  * <p>
121  * 
122  * <p>
123  * For a discussion of the relative advantages and disadvantages of using
124  * physical or logical fonts, see the
125  * <a href="../../../guide/intl/faq.html#fonts">Internationalization FAQ</a>
126  * document.
127  *
128  * <h4>Font Faces and Names</h4>
129  * 
130  * A <code>Font</code> 
131  * can have many faces, such as heavy, medium, oblique, gothic and
132  * regular. All of these faces have similar typographic design.
133  * <p>
134  * There are three different names that you can get from a 
135  * <code>Font</code> object.  The <em>logical font name</em> is simply the
136  * name that was used to construct the font.
137  * The <em>font face name</em>, or just <em>font name</em> for
138  * short, is the name of a particular font face, like Helvetica Bold. The
139  * <em>family name</em> is the name of the font family that determines the
140  * typographic design across several faces, like Helvetica.
141  * <p>
142  * The <code>Font</code> class represents an instance of a font face from
143  * a collection of  font faces that are present in the system resources
144  * of the host system.  As examples, Arial Bold and Courier Bold Italic
145  * are font faces.  There can be several <code>Font</code> objects
146  * associated with a font face, each differing in size, style
147  *  and font features.  
148  *
