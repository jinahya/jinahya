44  /** 
45   * The <code>Shape</code> interface provides definitions for objects 
46   * that represent some form of geometric shape.
47  
48   * <p>
49   * <b>Definition of insideness:</b>
50   * A point is considered to lie inside a 
51   * <code>Shape</code> if and only if:
52   * <ul>
53   * <li> it lies completely
54   * inside the<code>Shape</code> boundary <i>or</i> 
55   * <li>
56   * it lies exactly on the <code>Shape</code> boundary <i>and</i> the 
57   * space immediately adjacent to the
58   * point in the increasing <code>X</code> direction is 
59   * entirely inside the boundary <i>or</i>
60   * <li>
61   * it lies exactly on a horizontal boundary segment <b>and</b> the
62   * space immediately adjacent to the point in the 
63   * increasing <code>Y</code> direction is inside the boundary.
64   * </ul>
65   * <p>The <code>contains</code> and <code>intersects</code> methods
66   * consider the interior of a <code>Shape</code> to be the area it
67   * encloses as if it were filled.  This means that these methods
68   * consider
69   * unclosed shapes to be implicitly closed for the purpose of
70   * determining if a shape contains or intersects a rectangle or if a
71   * shape contains a point.
72   * 
73   * @see 
74   *
75   * @version 1.19 06/24/98
