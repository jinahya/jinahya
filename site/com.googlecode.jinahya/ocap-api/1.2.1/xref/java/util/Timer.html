42  /** 
43   * A facility for threads to schedule tasks for future execution in a
44   * background thread.  Tasks may be scheduled for one-time execution, or for
45   * repeated execution at regular intervals.
46   *
47   * <p>Corresponding to each <tt>Timer</tt> object is a single background
48   * thread that is used to execute all of the timer's tasks, sequentially.
49   * Timer tasks should complete quickly.  If a timer task takes excessive time
50   * to complete, it "hogs" the timer's task execution thread.  This can, in
51   * turn, delay the execution of subsequent tasks, which may "bunch up" and
52   * execute in rapid succession when (and if) the offending task finally
53   * completes.
54   *
55   * <p>After the last live reference to a <tt>Timer</tt> object goes away
56   * <i>and</i> all outstanding tasks have completed execution, the timer's task
57   * execution thread terminates gracefully (and becomes subject to garbage
58   * collection).  However, this can take arbitrarily long to occur.  By
59   * default, the task execution thread does not run as a <i>daemon thread</i>,
60   * so it is capable of keeping an application from terminating.  If a caller
61   * wants to terminate a timer's task execution thread rapidly, the caller
62   * should invoke the the timer's <tt>cancel</tt> method.
63   *
64   * <p>If the timer's task execution thread terminates unexpectedly, for
65   * example, because its <tt>stop</tt> method is invoked, any further
66   * attempt to schedule a task on the timer will result in an
67   * <tt>IllegalStateException</tt>, as if the timer's <tt>cancel</tt>
68   * method had been invoked.
69   *
70   * <p>This class is thread-safe: multiple threads can share a single
71   * <tt>Timer</tt> object without the need for external synchronization.
72   *
73   * <p>This class does <i>not</i> offer real-time guarantees: it schedules
74   * tasks using the <tt>Object.wait(long)</tt> method.
75   *
76   * <p>Implementation note: This class scales to large numbers of concurrently
77   * scheduled tasks (thousands should present no problem).  Internally,
78   * it uses a binary heap to represent its task queue, so the cost to schedule
79   * a task is O(log n), where n is the number of concurrently scheduled tasks.
80   *
81   * @author  Josh Bloch
82   * @version 1.8, 05/17/00
83   * @see     TimerTask
84   * @see     Object#wait(long)
85   * @since   1.3
86   */

154      * tasks, such as blinking a cursor at regular intervals.  It also includes
155      * tasks wherein regular activity is performed in response to human
156      * input, such as automatically repeating a character as long as a key
157      * is held down.
