44  /** 
45   * A <code>Locale</code> object represents a specific geographical, political,
46   * or cultural region. An operation that requires a <code>Locale</code> to perform
47   * its task is called <em>locale-sensitive</em> and uses the <code>Locale</code>
48   * to tailor information for the user. For example, displaying a number
49   * is a locale-sensitive operation--the number should be formatted
50   * according to the customs/conventions of the user's native country,
51   * region, or culture.
52   *
53   * <P>
54   * Create a <code>Locale</code> object using the constructors in this class:
55   * <blockquote>
56   * <pre>
57   * Locale(String language)
58   * Locale(String language, String country)
59   * Locale(String language, String country, String variant)
60   * </pre>
61   * </blockquote>
62   * The language argument is a valid <STRONG>ISO Language Code.</STRONG> 
63   * These codes are the lower-case, two-letter codes as defined by ISO-639.
64   * You can find a full list of these codes at a number of sites, such as:
65   * <BR><a href ="http://www.ics.uci.edu/pub/ietf/http/related/iso639.txt">
66   * <code>http://www.ics.uci.edu/pub/ietf/http/related/iso639.txt</code></a>
67   *
68   * <P>
69   * The country argument is a valid <STRONG>ISO Country Code.</STRONG> These 
70   * codes are the upper-case, two-letter codes as defined by ISO-3166.
71   * You can find a full list of these codes at a number of sites, such as:
72   * <BR><a href="http://www.chemie.fu-berlin.de/diverse/doc/ISO_3166.html">
73   * <code>http://www.chemie.fu-berlin.de/diverse/doc/ISO_3166.html</code></a>
74   *
75   * <P>
76   * The variant argument is a vendor or browser-specific code.
77   * For example, use WIN for Windows, MAC for Macintosh, and POSIX for POSIX.
78   * Where there are two variants, separate them with an underscore, and
79   * put the most important one first. For example, a Traditional Spanish collation 
80   * might construct a locale with parameters for language, country and variant as: 
81   * "es", "ES", "Traditional_WIN".
82   *
83   * <P>
84   * Because a <code>Locale</code> object is just an identifier for a region,
85   * no validity check is performed when you construct a <code>Locale</code>.
86   * If you want to see whether particular resources are available for the
87   * <code>Locale</code> you construct, you must query those resources. For
88   * example, ask the <code>NumberFormat</code> for the locales it supports
89   * using its <code>getAvailableLocales</code> method.
90   * <BR><STRONG>Note:</STRONG> When you ask for a resource for a particular
91   * locale, you get back the best available match, not necessarily
92   * precisely what you asked for. For more information, look at
93   * {@link ResourceBundle}.
94   *
95   * <P>
96   * The <code>Locale</code> class provides a number of convenient constants
97   * that you can use to create <code>Locale</code> objects for commonly used
98   * locales. For example, the following creates a <code>Locale</code> object
99   * for the United States:
100  * <blockquote>
101  * <pre>
102  * Locale.US
103  * </pre>
104  * </blockquote>
105  *
106  * <P>
107  * Once you've created a <code>Locale</code> you can query it for information about
108  * itself. Use <code>getCountry</code> to get the ISO Country Code and
109  * <code>getLanguage</code> to get the ISO Language Code. You can
110  * use <code>getDisplayCountry</code> to get the
111  * name of the country suitable for displaying to the user. Similarly,
112  * you can use <code>getDisplayLanguage</code> to get the name of
113  * the language suitable for displaying to the user. Interestingly,
114  * the <code>getDisplayXXX</code> methods are themselves locale-sensitive
115  * and have two versions: one that uses the default locale and one
116  * that uses the locale specified as an argument.
117  *
118  * <P>
119  * The Java 2 platform provides a number of classes that perform locale-sensitive
120  * operations. For example, the <code>NumberFormat</code> class formats
121  * numbers, currency, or percentages in a locale-sensitive manner. Classes
122  * such as <code>NumberFormat</code> have a number of convenience methods
123  * for creating a default object of that type. For example, the
124  * <code>NumberFormat</code> class provides these three convenience methods
125  * for creating a default <code>NumberFormat</code> object:
126  * <blockquote>
127  * <pre>
128  * NumberFormat.getInstance()
129  * NumberFormat.getCurrencyInstance()
130  * NumberFormat.getPercentInstance()
131  * </pre>
132  * </blockquote>
133  * These methods have two variants; one with an explicit locale
134  * and one without; the latter using the default locale.
135  * <blockquote>
136  * <pre>
137  * NumberFormat.getInstance(myLocale)
138  * NumberFormat.getCurrencyInstance(myLocale)
139  * NumberFormat.getPercentInstance(myLocale)
140  * </pre>
141  * </blockquote>
142  * A <code>Locale</code> is the mechanism for identifying the kind of object
143  * (<code>NumberFormat</code>) that you would like to get. The locale is
144  * <STRONG>just</STRONG> a mechanism for identifying objects,
145  * <STRONG>not</STRONG> a container for the objects themselves.
146  *
147  * <P>
148  * Each class that performs locale-sensitive operations allows you
149  * to get all the available objects of that type. You can sift
150  * through these objects by language, country, or variant,
151  * and use the display names to present a menu to the user.
152  * For example, you can create a menu of all the collation objects
153  * suitable for a given language. Such classes must implement these
154  * three class methods:
155  * <blockquote>
156  * <pre>
157  * public static Locale[] getAvailableLocales()
158  * public static String getDisplayName(Locale objectLocale,
159  *                                     Locale displayLocale)
160  * public static final String getDisplayName(Locale objectLocale)
161  *     // getDisplayName will throw MissingResourceException if the locale
162  *     // is not one of the available locales.
163  * </pre>
164  * </blockquote>
165  *
166  * @see         ResourceBundle
167  * @see         java.text.Format
168  * @see         java.text.NumberFormat
169  * @see         java.text.Collator
170  * @version     1.55, 01/19/00
171  * @author      Mark Davis
172  * @since       1.1
173  */
