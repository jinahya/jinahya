39  /** 
40   * A collection that contains no duplicate elements.  More formally, sets
41   * contain no pair of elements <code>e1</code> and <code>e2</code> such that
42   * <code>e1.equals(e2)</code>, and at most one null element.  As implied by
43   * its name, this interface models the mathematical <i>set</i> abstraction.<p>
44   *
45   * The <tt>Set</tt> interface places additional stipulations, beyond those
46   * inherited from the <tt>Collection</tt> interface, on the contracts of all
47   * constructors and on the contracts of the <tt>add</tt>, <tt>equals</tt> and
48   * <tt>hashCode</tt> methods.  Declarations for other inherited methods are
49   * also included here for convenience.  (The specifications accompanying these
50   * declarations have been tailored to the <tt>Set</tt> interface, but they do
51   * not contain any additional stipulations.)<p>
52   *
53   * The additional stipulation on constructors is, not surprisingly,
54   * that all constructors must create a set that contains no duplicate elements
55   * (as defined above).<p>
56   *
57   * Note: Great care must be exercised if mutable objects are used as set
58   * elements.  The behavior of a set is not specified if the value of an object
59   * is changed in a manner that affects equals comparisons while the object is
60   * an element in the set.  A special case of this prohibition is that it is
61   * not permissible for a set to contain itself as an element.
62   *
63   * <p>Some set implementations have restrictions on the elements that
64   * they may contain.  For example, some implementations prohibit null elements,
65   * and some have restrictions on the types of their elements.  Attempting to
66   * add an ineligible element throws an unchecked exception, typically
67   * <tt>NullPointerException</tt> or <tt>ClassCastException</tt>.  Attempting
68   * to query the presence of an ineligible element may throw an exception,
69   * or it may simply return false; some implementations will exhibit the former
70   * behavior and some will exhibit the latter.  More generally, attempting an
71   * operation on an ineligible element whose completion would not result in
72   * the insertion of an ineligible element into the set may throw an
73   * exception or it may succeed, at the option of the implementation.
74   * Such exceptions are marked as "optional" in the specification for this
75   * interface. 
76   *
77   * <p>This interface is a member of the 
78   * <a href="{@docRoot}/../guide/collections/index.html">
79   * Java Collections Framework</a>.
80   *
81   * @author  Josh Bloch
82   * @version 1.23, 02/02/00
83   * @see Collection
84   * @see List
85   * @see SortedSet
86   * @see HashSet
87   * @see TreeSet
88   * @see AbstractSet
89   * @see Collections#singleton(java.lang.Object)
90   * @see Collections#EMPTY_SET
91   * @since 1.2
92   */
