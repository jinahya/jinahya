44  /** 
45   * Resource bundles contain locale-specific objects.
46   * When your program needs a locale-specific resource,
47   * a <code>String</code> for example, your program can load it
48   * from the resource bundle that is appropriate for the
49   * current user's locale. In this way, you can write
50   * program code that is largely independent of the user's
51   * locale isolating most, if not all, of the locale-specific
52   * information in resource bundles.
53   *
54   * <p>
55   * This allows you to write programs that can:
56   * <UL type=SQUARE>
57   * <LI> be easily localized, or translated, into different languages
58   * <LI> handle multiple locales at once
59   * <LI> be easily modified later to support even more locales
60   * </UL>
61   *
62   * <P>
63   * Resource bundles belong to families whose members share a common base 
64   * name, but whose names also have additional components that identify 
65   * their locales. For example, the base name of a family of resource 
66   * bundles might be "MyResources". The family should have a default 
67   * resource bundle which simply has the same name as its family - 
68   * "MyResources" - and will be used as the bundle of last resort if a
69   * specific locale is not supported. The family can then provide as
70   * many locale-specific members as needed, for example a German one
71   * named "MyResources_de".
72   *
73   * <P>
74   * Each resource bundle in a family contains the same items, but the items have
75   * been translated for the locale represented by that resource bundle.
76   * For example, both "MyResources" and "MyResources_de" may have a
77   * <code>String</code> that's used on a button for canceling operations.
78   * In "MyResources" the <code>String</code> may contain "Cancel" and in
79   * "MyResources_de" it may contain "Abbrechen".
80   *
81   * <P>
82   * If there are different resources for different countries, you
83   * can make specializations: for example, "MyResources_de_CH" contains objects for
84   * the German language (de) in Switzerland (CH). If you want to only
85   * modify some of the resources
86   * in the specialization, you can do so.
87   *
88   * <P>
89   * When your program needs a locale-specific object, it loads
90   * the <code>ResourceBundle</code> class using the
91   * {@link #getBundle(java.lang.String, java.util.Locale) getBundle}
92   * method:
93   * <blockquote>
94   * <pre>
95   * ResourceBundle myResources =
96   *      ResourceBundle.getBundle("MyResources", currentLocale);
97   * </pre>
98   * </blockquote>
99   *
100  * <P>
101  * Resource bundles contain key/value pairs. The keys uniquely
102  * identify a locale-specific object in the bundle. Here's an
103  * example of a <code>ListResourceBundle</code> that contains
104  * two key/value pairs:
105  * <blockquote>
106  * <pre>
107  * public class MyResources extends ListResourceBundle {
108  *      public Object[][] getContents() {
109  *              return contents;
110  *      }
111  *      static final Object[][] contents = {
112  *      // LOCALIZE THIS
113  *              {"OkKey", "OK"},
114  *              {"CancelKey", "Cancel"},
115  *      // END OF MATERIAL TO LOCALIZE
116  *      };
117  * }
118  * </pre>
119  * </blockquote>
120  * Keys are always <code>String</code>s.
121  * In this example, the keys are "OkKey" and "CancelKey".
122  * In the above example, the values
123  * are also <code>String</code>s--"OK" and "Cancel"--but
124  * they don't have to be. The values can be any type of object.
125  *
126  * <P>
127  * You retrieve an object from resource bundle using the appropriate
128  * getter method. Because "OkKey" and "CancelKey"
129  * are both strings, you would use <code>getString</code> to retrieve them:
130  * <blockquote>
131  * <pre>
132  * button1 = new Button(myResources.getString("OkKey"));
133  * button2 = new Button(myResources.getString("CancelKey"));
134  * </pre>
135  * </blockquote>
136  * The getter methods all require the key as an argument and return
137  * the object if found. If the object is not found, the getter method
138  * throws a <code>MissingResourceException</code>.
139  *
140  * <P>
141  * Besides <code>getString</code>, ResourceBundle also provides
142  * a method for getting string arrays, <code>getStringArray</code>,
143  * as well as a generic <code>getObject</code> method for any other
144  * type of object. When using <code>getObject</code>, you'll
145  * have to cast the result to the appropriate type. For example:
146  * <blockquote>
147  * <pre>
148  * int[] myIntegers = (int[]) myResources.getObject("intList");
149  * </pre>
150  * </blockquote>
151  *
152  * <P>
153  * The Java 2 platform provides two subclasses of <code>ResourceBundle</code>,
154  * <code>ListResourceBundle</code> and <code>PropertyResourceBundle</code>,
155  * that provide a fairly simple way to create resources.
156  * As you saw briefly in a previous example, <code>ListResourceBundle</code>
157  * manages its resource as a List of key/value pairs.
158  * <code>PropertyResourceBundle</code> uses a properties file to manage
159  * its resources.
160  *
161  * <p>
162  * If <code>ListResourceBundle</code> or <code>PropertyResourceBundle</code>
163  * do not suit your needs, you can write your own <code>ResourceBundle</code>
164  * subclass.  Your subclasses must override two methods: <code>handleGetObject</code>
165  * and <code>getKeys()</code>.
166  *
167  * <P>
168  * The following is a very simple example of a <code>ResourceBundle</code>
169  * subclass, MyResources, that manages two resources (for a larger number of
170  * resources you would probably use a <code>Hashtable</code>).
171  * Notice that you don't need to supply a value if 
172  * a "parent-level" <code>ResourceBundle</code> handles the same
173  * key with the same value (as for the okKey below).
174  * <p><strong>Example:</strong>
175  * <blockquote>
176  * <pre>
177  * // default (English language, United States)
178  * public class MyResources extends ResourceBundle {
179  *     public Object handleGetObject(String key) {
180  *         if (key.equals("okKey")) return "Ok";
181  *         if (key.equals("cancelKey")) return "Cancel";
182  *         return null;
183  *     }
184  * }
185  *
186  * // German language
187  * public class MyResources_de extends MyResources {
188  *     public Object handleGetObject(String key) {
189  *         // don't need okKey, since parent level handles it.
190  *         if (key.equals("cancelKey")) return "Abbrechen";
191  *         return null;
192  *     }
193  * }
194  * </pre>
195  * </blockquote>
196  * You do not have to restrict yourself to using a single family of
197  * <code>ResourceBundle</code>s. For example, you could have a set of bundles for
198  * exception messages, <code>ExceptionResources</code>
199  * (<code>ExceptionResources_fr</code>, <code>ExceptionResources_de</code>, ...),
200  * and one for widgets, <code>WidgetResource</code> (<code>WidgetResources_fr</code>,
201  * <code>WidgetResources_de</code>, ...); breaking up the resources however you like.
202  *
203  * @see ListResourceBundle
204  * @see PropertyResourceBundle
205  * @see MissingResourceException
206  * @since JDK1.1
207  */

340     /** 
341      * Gets a resource bundle using the specified base name, locale, and class loader.
342      *
343      * <p>
344      * Conceptually, <code>getBundle</code> uses the following strategy for locating and instantiating
345      * resource bundles:
346      * <p>
347      * <code>getBundle</code> uses the base name, the specified locale, and the default
348      * locale (obtained from {@link java.util.Locale#getDefault() Locale.getDefault})
349      * to generate a sequence of <em>candidate bundle names</em>.
350      * If the specified locale's language, country, and variant are all empty
351      * strings, then the base name is the only candidate bundle name.
352      * Otherwise, the following sequence is generated from the attribute
353      * values of the specified locale (language1, country1, and variant1)
354      * and of the default locale (language2, country2, and variant2):
355      * <ul>
356      * <li> baseName + "_" + language1 + "_" + country1 + "_" + variant1
357      * <li> baseName + "_" + language1 + "_" + country1
358      * <li> baseName + "_" + language1
359      * <li> baseName + "_" + language2 + "_" + country2 + "_" + variant2
360      * <li> baseName + "_" + language2 + "_" + country2
361      * <li> baseName + "_" + language2
362      * <li> baseName
363      * </ul>
364      * <p>
365      * Candidate bundle names where the final component is an empty string are omitted.
366      * For example, if country1 is an empty string, the second candidate bundle name is omitted.
367      *
368      * <p>
369      * <code>getBundle</code> then iterates over the candidate bundle names to find the first
370      * one for which it can <em>instantiate</em> an actual resource bundle. For each candidate
371      * bundle name, it attempts to create a resource bundle:
372      * <ul>
373      * <li>
374      * First, it attempts to load a class using the candidate bundle name.
375      * If such a class can be found and loaded using the specified class loader, is assignment
376      * compatible with ResourceBundle, is accessible from ResourceBundle, and can be instantiated,
377      * <code>getBundle</code> creates a new instance of this class and uses it as the <em>result
378      * resource bundle</em>.
379      * <li>
380      * Otherwise, <code>getBundle</code> attempts to locate a property resource file.
381      * It generates a path name from the candidate bundle name by replacing all "." characters
382      * with "/" and appending the string ".properties".
383      * It attempts to find a "resource" with this name using
384      * {@link java.lang.ClassLoader#getResource(java.lang.String) ClassLoader.getResource}.
385      * (Note that a "resource" in the sense of <code>getResource</code> has nothing to do with
386      * the contents of a resource bundle, it is just a container of data, such as a file.)
387      * If it finds a "resource", it attempts to create a new
388      * {@link PropertyResourceBundle} instance from its contents.
389      * If successful, this instance becomes the <em>result resource bundle</em>.
390      * </ul>
391      *
392      * <p>
393      * If no result resource bundle has been found, a <code>MissingResourceException</code>
394      * is thrown.
395      *
396      * <p>
397      * Once a result resource bundle has been found, its parent chain is instantiated.
398      * <code>getBundle</code> iterates over the candidate bundle names that can be
399      * obtained by successively removing variant, country, and language
400      * (each time with the preceding "_") from the bundle name of the result resource bundle.
401      * As above, candidate bundle names where the final component is an empty string are omitted.
402      * With each of the candidate bundle names it attempts to instantiate a resource bundle, as
403      * described above.
404      * Whenever it succeeds, it calls the previously instantiated resource
405      * bundle's {@link #setParent(java.util.ResourceBundle) setParent} method
406      * with the new resource bundle, unless the previously instantiated resource
407      * bundle already has a non-null parent.
408      *
409      * <p>
410      * Implementations of <code>getBundle</code> may cache instantiated resource bundles
411      * and return the same resource bundle instance multiple times. They may also
412      * vary the sequence in which resource bundles are instantiated as long as the
413      * selection of the result resource bundle and its parent chain are compatible with
414      * the description above.
415      *
416      * <p>
417      * The <code>baseName</code> argument should be a fully qualified class name. However, for
418      * compatibility with earlier versions, Sun's Java 2 runtime environments do not verify this,
419      * and so it is possible to access <code>PropertyResourceBundle</code>s by specifying a
420      * path name (using "/") instead of a fully qualified class name (using ".").
421      *
422      * <p>
423      * <strong>Example:</strong> The following class and property files are provided:
424      * MyResources.class, MyResources_fr_CH.properties, MyResources_fr_CH.class,
425      * MyResources_fr.properties, MyResources_en.properties, MyResources_es_ES.class.
426      * The contents of all files are valid (that is, public non-abstract subclasses of ResourceBundle for
427      * the ".class" files, syntactically correct ".properties" files).
428      * The default locale is <code>Locale("en", "GB")</code>.
429      * <p>
430      * Calling <code>getBundle</code> with the shown locale argument values instantiates
431      * resource bundles from the following sources:
432      * <ul>
433      * <li>Locale("fr", "CH"): result MyResources_fr_CH.class, parent MyResources_fr.properties, parent MyResources.class
434      * <li>Locale("fr", "FR"): result MyResources_fr.properties, parent MyResources.class
435      * <li>Locale("de", "DE"): result MyResources_en.properties, parent MyResources.class
436      * <li>Locale("en", "US"): result MyResources_en.properties, parent MyResources.class
437      * <li>Locale("es", "ES"): result MyResources_es_ES.class, parent MyResources.class
438      * </ul>
439      * The file MyResources_fr_CH.properties is never used because it is hidden by
440      * MyResources_fr_CH.class.
441      *
442      * <p>
443      *
444      * @param baseName the base name of the resource bundle, a fully qualified class name
445      * @param locale the locale for which a resource bundle is desired
446      * @param loader the class loader from which to load the resource bundle
447      * @exception java.lang.NullPointerException
448      *     if <code>baseName</code>, <code>locale</code>, or <code>loader</code> is <code>null</code>
449      * @exception MissingResourceException
450      *     if no resource bundle for the specified base name can be found
451      * @return a resource bundle for the given base name and locale
452      * @since 1.2
453      */
