39  /** 
40   * An ordered collection (also known as a <i>sequence</i>).  The user of this
41   * interface has precise control over where in the list each element is
42   * inserted.  The user can access elements by their integer index (position in
43   * the list), and search for elements in the list.<p>
44   *
45   * Unlike sets, lists typically allow duplicate elements.  More formally,
46   * lists typically allow pairs of elements <tt>e1</tt> and <tt>e2</tt>
47   * such that <tt>e1.equals(e2)</tt>, and they typically allow multiple
48   * null elements if they allow null elements at all.  It is not inconceivable
49   * that someone might wish to implement a list that prohibits duplicates, by
50   * throwing runtime exceptions when the user attempts to insert them, but we
51   * expect this usage to be rare.<p>
52   *
53   * The <tt>List</tt> interface places additional stipulations, beyond those
54   * specified in the <tt>Collection</tt> interface, on the contracts of the
55   * <tt>iterator</tt>, <tt>add</tt>, <tt>remove</tt>, <tt>equals</tt>, and
56   * <tt>hashCode</tt> methods.  Declarations for other inherited methods are
57   * also included here for convenience.<p>
58   *
59   * The <tt>List</tt> interface provides four methods for positional (indexed)
60   * access to list elements.  Lists (like Java arrays) are zero based.  Note
61   * that these operations may execute in time proportional to the index value
62   * for some implementations (the <tt>LinkedList</tt> class, for
63   * example). Thus, iterating over the elements in a list is typically
64   * preferable to indexing through it if the caller does not know the
65   * implementation.<p>
66   *
67   * The <tt>List</tt> interface provides a special iterator, called a
68   * <tt>ListIterator</tt>, that allows element insertion and replacement, and
69   * bidirectional access in addition to the normal operations that the
70   * <tt>Iterator</tt> interface provides.  A method is provided to obtain a
71   * list iterator that starts at a specified position in the list.<p>
72   *
73   * The <tt>List</tt> interface provides two methods to search for a specified
74   * object.  From a performance standpoint, these methods should be used with
75   * caution.  In many implementations they will perform costly linear
76   * searches.<p>
77   *
78   * The <tt>List</tt> interface provides two methods to efficiently insert and
79   * remove multiple elements at an arbitrary point in the list.<p>
80   *
81   * Note: While it is permissible for lists to contain themselves as elements,
82   * extreme caution is advised: the <tt>equals</tt> and <tt>hashCode</tt>
83   * methods are no longer well defined on a such a list.
84   *
85   * <p>Some list implementations have restrictions on the elements that
86   * they may contain.  For example, some implementations prohibit null elements,
87   * and some have restrictions on the types of their elements.  Attempting to
88   * add an ineligible element throws an unchecked exception, typically
89   * <tt>NullPointerException</tt> or <tt>ClassCastException</tt>.  Attempting
90   * to query the presence of an ineligible element may throw an exception,
91   * or it may simply return false; some implementations will exhibit the former
92   * behavior and some will exhibit the latter.  More generally, attempting an
93   * operation on an ineligible element whose completion would not result in
94   * the insertion of an ineligible element into the list may throw an
95   * exception or it may succeed, at the option of the implementation.
96   * Such exceptions are marked as "optional" in the specification for this
97   * interface. 
98   *
99   * <p>This interface is a member of the 
100  * <a href="{@docRoot}/../guide/collections/index.html">
101  * Java Collections Framework</a>.
102  *
103  * @author  Josh Bloch
104  * @version 1.32, 02/02/00
105  * @see Collection
106  * @see Set
107  * @see ArrayList
108  * @see LinkedList
109  * @see Vector
110  * @see Arrays#asList(Object[])
111  * @see Collections#nCopies(int, Object)
112  * @see Collections#EMPTY_LIST
113  * @see AbstractList
114  * @see AbstractSequentialList
115  * @since 1.2
116  */
