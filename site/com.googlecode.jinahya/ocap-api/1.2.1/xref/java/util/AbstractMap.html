42   * This class provides a skeletal implementation of the <tt>Map</tt>
43   * interface, to minimize the effort required to implement this interface. <p>
44   *
45   * To implement an unmodifiable map, the programmer needs only to extend this
46   * class and provide an implementation for the <tt>entrySet</tt> method, which
47   * returns a set-view of the map's mappings.  Typically, the returned set
48   * will, in turn, be implemented atop <tt>AbstractSet</tt>.  This set should
49   * not support the <tt>add</tt> or <tt>remove</tt> methods, and its iterator
50   * should not support the <tt>remove</tt> method.<p>
51   *
52   * To implement a modifiable map, the programmer must additionally override
53   * this class's <tt>put</tt> method (which otherwise throws an
54   * <tt>UnsupportedOperationException</tt>), and the iterator returned by
55   * <tt>entrySet().iterator()</tt> must additionally implement its
56   * <tt>remove</tt> method.<p>
57   *
58   * The programmer should generally provide a void (no argument) and map
59   * constructor, as per the recommendation in the <tt>Map</tt> interface
60   * specification.<p>

63   * implementation in detail.  Each of these methods may be overridden if the
