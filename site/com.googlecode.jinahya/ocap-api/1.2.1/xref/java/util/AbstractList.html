40   * This class provides a skeletal implementation of the <tt>List</tt>
41   * interface to minimize the effort required to implement this interface
42   * backed by a "random access" data store (such as an array).  For sequential
43   * access data (such as a linked list), <tt>AbstractSequentialList</tt> should
44   * be used in preference to this class.<p>
45   *
46   * To implement an unmodifiable list, the programmer needs only to extend this
47   * class and provide implementations for the <tt>get(int index)</tt> and
48   * <tt>size()</tt> methods.<p>
49   *
50   * To implement a modifiable list, the programmer must additionally override
51   * the <tt>set(int index, Object element)</tt> method (which otherwise throws
52   * an <tt>UnsupportedOperationException</tt>.  If the list is variable-size
53   * the programmer must additionally override the <tt>add(int index, Object
54   * element)</tt> and <tt>remove(int index)</tt> methods.<p>
55   *
56   * The programmer should generally provide a void (no argument) and collection
57   * constructor, as per the recommendation in the <tt>Collection</tt> interface
58   * specification.<p>
59   *
60   * Unlike the other abstract collection implementations, the programmer does
61   * <i>not</i> have to provide an iterator implementation; the iterator and
62   * list iterator are implemented by this class, on top the "random access"
63   * methods: <tt>get(int index)</tt>, <tt>set(int index, Object element)</tt>,
64   * <tt>set(int index, Object element)</tt>, <tt>add(int index, Object
65   * element)</tt> and <tt>remove(int index)</tt>.<p>
66   *
67   * The documentation for each non-abstract methods in this class describes its
68   * implementation in detail.  Each of these methods may be overridden if the
69   * collection being implemented admits a more efficient implementation.<p>
70   *
71   * This class is a member of the 
72   * <a href="{@docRoot}/../guide/collections/index.html">
73   * Java Collections Framework</a>.

90       *
91       * This field is used by the iterator and list iterator implementation
92       * returned by the <tt>iterator</tt> and <tt>listIterator</tt> methods.

101      * wishes to provide fail-fast iterators (and list iterators), then it
102      * merely has to increment this field in its <tt>add(int, Object)</tt> and

407      * Returns a view of the portion of this list between <tt>fromIndex</tt>,
408      * inclusive, and <tt>toIndex</tt>, exclusive.  (If <tt>fromIndex</tt> and

418      * range of elements from a list:
419      * <pre>
420      *     list.subList(from, to).clear();

422      * Similar idioms may be constructed for <tt>indexOf</tt> and
