39  /** 
40   * A <code>PushbackInputStream</code> adds
41   * functionality to another input stream, namely
42   * the  ability to "push back" or "unread"
43   * one byte. This is useful in situations where
44   * it is  convenient for a fragment of code
45   * to read an indefinite number of data bytes
46   * that  are delimited by a particular byte
47   * value; after reading the terminating byte,
48   * the  code fragment can "unread" it, so that
49   * the next read operation on the input stream
50   * will reread the byte that was pushed back.
51   * For example, bytes representing the  characters
52   * constituting an identifier might be terminated
53   * by a byte representing an  operator character;
54   * a method whose job is to read just an identifier
55   * can read until it  sees the operator and
56   * then push the operator back to be re-read.
57   *
58   * @author  David Connelly
59   * @author  Jonathan Payne
60   * @version 1.31, 02/02/00
61   * @since   JDK1.0
62   */
