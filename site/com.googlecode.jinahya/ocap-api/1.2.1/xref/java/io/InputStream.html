121      * an array of bytes.  An attempt is made to read as many as
122      * <code>len</code> bytes, but a smaller number may be read, possibly

124      *
125      * <p> This method blocks until input data is available, end of file is

134      * thrown.
135      *
136      * <p> If <code>len</code> is zero, then no bytes are read and
137      * <code>0</code> is returned; otherwise, there is an attempt to read at
138      * least one byte. If no byte is available because the stream is at end of
139      * file, the value <code>-1</code> is returned; otherwise, at least one
140      * byte is read and stored into <code>b</code>.
141      *
142      * <p> The first byte read is stored into element <code>b[off]</code>, the
143      * next one into <code>b[off+1]</code>, and so on. The number of bytes read
144      * is, at most, equal to <code>len</code>. Let <i>k</i> be the number of
145      * bytes actually read; these bytes will be stored in elements
146      * <code>b[off]</code> through <code>b[off+</code><i>k</i><code>-1]</code>,

148      * <code>b[off+len-1]</code> unaffected.
149      *
150      * <p> In every case, elements <code>b[0]</code> through
151      * <code>b[off]</code> and elements <code>b[off+len]</code> through

153      *
154      * <p> If the first byte cannot be read for any reason other than end of
155      * file, then an <code>IOException</code> is thrown. In particular, an
