39  /** 
40   * Instances of this class support both reading and writing to a 
41   * random access file. A random access file behaves like a large 
42   * array of bytes stored in the file system. There is a kind of cursor, 
43   * or index into the implied array, called the <em>file pointer</em>; 
44   * input operations read bytes starting at the file pointer and advance 
45   * the file pointer past the bytes read. If the random access file is 
46   * created in read/write mode, then output operations are also available; 
47   * output operations write bytes starting at the file pointer and advance 
48   * the file pointer past the bytes written. Output operations that write 
49   * past the current end of the implied array cause the array to be 
50   * extended. The file pointer can be read by the 
51   * <code>getFilePointer</code> method and set by the <code>seek</code> 
52   * method. 
53   * <p>
54   * It is generally true of all the reading routines in this class that 
55   * if end-of-file is reached before the desired number of bytes has been 
56   * read, an <code>EOFException</code> (which is a kind of 
57   * <code>IOException</code>) is thrown. If any byte cannot be read for 
58   * any reason other than end-of-file, an <code>IOException</code> other 
59   * than <code>EOFException</code> is thrown. In particular, an 
60   * <code>IOException</code> may be thrown if the stream has been closed.
61   *
62   * @author  unascribed
63   * @version 1.57, 05/03/00
64   * @since   JDK1.0
65   */
66  public class RandomAccessFile implements DataOutput, DataInput
67  {

70       * Creates a random access file stream to read from, and optionally 
71       * to write to, a file with the specified name. A new 
72       * {@link FileDescriptor} object is created to represent the 
73       * connection to the file.
74       * 
75       * <p> The <tt>mode</tt> argument specifies the access mode with which the
76       * file is to be opened.  The permitted values and their meanings are as
77       * specified for the <a
78       * href="#mode"><tt>RandomAccessFile(File,String)</tt></a> constructor.
79       *
80       * <p>
81       * If there is a security manager, its <code>checkRead</code> method
82       * is called with the <code>name</code> argument
83       * as its argument to see if read access to the file is allowed.
84       * If the mode allows writing, the security manager's
85       * <code>checkWrite</code> method
86       * is also called with the <code>name</code> argument
87       * as its argument to see if write access to the file is allowed.
88       *
89       * @param      name   the system-dependent filename
90       * @param      mode   the access <a href="#mode">mode</a>
91       * @exception  IllegalArgumentException  if the mode argument is not equal
92       *               to one of <tt>"r"</tt>, <tt>"rw"</tt>, <tt>"rws"</tt>, or
93       *               <tt>"rwd"</tt>
94       * @exception  FileNotFoundException  if the file exists but is a directory
95       *                   rather than a regular file, or cannot be opened or
96       *                   created for any other reason
97       * @exception  SecurityException         if a security manager exists and its
98       *               <code>checkRead</code> method denies read access to the file
99       *               or the mode is "rw" and the security manager's
100      *               <code>checkWrite</code> method denies write access to the file
101      * @see        java.lang.SecurityException
102      * @see        java.lang.SecurityManager#checkRead(java.lang.String)
103      * @see        java.lang.SecurityManager#checkWrite(java.lang.String)
104      * @revised 1.4
105      * @spec JSR-51
106      */
107     public RandomAccessFile(String name, String mode)
108         throws FileNotFoundException
109     { }

111     /** 
112      * Creates a random access file stream to read from, and optionally to
113      * write to, the file specified by the {@link File} argument.  A new {@link
114      * FileDescriptor} object is created to represent this file connection.
115      *
116      * <a name="mode"><p> The <tt>mode</tt> argument specifies the access mode
117      * in which the file is to be opened.  The permitted values and their
118      * meanings are:
119      *
120      * <blockquote><table summary="Access mode permitted values and meanings">
121      * <tr><th><p align="left">Value</p></th><th><p align="left">Meaning</p></th></tr>
122      * <tr><td valign="top"><tt>"r"</tt></td>
123      *     <td> Open for reading only.  Invoking any of the <tt>write</tt>
124      *     methods of the resulting object will cause an {@link
125      *     java.io.IOException} to be thrown. </td></tr>
126      * <tr><td valign="top"><tt>"rw"</tt></td>
127      *     <td> Open for reading and writing.  If the file does not already
128      *     exist then an attempt will be made to create it. </td></tr>
129      * <tr><td valign="top"><tt>"rws"</tt></td>
130      *     <td> Open for reading and writing, as with <tt>"rw"</tt>, and also
131      *     require that every update to the file's content or metadata be
132      *     written synchronously to the underlying storage device.  </td></tr>
133      * <tr><td valign="top"><tt>"rwd"&nbsp;&nbsp;</tt></td>
134      *     <td> Open for reading and writing, as with <tt>"rw"</tt>, and also
135      *     require that every update to the file's content be written
136      *     synchronously to the underlying storage device. </td></tr>
137      * </table></blockquote>
138      *
139      * If
140      * the file resides on a local storage device then when an invocation of a
141      * method of this class returns it is guaranteed that all changes made to
142      * the file by that invocation will have been written to that device.  This
143      * is useful for ensuring that critical information is not lost in the
144      * event of a system crash.  If the file does not reside on a local device
145      * then no such guarantee is made.
146      *
147      * <p> The <tt>"rwd"</tt> mode can be used to reduce the number of I/O
148      * operations performed.  Using <tt>"rwd"</tt> only requires updates to the
149      * file's content to be written to storage; using <tt>"rws"</tt> requires
150      * updates to both the file's content and its metadata to be written, which
151      * generally requires at least one more low-level I/O operation.
152      *
153      * <p> If there is a security manager, its <code>checkRead</code> method is
154      * called with the pathname of the <code>file</code> argument as its
155      * argument to see if read access to the file is allowed.  If the mode
156      * allows writing, the security manager's <code>checkWrite</code> method is
157      * also called with the path argument to see if write access to the file is
158      * allowed.
159      *
160      * @param      file   the file object
161      * @param      mode   the access mode, as described
162      *                    <a href="#mode">above</a>
163      * @exception  IllegalArgumentException  if the mode argument is not equal
164      *               to one of <tt>"r"</tt>, <tt>"rw"</tt>, <tt>"rws"</tt>, or
165      *               <tt>"rwd"</tt>
166      * @exception  FileNotFoundException  if the file exists but is a directory
167      *                   rather than a regular file, or cannot be opened or
168      *                   created for any other reason
169      * @exception  SecurityException         if a security manager exists and its
170      *               <code>checkRead</code> method denies read access to the file
171      *               or the mode is "rw" and the security manager's
172      *               <code>checkWrite</code> method denies write access to the file
173      * @see        java.lang.SecurityManager#checkRead(java.lang.String)
174      * @see        java.lang.SecurityManager#checkWrite(java.lang.String)
175      * @revised 1.4
176      * @spec JSR-51
177      */
178     public RandomAccessFile(File file, String mode) throws FileNotFoundException
179     { }
