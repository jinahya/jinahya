39  /** 
40   * Phantom reference objects, which are enqueued after the collector
41   * determines that their referents may otherwise be reclaimed.  Phantom
42   * references are most often used for scheduling pre-mortem cleanup actions in
43   * a more flexible way than is possible with the Java finalization mechanism.
44   *
45   * <p> If the garbage collector determines at a certain point in time that the
46   * referent of a phantom reference is <a
47   * href="package-summary.html#reachability">phantom reachable</a>, then at that
48   * time or at some later time it will enqueue the reference.
49   *
50   * <p> In order to ensure that a reclaimable object remains so, the referent of
51   * a phantom reference may not be retrieved: The <code>get</code> method of a
52   * phantom reference always returns <code>null</code>.
53   *
54   * <p> Unlike soft and weak references, phantom references are not
55   * automatically cleared by the garbage collector as they are enqueued.  An
56   * object that is reachable via phantom references will remain so until all
57   * such references are cleared or themselves become unreachable.
58   *
59   * @version  1.12, 02/02/00
60   * @author   Mark Reinhold
61   * @since    1.2
62   */
63  public class PhantomReference extends Reference
64  {
65  
66      /** 
67       * Creates a new phantom reference that refers to the given object and
68       * is registered with the given queue.
69       *
70       * @param referent the object the new phantom reference will refer to
71       * @param q queue the phantom reference is registered with
72       * @throws  NullPointerException  If the <code>queue</code> argument
73       *                                is <code>null</code>
74       */
75      public PhantomReference(Object referent, ReferenceQueue q) { 
76  	super(referent, q);    
77      }
78  
79      /** 
80       * Returns this reference object's referent.  Because the referent of a
81       * phantom reference is always inaccessible, this method always returns
82       * <code>null</code>.
83       *
84       * @return  <code>null</code>
85       */
86      public Object get() {
87          return null;
88      }
89  }
