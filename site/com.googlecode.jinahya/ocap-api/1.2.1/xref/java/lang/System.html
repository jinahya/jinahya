241     /** 
242      * Copies an array from the specified source array, beginning at the
243      * specified position, to the specified position of the destination array.
244      * A subsequence of array components are copied from the source
245      * array referenced by <code>src</code> to the destination array
246      * referenced by <code>dest</code>. The number of components copied is
247      * equal to the <code>length</code> argument. The components at
248      * positions <code>srcPos</code> through
249      * <code>srcPos+length-1</code> in the source array are copied into
250      * positions <code>destPos</code> through
251      * <code>destPos+length-1</code>, respectively, of the destination
252      * array.
253      * <p>
254      * If the <code>src</code> and <code>dest</code> arguments refer to the
255      * same array object, then the copying is performed as if the
256      * components at positions <code>srcPos</code> through
257      * <code>srcPos+length-1</code> were first copied to a temporary
258      * array with <code>length</code> components and then the contents of
259      * the temporary array were copied into positions
260      * <code>destPos</code> through <code>destPos+length-1</code> of the
261      * destination array.
262      * <p>
263      * If <code>dest</code> is <code>null</code>, then a
264      * <code>NullPointerException</code> is thrown.
265      * <p>
266      * If <code>src</code> is <code>null</code>, then a
267      * <code>NullPointerException</code> is thrown and the destination
268      * array is not modified.
269      * <p>
270      * Otherwise, if any of the following is true, an
271      * <code>ArrayStoreException</code> is thrown and the destination is
272      * not modified:
273      * <ul>
274      * <li>The <code>src</code> argument refers to an object that is not an
275      *     array.
276      * <li>The <code>dest</code> argument refers to an object that is not an
277      *     array.
278      * <li>The <code>src</code> argument and <code>dest</code> argument refer
279      *     to arrays whose component types are different primitive types.
280      * <li>The <code>src</code> argument refers to an array with a primitive
281      *    component type and the <code>dest</code> argument refers to an array
282      *     with a reference component type.
283      * <li>The <code>src</code> argument refers to an array with a reference
284      *    component type and the <code>dest</code> argument refers to an array
285      *     with a primitive component type.
286      * </ul>
287      * <p>
288      * Otherwise, if any of the following is true, an
289      * <code>IndexOutOfBoundsException</code> is
290      * thrown and the destination is not modified:
291      * <ul>
292      * <li>The <code>srcPos</code> argument is negative.
293      * <li>The <code>destPos</code> argument is negative.
294      * <li>The <code>length</code> argument is negative.
295      * <li><code>srcPos+length</code> is greater than
296      *     <code>src.length</code>, the length of the source array.
297      * <li><code>destPos+length</code> is greater than
298      *     <code>dest.length</code>, the length of the destination array.
299      * </ul>
300      * <p>
301      * Otherwise, if any actual component of the source array from
302      * position <code>srcPos</code> through
303      * <code>srcPos+length-1</code> cannot be converted to the component
304      * type of the destination array by assignment conversion, an
305      * <code>ArrayStoreException</code> is thrown. In this case, let
306      * <b><i>k</i></b> be the smallest nonnegative integer less than
307      * length such that <code>src[srcPos+</code><i>k</i><code>]</code>
308      * cannot be converted to the component type of the destination
309      * array; when the exception is thrown, source array components from
310      * positions <code>srcPos</code> through
311      * <code>srcPos+</code><i>k</i><code>-1</code>
312      * will already have been copied to destination array positions
313      * <code>destPos</code> through
314      * <code>destPos+</code><i>k</I><code>-1</code> and no other
315      * positions of the destination array will have been modified.
316      * (Because of the restrictions already itemized, this
317      * paragraph effectively applies only to the situation where both
318      * arrays have component types that are reference types.)
319      *
320      * @param      src      the source array.
321      * @param      srcPos   starting position in the source array.
322      * @param      dest     the destination array.
323      * @param      destPos  starting position in the destination data.
324      * @param      length   the number of array elements to be copied.
325      * @exception  IndexOutOfBoundsException  if copying would cause
326      *               access of data outside array bounds.
327      * @exception  ArrayStoreException  if an element in the <code>src</code>
328      *               array could not be stored into the <code>dest</code> array
329      *               because of a type mismatch.
330      * @exception  NullPointerException if either <code>src</code> or
331      *               <code>dest</code> is <code>null</code>.
332      */
333     public static void arraycopy(java.lang.Object src, int srcPos,
334         java.lang.Object dest, int destPos, int length)
335     { }

567     /** 
568      * Terminates the currently running Java Virtual Machine. The
569      * argument serves as a status code; by convention, a nonzero status
570      * code indicates abnormal termination.
571      * <p>
572      * This method calls the <code>exit</code> method in class
573      * <code>Runtime</code>. This method never returns normally.
574      * <p>
575      * The call <code>System.exit(n)</code> is effectively equivalent to
576      * the call:
577      * <blockquote><pre>
578      * Runtime.getRuntime().exit(n)
579      * </pre></blockquote>
580      * <p>
581      * If process model exists, System.exit() must rely on the process 
582      * exiting to release resources.
583      * <p>
584      * If process model does not exist, and a security manager exists,
585      * the security manager by default must prohibit calls to System.exit()
