241      * When the keepalive option is set for a TCP socket and no data
242      * has been exchanged across the socket in either direction for 
243      * 2 hours (NOTE: the actual value is implementation dependent),
244      * TCP automatically sends a keepalive probe to the peer. This probe is a 
245      * TCP segment to which the peer must respond. 
246      * One of three responses is expected:
247      * 1. The peer responds with the expected ACK. The application is not 
248      *    notified (since everything is OK). TCP will send another probe 
249      *    following another 2 hours of inactivity.
250      * 2. The peer responds with an RST, which tells the local TCP that
251      *    the peer host has crashed and rebooted. The socket is closed.
252      * 3. There is no response from the peer. The socket is closed. 
253      *
254      * The purpose of this option is to detect if the peer host crashes. 
255      *
256      * Valid only for TCP socket: SocketImpl
257      * NOTE: <B>java.net.SocketImpl</B> is found in J2ME CDC profiles such as 
258      * J2ME Foundation Profile.
259      *
260      * @see Socket#setKeepAlive
261      * @see Socket#getKeepAlive

277      * Enable/disable the option specified by <I>optID</I>.  If the option
278      * is to be enabled, and it takes an option-specific "value",  this is
279      * passed in <I>value</I>.  The actual type of value is option-specific,
280      * and it is an error to pass something that isn't of the expected type:
281      * <BR><PRE>
282      * SocketImpl s;
283      * ...
284      * s.setOption(SO_LINGER, new Integer(10));
285      *    // OK - set SO_LINGER w/ timeout of 10 sec.
286      * s.setOption(SO_LINGER, new Double(10));
287      *    // ERROR - expects java.lang.Integer
288      *</PRE>
289      * If the requested option is binary, it can be set using this method by
290      * a java.lang.Boolean:
291      * <BR><PRE>
292      * s.setOption(TCP_NODELAY, new Boolean(true));
293      *    // OK - enables TCP_NODELAY, a binary option
294      * </PRE>
295      * <BR>
296      * Any option can be disabled using this method with a Boolean(false):
297      * <BR><PRE>
298      * s.setOption(TCP_NODELAY, new Boolean(false));
299      *    // OK - disables TCP_NODELAY
300      * s.setOption(SO_LINGER, new Boolean(false));
301      *    // OK - disables SO_LINGER
302      * </PRE>
303      * <BR>
304      * NOTE: <B>java.net.SocketImpl</B> is found in J2ME CDC profiles such as 
305      * J2ME Foundation Profile.
306      *
307      * For an option that has a notion of on and off, and requires 
308      * a non-boolean parameter, setting its value to anything other than
309      * <I>Boolean(false)</I> implicitly enables it.
310      * <BR>
311      * Throws SocketException if the option is unrecognized,
312      * the socket is closed, or some low-level error occurred
313      * <BR>
314      * @param optID identifies the option
315      * @param value the parameter of the socket option
316      * @throws SocketException if the option is unrecognized,
317      * the socket is closed, or some low-level error occurred
318      * @see #getOption(int)
319      */
320     public void setOption(int optID, Object value) throws SocketException;

326      * <BR><PRE>
327      * SocketImpl s;

342      *     System.out.print("Linger time is " + ((Integer)o).intValue());

344      *   // the true type of o is java.lang.Boolean(false);
345      * }
346      * </PRE>

348      * NOTE: <B>java.net.SocketImpl</B> is found in J2ME CDC profiles such as

350      *

352      * @return the value of the option
