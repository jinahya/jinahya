50  /** 
51   * The abstract class <code>URLConnection</code> is the superclass
52   * of all classes that represent a communications link between the
53   * application and a URL. Instances of this class can be used both to
54   * read from and to write to the resource referenced by the URL. In
55   * general, creating a connection to a URL is a multistep process:
56   * <p>
57   * <center><table border=2 summary="Describes the process of creating a connection to a URL: openConnection() and connect() over time.">
58   * <tr><th><code>openConnection()</code></th>
59   *     <th><code>connect()</code></th></tr>
60   * <tr><td>Manipulate parameters that affect the connection to the remote
61   *         resource.</td>
62   *     <td>Interact with the resource; query header fields and
63   *         contents.</td></tr>
64   * </table>
65   * ----------------------------&gt;
66   * <br>time</center>
67   *
68   * <ol>
69   * <li>The connection object is created by invoking the
70   *     <code>openConnection</code> method on a URL.
71   * <li>The setup parameters and general request properties are manipulated.
72   * <li>The actual connection to the remote object is made, using the
73   *    <code>connect</code> method.
74   * <li>The remote object becomes available. The header fields and the contents
75   *     of the remote object can be accessed.
76   * </ol>
77   * <p>
78   * The setup parameters are modified using the following methods:
79   * <ul>
80   *   <li><code>setAllowUserInteraction</code>
81   *   <li><code>setDoInput</code>
82   *   <li><code>setDoOutput</code>
83   *   <li><code>setIfModifiedSince</code>
84   *   <li><code>setUseCaches</code>
85   * </ul>
86   * <p>
87   * and the general request properties are modified using the method:
88   * <ul>
89   *   <li><code>setRequestProperty</code>
90   * </ul>
91   * <p>
92   * Default values for the <code>AllowUserInteraction</code> and
93   * <code>UseCaches</code> parameters can be set using the methods
94   * <code>setDefaultAllowUserInteraction</code> and
95   * <code>setDefaultUseCaches</code>.
96   * <p>
97   * Each of the above <code>set</code> methods has a corresponding
98   * <code>get</code> method to retrieve the value of the parameter or
99   * general request property. The specific parameters and general
100  * request properties that are applicable are protocol specific. 
101  * <p>
102  * The following methods are used to access the header fields and 
103  * the contents after the connection is made to the remote object:
104  * <ul>
105  *   <li><code>getContent</code>
106  *   <li><code>getHeaderField</code>
107  *   <li><code>getInputStream</code>
108  *   <li><code>getOutputStream</code>
109  * </ul>
110  * <p>
111  * Certain header fields are accessed frequently. The methods:
112  * <ul>
113  *   <li><code>getContentEncoding</code>
114  *   <li><code>getContentLength</code>
115  *   <li><code>getContentType</code>
116  *   <li><code>getDate</code>
117  *   <li><code>getExpiration</code>
118  *   <li><code>getLastModifed</code>
119  * </ul>
120  * <p>
121  * provide convenient access to these fields. The 
122  * <code>getContentType</code> method is used by the 
123  * <code>getContent</code> method to determine the type of the remote 
124  * object; subclasses may find it convenient to override the 
125  * <code>getContentType</code> method. 
126  * <p>
127  * In the common case, all of the pre-connection parameters and 
128  * general request properties can be ignored: the pre-connection 
129  * parameters and request properties default to sensible values. For
130  * most clients of this interface, there are only two interesting
131  * methods: <code>getInputStream</code> and <code>getContent</code>,
132  * which are mirrored in the <code>URL</code> class by convenience methods.
133  * <p>
134  * More information on the request properties and header fields of
135  * an <code>http</code> connection can be found at:
136  * <blockquote><pre>
137  * <a href="http://www.ietf.org/rfc/rfc2068.txt">http://www.ietf.org/rfc/rfc2068.txt</a>
138  * </pre></blockquote>
139  *
140  * Note about <code>fileNameMap</code>: In versions prior to JDK 1.1.6, 
141  * field <code>fileNameMap</code> of <code>URLConnection</code> was public.
142  * In JDK 1.1.6 and later, <code>fileNameMap</code> is private; accessor 
143  * and mutator methods {@link #getFileNameMap() getFileNameMap} and 
144  * {@link #setFileNameMap(java.net.FileNameMap) setFileNameMap} are added
145  * to access it.  This change is also described on the <a href=
146  * "http://java.sun.com/products/jdk/1.2/compatibility.html#incompatibilities1.2">
147  * Compatibility</a> page.
148  *
149  * Calling the <tt>close()</tt> methods on the <tt>InputStream</tt> or <tt>OutputStream</tt> of an 
150  * <tt>URLConnection</tt> after a request may free network resources associated with this 
151  * instance, unless particular protocol specifications specify different behaviours 
152  * for it.
153  *
154  * @author  James Gosling
155  * @version 1.75, 05/03/00
156  * @see     java.net.URL#openConnection()
157  * @see     java.net.URLConnection#connect()
158  * @see     java.net.URLConnection#getContent()
159  * @see     java.net.URLConnection#getContentEncoding()
160  * @see     java.net.URLConnection#getContentLength()
161  * @see     java.net.URLConnection#getContentType()
162  * @see     java.net.URLConnection#getDate()
163  * @see     java.net.URLConnection#getExpiration()
164  * @see     java.net.URLConnection#getHeaderField(int)
165  * @see     java.net.URLConnection#getHeaderField(java.lang.String)
166  * @see     java.net.URLConnection#getInputStream()
167  * @see     java.net.URLConnection#getLastModified()
168  * @see     java.net.URLConnection#getOutputStream()
169  * @see     java.net.URLConnection#setAllowUserInteraction(boolean)
170  * @see     java.net.URLConnection#setDefaultUseCaches(boolean)
171  * @see     java.net.URLConnection#setDoInput(boolean)
