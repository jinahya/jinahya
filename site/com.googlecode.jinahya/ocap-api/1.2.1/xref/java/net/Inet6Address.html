45  /** 
46   * This class represents an Internet Protocol version 6 (IPv6) address.
47   * Defined by <a href="http://www.ietf.org/rfc/rfc2373.txt">
48   * <i>RFC&nbsp;2373: IP Version 6 Addressing Architecture</i></a>.
49   *
50   * <h4> <A NAME="format">Textual representation of IP addresses<a> </h4>
51   *
52   * Textual representation of IPv6 address used as input to methods
53   * takes one of the following forms:
54   * 
55   * <ol>
56   *   <li><p> <A NAME="lform">The preferred form<a> is x:x:x:x:x:x:x:x, where the 'x's are
57   *   the hexadecimal values of the eight 16-bit pieces of the
58   *   address. This is the full form.  For example,
59   *
60   *   <blockquote><table cellpadding=0 cellspacing=0 summary="layout">
61   *   <tr><td><tt>1080:0:0:0:8:800:200C:417A</tt><td></tr>
62   *   </table></blockquote>
63   *
64   *   <p> Note that it is not necessary to write the leading zeros in
65   *   an individual field. However, there must be at least one numeral
66   *   in every field, except as described below.</li>
67   *
68   *   <li><p> Due to some methods of allocating certain styles of IPv6
69   *   addresses, it will be common for addresses to contain long
70   *   strings of zero bits. In order to make writing addresses
71   *   containing zero bits easier, a special syntax is available to
72   *   compress the zeros. The use of "::" indicates multiple groups
73   *   of 16-bits of zeros. The "::" can only appear once in an address.
74   *   The "::" can also be used to compress the leading and/or trailing
75   *   zeros in an address. For example,
76   *
77   *   <blockquote><table cellpadding=0 cellspacing=0 summary="layout">
78   *   <tr><td><tt>1080::8:800:200C:417A</tt><td></tr>
79   *   </table></blockquote>
80   *
81   *   <li><p> An alternative form that is sometimes more convenient
82   *   when dealing with a mixed environment of IPv4 and IPv6 nodes is
83   *   x:x:x:x:x:x:d.d.d.d, where the 'x's are the hexadecimal values
84   *   of the six high-order 16-bit pieces of the address, and the 'd's
85   *   are the decimal values of the four low-order 8-bit pieces of the
86   *   standard IPv4 representation address, for example,
87   *
88   *   <blockquote><table cellpadding=0 cellspacing=0 summary="layout">
89   *   <tr><td><tt>::FFFF:129.144.52.38</tt><td></tr>
90   *   <tr><td><tt>::129.144.52.38</tt><td></tr>
91   *   </table></blockquote>
92   *
93   *   <p> where "::FFFF:d.d.d.d" and "::d.d.d.d" are, respectively, the
94   *   general forms of an IPv4-mapped IPv6 address and an
95   *   IPv4-compatible IPv6 address. Note that the IPv4 portion must be
96   *   in the "d.d.d.d" form. The following forms are invalid:
97   *
98   *   <blockquote><table cellpadding=0 cellspacing=0 summary="layout">
99   *   <tr><td><tt>::FFFF:d.d.d</tt><td></tr>
100  *   <tr><td><tt>::FFFF:d.d</tt><td></tr>
101  *   <tr><td><tt>::d.d.d</tt><td></tr>
102  *   <tr><td><tt>::d.d</tt><td></tr>
103  *   </table></blockquote>
104  *
105  *   <p> The following form:
106  *
107  *   <blockquote><table cellpadding=0 cellspacing=0 summary="layout">
108  *   <tr><td><tt>::FFFF:d</tt><td></tr>
109  *   </table></blockquote>
110  *
111  *   <p> is valid, however it is an unconventional representation of
112  *   the IPv4-compatible IPv6 address,
113  *
114  *   <blockquote><table cellpadding=0 cellspacing=0 summary="layout">
115  *   <tr><td><tt>::255.255.0.d</tt><td></tr>
116  *   </table></blockquote>
117  *
118  *   <p> while "::d" corresponds to the general IPv6 address
119  *   "0:0:0:0:0:0:0:d".</li>
120  * </ol>
121  *
122  * <p> For methods that return a textual representation as output
123  * value, the full form is used. Inet6Address will return the full
124  * form because it is unambiguous when used in combination with other
125  * textual data.
126  *
127  * <h4> Special IPv6 address </h4>
128  *
129  * <blockquote>
130  * <table cellspacing=2 summary="Description of IPv4-mapped address"> <tr><th valign=top><i>IPv4-mapped address</i></th>
131  *         <td>Of the form::ffff:w.x.y.z, this IPv6 address is used to
132  *         represent an IPv4 address. It allows the native program to
133  *         use the same address data structure and also the same
134  *         socket when communicating with both IPv4 and IPv6 nodes.
135  *
136  *         <p>In InetAddress and Inet6Address, it is used for internal
137  *         representation; it has no functional role. Java will never
138  *         return an IPv4-mapped address.  These classes can take an
139  *         IPv4-mapped address as input, both in byte array and text
140  *         representation. However, it will be converted into an IPv4
141  *         address.</td></tr>
142  * </table></blockquote>
143  */
