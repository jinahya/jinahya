45  /** 
46   * Class <code>URL</code> represents a Uniform Resource
47   * Locator, a pointer to a "resource" on the World
48   * Wide Web. A resource can be something as simple as a file or a
49   * directory, or it can be a reference to a more complicated object,
50   * such as a query to a database or to a search engine. More
51   * information on the types of URLs and their formats can be found at:
52   * <blockquote>
53   *     <a href="http://archive.ncsa.uiuc.edu/SDG/Software/Mosaic/Demo/url-primer.html">
54   *    <i>http://archive.ncsa.uiuc.edu/SDG/Software/Mosaic/Demo/url-primer.html</i></a>
55   * </blockquote>
56   * <p>
57   * In general, a URL can be broken into several parts. The previous
58   * example of a URL indicates that the protocol to use is
59   * <code>http</code> (HyperText Transfer Protocol) and that the
60   * information resides on a host machine named
61   * <code>www.ncsa.uiuc.edu</code>. The information on that host
62   * machine is named <code>/SDG/Software/Mosaic/Demo/url-primer.html</code>. The exact
63   * meaning of this name on the host machine is both protocol
64   * dependent and host dependent. The information normally resides in
65   * a file, but it could be generated on the fly. This component of
66   * the URL is called the <i>path</i> component.
67   * <p>
68   * A URL can optionally specify a "port", which is the
69   * port number to which the TCP connection is made on the remote host
70   * machine. If the port is not specified, the default port for
71   * the protocol is used instead. For example, the default port for
72   * <code>http</code> is <code>80</code>. An alternative port could be
73   * specified as:
74   * <blockquote><pre>
75   *     http://archive.ncsa.uiuc.edu:80/SDG/Software/Mosaic/Demo/url-primer.html
76   * </pre></blockquote>
77   * <p>
78   * The syntax of <code>URL</code> is defined by  <a
79   * href="http://www.ietf.org/rfc/rfc2396.txt""><i>RFC&nbsp;2396: Uniform
80   * Resource Identifiers (URI): Generic Syntax</i></a>, amended by <a
81   * href="http://www.ietf.org/rfc/rfc2732.txt"><i>RFC&nbsp;2732: Format for
82   * Literal IPv6 Addresses in URLs</i></a>.
83   * <p>
84   * A URL may have appended to it a "fragment", also known
85   * as a "ref" or a "reference". The fragment is indicated by the sharp
86   * sign character "#" followed by more characters. For example,
87   * <blockquote><pre>
88   *     http://java.sun.com/index.html#chapter1
89   * </pre></blockquote>
90   * <p>
91   * This fragment is not technically part of the URL. Rather, it
92   * indicates that after the specified resource is retrieved, the
93   * application is specifically interested in that part of the
94   * document that has the tag <code>chapter1</code> attached to it. The
95   * meaning of a tag is resource specific.
96   * <p>
97   * An application can also specify a "relative URL",
98   * which contains only enough information to reach the resource
99   * relative to another URL. Relative URLs are frequently used within
100  * HTML pages. For example, if the contents of the URL:
101  * <blockquote><pre>
102  *     http://java.sun.com/index.html
103  * </pre></blockquote>
104  * contained within it the relative URL:
105  * <blockquote><pre>
106  *     FAQ.html
107  * </pre></blockquote>
108  * it would be a shorthand for:
109  * <blockquote><pre>
110  *     http://java.sun.com/FAQ.html
111  * </pre></blockquote>
112  * <p>
113  * The relative URL need not specify all the components of a URL. If
114  * the protocol, host name, or port number is missing, the value is
115  * inherited from the fully specified URL. The file component must be
116  * specified. The optional fragment is not inherited.
117  *
118  * @author  James Gosling
119  * @version 1.106, 03/12/05
120  * @since JDK1.0 
121  */

165     /** 
166      * Creates a <code>URL</code> object from the specified
167      * <code>protocol</code>, <code>host</code>, <code>port</code>
168      * number, and <code>file</code>.<p>
169      *
170      * <code>host</code> can be expressed as a host name or a literal
171      * IP address. If IPv6 literal address is used, it should be
172      * enclosed in square brackets (<tt>'['</tt> and <tt>']'</tt>), as
173      * specified by <a
174      * href="http://www.ietf.org/rfc/rfc2732.txt">RFC&nbsp;2732</a>;
175      * However, the literal IPv6 address format defined in <a
176      * href="http://www.ietf.org/rfc/rfc2373.txt"><i>RFC&nbsp;2373: IP
177      * Version 6 Addressing Architecture</i></a> is also accepted.<p>
178      *
179      * Specifying a <code>port</code> number of <code>-1</code>
180      * indicates that the URL should use the default port for the
181      * protocol.<p>
182      *
183      * If this is the first URL object being created with the specified
184      * protocol, a <i>stream protocol handler</i> object, an instance of
185      * class <code>URLStreamHandler</code>, is created for that protocol:
186      * <ol>
187      * <li>If the application has previously set up an instance of
188      *     <code>URLStreamHandlerFactory</code> as the stream handler factory,
189      *     then the <code>createURLStreamHandler</code> method of that instance
190      *     is called with the protocol string as an argument to create the
191      *     stream protocol handler.
192      * <li>If no <code>URLStreamHandlerFactory</code> has yet been set up,
193      *     or if the factory's <code>createURLStreamHandler</code> method
194      *     returns <code>null</code>, then the constructor finds the
195      *     value of the system property:
196      *     <blockquote><pre>
197      *         java.protocol.handler.pkgs
198      *     </pre></blockquote>
199      *     If the value of that system property is not <code>null</code>,
200      *     it is interpreted as a list of packages separated by a vertical
201      *     slash character '<code>|</code>'. The constructor tries to load
202      *     the class named:
203      *     <blockquote><pre>
204      *         &lt;<i>package</i>&gt;.&lt;<i>protocol</i>&gt;.Handler
205      *     </pre></blockquote>
206      *     where &lt;<i>package</i>&gt; is replaced by the name of the package
207      *     and &lt;<i>protocol</i>&gt; is replaced by the name of the protocol.
208      *     If this class does not exist, or if the class exists but it is not
209      *     a subclass of <code>URLStreamHandler</code>, then the next package
210      *     in the list is tried.
211      * <li>If the previous step fails to find a protocol handler, then the
212      *     constructor tries to load from a system default package.
213      *     <blockquote><pre>
214      *         &lt;<i>system default package</i>&gt;.&lt;<i>protocol</i>&gt;.Handler
215      *     </pre></blockquote>
216      *     If this class does not exist, or if the class exists but it is not a
217      *     subclass of <code>URLStreamHandler</code>, then a
218      *     <code>MalformedURLException</code> is thrown.
219      * </ol>
220      *
221      * <p>No validation of the inputs is performed by this constructor.
222      *
223      * @param      protocol   the name of the protocol to use.
224      * @param      host       the name of the host.
225      * @param      port       the port number on the host.
226      * @param      file       the file on the host
227      * @exception  MalformedURLException  if an unknown protocol is specified.
228      * @see        java.lang.System#getProperty(java.lang.String)
229      * @see        java.net.URL#setURLStreamHandlerFactory(
230      *			java.net.URLStreamHandlerFactory)
231      * @see        java.net.URLStreamHandler
232      * @see        java.net.URLStreamHandlerFactory#createURLStreamHandler(
233      *                  java.lang.String)
234      */
235     public URL(String protocol, String host, int port, String file)
236         throws MalformedURLException
237     { }

318     /** 
319      * Creates a URL by parsing the given spec within a specified context.
320      *
321      * The new URL is created from the given context URL and the spec
322      * argument as described in
323      * RFC2396 &quot;Uniform Resource Identifiers : Generic * Syntax&quot; :
324      * <blockquote><pre>
325      *          &lt;scheme&gt;://&lt;authority&gt;&lt;path&gt;?&lt;query&gt;#&lt;fragment&gt;
326      * </pre></blockquote>
327      * The reference is parsed into the scheme, authority, path, query and
328      * fragment parts. If the path component is empty and the scheme,
329      * authority, and query components are undefined, then the new URL is a
330      * reference to the current document. Otherwise, the fragment and query
331      * parts present in the spec are used in the new URL.
332      * <p>
333      * If the scheme component is defined in the given spec and does not match
334      * the scheme of the context, then the new URL is created as an absolute
335      * URL based on the spec alone. Otherwise the scheme component is inherited
336      * from the context URL.
337      * <p>
338      * If the authority component is present in the spec then the spec is
339      * treated as absolute and the spec authority and path will replace the
340      * context authority and path. If the authority component is absent in the
341      * spec then the authority of the new URL will be inherited from the
342      * context.
343      * <p>
344      * If the spec's path component begins with a slash character
345      * &quot;/&quot; then the
346      * path is treated as absolute and the spec path replaces the context path.
347      * <p>
348      * Otherwise, the path is treated as a relative path and is appended to the
349      * context path, as described in RFC2396. Also, in this case, 
350      * the path is canonicalized through the removal of directory 
351      * changes made by occurences of &quot;..&quot; and &quot;.&quot;.
352      * <p>
353      * For a more detailed description of URL parsing, refer to RFC2396.
354      *
355      * @param      context   the context in which to parse the specification.
356      * @param      spec      the <code>String</code> to parse as a URL.
357      * @exception  MalformedURLException  if no protocol is specified, or an
358      *               unknown protocol is found.
359      * @see        java.net.URL#URL(java.lang.String, java.lang.String,
360      *			int, java.lang.String)
361      * @see        java.net.URLStreamHandler
362      * @see        java.net.URLStreamHandler#parseURL(java.net.URL,
363      *			java.lang.String, int, int)
364      */
365     public URL(URL context, String spec) throws MalformedURLException { }
