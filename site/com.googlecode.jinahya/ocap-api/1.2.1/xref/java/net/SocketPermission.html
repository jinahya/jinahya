54  /** 
55   * This class represents access to a network via sockets.
56   * A SocketPermission consists of a 
57   * host specification and a set of "actions" specifying ways to
58   * connect to that host. The host is specified as
59   * <pre>
60   *    host = (hostname | IPv4address | iPv6reference) [:portrange]
61   *    portrange = portnumber | -portnumber | portnumber-[portnumber]
62   * </pre>
63   * The host is expressed as a DNS name, as a numerical IP address,
64   * or as "localhost" (for the local machine).
65   * The wildcard "*" may be included once in a DNS name host
66   * specification. If it is included, it must be in the leftmost 
67   * position, as in "*.sun.com".
68   * <p>
69   * The format of the IPv6reference should follow that specified in <a
70   * href="http://www.ietf.org/rfc/rfc2732.txt"><i>RFC&nbsp;2732: Format
71   * for Literal IPv6 Addresses in URLs</i></a>:
72   * <pre>
73   *    ipv6reference = "[" IPv6address "]"
74   *</pre>
75   * For example, you can construct a SocketPermission instance
76   * as the following:
77   * <pre>
78   *    String hostAddress = inetaddress.getHostAddress();
79   *    if (inetaddress instanceof Inet6Address) {
80   *        sp = new SocketPermission("[" + hostAddress + "]:" + port, action);
81   *    } else {
82   *        sp = new SocketPermission(hostAddress + ":" + port, action);
83   *    }
84   * </pre>
85   * or
86   * <pre>
87   *    String host = url.getHost();
88   *    sp = new SocketPermission(host + ":" + port, action);
89   * </pre>
90   * <p>
91   * The <A HREF="Inet6Address.html#lform">full uncompressed form</A> of
92   * an IPv6 literal address is also valid.
93   * <p>
94   * The port or portrange is optional. A port specification of the 
95   * form "N-", where <i>N</i> is a port number, signifies all ports
96   * numbered <i>N</i> and above, while a specification of the
97   * form "-N" indicates all ports numbered <i>N</i> and below.
98   * <p>
99   * The possible ways to connect to the host are 
100  * <pre>
101  * accept
102  * connect
103  * listen
104  * resolve
105  * </pre>
106  * The "listen" action is only meaningful when used with "localhost". 
107  * The "resolve" action is implied when any of the other actions are present.
108  * The action "resolve" refers to host/ip name service lookups.
109  * 
110  * <p>As an example of the creation and meaning of SocketPermissions,  
111  * note that if the following permission:
112  * 
113  * <pre>
114  *   p1 = new SocketPermission("puffin.eng.sun.com:7777", "connect,accept");
115  * </pre>
116  * 
117  * is granted to some code, it allows that code to connect to port 7777 on
118  * <code>puffin.eng.sun.com</code>, and to accept connections on that port.
119  * 
120  * <p>Similarly, if the following permission:
121  * 
122  * <pre>
123  *   p1 = new SocketPermission("puffin.eng.sun.com:7777", "connect,accept");
124  *   p2 = new SocketPermission("localhost:1024-", "accept,connect,listen");
125  * </pre>
126  * 
127  * is granted to some code, it allows that code to 
128  * accept connections on, connect to, or listen on any port between
129  * 1024 and 65535 on the local host.
130  *
131  * <p>Note: Granting code permission to accept or make connections to remote
132  * hosts may be dangerous because malevolent code can then more easily
133  * transfer and share confidential data among parties who may not
134  * otherwise have access to the data.
135  * 
136  * @see java.security.Permissions
137  * @see SocketPermission
138  *
139  * @version 1.41 10/27/00
140  *
141  * @author Marianne Mueller
142  * @author Roland Schemers
