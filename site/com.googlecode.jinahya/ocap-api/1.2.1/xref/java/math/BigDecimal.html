39  /** 
40   * Immutable, arbitrary-precision signed decimal numbers.  A BigDecimal
41   * consists of an arbitrary precision integer <i>unscaled value</i> and a
42   * non-negative 32-bit integer <i>scale</i>, which represents the number of
43   * digits to the right of the decimal point.  The number represented by the
44   * BigDecimal is <tt>(unscaledValue/10<sup>scale</sup>)</tt>.  BigDecimal
45   * provides operations for basic arithmetic, scale manipulation, comparison,
46   * hashing, and format conversion.
47   * <p>
48   * The BigDecimal class gives its user complete control over rounding
49   * behavior, forcing the user to explicitly specify a rounding
50   * behavior for operations capable of discarding precision ({@link
51   * #divide(BigDecimal, int)}, {@link #divide(BigDecimal, int, int)},
52   * and {@link #setScale}).  Eight <em>rounding modes</em> are provided
53   * for this purpose.
54   * <p>
55   * Two types of operations are provided for manipulating the scale of a
56   * BigDecimal: scaling/rounding operations and decimal point motion
57   * operations.  Scaling/rounding operations (<tt>setScale</tt>) return a
58   * BigDecimal whose value is approximately (or exactly) equal to that of the
59   * operand, but whose scale is the specified value; that is, they increase or
60   * decrease the precision of the number with minimal effect on its value.
61   * Decimal point motion operations ({@link #movePointLeft} and
62   * {@link #movePointRight}) return a BigDecimal created from the operand by
63   * moving the decimal point a specified distance in the specified direction;
64   * that is, they change a number's value without affecting its precision.
65   * <p>
66   * For the sake of brevity and clarity, pseudo-code is used throughout the
67   * descriptions of BigDecimal methods.  The pseudo-code expression
68   * <tt>(i + j)</tt> is shorthand for "a BigDecimal whose value is
69   * that of the BigDecimal <tt>i</tt> plus that of the BigDecimal <tt>j</tt>."
70   * The pseudo-code expression <tt>(i == j)</tt> is shorthand for
71   * "<tt>true</tt> if and only if the BigDecimal <tt>i</tt> represents the same
72   * value as the the BigDecimal <tt>j</tt>."  Other pseudo-code expressions are
73   * interpreted similarly. 
74   * <p>
75   * Note: care should be exercised if BigDecimals are to be used as
76   * keys in a {@link java.util.SortedMap} or elements in a {@link
77   * java.util.SortedSet}, as BigDecimal's <i>natural ordering</i> is
78   * <i>inconsistent with equals</i>.  See {@link Comparable}, {@link
79   * java.util.SortedMap} or {@link java.util.SortedSet} for more
80   * information.
81   * <p>
82   * All methods and constructors for this class
83   * throw <CODE>NullPointerException</CODE> when passed
84   * a null object reference for any input parameter.
85   *
86   * @see     BigInteger
87   * @see	    java.util.SortedMap
88   * @see	    java.util.SortedSet
89   * @version 1.33, 08/19/02
90   * @author Josh Bloch
91   */

112      * decreases the calculated value.

120      * increases the calculated value.

126      * neighbors are equidistant, in which case round up.
127      * Behaves as for <tt>ROUND_UP</tt> if the discarded fraction is &gt;= .5;
128      * otherwise, behaves as for <tt>ROUND_DOWN</tt>.  Note that this is the
129      * rounding mode that most of us were taught in grade school.
130      */
131     public static final int ROUND_HALF_UP = 4;

135      * neighbors are equidistant, in which case round down.

137      * otherwise, behaves as for <tt>ROUND_DOWN</tt>.
138      */

141     /**

143      * neighbors are equidistant, in which case, round towards the even

177     /** 
178      * Translates the String representation of a BigDecimal into a
179      * BigDecimal.  The String representation consists of an optional
180      * sign, <tt>'+'</tt> (<tt>'&#92;u002B'</tt>) or <tt>'-'</tt>
181      * (<tt>'&#92;u002D'</tt>), followed by a sequence of zero or more
182      * decimal digits ("the integer"), optionally followed by a
183      * fraction, optionally followed by an exponent.
184      *
185      * <p>The fraction consists of of a decimal point followed by zero or more
186      * decimal digits.  The string must contain at least one digit in either
187      * the integer or the fraction.  The number formed by the sign, the
188      * integer and the fraction is referred to as the <i>significand</i>.
189      *
190      * <p>The exponent consists of the character <tt>'e'</tt>
191      * (<tt>'&#92;u0075'</tt>) or <tt>'E'</tt> (<tt>'&#92;u0045'</tt>)
192      * followed by one or more decimal digits.  The value of the
193      * exponent must lie between -{@link Integer#MAX_VALUE} ({@link
194      * Integer#MIN_VALUE}+1) and {@link Integer#MAX_VALUE}, inclusive.
195      *
196      * <p>More formally, the strings this constructor accepts are
197      * described by the following grammar:
198      * <blockquote>
199      * <dl>
200      * <dt><i>BigDecimalString:</i>
201      * <dd><i>Sign<sub>opt</sub> Significand Exponent<sub>opt</sub></i>
202      * <p>
203      * <dt><i>Sign:</i>
204      * <dd><code>+</code>
205      * <dd><code>-</code>
206      * <p>
207      * <dt><i>Significand:</i>
208      * <dd><i>IntegerPart</i> <code>.</code> <i>FractionPart<sub>opt</sub></i>
209      * <dd><code>.</code> <i>FractionPart</i>
210      * <dd><i>IntegerPart</i>
211      * <p>
212      * <dt><i>IntegerPart:
213      * <dd>Digits</i>
214      * <p>
215      * <dt><i>FractionPart:
216      * <dd>Digits</i>
217      * <p>
218      * <dt><i>Exponent:
219      * <dd>ExponentIndicator SignedInteger</i>
220      * <p>
221      * <dt><i>ExponentIndicator:</i>
222      * <dd><code>e</code>
223      * <dd><code>E</code>
224      * <p>
225      * <dt><i>SignedInteger:
226      * <dd>Sign<sub>opt</sub> Digits</i>
227      * <p>
228      * <dt><i>Digits:
229      * <dd>Digit
230      * <dd>Digits Digit</i>
231      * <p>
232      * <dt><i>Digit:</i>
233      * <dd>any character for which {@link Character#isDigit}
234      * returns <code>true</code>, including 0, 1, 2 ...
235      * </dl>
236      * </blockquote>
237      *
238      * <p>The scale of the returned BigDecimal will be the number of digits in
239      * the fraction, or zero if the string contains no decimal point, subject
240      * to adjustment for any exponent:  If the string contains an exponent, the
241      * exponent is subtracted from the scale.  If the resulting scale is
242      * negative, the scale of the returned BigDecimal is zero and the unscaled
243      * value is multiplied by the appropriate power of ten so that, in every
244      * case, the resulting BigDecimal is equal to <i>significand</i> &times;
245      * 10<i><sup>exponent</sup></i>. (If in the future this specification is 
246      * amended to permit negative scales, the final step of zeroing the scale
247      * and adjusting the unscaled value will be eliminated.)
248      *
249      * <p>The character-to-digit mapping is provided by {@link
250      * java.lang.Character#digit} set to convert to radix 10.  The
251      * String may not contain any extraneous characters (whitespace,
252      * for example).
253      *
254      * <p>Note: For values other <tt>float</tt> and <tt>double</tt>
255      * NaN and &plusmn;Infinity, this constructor is compatible with
256      * the values returned by {@link Float#toString} and {@link
257      * Double#toString}.  This is generally the preferred way to
258      * convert a <tt>float</tt> or <tt>double</tt> into a BigDecimal,
259      * as it doesn't suffer from the unpredictability of the {@link
260      * #BigDecimal(double)} constructor.
261      *
262      * <p>Note: the optional leading plus sign and trailing exponent were
263      * added in release 1.3.
264      *
265      * @param val String representation of BigDecimal.
266      * @throws NumberFormatException <tt>val</tt> is not a valid representation
267      *	       of a BigDecimal.
268      */
269     public BigDecimal(String val) { }

272      * Translates a <code>double</code> into a BigDecimal.  The scale
273      * of the BigDecimal is the smallest value such that
274      * <tt>(10<sup>scale</sup> * val)</tt> is an integer.
275      * <p>
276      * Note: the results of this constructor can be somewhat unpredictable.
277      * One might assume that <tt>new BigDecimal(.1)</tt> is exactly equal
278      * to .1, but it is actually equal
279      * to .1000000000000000055511151231257827021181583404541015625.
280      * This is so because .1 cannot be represented exactly as a double
281      * (or, for that matter, as a binary fraction of any finite length).
282      * Thus, the long value that is being passed <i>in</i> to the constructor

284      * <p>

286      * <tt>new BigDecimal(".1")</tt> is <i>exactly</i> equal to .1, as one
287      * would expect.  Therefore, it is generally recommended that the (String)
288      * constructor be used in preference to this one.
