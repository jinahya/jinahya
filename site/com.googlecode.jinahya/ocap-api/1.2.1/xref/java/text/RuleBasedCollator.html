39  import java.util.Vector;
40  import java.util.Locale;

42  /** 
43   * The <code>RuleBasedCollator</code> class is a concrete subclass of
44   * <code>Collator</code> that provides a simple, data-driven, table
45   * collator.  With this class you can create a customized table-based
46   * <code>Collator</code>.  <code>RuleBasedCollator</code> maps
47   * characters to sort keys.
48   *
49   * <p>
50   * <code>RuleBasedCollator</code> has the following restrictions
51   * for efficiency (other subclasses may be used for more complex languages) :
52   * <ol>
53   * <li>If a special collation rule controlled by a &lt;modifier&gt; is
54   *      specified it applies to the whole collator object.
55   * <li>All non-mentioned characters are at the end of the
56   *     collation order.
57   * </ol>
58   *
59   * <p>
60   * The collation table is composed of a list of collation rules, where each
61   * rule is of one of three forms:
62   * <pre>
63   *    &lt;modifier&gt;
64   *    &lt;relation&gt; &lt;text-argument&gt;
65   *    &lt;reset&gt; &lt;text-argument&gt;
66   * </pre>
67   * The definitions of the rule elements is as follows:
68   * <UL Type=disc>
69   *    <LI><strong>Text-Argument</strong>: A text-argument is any sequence of
70   *        characters, excluding special characters (that is, common
71   *        whitespace characters [0009-000D, 0020] and rule syntax characters
72   *        [0021-002F, 003A-0040, 005B-0060, 007B-007E]). If those
73   *        characters are desired, you can put them in single quotes
74   *        (e.g. ampersand => '&'). Note that unquoted white space characters
75   *        are ignored; e.g. <code>b c</code> is treated as <code>bc</code>.
76   *    <LI><strong>Modifier</strong>: There are currently two modifiers that 
77   *        turn on special collation rules.
78   *        <UL Type=square>
79   *            <LI>'@' : Turns on backwards sorting of accents (secondary
80   *                      differences), as in French.
81   *            <LI>'!' : Turns on Thai/Lao vowel-consonant swapping.  If this
82   *                      rule is in force when a Thai vowel of the range
83   *                      &#92;U0E40-&#92;U0E44 precedes a Thai consonant of the range
84   *                      &#92;U0E01-&#92;U0E2E OR a Lao vowel of the range &#92;U0EC0-&#92;U0EC4
85   *                      precedes a Lao consonant of the range &#92;U0E81-&#92;U0EAE then
86   *                      the vowel is placed after the consonant for collation
87   *                      purposes.
88   *        </UL>
89   *        <p>'@' : Indicates that accents are sorted backwards, as in French.
90   *    <LI><strong>Relation</strong>: The relations are the following:
91   *        <UL Type=square>
92   *            <LI>'&lt;' : Greater, as a letter difference (primary)
93   *            <LI>';' : Greater, as an accent difference (secondary)
94   *            <LI>',' : Greater, as a case difference (tertiary)
95   *            <LI>'=' : Equal
96   *        </UL>
97   *    <LI><strong>Reset</strong>: There is a single reset
98   *        which is used primarily for contractions and expansions, but which
99   *        can also be used to add a modification at the end of a set of rules.
100  *        <p>'&' : Indicates that the next rule follows the position to where
101  *            the reset text-argument would be sorted.
102  * </UL>
103  *
104  * <p>
105  * This sounds more complicated than it is in practice. For example, the
106  * following are equivalent ways of expressing the same thing:
107  * <blockquote>
108  * <pre>
109  * a &lt; b &lt; c
110  * a &lt; b &amp; b &lt; c
111  * a &lt; c &amp; a &lt; b
112  * </pre>
113  * </blockquote>
114  * Notice that the order is important, as the subsequent item goes immediately
115  * after the text-argument. The following are not equivalent:
116  * <blockquote>
117  * <pre>
118  * a &lt; b &amp; a &lt; c
119  * a &lt; c &amp; a &lt; b
120  * </pre>
121  * </blockquote>
122  * Either the text-argument must already be present in the sequence, or some
123  * initial substring of the text-argument must be present. (e.g. "a &lt; b &amp; ae &lt; 
124  * e" is valid since "a" is present in the sequence before "ae" is reset). In
125  * this latter case, "ae" is not entered and treated as a single character;
126  * instead, "e" is sorted as if it were expanded to two characters: "a"
127  * followed by an "e". This difference appears in natural languages: in
128  * traditional Spanish "ch" is treated as though it contracts to a single
129  * character (expressed as "c &lt; ch &lt; d"), while in traditional German
130  * a-umlaut is treated as though it expanded to two characters
131  * (expressed as "a,A &lt; b,B ... &amp;ae;&#92;u00e3&amp;AE;&#92;u00c3").
132  * [&#92;u00e3 and &#92;u00c3 are, of course, the escape sequences for a-umlaut.]
133  * <p>
134  * <strong>Ignorable Characters</strong>
135  * <p>
136  * For ignorable characters, the first rule must start with a relation (the
137  * examples we have used above are really fragments; "a &lt; b" really should be
138  * "&lt; a &lt; b"). If, however, the first relation is not "&lt;", then all the all
139  * text-arguments up to the first "&lt;" are ignorable. For example, ", - &lt; a &lt; b"
140  * makes "-" an ignorable character, as we saw earlier in the word
141  * "black-birds". In the samples for different languages, you see that most
142  * accents are ignorable.
143  *
144  * <p><strong>Normalization and Accents</strong>
145  * <p>
146  * <code>RuleBasedCollator</code> automatically processes its rule table to
147  * include both pre-composed and combining-character versions of
148  * accented characters.  Even if the provided rule string contains only
149  * base characters and separate combining accent characters, the pre-composed
150  * accented characters matching all canonical combinations of characters from
151  * the rule string will be entered in the table.
152  * <p>
153  * This allows you to use a RuleBasedCollator to compare accented strings
154  * even when the collator is set to NO_DECOMPOSITION.  There are two caveats,
155  * however.  First, if the strings to be collated contain combining
156  * sequences that may not be in canonical order, you should set the collator to
157  * CANONICAL_DECOMPOSITION or FULL_DECOMPOSITION to enable sorting of
158  * combining sequences.  Second, if the strings contain characters with
159  * compatibility decompositions (such as full-width and half-width forms),
160  * you must use FULL_DECOMPOSITION, since the rule tables only include
161  * canonical mappings.
162  *
163  * <p><strong>Errors</strong>
164  * <p>
165  * The following are errors:
166  * <UL Type=disc>
167  *     <LI>A text-argument contains unquoted punctuation symbols
168  *        (e.g. "a &lt; b-c &lt; d").
169  *     <LI>A relation or reset character not followed by a text-argument
170  *        (e.g. "a &lt; ,b").
171  *     <LI>A reset where the text-argument (or an initial substring of the
172  *         text-argument) is not already in the sequence.
173  *         (e.g. "a &lt; b &amp; e &lt; f")
174  * </UL>
175  * If you produce one of these errors, a <code>RuleBasedCollator</code> throws
176  * a <code>ParseException</code>.
177  * 
178  * <p><strong>Examples</strong>
179  * <p>Simple:     "&lt; a &lt; b &lt; c &lt; d"
180  * <p>Norwegian:  "&lt; a,A&lt; b,B&lt; c,C&lt; d,D&lt; e,E&lt; f,F&lt; g,G&lt; h,H&lt; i,I&lt; j,J
181  *                 &lt; k,K&lt; l,L&lt; m,M&lt; n,N&lt; o,O&lt; p,P&lt; q,Q&lt; r,R&lt; s,S&lt; t,T
182  *                 &lt; u,U&lt; v,V&lt; w,W&lt; x,X&lt; y,Y&lt; z,Z
183  *                 &lt; &#92;u00E5=a&#92;u030A,&#92;u00C5=A&#92;u030A
184  *                 ;aa,AA&lt; &#92;u00E6,&#92;u00C6&lt; &#92;u00F8,&#92;u00D8"
185  *
186  * <p>
187  * Normally, to create a rule-based Collator object, you will use
188  * <code>Collator</code>'s factory method <code>getInstance</code>.
189  * However, to create a rule-based Collator object with specialized
190  * rules tailored to your needs, you construct the <code>RuleBasedCollator</code>
191  * with the rules contained in a <code>String</code> object. For example:
192  * <blockquote>
193  * <pre>
194  * String Simple = "&lt; a&lt; b&lt; c&lt; d";
195  * RuleBasedCollator mySimple = new RuleBasedCollator(Simple);
196  * </pre>
197  * </blockquote>
198  * Or:
199  * <blockquote>
200  * <pre>
201  * String Norwegian = "&lt; a,A&lt; b,B&lt; c,C&lt; d,D&lt; e,E&lt; f,F&lt; g,G&lt; h,H&lt; i,I&lt; j,J" +
202  *                 "&lt; k,K&lt; l,L&lt; m,M&lt; n,N&lt; o,O&lt; p,P&lt; q,Q&lt; r,R&lt; s,S&lt; t,T" +
203  *                 "&lt; u,U&lt; v,V&lt; w,W&lt; x,X&lt; y,Y&lt; z,Z" +
204  *                 "&lt; &#92;u00E5=a&#92;u030A,&#92;u00C5=A&#92;u030A" +
205  *                 ";aa,AA&lt; &#92;u00E6,&#92;u00C6&lt; &#92;u00F8,&#92;u00D8";
206  * RuleBasedCollator myNorwegian = new RuleBasedCollator(Norwegian);
207  * </pre>
208  * </blockquote>
209  *
210  * <p>
211  * Combining <code>Collator</code>s is as simple as concatenating strings.
212  * Here's an example that combines two <code>Collator</code>s from two
213  * different locales:
214  * <blockquote>
215  * <pre>
216  * // Create an en_US Collator object
217  * RuleBasedCollator en_USCollator = (RuleBasedCollator)
218  *     Collator.getInstance(new Locale("en", "US", ""));
219  * // Create a da_DK Collator object
220  * RuleBasedCollator da_DKCollator = (RuleBasedCollator)
221  *     Collator.getInstance(new Locale("da", "DK", ""));
222  * // Combine the two
223  * // First, get the collation rules from en_USCollator
224  * String en_USRules = en_USCollator.getRules();
225  * // Second, get the collation rules from da_DKCollator
226  * String da_DKRules = da_DKCollator.getRules();
227  * RuleBasedCollator newCollator =
228  *     new RuleBasedCollator(en_USRules + da_DKRules);
229  * // newCollator has the combined rules
230  * </pre>
231  * </blockquote>
232  *
233  * <p>
234  * Another more interesting example would be to make changes on an existing
235  * table to create a new <code>Collator</code> object.  For example, add
236  * "&amp;C&lt; ch, cH, Ch, CH" to the <code>en_USCollator</code> object to create
237  * your own:
238  * <blockquote>
239  * <pre>
240  * // Create a new Collator object with additional rules
241  * String addRules = "&amp;C&lt; ch, cH, Ch, CH";
242  * RuleBasedCollator myCollator =
243  *     new RuleBasedCollator(en_USCollator + addRules);
244  * // myCollator contains the new rules
245  * </pre>
246  * </blockquote>
247  *
248  * <p>
249  * The following example demonstrates how to change the order of
250  * non-spacing accents,
251  * <blockquote>
252  * <pre>
253  * // old rule
254  * String oldRules = "=&#92;u0301;&#92;u0300;&#92;u0302;&#92;u0308"    // main accents
255  *                 + ";&#92;u0327;&#92;u0303;&#92;u0304;&#92;u0305"    // main accents
256  *                 + ";&#92;u0306;&#92;u0307;&#92;u0309;&#92;u030A"    // main accents
257  *                 + ";&#92;u030B;&#92;u030C;&#92;u030D;&#92;u030E"    // main accents
258  *                 + ";&#92;u030F;&#92;u0310;&#92;u0311;&#92;u0312"    // main accents
259  *                 + "&lt; a , A ; ae, AE ; &#92;u00e6 , &#92;u00c6"
260  *                 + "&lt; b , B &lt; c, C &lt; e, E & C &lt; d, D";
261  * // change the order of accent characters
262  * String addOn = "& &#92;u0300 ; &#92;u0308 ; &#92;u0302";
263  * RuleBasedCollator myCollator = new RuleBasedCollator(oldRules + addOn);
264  * </pre>
265  * </blockquote>
266  *
267  * <p>
268  * The last example shows how to put new primary ordering in before the
269  * default setting. For example, in Japanese <code>Collator</code>, you
270  * can either sort English characters before or after Japanese characters,
271  * <blockquote>
272  * <pre>
273  * // get en_US Collator rules
274  * RuleBasedCollator en_USCollator = (RuleBasedCollator)Collator.getInstance(Locale.US);
275  * // add a few Japanese character to sort before English characters
276  * // suppose the last character before the first base letter 'a' in
277  * // the English collation rule is &#92;u2212
278  * String jaString = "& &#92;u2212 &lt; &#92;u3041, &#92;u3042 &lt; &#92;u3043, &#92;u3044";
279  * RuleBasedCollator myJapaneseCollator = new
280  *     RuleBasedCollator(en_USCollator.getRules() + jaString);
281  * </pre>
282  * </blockquote>
283  *
284  * @see        Collator
