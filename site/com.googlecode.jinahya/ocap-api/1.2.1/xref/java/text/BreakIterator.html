52  /** 
53   * The <code>BreakIterator</code> class implements methods for finding
54   * the location of boundaries in text. Instances of <code>BreakIterator</code>
55   * maintain a current position and scan over text
56   * returning the index of characters where boundaries occur.
57   * Internally, <code>BreakIterator</code> scans text using a
58   * <code>CharacterIterator</code>, and is thus able to scan text held
59   * by any object implementing that protocol. A <code>StringCharacterIterator</code>
60   * is used to scan <code>String</code> objects passed to <code>setText</code>.
61   *
62   * <p>
63   * You use the factory methods provided by this class to create
64   * instances of various types of break iterators. In particular,
65   * use <code>getWordIterator</code>, <code>getLineIterator</code>,
66   * <code>getSentenceIterator</code>, and <code>getCharacterIterator</code>
67   * to create <code>BreakIterator</code>s that perform
68   * word, line, sentence, and character boundary analysis respectively.
69   * A single <code>BreakIterator</code> can work only on one unit
70   * (word, line, sentence, and so on). You must use a different iterator
71   * for each unit boundary analysis you wish to perform.
72   *
73   * <p>
74   * Line boundary analysis determines where a text string can be
75   * broken when line-wrapping. The mechanism correctly handles
76   * punctuation and hyphenated words.
77   *
78   * <p>
79   * Sentence boundary analysis allows selection with correct interpretation
80   * of periods within numbers and abbreviations, and trailing punctuation
81   * marks such as quotation marks and parentheses.
82   *
83   * <p>
84   * Word boundary analysis is used by search and replace functions, as
85   * well as within text editing applications that allow the user to
86   * select words with a double click. Word selection provides correct
87   * interpretation of punctuation marks within and following
88   * words. Characters that are not part of a word, such as symbols
89   * or punctuation marks, have word-breaks on both sides.
90   *
91   * <p>
92   * Character boundary analysis allows users to interact with characters
93   * as they expect to, for example, when moving the cursor through a text
94   * string. Character boundary analysis provides correct navigation of
95   * through character strings, regardless of how the character is stored.
96   * For example, an accented character might be stored as a base character
97   * and a diacritical mark. What users consider to be a character can
98   * differ between languages.
99   *
100  * <p>
101  * <code>BreakIterator</code> is intended for use with natural
102  * languages only. Do not use this class to tokenize a programming language.
103  *
104  * <P>
105  * <strong>Examples</strong>:<P>
106  * Creating and using text boundaries
107  * <blockquote>
108  * <pre>
109  * public static void main(String args[]) {
110  *      if (args.length == 1) {
111  *          String stringToExamine = args[0];
112  *          //print each word in order
113  *          BreakIterator boundary = BreakIterator.getWordInstance();
114  *          boundary.setText(stringToExamine);
115  *          printEachForward(boundary, stringToExamine);
116  *          //print each sentence in reverse order
117  *          boundary = BreakIterator.getSentenceInstance(Locale.US);
118  *          boundary.setText(stringToExamine);
119  *          printEachBackward(boundary, stringToExamine);
120  *          printFirst(boundary, stringToExamine);
121  *          printLast(boundary, stringToExamine);
122  *      }
123  * }
124  * </pre>
125  * </blockquote>
126  *
127  * Print each element in order
128  * <blockquote>
129  * <pre>
130  * public static void printEachForward(BreakIterator boundary, String source) {
131  *     int start = boundary.first();
132  *     for (int end = boundary.next();
133  *          end != BreakIterator.DONE;
134  *          start = end, end = boundary.next()) {
135  *          System.out.println(source.substring(start,end));
136  *     }
137  * }
138  * </pre>
139  * </blockquote>
140  *
141  * Print each element in reverse order
142  * <blockquote>
143  * <pre>
144  * public static void printEachBackward(BreakIterator boundary, String source) {
145  *     int end = boundary.last();
146  *     for (int start = boundary.previous();
147  *          start != BreakIterator.DONE;
148  *          end = start, start = boundary.previous()) {
149  *         System.out.println(source.substring(start,end));
150  *     }
151  * }
152  * </pre>
153  * </blockquote>
154  *
155  * Print first element
156  * <blockquote>
157  * <pre>
158  * public static void printFirst(BreakIterator boundary, String source) {
159  *     int start = boundary.first();
160  *     int end = boundary.next();
161  *     System.out.println(source.substring(start,end));
162  * }
163  * </pre>
164  * </blockquote>
165  *
166  * Print last element
167  * <blockquote>
168  * <pre>
169  * public static void printLast(BreakIterator boundary, String source) {
170  *     int end = boundary.last();
171  *     int start = boundary.previous();
172  *     System.out.println(source.substring(start,end));
173  * }
174  * </pre>
175  * </blockquote>
176  *
177  * Print the element at a specified position
178  * <blockquote>
179  * <pre>
180  * public static void printAt(BreakIterator boundary, int pos, String source) {
181  *     int end = boundary.following(pos);
182  *     int start = boundary.previous();
183  *     System.out.println(source.substring(start,end));
184  * }
185  * </pre>
186  * </blockquote>
187  *
188  * Find the next word
189  * <blockquote>
190  * <pre>
191  * public static int nextWordStartAfter(int pos, String text) {
192  *     BreakIterator wb = BreakIterator.getWordInstance();
193  *     wb.setText(text);
194  *     int last = wb.following(pos);
195  *     int current = wb.next();
196  *     while (current != BreakIterator.DONE) {
197  *         for (int p = last; p < current; p++) {
198  *             if (Character.isLetter(text.charAt(p))
199  *                 return last;
200  *         }
201  *         last = current;
202  *         current = wb.next();
203  *     }
204  *     return BreakIterator.DONE;
205  * }
206  * </pre>
207  * (The iterator returned by BreakIterator.getWordInstance() is unique in that
208  * the break positions it returns don't represent both the start and end of the
209  * thing being iterated over.  That is, a sentence-break iterator returns breaks
210  * that each represent the end of one sentence and the beginning of the next.
211  * With the word-break iterator, the characters between two boundaries might be a
212  * word, or they might be the punctuation or whitespace between two words.  The
213  * above code uses a simple heuristic to determine which boundary is the beginning
214  * of a word: If the characters between this boundary and the next boundary
215  * include at least one letter (this can be an alphabetical letter, a CJK ideograph,
216  * a Hangul syllable, a Kana character, etc.), then the text between this boundary
217  * and the next is a word; otherwise, it's the material between words.)
218  * </blockquote>
