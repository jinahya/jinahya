49  /** 
50   * <code>DecimalFormat</code> is a concrete subclass of
51   * <code>NumberFormat</code> that formats decimal numbers. It has a variety of
52   * features designed to make it possible to parse and format numbers in any
53   * locale, including support for Western, Arabic, and Indic digits.  It also
54   * supports different kinds of numbers, including integers (123), fixed-point
55   * numbers (123.4), scientific notation (1.23E4), percentages (12%), and
56   * currency amounts ($123).  All of these can be localized.
57   *
58   * <p>To obtain a <code>NumberFormat</code> for a specific locale, including the
59   * default locale, call one of <code>NumberFormat</code>'s factory methods, such
60   * as <code>getInstance()</code>.  In general, do not call the
61   * <code>DecimalFormat</code> constructors directly, since the
62   * <code>NumberFormat</code> factory methods may return subclasses other than
63   * <code>DecimalFormat</code>. If you need to customize the format object, do
64   * something like this:
65   *
66   * <blockquote><pre>
67   * NumberFormat f = NumberFormat.getInstance(loc);
68   * if (f instanceof DecimalFormat) {
69   *     ((DecimalFormat) f).setDecimalSeparatorAlwaysShown(true);
70   * }
71   * </pre></blockquote>
72   *
73   * <p>A <code>DecimalFormat</code> comprises a <em>pattern</em> and a set of
74   * <em>symbols</em>.  The pattern may be set directly using
75   * <code>applyPattern()</code>, or indirectly using the API methods.  The
76   * symbols are stored in a <code>DecimalFormatSymbols</code> object.  When using
77   * the <code>NumberFormat</code> factory methods, the pattern and symbols are
78   * read from localized <code>ResourceBundle</code>s.
79   *
80   * <h4>Patterns</h4>
81   *
82   * <code>DecimalFormat</code> patterns have the following syntax:
83   * <blockquote><pre>
84   * <i>Pattern:</i>
85   *         <i>PositivePattern</i>
86   *         <i>PositivePattern</i> ; <i>NegativePattern</i>
87   * <i>PositivePattern:</i>
88   *         <i>Prefix<sub>opt</sub></i> <i>Number</i> <i>Suffix<sub>opt</sub></i>
89   * <i>NegativePattern:</i>
90   *         <i>Prefix<sub>opt</sub></i> <i>Number</i> <i>Suffix<sub>opt</sub></i>
91   * <i>Prefix:</i>
92   *         any Unicode characters except &#92;uFFFE, &#92;uFFFF, and special characters
93   * <i>Suffix:</i>
94   *         any Unicode characters except &#92;uFFFE, &#92;uFFFF, and special characters
95   * <i>Number:</i>
96   *         <i>Integer</i> <i>Exponent<sub>opt</sub></i>
97   *         <i>Integer</i> . <i>Fraction</i> <i>Exponent<sub>opt</sub></i>
98   * <i>Integer:</i>
99   *         <i>MinimumInteger</i>
100  *         #
101  *         # <i>Integer</i>
102  *         # , <i>Integer</i>
103  * <i>MinimumInteger:</i>
104  *         0
105  *         0 <i>MinimumInteger</i>
106  *         0 , <i>MinimumInteger</i>
107  * <i>Fraction:</i>
108  *         <i>MinimumFraction<sub>opt</sub></i> <i>OptionalFraction<sub>opt</sub></i>
109  * <i>MinimumFraction:</i>
110  *         0 <i>MinimumFraction<sub>opt</sub></i>
111  * <i>OptionalFraction:</i>
112  *         # <i>OptionalFraction<sub>opt</sub></i>
113  * <i>Exponent:</i>
114  *         E <i>MinimumExponent</i>
115  * <i>MinimumExponent:</i>
116  *         0 <i>MinimumExponent<sub>opt</sub></i>
117  * </pre></blockquote>
118  *
119  * <p>A <code>DecimalFormat</code> pattern contains a positive and negative
120  * subpattern, for example, <code>"#,##0.00;(#,##0.00)"</code>.  Each
121  * subpattern has a prefix, numeric part, and suffix. The negative subpattern
122  * is optional; if absent, then the positive subpattern prefixed with the
123  * localized minus sign (code>'-'</code> in most locales) is used as the
124  * negative subpattern. That is, <code>"0.00"</code> alone is equivalent to
125  * <code>"0.00;-0.00"</code>.  If there is an explicit negative subpattern, it
126  * serves only to specify the negative prefix and suffix; the number of digits,
127  * minimal digits, and other characteristics are all the same as the positive
128  * pattern. That means that <code>"#,##0.0#;(#)"</code> produces precisely
129  * the same behavior as <code>"#,##0.0#;(#,##0.0#)"</code>.
130  *
131  * <p>The prefixes, suffixes, and various symbols used for infinity, digits,
132  * thousands separators, decimal separators, etc. may be set to arbitrary
133  * values, and they will appear properly during formatting.  However, care must
134  * be taken that the symbols and strings do not conflict, or parsing will be
135  * unreliable.  For example, either the positive and negative prefixes or the
136  * suffixes must be distinct for <code>DecimalFormat.parse()</code> to be able
137  * to distinguish positive from negative values.  (If they are identical, then
138  * <code>DecimalFormat</code> will behave as if no negative subpattern was
139  * specified.)  Another example is that the decimal separator and thousands
140  * separator should be distinct characters, or parsing will be impossible.
141  *
142  * <p>The grouping separator is commonly used for thousands, but in some
143  * countries it separates ten-thousands. The grouping size is a constant number
144  * of digits between the grouping characters, such as 3 for 100,000,000 or 4 for
145  * 1,0000,0000.  If you supply a pattern with multiple grouping characters, the
146  * interval between the last one and the end of the integer is the one that is
147  * used. So <code>"#,##,###,####"</code> == <code>"######,####"</code> ==
148  * <code>"##,####,####"</code>.
149  *
150  * <h4>Special Pattern Characters</h4>
151  *
152  * <p>Many characters in a pattern are taken literally; they are matched during
153  * parsing and output unchanged during formatting.  Special characters, on the
154  * other hand, stand for other characters, strings, or classes of characters.
155  * They must be quoted, unless noted otherwise, if they are to appear in the
156  * prefix or suffix as literals.
157  *
158  * <p>The characters listed here are used in non-localized patterns.  Localized
159  * patterns use the corresponding characters taken from this formatter's
160  * <code>DecimalFormatSymbols</code> object instead, and these characters lose
161  * their special status.  Two exceptions are the currency sign and quote, which
162  * are not localized.

237  * <h4>Scientific Notation</h4>
238  *
239  * <p>Numbers in scientific notation are expressed as the product of a mantissa
240  * and a power of ten, for example, 1234 can be expressed as 1.234 x 10^3.  The
241  * mantissa is often in the range 1.0 <= x < 10.0, but it need not be.
242  * <code>DecimalFormat</code> can be instructed to format and parse scientific
243  * notation <em>only via a pattern</em>; there is currently no factory method
244  * that creates a scientific notation format.  In a pattern, the exponent
245  * character immediately followed by one or more digit characters indicates
246  * scientific notation.  Example: <code>"0.###E0"</code> formats the number
247  * 1234 as <code>"1.234E3"</code>.
248  *
249  * <ul>
250  * <li>The number of digit characters after the exponent character gives the
251  * minimum exponent digit count.  There is no maximum.  Negative exponents are
252  * formatted using the localized minus sign, <em>not</em> the prefix and suffix
253  * from the pattern.  This allows patterns such as <code>"0.###E0 m/s"</code>.
254  *
255  * <li>The minimum and maximum number of integer digits are interpreted
256  * together:
257  *
258  * <ul>
259  * <li>If the maximum number of integer digits is greater than their minimum number
260  * and greater than 1, it forces the exponent to be a multiple of the maximum
261  * number of integer digits, and the minimum number of integer digits to be
262  * interpreted as 1.  The most common use of this is to generate
263  * <em>engineering notation</em>, in which the exponent is a multiple of three,
264  * e.g., <code>"##0.#####E0"</code>. Using this pattern, the number 12345
265  * formats to <code>"12.345E3"</code>, and 123456 formats to
266  * <code>"123.456E3"</code>.
267  *
268  * <li>Otherwise, the minimum number of integer digits is achieved by adjusting the
269  * exponent.  Example: 0.00123 formatted with <code>"00.###E0"</code> yields
270  * <code>"12.3E-4"</code>.
271  * </ul>
272  *
273  * <li>The number of significant digits in the mantissa is the sum of the
274  * <em>minimum integer</em> and <em>maximum fraction</em> digits, and is
275  * unaffected by the maximum integer digits.  For example, 12345 formatted with
276  * <code>"##0.##E0"</code> is <code>"12.3E3"</code>. To show all digits, set
277  * the significant digits count to zero.  The number of significant digits
278  * does not affect parsing.
279  *
280  * <li>Exponential patterns may not contain grouping separators.
281  * </ul>
282  *
283  * <h4>Rounding</h4>
284  *
285  * <code>DecimalFormat</code> uses half-even rounding (see
286  * {@link java.math.BigDecimal#ROUND_HALF_EVEN ROUND_HALF_EVEN}) for
287  * formatting.
288  * 
289  * <h4>Digits</h4>
290  *
291  * For formatting, <code>DecimalFormat</code> uses the ten consecutive
292  * characters starting with the localized zero digit defined in the
293  * <code>DecimalFormatSymbols</code> object as digits. For parsing, these
294  * digits as well as all Unicode decimal digits, as defined by
295  * {@link Character#digit Character.digit}, are recognized.
296  *
297  * <h4>Special Values</h4>
298  *
299  * <p><code>NaN</code> is formatted as a single character, typically
300  * <code>&#92;uFFFD</code>.  This character is determined by the
301  * <code>DecimalFormatSymbols</code> object.  This is the only value for which
302  * the prefixes and suffixes are not used.
303  *
304  * <p>Infinity is formatted as a single character, typically
305  * <code>&#92;u221E</code>, with the positive or negative prefixes and suffixes
306  * applied.  The infinity character is determined by the
307  * <code>DecimalFormatSymbols</code> object.
308  *
309  * <p>Negative zero (<code>"-0"</code>) parses to <code>Double(-0.0)</code>,
310  * unless <code>isParseIntegerOnly()</code> is true, in which case it parses to
311  * <code>Long(0)</code>.
312  *
313  * <h4><a name="synchronization">Synchronization</a></h4>
314  *
315  * <p>
316  * Decimal formats are generally not synchronized.
317  * It is recommended to create separate format instances for each thread.
318  * If multiple threads access a format concurrently, it must be synchronized
319  * externally.
320  *
321  * <h4>Example</h4>
322  *
323  * <blockquote><pre>
324  * <strong>// Print out a number using the localized number, integer, currency,
325  * // and percent format for each locale</strong>
326  * Locale[] locales = NumberFormat.getAvailableLocales();
327  * double myNumber = -1234.56;
328  * NumberFormat form;
329  * for (int j=0; j<4; ++j) {
330  *     System.out.println("FORMAT");
331  *     for (int i = 0; i < locales.length; ++i) {
332  *         if (locales[i].getCountry().length() == 0) {
333  *            continue; // Skip language-only locales
334  *         }
335  *         System.out.print(locales[i].getDisplayName());
336  *         switch (j) {
337  *         case 0:
338  *             form = NumberFormat.getInstance(locales[i]); break;
339  *         case 1:
340  *             form = NumberFormat.getIntegerInstance(locales[i]); break;
341  *         case 2:
342  *             form = NumberFormat.getCurrencyInstance(locales[i]); break;
343  *         default:
344  *             form = NumberFormat.getPercentInstance(locales[i]); break;
345  *         }
346  *         if (form instanceof DecimalFormat) {
347  *             System.out.print(": " + ((DecimalFormat) form).toPattern());
348  *         }
349  *         System.out.print(" -> " + form.format(myNumber));
350  *         try {
351  *             System.out.println(" -> " + form.parse(form.format(myNumber)));
352  *         } catch (ParseException e) {}
353  *     }
354  * }
355  * </pre></blockquote>
356  *
357  * @see          <a href="http://java.sun.com/docs/books/tutorial/i18n/format/decimalFormat.html">Java Tutorial</a>
358  * @see          NumberFormat
359  * @see          DecimalFormatSymbols
360  * @see          ParsePosition
361  * @version      1.65, 01/12/04
362  * @author       Mark Davis
363  * @author       Alan Liu
