47  /** 
48   * <code>DateFormatSymbols</code> is a public class for encapsulating
49   * localizable date-time formatting data, such as the names of the
50   * months, the names of the days of the week, and the time zone data.
51   * <code>DateFormat</code> and <code>SimpleDateFormat</code> both use
52   * <code>DateFormatSymbols</code> to encapsulate this information.
53   *
54   * <p>
55   * Typically you shouldn't use <code>DateFormatSymbols</code> directly.
56   * Rather, you are encouraged to create a date-time formatter with the
57   * <code>DateFormat</code> class's factory methods: <code>getTimeInstance</code>,
58   * <code>getDateInstance</code>, or <code>getDateTimeInstance</code>.
59   * These methods automatically create a <code>DateFormatSymbols</code> for
60   * the formatter so that you don't have to. After the
61   * formatter is created, you may modify its format pattern using the
62   * <code>setPattern</code> method. For more information about
63   * creating formatters using <code>DateFormat</code>'s factory methods,
64   * see {@link DateFormat}.
65   *
66   * <p>
67   * If you decide to create a date-time formatter with a specific
68   * format pattern for a specific locale, you can do so with:
69   * <blockquote>
70   * <pre>
71   * new SimpleDateFormat(aPattern, new DateFormatSymbols(aLocale)).
72   * </pre>
73   * </blockquote>
74   *
75   * <p>
76   * <code>DateFormatSymbols</code> objects are cloneable. When you obtain
77   * a <code>DateFormatSymbols</code> object, feel free to modify the
78   * date-time formatting data. For instance, you can replace the localized
79   * date-time format pattern characters with the ones that you feel easy
80   * to remember. Or you can change the representative cities
81   * to your favorite ones.
82   *
83   * <p>
84   * New <code>DateFormatSymbols</code> subclasses may be added to support
85   * <code>SimpleDateFormat</code> for date-time formatting for additional locales.
86   *
87   * @see          DateFormat
88   * @see          SimpleDateFormat
89   * @see          java.util.SimpleTimeZone
