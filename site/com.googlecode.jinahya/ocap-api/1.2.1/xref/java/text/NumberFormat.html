51  /** 
52   * <code>NumberFormat</code> is the abstract base class for all number
53   * formats. This class provides the interface for formatting and parsing
54   * numbers. <code>NumberFormat</code> also provides methods for determining
55   * which locales have number formats, and what their names are.
56   *
57   * <p>
58   * <code>NumberFormat</code> helps you to format and parse numbers for any locale.
59   * Your code can be completely independent of the locale conventions for
60   * decimal points, thousands-separators, or even the particular decimal
61   * digits used, or whether the number format is even decimal.
62   *
63   * <p>
64   * To format a number for the current Locale, use one of the factory
65   * class methods:
66   * <blockquote>
67   * <pre>
68   *  myString = NumberFormat.getInstance().format(myNumber);
69   * </pre>
70   * </blockquote>
71   * If you are formatting multiple numbers, it is
72   * more efficient to get the format and use it multiple times so that
73   * the system doesn't have to fetch the information about the local
74   * language and country conventions multiple times.
75   * <blockquote>
76   * <pre>
77   * NumberFormat nf = NumberFormat.getInstance();
78   * for (int i = 0; i < a.length; ++i) {
79   *     output.println(nf.format(myNumber[i]) + "; ");
80   * }
81   * </pre>
82   * </blockquote>
83   * To format a number for a different Locale, specify it in the
84   * call to <code>getInstance</code>.
85   * <blockquote>
86   * <pre>
87   * NumberFormat nf = NumberFormat.getInstance(Locale.FRENCH);
88   * </pre>
89   * </blockquote>
90   * You can also use a <code>NumberFormat</code> to parse numbers:
91   * <blockquote>
92   * <pre>
93   * myNumber = nf.parse(myString);
94   * </pre>
95   * </blockquote>
96   * Use <code>getInstance</code> or <code>getNumberInstance</code> to get the
97   * normal number format. Use <code>getIntegerInstance</code> to get an
98   * integer number format. Use <code>getCurrencyInstance</code> to get the
99   * currency number format. And use <code>getPercentInstance</code> to get a
100  * format for displaying percentages. With this format, a fraction like
101  * 0.53 is displayed as 53%.
102  *
103  * <p>
104  * You can also control the display of numbers with such methods as
105  * <code>setMinimumFractionDigits</code>.
106  * If you want even more control over the format or parsing,
107  * or want to give your users more control,
108  * you can try casting the <code>NumberFormat</code> you get from the factory methods
109  * to a <code>DecimalFormat</code>. This will work for the vast majority
110  * of locales; just remember to put it in a <code>try</code> block in case you
111  * encounter an unusual one.
112  *
113  * <p>
114  * NumberFormat and DecimalFormat are designed such that some controls
115  * work for formatting and others work for parsing.  The following is
116  * the detailed description for each these control methods,
117  * <p>
118  * setParseIntegerOnly : only affects parsing, e.g.
119  * if true,  "3456.78" -> 3456 (and leaves the parse position just after index 6)
120  * if false, "3456.78" -> 3456.78 (and leaves the parse position just after index 8)
121  * This is independent of formatting.  If you want to not show a decimal point
122  * where there might be no digits after the decimal point, use
123  * setDecimalSeparatorAlwaysShown.
124  * <p>
125  * setDecimalSeparatorAlwaysShown : only affects formatting, and only where
126  * there might be no digits after the decimal point, such as with a pattern
127  * like "#,##0.##", e.g.,
128  * if true,  3456.00 -> "3,456."
129  * if false, 3456.00 -> "3456"
130  * This is independent of parsing.  If you want parsing to stop at the decimal
131  * point, use setParseIntegerOnly.
132  *
133  * <p>
134  * You can also use forms of the <code>parse</code> and <code>format</code>
135  * methods with <code>ParsePosition</code> and <code>FieldPosition</code> to
136  * allow you to:
137  * <ul>
138  * <li> progressively parse through pieces of a string
139  * <li> align the decimal point and other areas
140  * </ul>
141  * For example, you can align numbers in two ways:
142  * <ol>
143  * <li> If you are using a monospaced font with spacing for alignment,
144  *      you can pass the <code>FieldPosition</code> in your format call, with
145  *      <code>field</code> = <code>INTEGER_FIELD</code>. On output,
146  *      <code>getEndIndex</code> will be set to the offset between the
147  *      last character of the integer and the decimal. Add
148  *      (desiredSpaceCount - getEndIndex) spaces at the front of the string.
149  *
150  * <li> If you are using proportional fonts,
151  *      instead of padding with spaces, measure the width
152  *      of the string in pixels from the start to <code>getEndIndex</code>.
153  *      Then move the pen by
154  *      (desiredPixelWidth - widthToAlignmentPoint) before drawing the text.
155  *      It also works where there is no decimal, but possibly additional
156  *      characters at the end, e.g., with parentheses in negative
157  *      numbers: "(12)" for -12.
158  * </ol>
159  *

163  * Number formats are generally not synchronized.
164  * It is recommended to create separate format instances for each thread.
165  * If multiple threads access a format concurrently, it must be synchronized
166  * externally.
