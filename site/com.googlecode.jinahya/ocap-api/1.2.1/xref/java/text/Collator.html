43  /** 
44   * The <code>Collator</code> class performs locale-sensitive
45   * <code>String</code> comparison. You use this class to build
46   * searching and sorting routines for natural language text.
47   *
48   * <p>
49   * <code>Collator</code> is an abstract base class. Subclasses
50   * implement specific collation strategies. One subclass,
51   * <code>RuleBasedCollator</code>, is currently provided with
52   * the Java 2 platform and is applicable to a wide set of languages. Other
53   * subclasses may be created to handle more specialized needs.
54   *
55   * <p>
56   * Like other locale-sensitive classes, you can use the static
57   * factory method, <code>getInstance</code>, to obtain the appropriate
58   * <code>Collator</code> object for a given locale. You will only need
59   * to look at the subclasses of <code>Collator</code> if you need
60   * to understand the details of a particular collation strategy or
61   * if you need to modify that strategy.
62   *
63   * <p>
64   * The following example shows how to compare two strings using
65   * the <code>Collator</code> for the default locale.
66   * <blockquote>
67   * <pre>
68   * // Compare two strings in the default locale
69   * Collator myCollator = Collator.getInstance();
70   * if( myCollator.compare("abc", "ABC") < 0 )
71   *     System.out.println("abc is less than ABC");
72   * else
73   *     System.out.println("abc is greater than or equal to ABC");
74   * </pre>
75   * </blockquote>
76   *
77   * <p>
78   * You can set a <code>Collator</code>'s <em>strength</em> property
79   * to determine the level of difference considered significant in
80   * comparisons. Four strengths are provided: <code>PRIMARY</code>,
81   * <code>SECONDARY</code>, <code>TERTIARY</code>, and <code>IDENTICAL</code>.
82   * The exact assignment of strengths to language features is
83   * locale dependant.  For example, in Czech, "e" and "f" are considered
84   * primary differences, while "e" and "?" are secondary differences,
85   * "e" and "E" are tertiary differences and "e" and "e" are identical.
86   * The following shows how both case and accents could be ignored for
87   * US English.
88   * <blockquote>
89   * <pre>
90   * //Get the Collator for US English and set its strength to PRIMARY
91   * Collator usCollator = Collator.getInstance(Locale.US);
92   * usCollator.setStrength(Collator.PRIMARY);
93   * if( usCollator.compare("abc", "ABC") == 0 ) {
94   *     System.out.println("Strings are equivalent");
95   * }
96   * </pre>
97   * </blockquote>
98   * <p>
99   * For comparing <code>String</code>s exactly once, the <code>compare</code>
100  * method provides the best performance. When sorting a list of
101  * <code>String</code>s however, it is generally necessary to compare each
102  * <code>String</code> multiple times. In this case, <code>CollationKey</code>s
103  * provide better performance. The <code>CollationKey</code> class converts
104  * a <code>String</code> to a series of bits that can be compared bitwise
105  * against other <code>CollationKey</code>s. A <code>CollationKey</code> is
106  * created by a <code>Collator</code> object for a given <code>String</code>.
107  * <br>
108  * <strong>Note:</strong> <code>CollationKey</code>s from different
109  * <code>Collator</code>s can not be compared. See the class description
110  * for {@link CollationKey}
111  * for an example using <code>CollationKey</code>s.
112  *
113  * @see         RuleBasedCollator
114  * @see         CollationKey
115  * @see         CollationElementIterator
116  * @see         Locale
117  * @version     1.34, 01/27/03
118  * @author      Helena Shih, Laura Werner, Richard Gillam
119  */
